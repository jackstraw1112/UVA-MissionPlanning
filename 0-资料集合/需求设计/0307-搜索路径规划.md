# 需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 搜索路径规划 |
| **需求标识**     | FR-SEARCH-01 |
| **需求概述**     | 提供反辐射无人机搜索路径规划算法，能够依据任务类型、目标区域和任务要达到的目标（如搜索覆盖率、发现概率等），综合考虑无人机作战性能（航程、速度、续航时间、传感器性能）、敌方威胁、地形、气象等战场环境因素，自动生成满足协同作战任务要求且优化的协同作战搜索路径，支持多机协同区域覆盖搜索。 |
| **需求来源**     | 系统总体设计要求 / 协同任务规划管理CSCI能力指标 |
| **前置条件**     | 1. 任务规划已确定任务类型（如区域侦察、目标搜索、电子侦察等）和搜索目标区域（多边形区域或点集）。<br>2. 无人机资源信息已加载（包含每架无人机的性能参数：最大航程、巡航速度、续航时间、传感器类型及探测宽度/角度、机动性能等）。<br>3. 战场环境信息已从环境参数设置模块获取（地形高程数据、障碍物分布、威胁区、气象条件）。<br>4. 任务目标要求已明确（如最低覆盖率、搜索时限、发现概率要求等）。<br>5. 所有输入数据通过合法性校验。<br>6. 搜索路径规划算法所需参数（如搜索模式、覆盖策略、优化目标权重等）已配置。 |
| **触发条件**     | 1. 用户点击“开始搜索路径规划”按钮。<br>2. 任务区域或无人机资源变更后自动触发重新规划（可选）。 |
| **退出条件**     | 1. 搜索路径规划完成，所有无人机的搜索路径生成并显示。<br>2. 用户手动停止规划（若计算时间过长）。<br>3. 规划过程中发生严重错误，模块退出并报告错误。 |
| **功能参与者**   | 任务规划人员（用户）、环境参数设置模块（提供环境数据）、无人机资源数据库（提供性能参数）、任务管理模块（提供任务类型与目标区域） |
| **输入**         | 任务类型（区域搜索、目标确认、电子侦察等）、搜索目标区域（多边形顶点坐标列表）、任务目标要求（最低覆盖率、搜索时限、期望发现概率）、无人机资源列表（性能参数、可用数量）、战场环境数据（地形栅格、障碍物、威胁区、气象）、搜索路径规划算法参数（搜索模式：平行扫海/螺旋/随机等，覆盖策略：分区域/协同覆盖，优化目标权重：覆盖率/时间/风险等） |
| **输出**         | 每架无人机的搜索路径（三维航点序列，包含经纬度、高度、预计到达时间、搜索动作指令（如传感器开关））、总搜索覆盖率预计值、预计总搜索时间、总航程、燃油/能量消耗估计、路径安全评估（是否规避威胁/障碍）、协同覆盖重叠率、搜索效能评估报告 |
| **处理流程**     | 1. 获取输入数据，构建综合环境模型（地形高程、威胁区代价场、气象影响因子）。<br>2. 根据任务类型和目标区域，确定搜索区域范围和搜索需求。<br>3. 采用区域分解算法将整个搜索区域划分为若干子区域（考虑无人机数量、传感器探测宽度、续航能力），确保子区域负载均衡且满足协同要求。<br>4. 对每个子区域（或直接对整体区域），采用覆盖路径规划（CPP）算法生成单架无人机的搜索路径，路径需满足：<br>   - 完整覆盖子区域（或满足覆盖率要求），避免重复或遗漏。<br>   - 路径长度在无人机航程内。<br>   - 规避威胁区和障碍物，考虑地形跟随。<br>5. 若多机协同搜索（如同一区域同时搜索），需考虑路径间的时空协同（避免同时同地冲突，控制重叠率）。<br>6. 对生成的路径进行平滑处理，满足无人机机动性能。<br>7. 计算路径的覆盖效能指标：覆盖率、重叠率、搜索时间、暴露风险等。<br>8. 验证每架无人机路径是否满足性能约束。<br>9. 输出所有无人机的搜索路径数据及评估指标。 |
| **成功判据**     | 1. 每架无人机的搜索路径均满足自身性能约束（航程、时间、机动性）。<br>2. 整体搜索覆盖率（或发现概率）达到任务要求。<br>3. 路径有效规避所有已知威胁区和障碍物。<br>4. 多机路径之间无时空冲突，协同覆盖重叠率控制在合理范围（如不超过20%）。<br>5. 路径规划的优化目标（如最小化总时间、最大化覆盖率）达到算法设定的阈值或用户可接受。 |
| **异常处理**     | 1. 输入数据缺失或格式错误：弹出提示框，高亮错误字段，禁止规划。<br>2. 无法在满足航程约束下完整覆盖目标区域（区域过大）：提示用户调整区域、增加无人机或接受部分覆盖，并提供最大可能覆盖率估计。<br>3. 某无人机无法找到可行路径（如区域被威胁完全阻断）：提示用户该无人机无法参与搜索，建议调整区域或资源。<br>4. 规划算法超时：提示用户是否继续等待，允许手动终止。<br>5. 多机路径冲突无法解耦：提示用户尝试调整搜索模式或放宽协同约束。 |
| **后置条件**     | 1. 生成的搜索路径数据保存于内存中，可供后续仿真推演、飞行控制模块调用。<br>2. 用户可保存当前搜索路径规划结果为项目文件（含路径点及评估数据）。<br>3. 用户可导出路径报告或KML文件用于外部查看。 |
| **显示约束**     | 1. 在二维/三维地图上以不同颜色绘制每架无人机的搜索路径，并标注搜索起始点和转向点。<br>2. 可叠加显示搜索覆盖区域（如传感器覆盖带），直观展示覆盖效果。<br>3. 列表展示每架无人机的路径统计：总航程、总耗时、预计覆盖面积、剩余燃油等。<br>4. 应提供整体覆盖率、重叠率等指标的数值显示。<br>5. 支持区域分解结果的可视化（不同子区域用不同颜色区分）。 |
| **其他约束**     | 1. 算法应支持多种搜索模式（平行扫海、向内/外螺旋、扇形搜索、随机搜索等），以适应不同任务类型。<br>2. 需支持动态调整：若搜索过程中发现目标，可触发局部路径重规划（如转为确认搜索）。<br>3. 优化目标应支持多目标权衡（如搜索时间与覆盖率之间的平衡）。<br>4. 算法应考虑传感器探测宽度随高度、气象的变化。<br>5. 需具备可扩展性，以适应未来更多无人机数量或更复杂环境。 |

## 3. 分析流程

搜索路径规划采用分层、混合算法，综合考虑区域覆盖、环境约束和无人机性能，具体流程如下：

### 3.1 环境与任务建模
1. **目标区域离散化**：将搜索目标多边形区域划分为网格单元（栅格），每个单元尺寸根据传感器探测宽度设定（如探测宽度的一半），作为覆盖的基本单元。
2. **环境代价场构建**：在地形高程基础上，叠加威胁区代价、障碍物代价，生成综合代价地图，用于路径搜索时规避高风险区域。
3. **传感器模型**：根据无人机飞行高度、传感器探测宽度，计算每个栅格单元被覆盖的条件（如进入探测范围且无遮挡）。

### 3.2 区域分解（针对多机协同）
- **等面积分割**：将目标区域按无人机数量等分为若干子区域，确保每架无人机的搜索面积相近，考虑航程约束调整。
- **基于负载均衡的聚类**：使用K-means或谱聚类将目标栅格聚为N类（N为无人机数），每类作为一个子区域。
- **考虑航程的区域分配**：结合无人机起始位置，采用匈牙利算法或遗传算法分配子区域，最小化总飞行距离。

### 3.3 单机覆盖路径生成
对于每个子区域（或整体区域），采用覆盖路径规划（CPP）算法生成搜索路径。常用方法：
- **平行扫海（Boustrophedon）**：
  - 将子区域分解为若干个条带（基于障碍物或边界）。
  - 在每个条带内生成来回往复的平行路径，间距为传感器探测宽度。
  - 用最短路径连接各条带间的过渡。
- **螺旋搜索**：适用于圆形或不规则区域，从外向内或从内向外生成螺旋线。
- **生成树覆盖（STC）**：将区域栅格化后，构建生成树，沿树遍历实现覆盖。

生成路径后，进行威胁规避和地形跟随调整：
- 将路径点映射到代价地图上，若某点代价过高（靠近威胁），则局部调整偏移。
- 确保路径高度满足地形跟随要求（如离地高度≥安全高度）。

### 3.4 多机协同与冲突处理
- **时空协同**：若要求同时开始或同时结束搜索，需调整各机路径长度（通过速度调节或添加盘旋等待）。
- **冲突检测**：检查各路径在时间和空间上是否可能发生碰撞（如交叉点时间相近）。若有冲突，可采用：
  - **速度调整**：微调速度错开时间。
  - **局部重规划**：在冲突区域为其中一架无人机重新规划绕行路径。
- **重叠率控制**：计算子区域边界处的重叠，若重叠率过高，可调整区域分解参数重新规划。

### 3.5 后处理与评估
1. **路径平滑**：使用Dubins曲线或B样条平滑转弯处，满足最小转弯半径。
2. **覆盖效能计算**：
   - 通过栅格覆盖模拟，统计被覆盖的栅格数量，计算覆盖率。
   - 统计重叠栅格数量，计算重叠率。
   - 结合传感器探测概率模型，估算发现概率。
3. **输出结果**：生成每架无人机的航点序列，附带预计时间，输出评估报告。

### 3.6 大规模并行化策略（针对240架）
- **分组并行**：将无人机分组（如按区域），各组独立进行区域分解和路径规划，组间协调通过冲突检测模块处理。
- **GPU加速**：栅格覆盖模拟、代价计算等可并行操作使用CUDA加速。
- **分布式计算**：若硬件允许，可将不同区域分配到不同计算节点，最后汇总。

## 4. 性能需求（支持240架无人机）

| **需求标识**        | **描述** | **指标** |
|---------------------|----------|----------|
| FR-SEARCH-PERF-01   | 支持的最大反辐射无人机数量 | **240架** |
| FR-SEARCH-PERF-02   | 支持的最大搜索区域面积 | 10000 km²（可配置） |
| FR-SEARCH-PERF-03   | 搜索路径规划算法总求解时间（240架无人机，区域面积5000 km²，考虑基本威胁规避） | ≤180秒（采用并行计算，多核CPU或GPU加速） |
| FR-SEARCH-PERF-04   | 单架无人机路径规划平均时间（在并行环境下） | ≤0.8秒 |
| FR-SEARCH-PERF-05   | 区域分解与分配时间 | ≤30秒 |
| FR-SEARCH-PERF-06   | 多机协同冲突检测与解脱时间 | ≤40秒 |

```cpp
class RegionDecomposition {
public:
    // 根据无人机数量、性能、起始位置进行区域分解
    QVector<Polygon> decompose(const Polygon& searchArea, 
                               const QVector<UAVResource>& uavs) {
        // 1. 网格化搜索区域
        GridMap grid = discretizeArea(searchArea, cellSize);
        
        // 2. 基于无人机负载均衡的聚类
        // 采用K-means++初始化，考虑起始位置约束
        QVector<Cluster> clusters = kMeansWithConstraints(grid.cells, uavs);
        
        // 3. 边界优化，减少区域间耦合
        optimizeBoundaries(clusters);
        
        // 4. 生成各无人机的搜索子区域
        return extractPolygons(clusters);
    }
    
private:
    // 基于密度峰值聚类的改进算法 
    // 通过计算任务与簇中心的距离，实现高效任务分组
```

```cpp
class CoveragePathPlanner {
public:
    SearchPath planPath(const Polygon& region, const UAVResource& uav) {
        // 1. 确定扫描方向（考虑地形、风向）
        double scanAngle = determineOptimalDirection(region);
        
        // 2. 生成平行扫描线
        // 扫描线间距 = 传感器探测宽度 × 重叠系数（通常0.8~0.9）
        double swathWidth = uav.sensorSwath * overlapFactor;
        QVector<LineString> scanlines = generateScanlines(region, scanAngle, swathWidth);
        
        // 3. 连接扫描线形成连续路径（Boustrophedon模式）
        QVector<Point> pathPoints = connectScanlines(scanlines);
        
        // 4. 威胁规避与地形跟随调整
        pathPoints = avoidThreats(pathPoints, threatMap);
        pathPoints = terrainFollowing(pathPoints, dem);
        
        // 5. 路径平滑（满足最小转弯半径）
        pathPoints = smoothPath(pathPoints, uav.minTurnRadius);
        
        return SearchPath(pathPoints);
    }
    
private:
    // 基于改进人工势场法的局部避障 
    // 解决目标不可达问题，提高搜索压制效率
```

```cpp
class ConsensusController {
public:
    // 一阶一致性算法实现
    void adjustSpeeds(QVector<UAVState>& uavs, double targetTime) {
        // 通信拓扑关系建立
        Graph commGraph = buildCommunicationTopology(uavs);
        
        // 迭代调整速度直到收敛
        while (!converged) {
            for (auto& uav : uavs) {
                // 基于邻居状态更新速度
                double sum = 0;
                for (int neighbor : commGraph.neighbors(uav.id)) {
                    sum += (uavs[neighbor].eta - uav.eta);
                }
                uav.eta_dot = -alpha * sum;  // 一致性协议
            }
            // 更新速度
            integrate();
        }
    }
};
```

| **电子侦察** | 随机徘徊 | 信号稀疏区域 | 路径随机性、停留时间 |
| **信号跟踪** | 8字盘旋 | 跟踪移动信号 | 盘旋半径、周期 |
| **多机协同** | 分区并行 | 大面积快速搜索 | 分区数、交接区 |

### 3.2 搜索模式选择逻辑

```
if 任务类型 == "区域搜索":
    if 区域形状 == "矩形":
        选择 平行扫海
    elif 区域形状 == "圆形":
        选择 螺旋搜索
    else:
        选择 生成树覆盖

if 任务类型 == "目标确认":
    if 信号稳定度 == "高":
        选择 8字盘旋
    else:
        选择 螺旋搜索 + 逐步缩小

if 无人机数量 > 1:
    启用 多机协同策略
```

## 4. 核心算法体系

### 4.1 区域分解算法

| 算法 | 原理 | 适用场景 | 优势 |
|------|------|----------|------|
| **等面积分割** | 将区域按面积等分 | 均匀分布搜索 | 负载均衡 |
| **K-means聚类** | 基于概率密度聚类 | 非均匀分布目标 | 重点区域加强 |
| **Voronoi图划分** | 基于距离划分 | 多机协同搜索 | 边界清晰 |
| **概率加权分割** | 按先验概率分配 | 已知目标位置概率 | 效率最高 |

### 4.2 单机搜索路径算法

| 算法 | 原理 | 适用区域 | 转弯次数 | 覆盖均匀度 |
|------|------|----------|----------|------------|
| **平行扫海 (Boustrophedon)** | 来回平行扫描 | 矩形/多边形 | 中等 | 高 |
| **螺旋搜索** | 由内向外/由外向内螺旋 | 圆形/椭圆形 | 低 | 中等 |
| **生成树覆盖 (STC)** | 基于生成树的遍历 | 任意形状 | 高 | 高 |
| **栅格扫描** | 逐格遍历 | 小面积区域 | 高 | 非常高 |
| **随机搜索** | 马尔可夫随机游走 | 未知环境 | 不定 | 低 |

### 4.3 平行扫海算法流程

```
1. 确定扫描方向（通常沿区域长边）
2. 计算扫描线间距 = 传感器探测宽度 × (1 - 重叠率)
3. 生成平行扫描线集合
4. 连接相邻扫描线形成连续路径（之字形）
5. 优化路径起点（从最近点开始）
6. 检查与障碍物的交点，局部调整
```

### 4.4 螺旋搜索算法流程

```
1. 确定螺旋中心（目标概率最高点）
2. 计算初始半径 = 传感器探测宽度 / 2
3. 半径递增步长 = 传感器探测宽度 × 重叠系数
4. 生成螺旋线点集（阿基米德螺旋）
5. 平滑路径（满足转弯半径约束）
6. 根据信号强度动态调整螺旋密度
```

## 5. 多机协同搜索策略

### 5.1 协同模式

| 协同模式 | 实现方式 | 适用场景 | 优点 |
|----------|----------|----------|------|
| **分区搜索** | 区域分割，各机独立 | 大面积均匀搜索 | 简单高效 |
| **分扇区搜索** | 以目标点为中心分扇区 | 圆形区域 | 方向明确 |
| **重叠搜索** | 区域重叠覆盖 | 高价值区域 | 提高截获概率 |
| **接力搜索** | 依次搜索不同区域 | 续航有限 | 延长覆盖时间 |
| **分层搜索** | 不同高度层同时搜索 | 立体空间 | 提高效率 |

### 5.2 协同优化目标

```
总目标 = max(覆盖率) - w1 × 重复率 - w2 × 总时间 - w3 × 能耗
```

### 5.3 冲突避免机制

```
1. 时空冲突检测：检查各机路径是否存在同时同地
2. 优先级调度：高优先级任务优先
3. 动态调整：速度调整、局部绕行
4. 通信协同：实时共享位置信息
```

## 6. 环境适应性策略

### 6.1 地形影响处理

- **地形遮蔽**：在遮蔽区域降低搜索高度
- **山体阻挡**：绕行或分段搜索
- **谷地聚焦**：增加搜索密度

### 6.2 威胁规避

- **已知威胁**：路径完全避开威胁区
- **动态威胁**：实时更新威胁地图，局部重规划
- **低威胁区**：增加搜索密度补偿

### 6.3 气象影响

- **风速影响**：修正地速，调整搜索时间
- **能见度影响**：调整扫描间距（能见度差时缩小间距）
- **降水影响**：根据雨强调整探测概率模型

## 7. 搜索效能评估指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **区域覆盖率** | 已搜索面积 / 总面积 | ≥ 95% |
| **重复覆盖率** | 重复搜索面积 / 总面积 | ≤ 10% |
| **目标截获概率** | 探测到目标的概率 | ≥ 90% |
| **搜索时间** | 完成搜索所需时间 | ≤ 续航 × 40% |
| **能量消耗** | 搜索阶段能耗 | ≤ 总能耗 × 40% |

## 8. 动态重规划机制

### 8.1 触发条件

- **新目标发现**：转入目标跟踪/确认模式
- **目标消失**：扩大搜索范围
- **续航不足**：提前返航或请求接力
- **环境变化**：威胁出现/天气恶化

### 8.2 重规划流程

```
1. 保存当前搜索状态（已覆盖区域、剩余区域）
2. 根据新情况调整搜索策略
3. 局部重规划剩余区域路径
4. 与其他无人机协同调整
5. 无缝衔接当前路径
```
