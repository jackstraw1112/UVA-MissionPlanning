## 4. 分析流程

### 4.1 地理数据接入流程
1. **数据源识别**：根据文件扩展名或用户指定格式选择对应解析器。
2. **坐标系处理**：将数据统一转换到WGS84坐标系（若需要）。
3. **区域裁剪**：根据用户指定区域裁剪数据。
4. **重采样**：若用户指定分辨率，进行重采样。
5. **内存构建**：生成高程网格（二维数组），记录地理参考信息（原点、像元大小）。

### 4.2 障碍物处理流程
1. **矢量解析**：读取SHP/KML文件，提取几何要素。
2. **属性提取**：获取障碍物高度、类型等信息。
3. **栅格化**：将障碍物转换为高程网格上的附加高度信息，或保留为独立列表。
4. **影响范围计算**：根据障碍物高度和类型计算影响半径。

### 4.3 气象数据处理
- 手动输入直接存储为结构化数据。
- 文件导入需解析CSV/自定义格式，与时间关联。

```cpp
class GDALDatasetWrapper {
public:
    bool open(const QString& filePath) {
        GDALAllRegister();
        m_dataset = (GDALDataset*)GDALOpen(filePath.toLocal8Bit(), GA_ReadOnly);
        return m_dataset != nullptr;
    }
    
    QRectF getGeoExtent() {
        double adfGeoTransform[6];
        m_dataset->GetGeoTransform(adfGeoTransform);
        
        int width = m_dataset->GetRasterXSize();
        int height = m_dataset->GetRasterYSize();
        
        double minX = adfGeoTransform[0];
        double maxY = adfGeoTransform[3];
        double maxX = minX + width * adfGeoTransform[1];
        double minY = maxY + height * adfGeoTransform[5];
        
        return QRectF(minX, minY, maxX - minX, maxY - minY);
    }
    
    QVector<float> readElevationBand() {
        GDALRasterBand* band = m_dataset->GetRasterBand(1);
        int width = band->GetXSize();
        int height = band->GetYSize();
        
        QVector<float> data(width * height);
        band->RasterIO(GF_Read, 0, 0, width, height, data.data(), width, height,
                       GDT_Float32, 0, 0);
        return data;
    }
    
private:
    GDALDataset* m_dataset = nullptr;
};
```

#### 3.1.2 区域裁剪与重采样
```cpp
class TerrainManager {
public:
    bool loadTerrain(const QString& filePath, const QRectF& targetExtent, 
                     double targetResolution) {
        GDALDatasetWrapper ds;
        if (!ds.open(filePath)) return false;
        
        // 获取原始数据范围
        QRectF srcExtent = ds.getGeoExtent();
        
        // 计算目标网格大小
        int targetWidth = qRound(targetExtent.width() / targetResolution);
        int targetHeight = qRound(targetExtent.height() / targetResolution);
        
        // 使用GDAL的Warp API进行裁剪和重采样
        auto result = GDALWarp(targetWidth, targetHeight, targetExtent, ds);
        m_elevationData = result.buffer;
        m_geoTransform = result.geoTransform;
        
        return true;
    }
    
    double getElevation(double lon, double lat) {
        // 将地理坐标转换为网格坐标
        double col = (lon - m_geoTransform[0]) / m_geoTransform[1];
        double row = (lat - m_geoTransform[3]) / m_geoTransform[5];
        
        // 双线性插值
        return bilinearInterpolate(col, row);
    }
    
private:
    QVector<float> m_elevationData;
    double m_geoTransform[6];
    int m_width, m_height;
    
    double bilinearInterpolate(double col, double row) {
        int x1 = floor(col), x2 = x1 + 1;
        int y1 = floor(row), y2 = y1 + 1;
        
        // 边界处理
        if (x1 < 0 || x2 >= m_width || y1 < 0 || y2 >= m_height) {
            return m_elevationData[y1 * m_width + x1];
        }
        
        double q11 = m_elevationData[y1 * m_width + x1];
        double q12 = m_elevationData[y2 * m_width + x1];
        double q21 = m_elevationData[y1 * m_width + x2];
        double q22 = m_elevationData[y2 * m_width + x2];
        
        double dx = col - x1, dy = row - y1;
        
        double r1 = q11 * (1 - dx) + q21 * dx;
        double r2 = q12 * (1 - dx) + q22 * dx;
        
        return r1 * (1 - dy) + r2 * dy;
    }
};
```

### 3.2 障碍物处理

#### 3.2.1 矢量数据解析
```cpp
class ObstacleManager {
public:
    struct Obstacle {
        QString id;
        QString type;
        double lon, lat;
        double height;      // 障碍物高度
        double radius;      // 影响半径
        QPolygonF polygon;  // 面状障碍物轮廓
    };
    
    bool loadFromShapefile(const QString& filePath) {
        OGRDataSource* ds = OGRSFDriverRegistrar::Open(filePath.toLocal8Bit(), FALSE);
        OGRLayer* layer = ds->GetLayer(0);
        
        layer->ResetReading();
        OGRFeature* feature;
        while ((feature = layer->GetNextFeature()) != nullptr) {
            Obstacle obs;
            obs.id = QString::number(feature->GetFID());
            
            // 读取属性
            obs.type = feature->GetFieldAsString("type");
            obs.height = feature->GetFieldAsDouble("height");
            
            // 读取几何
            OGRGeometry* geom = feature->GetGeometryRef();
            if (wkbFlatten(geom->getGeometryType()) == wkbPoint) {
                OGRPoint* pt = (OGRPoint*)geom;
                obs.lon = pt->getX();
                obs.lat = pt->getY();
            } else if (wkbFlatten(geom->getGeometryType()) == wkbPolygon) {
                OGRPolygon* poly = (OGRPolygon*)geom;
                OGRLinearRing* ring = poly->getExteriorRing();
                for (int i = 0; i < ring->getNumPoints(); ++i) {
                    obs.polygon << QPointF(ring->getX(i), ring->getY(i));
                }
            }
            
            // 计算影响半径
            obs.radius = calculateInfluenceRadius(obs);
            
            m_obstacles.append(obs);
            OGRFeature::DestroyFeature(feature);
        }
        
        OGRDataSource::DestroyDataSource(ds);
        return true;
    }
    
private:
    QList<Obstacle> m_obstacles;
};
```

```cpp
void ObstacleManager::mergeWithTerrain(TerrainManager* terrain) {
    for (auto& obs : m_obstacles) {
        // 获取障碍物底部高程
        double baseHeight = terrain->getElevation(obs.lon, obs.lat);
        
        // 障碍物顶部高程 = 底部高程 + 障碍物高度
        obs.topHeight = baseHeight + obs.height;
        
        // 生成障碍物栅格掩码（用于路径规划）
        generateObstacleMask(obs, terrain);
    }
}
```

```cpp
class WindField {
public:
    virtual double getWindSpeed(double altitude) = 0;
    virtual double getWindDirection(double altitude) = 0;
};

class ConstantWind : public WindField {
public:
    double getWindSpeed(double altitude) override { return m_speed; }
    double getWindDirection(double altitude) override { return m_dir; }
};

class WindShear : public WindField {
public:
    double getWindSpeed(double altitude) override {
        // 风切变模型: v = v_ref * (altitude / alt_ref)^α
        return m_refSpeed * pow(altitude / m_refAlt, m_exponent);
    }
};

class Turbulence : public WindField {
public:
    double getWindSpeed(double altitude) override {
        // 在平均风上叠加随机扰动
        return m_baseSpeed + m_intensity * randomNormal();
    }
};
```

### 3.4 数据缓存与性能优化

```cpp
class TerrainCache {
public:
    float getElevation(double lon, double lat) {
        QMutexLocker locker(&m_mutex);
        
        // 将坐标离散化到缓存网格
        int x = floor((lon - m_minLon) / m_cellSize);
        int y = floor((lat - m_minLat) / m_cellSize);
        
        quint64 key = (quint64(x) << 32) | y;
        
        if (m_cache.contains(key)) {
            return m_cache[key];
        }
        
        // 从硬盘加载或计算
        float value = loadFromDisk(lon, lat);
        
        // LRU缓存管理
        if (m_cache.size() > MAX_CACHE_SIZE) {
            m_cache.remove(m_cache.keys().first());
        }
        m_cache.insert(key, value);
        
        return value;
    }
    
private:
    QHash<quint64, float> m_cache;
    QMutex m_mutex;
    double m_minLon, m_minLat, m_cellSize;
};
```

| **优化项**         | **实现方法**                                                | **效果**                     |

| 多线程加载         | 使用QtConcurrent并行读取多个瓦片                             | 加载速度提升3~5倍             |
| 延迟加载           | 只加载当前视口范围内的数据，动态扩展                         | 内存占用降低80%               |
| 数据压缩           | 将浮点高程转换为16位整数存储                                 | 内存减半                      |
| 缓存策略           | LRU缓存最近访问的区域                                       | 重复查询速度提升10倍          |
| 金字塔结构         | 构建多分辨率金字塔，根据缩放级别选择合适层级                 | 大幅提升渲染速度              |
| 障碍物空间索引     | 使用四叉树或网格索引快速查询                                 | 查询速度O(1) vs O(n)          |

## 6. 关键技术难点与解决方案

| **难点**                     | **解决方案**                                             |

## 3. 功能需求

### 3.1 地理环境数据接入（CM-TP-FUNC-01）
**描述**：模块应支持从本地文件或在线服务获取任务区域的地理环境数据，包含地形高程及障碍物分布。  
**细化需求**：
- **数据源选择**：用户可指定以下数据源之一：
  - 本地文件导入：支持格式 GeoTIFF、DTED（级别0/1/2）、ASCII Grid 用于地形；支持格式 SHP、KML、GeoJSON 用于障碍物。
  - 在线数据获取：支持从SRTM、ASTER GDEM等公开数据源下载指定地理矩形范围的数据。
  - 预设数据库：从本地缓存数据库中选取已导入的区域。
- **地形预处理**：
  - 重采样：支持设置采样间隔（1m~1000m，默认30m）。
  - 插值平滑：可选项，对缺失或噪声数据进行插值填充。
  - 基准转换：支持WGS84、EGM96等高程基准转换。
- **障碍物解析**：
  - 从矢量文件中提取点状或面状障碍物，计算其中心点坐标、高度、类型及影响半径。
  - 障碍物类型包括：建筑物、塔架、山脉、禁飞区等。
- **数据统计**：显示地形范围（最小/最大经纬度）、海拔区间（最小/最大高程）、平均坡度、障碍物总数等信息。

### 3.2 障碍物分布处理（CM-TP-FUNC-02）
**描述**：模块应允许用户对障碍物进行编辑和管理，并与地形融合生成综合环境模型。  
**细化需求**：
- **手动编辑**：支持增、删、改障碍物（ID、类型、经纬度、高度、半径）。
- **列表管理**：以表格形式展示所有障碍物，支持排序、筛选、导出。
- **融合计算**：将障碍物数据叠加到地形网格上，生成含障碍物高度场的复合地形，供后续航线规划使用。

### 3.3 气象环境设置（CM-TP-FUNC-03）
**描述**：模块应提供气象参数配置界面，支持手动输入和文件导入。  
**细化需求**：
- **基本气象参数**（手动输入）：
  - 温度（℃）：范围 -50～60，精度0.1。
  - 气压（hPa）：范围 800～1100，精度0.1。
  - 湿度（%）：范围 0～100，精度1。
  - 风速（m/s）：范围 0～50，精度0.1。
  - 风向（°）：范围 0～360，精度1。
  - 能见度（km）：范围 0～100，精度0.1。
- **降水/云层**：
  - 降水类型：无、小雨、中雨、大雨、雪。
  - 云底高度（m）：0～10000。
  - 云量（1～8）。
- **风场模型**：
  - 恒定风：风速风向恒定。
  - 风切变：输入风廓线指数（默认0.14），计算不同高度的风速。
  - 湍流模型：选择轻度/中度/重度湍流强度。
- **文件导入**：支持CSV格式气象数据文件，包含上述字段。
- **气象描述**：根据输入自动生成自然语言描述（如“东北风3级，能见度10km，多云”）。

### 3.4 环境数据与任务规划协同（CM-TP-FUNC-04）
**描述**：模块应将设置完成的环境数据封装为标准数据包，供任务规划模块调用。  
**细化需求**：
- **数据封装**：生成包含地形网格（二维浮点数组）、障碍物列表（结构体数组）、气象参数（结构体）的数据包。
- **接口方式**：
  - 共享内存（本机进程间通信）或网络Socket（分布式部署）。
  - 提供同步/异步两种调用模式，支持任务规划模块实时获取最新环境数据。
- **更新通知**：当用户修改环境参数后，自动向任务规划模块发送更新信号，并推送增量数据。

| **地理环境数据源** | 本地文件路径、在线数据源标识及地理范围、采样间隔、插值选项 | 无 | CM-TP-FUNC-01 |
| **地形高程数据** | 无 | 地形网格（二维数组，每个单元高程值）、地形统计信息 | CM-TP-FUNC-01 |
| **障碍物数据** | 矢量文件路径、手动输入的障碍物参数 | 障碍物列表（含ID、类型、坐标、高度、半径） | CM-TP-FUNC-02 |
| **气象参数** | 手动输入的气象数值、风场模型选择、CSV文件路径 | 气象参数结构体、气象文字描述 | CM-TP-FUNC-03 |
| **综合环境数据包** | 任务规划模块的请求指令 | 地形网格 + 障碍物列表 + 气象参数 | CM-TP-FUNC-04 |

## 5. 性能需求

### 5.1 地形数据处理性能（CM-TP-PERF-01）
- **加载时间**：对于1000km²范围（约30m分辨率，网格数约111万点），从本地文件加载至内存并完成预处理的时间≤30秒。
- **内存占用**：相同规模地形数据内存占用≤2GB。

### 5.2 障碍物处理性能（CM-TP-PERF-02）
- **容量**：支持不少于1000个障碍物的导入与显示。
- **响应时间**：界面增删改查操作响应时间≤2秒；障碍物与地形融合计算时间≤5秒。

### 5.3 气象参数响应性能（CM-TP-PERF-03）
- **参数更新**：气象参数修改后，风场计算及描述文本生成时间≤1秒。
- **实时性**：气象参数变化后，界面数值刷新≤0.5秒。

### 5.4 数据接口性能（CM-TP-PERF-04）
- **传输延迟**：综合环境数据包从生成到任务规划模块接收的延迟（共享内存）≤100ms；（网络通信）≤500ms。
- **并发支持**：支持同时为最多8架无人机规划任务的数据请求。

### 5.5 可靠性（CM-TP-REL-01）
- **异常处理**：输入数据格式错误、范围越界或文件损坏时，模块应给出明确错误提示，不崩溃，不产生错误结果。
- **数据持久化**：支持将当前环境配置保存为项目文件（XML/JSON），并能完整恢复。

### 5.6 精度要求
- **地形高程精度**：与原始数据源标称精度一致（如SRTM V3标称绝对高程精度≤16m）。
- **障碍物定位精度**：与导入源文件精度一致，界面显示保留小数点后6位（经纬度）。
- **气象参数精度**：满足输入控件设定的最小分辨率。

## 6. 接口需求

### 6.1 外部接口
- **用户界面接口**：基于Qt的图形界面，符合人机工程学要求。
- **文件系统接口**：读取/写入常见地理数据格式、CSV气象文件。
- **网络接口**：与在线数据服务（如OpenTopography）的HTTP/HTTPS通信。

### 6.2 内部接口
- **环境数据发布接口**：向任务规划核心模块提供数据订阅/推送服务，接口定义如下：
  - 数据结构 `TerrainGrid`：二维数组，每个元素为float型高程值。
  - 数据结构 `ObstacleList`：数组，每个元素包含 `id`（字符串），`type`（枚举），`lon`（double），`lat`（double），`height`（float），`radius`（float）。
  - 数据结构 `WeatherParams`：包含所有气象参数字段。
  - 函数 `getEnvironmentData()`：同步获取当前综合环境数据。
  - 信号 `environmentUpdated()`：环境数据变更时触发。
