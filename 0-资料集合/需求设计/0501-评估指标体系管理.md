## 4. 分析流程

### 4.1 指标体系结构设计
指标体系采用关系型数据结构，主要实体包括：
- **指标库**：存储所有指标定义，字段包括：`id`、`name`、`description`、`unit`、`formula`（字符串）、`weight`（浮点数）、`scenario_id`（适用场景，可为空）、`version`。
- **模板库**：存储模板信息，字段包括：`template_id`、`template_name`、`description`、`create_time`、`indicators`（关联指标ID列表）。
- **指标关系**：支持指标间的依赖，如一级指标由二级指标计算得出，可额外建立依赖表。

### 4.2 公式解析
系统需内置轻量级表达式解析器，支持：
- 算术运算：+、-、*、/、^（幂）、sqrt、log、sin、cos等。
- 引用其他指标：用指标ID或名称标识，如 `hit_rate = hits / launches`。
- 逻辑运算：if-else、and/or 等。
- 常量：PI、E等。

解析器需验证公式合法性，并能生成计算树用于后续评估。

### 4.3 权重管理
- 同一层级指标的权重之和应归一化（如总和为1或100%）。系统在保存时自动计算并提供警告/提示。
- 权重可影响最终综合评分的计算方式（如加权求和）。

### 4.4 模板管理
- 预置模板由系统管理员维护，普通用户可查看和基于模板创建新模板。
- 模板支持版本控制：每次修改保存为新版本，保留历史版本记录。
- 导出格式示例（JSON）：
```json
{
  "template_name": "反辐射攻击效能评估",
  "version": "1.0",
  "indicators": [
    {
      "id": "kill_prob",
      "name": "目标摧毁概率",
      "unit": "%",
      "formula": "kills / total_targets * 100",
      "weight": 0.4
    },
    {
      "id": "survival_rate",
      "name": "无人机生存率",
      "unit": "%",
      "formula": "survived_uavs / total_uavs * 100",
      "weight": 0.3
    }
  ]
}
```

### 4.5 用户交互流程
- 界面左侧显示模板列表，右侧显示选中模板的指标树。
- 指标树支持右键菜单（新增、编辑、删除、复制）。
- 双击指标弹出编辑对话框。
- 工具栏提供导入、导出、另存为按钮。

# 反辐射无人机仿真系统——评估指标体系管理模块开发实现方案

## 1. 总体架构

评估指标体系管理模块采用**模型-视图-控制器（MVC）**架构，后端基于C++/Qt，前端采用Qt Widgets实现管理界面。核心组件包括：

- **指标数据模型**：定义指标的结构、存储和关系。
- **指标公式解析器**：负责验证和解析用户定义的数学表达式。
- **模板管理器**：管理预置模板和用户自定义模板的增删改查、导入导出。
- **版本控制器**：维护指标和模板的版本历史。
- **持久化层**：使用SQLite或文件系统存储指标库和模板。

## 2. 数据结构设计

### 2.1 数据库表结构
使用SQLite作为本地存储，主要表如下：

```sql
-- 指标表
CREATE TABLE indicator (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,           -- 指标名称
    description TEXT,                     -- 描述
    unit TEXT,                            -- 单位（如 %、km、s）
    formula TEXT NOT NULL,                 -- 计算公式（字符串）
    weight REAL DEFAULT 0.0,                -- 权重（0~1）
    scenario_id INTEGER,                    -- 适用场景ID（可为空）
    version INTEGER DEFAULT 1,              -- 版本号
    created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    modified_time TIMESTAMP
);

-- 模板表
CREATE TABLE template (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,             -- 模板名称
    description TEXT,                       -- 描述
    version INTEGER DEFAULT 1,
    created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 模板-指标关联表（多对多）
CREATE TABLE template_indicator (
    template_id INTEGER,
    indicator_id INTEGER,
    weight_override REAL,                    -- 可覆盖指标权重
    FOREIGN KEY(template_id) REFERENCES template(id),
    FOREIGN KEY(indicator_id) REFERENCES indicator(id)
);

-- 场景表（可选）
CREATE TABLE scenario (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,               -- 场景名称（如“反辐射攻击”）
    description TEXT
);
```

### 2.2 内存数据结构
- `Indicator`类：封装指标属性，提供getter/setter，包含公式对象指针。
- `Template`类：包含模板基本信息和一个`QList<Indicator*>`（或存储指标ID）。
- `IndicatorFormula`类：解析后的公式表达式，支持求值。

## 3. 核心功能实现

### 3.1 指标定义与存储

**指标添加/编辑流程**：
1. 用户通过界面输入指标名称、描述、单位、公式、权重等。
2. 前端进行基础校验（非空、名称唯一）。
3. 调用后端`IndicatorManager::validateFormula(const QString& formula)`，使用公式解析器检查语法，返回错误信息或解析后的抽象语法树（AST）。
4. 若验证通过，创建`Indicator`对象，设置属性，调用`IndicatorManager::addIndicator(indicator)`插入数据库。
5. 数据库操作成功，刷新界面列表。

**指标删除**：
1. 检查指标是否被模板引用（通过查询`template_indicator`表）。
2. 若被引用，弹出提示框“指标正在使用中，是否强制删除？强制删除将同时从模板中移除该指标”。
3. 确认后，开启事务，删除关联记录，再删除指标记录。

### 3.2 公式解析器实现

采用**递归下降解析器**或集成**muParser**（C++数学表达式解析库）来验证和计算表达式。自定义解析器支持：
- 算术运算符：+, -, *, /, ^
- 函数：sqrt, log, log10, exp, sin, cos, tan, abs, max, min, if
- 引用其他指标：使用指标ID或名称，如 `[kill_count] / [total_targets] * 100`
- 常量：PI, E

**实现示例**（使用muParser）：
```cpp
#include "muParser.h"

bool IndicatorManager::validateFormula(const QString& formula, QString& errorMsg) {
    mu::Parser parser;
    try {
        // 定义可能出现的变量（指标引用），先预定义以便解析器识别
        parser.DefineVar("kill_count", &dummy);
        parser.DefineVar("total_targets", &dummy);
        // 也可以不预定义，通过检测未知变量来收集引用
        parser.SetExpr(formula.toStdString());
        parser.Eval(); // 尝试求值（变量未赋值会抛异常）
    } catch (mu::Parser::exception_type &e) {
        errorMsg = QString::fromStdString(e.GetMsg());
        return false;
    }
    return true;
}
```

**提取指标引用**：可通过解析器获取表达式中的变量名，用于建立指标间的依赖关系。

### 3.3 权重管理
- 权重在指标编辑时设置，类型为浮点数（0~1）。
- 在模板内，可覆盖单个指标的权重（`weight_override`字段）。
- 保存模板时，自动计算同层级指标权重和，若不为1则弹出警告，但允许保存（后续综合评分时可选择归一化）。

### 3.4 模板管理

**预置模板**：系统安装时通过SQL脚本插入`indicator`和`template`表，包含典型场景（如“反辐射攻击”、“协同压制”）的基础指标。

**模板操作**：
- 新建模板：用户指定名称、描述，可从现有指标库中勾选指标并设置权重。
- 基于模板创建：选择现有模板，复制其指标关联，生成新模板。
- 编辑模板：修改模板名称、描述，增加/移除指标，调整权重。
- 删除模板：检查模板是否被评估任务引用，若引用则提示。

**导入/导出**：
- 导出为JSON格式（见需求描述），包含模板信息和指标定义。
- 导入时解析JSON，检查指标是否已存在（按名称或ID），可选择覆盖或新建。

### 3.5 版本控制
每次修改指标定义（公式、权重等）时，`version`字段自增。模板修改也类似。历史版本可通过额外创建历史表存储，或直接更新当前记录并保留修改时间。

## 4. 用户界面设计

### 4.1 主界面布局
- 左侧树形视图：按场景分类显示模板，每个模板下可展开显示指标列表。

- 公式辅助按钮：可插入函数、运算符、引用其他指标的下拉列表。
- 验证按钮：实时验证公式，显示结果或错误。

### 4.3 模板编辑界面
- 模板名称、描述。
- 可用的指标列表（带复选框和权重输入框），支持搜索过滤。
- 保存后自动计算权重和提示。

### 4.4 导入/导出对话框
- 选择格式（JSON/XML），选择文件路径。
- 导入时显示预览，可选择冲突处理策略（覆盖/新建）。

## 5. 性能优化措施

- **索引**：对`indicator.name`、`template.name`建立唯一索引，加快查询。
- **缓存**：常用指标列表可缓存于内存，减少数据库访问。
- **延迟加载**：展开模板时才加载其关联的指标详情。
- **批量操作**：导入/导出时使用事务，加快速度。
