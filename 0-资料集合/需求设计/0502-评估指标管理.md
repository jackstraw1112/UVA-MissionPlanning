## 1. 功能需求描述

| **异常处理**     | 1. 循环依赖检测：若用户连线形成环路，弹出警告，禁止保存，并高亮环路上的连线。<br>2. 孤立节点提示：若存在未连接到顶层指标的节点，给出警告，但允许保存（可视为未使用指标）。<br>3. 权重校验：若父节点下子节点权重和不为1，提示用户调整。<br>4. 操作撤销/重做：支持Ctrl+Z/Y撤销误操作。 |
| **后置条件**     | 1. 构建的指标体系已保存到评估任务中，可供后续效能评估计算使用。<br>2. 画布状态被记录，下次打开可继续编辑。 |
| **显示约束**     | 1. 图形画布应支持缩放、平移。<br>2. 节点显示指标名称、权重等信息。<br>3. 连线应标注权重或聚合方式（可选）。<br>4. 提供节点和连线的选中、删除功能。<br>5. 提供缩略图导航，便于在大型指标体系内定位。 |
| **其他约束**     | 1. 指标体系层次深度不限，但应保证渲染性能。<br>2. 支持指标间的多种关系：父子、依赖（一个指标的计算需要另一个指标的值）。<br>3. 可导出指标体系为图片或文档格式（如PNG、PDF）。<br>4. 操作过程应有日志记录，便于追溯。 |

## 2. 关键术语解释

| **术语** | **解释** |
|----------|----------|
| **指标库** | 存储所有可用指标定义的中心库，包含指标的名称、描述、计算公式、单位、默认权重等信息。 |
| **层次结构** | 指标之间按照从宏观到微观的层级关系组织，例如顶层指标（综合效能）由多个二级指标（如摧毁概率、生存率）综合得出，二级指标又可细分为三级指标。 |
| **图形化界面** | 基于拖拽和连线的可视化编辑环境，用户通过直观的操作构建指标体系，无需手动编写代码或配置文本。 |
| **连线** | 在图形画布中，用线段连接两个节点，表示它们之间的关联关系（如父子、依赖）。通常箭头指向子节点或依赖节点。 |
| **关联关系** | 指标之间的内在联系，主要包括：<br>- 父子关系：父指标由子指标聚合计算得到（如加权求和）。<br>- 依赖关系：某个指标的计算需要引用另一个指标的值（如公式中引用）。 |

## 4. 分析流程

### 4.1 数据模型
构建的指标体系需存储为图结构。可采用JSON格式描述，例如：
```json
{
  "nodes": [
    { "id": 1, "indicatorId": "kill_prob", "name": "目标摧毁概率", "weight": 0.4, "x": 100, "y": 150 },
    { "id": 2, "indicatorId": "survival_rate", "name": "无人机生存率", "weight": 0.3, "x": 300, "y": 150 },
    { "id": 3, "name": "综合效能", "isCustom": true, "formula": "weightedSum", "x": 200, "y": 50 }
  ],
  "edges": [
    { "from": 3, "to": 1, "type": "parent-child", "weight": 0.4 },
    { "from": 3, "to": 2, "type": "parent-child", "weight": 0.3 }
  ]
}
```

### 4.2 图形交互实现
- 采用类似流程图编辑器的库（如GoJS、mxGraph、或Qt的Graphics View Framework）实现拖拽、连线。
- 节点显示：包含图标、名称、权重（可编辑）。
- 连线：箭头表示方向，鼠标悬停显示权重。
- 右键菜单：删除节点/连线、设置属性。

### 4.3 验证规则
- 无环检测：通过拓扑排序，确保图中无循环依赖。
- 权重和：对每个父节点，其所有出边（指向子节点）的权重之和应接近1（允许浮点误差）。
- 节点完整性：所有节点均可达（即存在从顶层到底层的路径，除非是孤立节点）。

### 4.4 与指标库的关联
- 节点可以是直接引用指标库中的指标（继承公式、单位），也可以是用户自定义的复合指标（无公式，需由子节点计算）。
- 当指标库中的指标更新时，应提示用户是否需要同步到已有指标体系。

### 4.5 保存与加载
- 保存时，将图形结构转换为上述JSON格式，存入数据库。
- 加载时，根据JSON重建节点和连线，恢复画布布局。

# 反辐射无人机仿真系统——评估指标管理模块开发实现方案

## 1. 总体架构

评估指标管理模块采用**MVC架构**，前端基于Qt的图形视图框架（Graphics View Framework）实现可视化编辑，后端使用C++处理数据逻辑和持久化。核心组件包括：

- **指标库数据模型**：管理指标定义，提供查询接口。
- **指标体系图模型**：表示用户构建的层次化指标体系（节点+边）。
- **图形视图组件**：`QGraphicsScene` + `QGraphicsView` 实现拖拽、连线、编辑。
- **验证引擎**：实时检测循环依赖、权重和等问题。
- **序列化模块**：将指标体系保存为JSON或XML，存入数据库。

## 2. 数据结构设计

### 2.1 指标体系图模型
使用自定义的图结构存储节点和边，便于拓扑分析和持久化。

```cpp
// 节点类（指标节点）
class IndicatorNode {
public:
    int id;                     // 节点唯一标识

class IndicatorEdge {
public:

// 指标体系类
class IndicatorGraph {
public:
    QMap<int, IndicatorNode> nodes;

    // 拓扑排序、环路检测等方法
    bool hasCycle();
    QList<int> getRootNodes();  // 获取顶层节点
    // ...
```

## 3. 图形界面实现

采用Qt的**Graphics View Framework**，提供流畅的拖拽和连线体验。

### 3.1 主要类设计
- **`IndicatorGraphicsScene`**：继承`QGraphicsScene`，管理所有节点和连线图元。
- **`IndicatorNodeItem`**：继承`QGraphicsItem`，表示一个指标节点，包含名称、权重显示，支持拖拽移动。
- **`IndicatorEdgeItem`**：继承`QGraphicsLineItem`，表示节点间的连线，支持箭头绘制和悬停显示权重。
- **`IndicatorLibraryWidget`**：右侧边栏，显示指标库列表，支持拖拽到场景。

### 3.2 交互实现

**拖拽添加节点**：
- 用户从指标库列表中按住鼠标左键拖拽，释放到场景时，创建`IndicatorNodeItem`，并弹出属性对话框设置初始权重等。

**连线操作**：
- 节点提供连接点（锚点），用户从一个节点的锚点拖拽到另一节点，释放后创建`IndicatorEdgeItem`。
- 连线时可弹出菜单选择关系类型（父子/依赖），并输入权重。

**节点编辑**：
- 双击节点弹出属性编辑对话框，可修改名称、权重、公式等。

**自动布局**：
- 实现力导向布局算法或层次布局算法，一键排列节点，使结构清晰。

### 3.3 视觉样式
- 节点：圆角矩形，显示指标名称和权重（如“目标摧毁概率 (0.4)”）。
- 连线：箭头指向子节点，线上标注权重。
- 颜色区分：顶层节点、普通节点、叶子节点使用不同背景色。

## 4. 验证引擎

### 4.1 循环依赖检测
使用拓扑排序算法，每次添加/删除边时触发检测：

```cpp
bool IndicatorGraph::hasCycle() {
    QMap<int, int> inDegree;
    QMap<int, QList<int>> adj;
    // 构建邻接表（仅考虑parent-child关系，因为依赖关系应允许循环？通常不允许）
    for (auto& edge : edges) {
        if (edge.type == "parent-child") {
            adj[edge.from].append(edge.to);
            inDegree[edge.to]++;
        }
    }
    QQueue<int> q;
    for (auto id : nodes.keys()) {
        if (inDegree[id] == 0) q.enqueue(id);
    }
    int visited = 0;
    while (!q.isEmpty()) {
        int u = q.dequeue();
        visited++;
        for (int v : adj[u]) {
            if (--inDegree[v] == 0) q.enqueue(v);
        }
    }
    return visited != nodes.size();
}
```

### 4.2 权重和校验
遍历所有父节点，计算其子节点权重和：

```cpp
void validateWeights(const IndicatorGraph& graph) {
    QMap<int, double> sumWeight;
    for (auto& edge : graph.edges) {
        if (edge.type == "parent-child") {
            sumWeight[edge.from] += edge.weight;
        }
    }
    for (auto it = sumWeight.begin(); it != sumWeight.end(); ++it) {
        if (abs(it.value() - 1.0) > 1e-6) {
            // 发出警告，高亮父节点
        }
    }
}
```

### 4.3 实时反馈
在用户操作时（如调整权重、新增连线），立即调用验证函数，若发现问题则在界面右下角显示警告列表，并高亮相关节点/连线。

## 6. 性能优化措施

- **懒加载**：当节点数超过50时，画布绘制可采用LOD（细节层次），远距离简化显示。
- **增量验证**：只对受影响的子图进行验证，而非全图扫描。
- **多线程**：自动布局算法可在后台线程运行，避免界面卡顿。
- **缓存**：指标库数据缓存在内存中，减少数据库查询。
