# 反辐射无人机仿真系统——仿真推演模块作战行动模拟功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 作战行动模拟 |
| **需求标识**     | FR-SIM-04 |
| **需求概述**     | 支持对反辐射无人机、雷达、防空系统、作战飞机（群）、弹道导弹等实体作战行动的模拟，包括实体的运动、探测、决策、交战、毁伤等行为，能够反映各实体在战场环境下的动态交互过程，为反辐射无人机协同作战仿真提供真实可信的实体行为表现。 |

| **触发条件**     | 1. 用户点击“开始仿真”按钮。<br>2. 仿真过程中根据时间步或事件驱动实体行为。 |
| **退出条件**     | 1. 仿真达到预设结束时间或任务完成。<br>2. 用户手动停止仿真。<br>3. 仿真过程中发生不可恢复错误。 |
| **功能参与者**   | 仿真引擎（驱动实体行为更新）、模型管理器（提供实体模型实例）、用户（监控、干预）、数据记录模块 |

| **异常处理**     | 1. 实体模型缺失或规则定义错误：跳过该实体并记录警告。<br>2. 实体行为冲突（如同时发射武器导致资源超限）：按优先级或预设规则解决。<br>3. 数值计算溢出或非法状态：终止该实体行为并记录错误。 |

| **显示约束**     | 1. 在二维/三维态势图中实时显示实体位置、轨迹、状态（如颜色表示敌我、模式）。<br>2. 支持事件日志的可视化列表显示。<br>3. 可选中实体查看详细属性（速度、高度、剩余弹药等）。 |
| **其他约束**     | 1. 实体行为模型应具备可扩展性，便于添加新的实体类型或行为规则。<br>2. 实体间的交互应考虑电磁环境、气象等外部因素的影响。<br>3. 支持用户干预（如手动调整实体状态、发射指令）以进行“人在回路”仿真。 |

## 3. 分析流程

### 3.1 关键术语解释
| **术语** | **解释** |
|----------|----------|
| **反辐射无人机** | 专门用于探测和攻击敌方雷达辐射源的无人机，通常配备被动雷达导引头，可在探测到雷达信号后自动飞向目标并摧毁之。 |
| **雷达** | 利用电磁波探测目标并测定其位置、速度等信息的设备，在仿真中包括预警雷达、火控雷达、制导雷达等，具有不同的探测范围、精度和抗干扰能力。 |
| **防空系统** | 由雷达、指挥控制和地空导弹/高炮组成的综合防御系统，用于拦截来袭的空中目标，如飞机、无人机、导弹等。 |
| **作战飞机（群）** | 执行空中作战任务的固定翼飞机，可单独或编队行动，进行空战、对地攻击、电子战等任务。 |
| **弹道导弹** | 按照预定弹道飞行，主要在大气层外或大气层内高超声速飞行的导弹，通常用于对地攻击。其仿真需考虑助推段、中段、末段的不同特性。 |

### 3.2 实体行为模拟方法
- **运动模拟**：基于动力学/运动学方程更新位置、速度、姿态。飞机和无人机采用六自由度或三自由度模型；导弹可采用比例导引等制导律；弹道导弹需考虑地球引力、空气动力学。
- **探测模拟**：根据传感器类型（雷达、红外、电子支援措施）建模，考虑探测概率、虚警、干扰影响。如雷达探测需计算信噪比、烧穿距离。
- **决策模拟**：采用有限状态机、行为树或规则引擎实现实体智能。例如，防空系统根据目标类型、距离、威胁等级决定是否发射导弹；反辐射无人机根据雷达信号选择攻击目标。
- **交战模拟**：包括武器发射、飞行、命中判定、毁伤计算。导弹飞行可简化为直线或比例导引；命中判定基于碰撞检测或概率模型；毁伤计算根据战斗部威力、目标易损性。
- **交互处理**：维护实体间的感知关系（如雷达探测链）、攻击关系（如导弹-目标配对），并实时更新。

### 3.3 实体间交互逻辑
- **反辐射无人机 vs 雷达**：无人机探测到雷达信号后，可进入攻击模式，沿信号方向飞行；雷达可能发现无人机并启动防空系统拦截。
- **防空系统 vs 反辐射无人机/作战飞机**：防空雷达探测目标，指挥所分配火力，发射导弹；导弹拦截目标，可能命中或脱靶。
- **作战飞机 vs 防空系统**：飞机可发射反辐射导弹压制雷达，或投放诱饵干扰；防空系统还击。
- **弹道导弹 vs 防空系统**：弹道导弹高速飞行，可能触发预警雷达，防空系统尝试拦截（但难度大）。
- **电子对抗**：干扰机对雷达实施压制，降低其探测能力；雷达可能采用跳频、反干扰措施。

### 3.4 数据流
- 输入：实体初始状态、行为规则、环境。
- 处理：每个时间步更新所有实体状态，检测交互，产生事件。
- 输出：状态数据、事件日志。

# 实现思路
# 反辐射无人机仿真系统——作战行动模拟模块开发实现方案

## 1. 总体技术架构

作战行动模拟模块采用**组件化、事件驱动**的架构，将实体的运动、探测、决策、交战等功能解耦为独立的组件，通过实体管理器统一调度。核心模块包括：

- **实体管理器**：维护所有仿真实体，提供快速查找（空间索引）。
- **组件系统**：每个实体由多个组件组成（如运动组件、传感器组件、决策组件、武器组件）。
- **事件总线**：处理实体间的交互事件（如探测事件、武器发射事件、命中事件）。
- **时间同步器**：管理仿真时钟，支持可变步长。

**技术选型**：
- 开发语言：C++17（性能关键），结合Qt（线程管理、事件系统）。
- 数学库：Eigen（向量、矩阵运算）。
- 并行计算：OpenMP（实体更新并行化）。
- 空间索引：四叉树/网格法（快速邻近查询）。

## 2. 各实体类型实现方法

### 2.1 反辐射无人机
**运动模型**：
- 采用六自由度（6DOF）或三自由度（3DOF）动力学模型，根据气动系数计算加速度。
- 支持预设航路点跟踪、比例导引、盘旋等机动模式。
- 代码示例：
```cpp
class UAVComponent : public IComponent {
    void update(double dt) override {
        // 根据当前控制指令计算加速度
        Vector3 acceleration = computeAcceleration(m_state.velocity, m_control);
        // 欧拉积分更新速度和位置
        m_state.velocity += acceleration * dt;
        m_state.position += m_state.velocity * dt;
    }
};
```

**探测模型**：
- 被动雷达导引头：接收目标雷达信号，计算信号强度（根据距离、雷达功率、天线增益）。
- 只有当信号强度超过灵敏度阈值时，才判定探测到目标，并获取目标方位。
- 考虑地形遮蔽（利用射线与地形网格碰撞检测）。

**决策模型**：
- 采用有限状态机（FSM）：搜索 → 跟踪 → 攻击 → 规避。
- 状态转移条件基于探测结果、任务规划、自身状态（如剩余弹药）。
- 行为树（Behavior Tree）也可用于更复杂的协同逻辑。

**武器系统**：
- 携带反辐射导弹或战斗部。导弹发射后作为独立实体加入仿真，其运动采用比例导引律。
- 引信模型：支持触发引信、近炸引信、延时引信，根据目标距离和相对速度判定引爆。

### 2.2 雷达
**探测模型**：
- 基于雷达方程计算信噪比（SNR），考虑目标RCS、距离、雷达功率、大气衰减、干扰等。
- 探测概率采用Swerling模型或经验公式：`P_d = f(SNR, P_fa)`。
- 若存在电子干扰，根据干扰功率计算烧穿距离。

**工作模式**：
- 支持搜索模式（360度扫描）、跟踪模式（窄波束跟踪特定目标）、静默模式。

**实现**：
```cpp
class RadarComponent : public IComponent {
    void update(double dt) override {
        // 获取当前扫描范围内的所有潜在目标
        auto targets = m_entityManager->getEntitiesInCone(m_owner->position(), m_azimuth, m_elevation, m_range);
        for (auto target : targets) {
            double snr = calculateSNR(target);
            double pd = detectionProbability(snr);
            if (random() < pd) {
                // 触发探测事件
                Event event(EventType::DETECTED, m_owner, target);
                m_eventBus->publish(event);
            }
        }
    }
};
```

### 2.3 防空系统
**组成**：
- 搜索雷达、火控雷达、发射单元、指挥控制单元。

**决策逻辑**：
- 搜索雷达探测到目标后，指挥单元评估威胁等级，分配火控雷达跟踪，并选择发射单元。
- 发射导弹后，导弹作为独立实体飞行，火控雷达提供中段制导或末端照射。

**导弹模型**：
- 采用比例导引法（PN）或增强比例导引法（APN）进行制导。
- 考虑导弹的最大过载、动力段/无动力段。

### 2.4 作战飞机（群）
**编队飞行**：
- 采用领航-僚机模式：领航飞机按规划路径飞行，僚机通过编队保持算法（如基于相对位置的PID控制）维持队形。
- 支持编队解散/重组。

**空战决策**：
- 基于态势评估（目标距离、角度、速度）选择机动动作（如高G转弯、发射导弹、投放诱饵）。
- 可采用专家系统或强化学习模型（预训练）。

**对地攻击**：
- 根据任务规划投掷炸弹或发射导弹，考虑投放精度（圆概率误差CEP）。

### 2.5 弹道导弹
**弹道计算**：
- 将弹道分为助推段、中段（大气层外）、末段（再入段）。
- 助推段：按推力曲线积分；中段：仅受地球引力，可近似为椭圆轨道；末段：考虑气动加热、控制舵面影响。

**制导**：
- 惯性制导 + 末端寻的（可选）。

**突防与拦截**：
- 可能释放诱饵或进行机动突防；防空系统拦截需计算拦截窗口。

## 3. 实体间交互处理

采用**事件驱动**机制，避免每帧全量碰撞检测：

- **探测事件**：由传感器组件产生，经事件总线广播，目标实体接收后可改变状态（如被跟踪）。
- **武器发射事件**：发射实体创建导弹实体，加入仿真。
- **命中事件**：导弹与目标距离小于阈值时触发，调用毁伤模型计算毁伤效果。
- **干扰事件**：干扰机发射干扰信号，影响雷达探测。

事件处理优先级：高优先级事件（如命中）先处理，可能触发连锁反应（如目标被摧毁，解除其威胁）。

## 4. 并行化与性能优化

针对240架无人机及其他实体（总数≥500），采用以下优化：

- **实体分组并行**：将实体按空间区域划分，每个区域由独立线程更新（OpenMP）。
- **空间索引**：使用网格法（如100x100网格）加速邻近查询，避免O(N²)复杂度。
- **LOD（细节层次）**：对远距离实体简化模型（如用质点代替六自由度），降低计算量。
- **事件队列**：采用无锁队列（moodycamel::ConcurrentQueue）处理事件，减少锁竞争。
- **内存池**：预分配实体和组件内存，避免频繁new/delete。

## 5. 数据记录与回放

- **状态记录**：每个时间步记录实体关键属性（位置、速度、模式），采用二进制格式（如Cap'n Proto）减少体积。
- **事件记录**：所有事件以文本形式存入日志，便于事后分析。
- **回放支持**：记录仿真初始状态和随机种子，可完全复现。

## 6. 关键代码结构

```cpp
// 实体组件基类
class IComponent {
public:
    virtual void update(double dt) = 0;
    virtual ~IComponent() = default;
};

// 实体类
class Entity {
public:
    void addComponent(std::unique_ptr<IComponent> comp);
    void update(double dt) {
        for (auto& comp : m_components) {
            comp->update(dt);
        }
    }
private:
    std::vector<std::unique_ptr<IComponent>> m_components;
};

// 实体管理器
class EntityManager {
public:
    void updateEntities(double dt) {
        #pragma omp parallel for
        for (int i = 0; i < m_entities.size(); ++i) {
            m_entities[i]->update(dt);
        }
        processEvents();
    }
private:
    std::vector<Entity*> m_entities;
    SpatialIndex m_spatialIndex;
};
```
