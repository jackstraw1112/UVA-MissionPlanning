# 反辐射无人机仿真系统——仿真推演模块态势回放功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 态势回放 |
| **需求标识**     | FR-SIM-09 |
| **需求概述**     | 具备态势回放功能，能够按照仿真时间顺序重现作战过程中的关键态势和事件，为用户提供事后分析和复盘能力，支持对作战过程的回顾、研究和评估。 |
| **需求来源**     | 系统总体设计要求 / 仿真推演模块能力指标 |
| **前置条件**     | 1. 仿真过程已完成，并已记录完整的态势数据和事件日志，数据文件可访问。<br>2. 三维/二维态势显示模块已初始化，能够正常渲染场景。<br>3. 用户已加载待回放的仿真数据文件。 |
| **触发条件**     | 1. 用户在仿真结束后点击“态势回放”按钮进入回放模式。<br>2. 用户在回放控制界面点击“开始回放”按钮启动回放。<br>3. 用户选择特定时间段或通过进度条跳转触发回放。 |
| **退出条件**     | 1. 回放进度到达设定的结束时间。<br>2. 用户点击“停止回放”按钮。<br>3. 用户关闭回放窗口或切换至其他功能模块。 |
| **功能参与者**   | 用户（观察者）、回放控制模块、数据加载模块、三维渲染引擎 |
| **输入**         | 回放数据文件（包含实体状态序列、事件日志、环境数据等）、回放控制参数（起始时间、结束时间、播放速度、循环模式、跳转时间等） |
| **输出**         | 三维场景中随时间动态变化的实体状态（位置、姿态、状态）、事件标注（文字提示、高亮、特效）、时间进度显示 |
| **处理流程**     | 1. **数据加载**：读取回放数据文件，解析实体状态序列和事件日志，建立时间索引。<br>2. **初始化**：根据用户设定的起始时间和结束时间，初始化回放时间轴，将当前时间设为起始时间。<br>3. **主循环**：按设定的播放速度逐帧推进回放时间，每帧执行：<br>   - 从数据中获取当前时间前后的两个状态点，通过线性插值计算实体当前时刻的位置、姿态等。<br>   - 更新三维场景中对应实体的模型变换和状态显示（如颜色、标签）。<br>   - 检查事件列表，若当前时间到达或超过事件时间戳，则在场景中显示事件标注（如闪烁、文字框、特效）。<br>4. **交互响应**：实时响应用户的暂停、继续、快进、快退、调速、拖拽进度条等操作，调整回放状态和时间。<br>5. **结束处理**：回放到达结束时间后停止更新，或根据循环模式重新开始；用户可重新设置时间段再次回放。 |
| **成功判据**     | 1. 回放过程中实体运动平滑连续，无跳变或卡顿。<br>2. 事件标注与仿真日志时间对应准确，无遗漏或错位。<br>3. 用户可灵活控制回放速度、暂停/继续、跳转时间，操作响应及时。<br>4. 加载不同时间段的数据正确，场景状态与原始仿真一致。 |
| **异常处理**     | 1. 数据文件损坏、格式错误或版本不匹配：提示用户“无法加载回放数据”，并记录错误日志。<br>2. 用户指定的时间点超出数据范围：自动调整到有效边界，并提示用户。<br>3. 回放过程中性能不足导致帧率下降：自动降低特效质量或提示用户降低播放速度。<br>4. 事件数据缺失：忽略该事件，继续回放，并记录警告。 |
| **后置条件**     | 1. 回放结束后，三维场景停留在最后时刻的状态。<br>2. 用户可重新选择时间段或加载其他回放文件进行新的回放。<br>3. 回放控制界面恢复初始状态。 |
| **显示约束**     | 1. 应显示当前回放时间/总时长进度条，并支持拖拽跳转。<br>2. 事件发生时应在场景中有时标注记（如闪烁的光点、文字框），并可点击查看详情。<br>3. 支持切换显示实体轨迹、标签等辅助信息。<br>4. 播放速度应在界面上明确显示（如1x、2x等）。 |
| **其他约束**     | 1. 回放速度应支持多级调节，建议包括：0.5x、1x、2x、4x、8x、16x、32x等。<br>2. 支持单步前进/后退（按帧或按事件）。<br>3. 数据记录应包含足够的时间分辨率（建议至少每秒10帧），以支持平滑回放。<br>4. 回放数据文件应采用压缩格式存储，减少存储空间和I/O开销。 |

## 4. 分析流程

### 4.1 数据组织
回放数据文件需要高效组织，以便快速随机访问任意时间点的数据。建议采用以下结构：
- **文件头**：包含版本、实体数量、时间范围、索引偏移等信息。
- **时间索引**：记录每个时间点（如每0.1秒）在文件中的偏移量，便于跳转。
- **实体状态块**：按时间顺序存储所有实体的状态（位置、姿态、模式等），可采用增量压缩或固定间隔采样。
- **事件块**：存储事件列表，每条事件包含时间戳、类型、参数（如关联实体ID）。

### 4.2 插值算法
为保证回放平滑，对于连续运动（如位置），采用线性插值；对于离散状态（如模式切换），在事件发生时刻直接跳变。

### 4.3 交互设计
- **时间轴**：提供可拖拽的进度条，实时显示当前时间/总时长。
- **速度控制**：下拉菜单或按钮循环切换倍率。
- **事件列表**：可弹出事件列表窗口，点击事件跳转到对应时间。

### 4.4 性能优化
- **预加载**：将时间索引和关键帧加载到内存，实际状态数据按需读取。
- **LOD**：远距离实体可简化更新频率或使用较低精度的模型。
- **多线程**：数据读取和解析在后台线程进行，渲染线程专注绘制。

# 反辐射无人机仿真系统——态势回放模块开发实现方案

## 1. 总体架构

态势回放模块采用**数据驱动 + 时间轴控制**的架构，与三维渲染引擎紧密集成。核心组件包括：

- **回放数据管理器**：负责读取、解析、索引仿真记录文件。
- **时间轴控制器**：管理回放时间、播放速度、暂停/继续、跳转等。
- **状态插值器**：根据当前时间计算实体的平滑状态。
- **事件调度器**：按时间触发事件标注。
- **渲染适配器**：将实体状态更新和事件传递给三维渲染引擎。

```plaintext
┌─────────────────────────────────────────────────────┐
│                   回放数据文件                        │
│  (压缩的二进制/HDF5格式，包含实体状态序列+事件日志)   │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│                回放数据管理器                         │
│  - 解析文件头，构建时间索引                            │
│  - 按需读取状态块（分页/预取）                         │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│                  时间轴控制器                          │
│  - 维护当前回放时间 t                                 │
│  - 处理用户指令（开始、暂停、调速、跳转）               │

└─────────────────────────────────────────────────────┘
                           ↓

│                 状态插值器                             │
│  - 获取 t 前后的两个状态点                             │
```

## 2. 数据记录与存储格式

为实现高效回放，仿真引擎需要以紧凑格式记录数据。推荐采用 **HDF5** 或 **自定义二进制** 格式。

### 2.1 文件结构设计
```
文件头 (Header)
├─ 版本号 (4 bytes)
├─ 实体数量 (4 bytes)
├─ 实体类型表 (变长)
├─ 时间范围 (开始时间, 结束时间)
├─ 时间索引偏移量 (指向索引块)
├─ 事件块偏移量
└─ 状态块偏移量

时间索引块
├─ 时间点列表 (例如每秒10个采样点)
└─ 每个时间点在状态块中的位置 (偏移量)

状态块 (按时间顺序存储)
├─ 时间戳
├─ 实体1的状态 (位置 x,y,z, 姿态四元数, 模式)
├─ 实体2的状态 ...
└─ ... (可采用差分编码压缩)

事件块
├─ 事件1: 时间戳, 类型, 关联实体ID, 参数
├─ 事件2 ...
└─ ...
```

### 2.2 压缩策略
- **状态增量编码**：只存储相对于上一帧的变化（对于连续量可使用Delta编码）。
- **量化**：将浮点数转换为半精度（2字节）或定点数。
- **通用压缩**：使用zstd或LZ4对块进行压缩，减少存储空间。

### 2.3 示例：二进制状态记录（简化）
```
struct StateRecord {
    double timestamp;
    float pos[3];          // 世界坐标 (经纬度转为笛卡尔)
    float quat[4];         // 姿态四元数
    uint8_t mode;          // 工作模式 (枚举)
    uint8_t damageLevel;   // 毁伤等级 0-255
};
```

## 3. 数据加载与索引

### 3.1 建立时间索引
读取文件时，首先解析文件头和时间索引块，构建内存中的索引表：

```javascript
class ReplayData {
    constructor(file) {
        this.index = []; // { time: 123.4, offset: 0x1234 }
        // 读取索引块填充this.index
    }
    
    // 根据时间t找到前后两个状态点
    findStates(t) {
        // 二分查找最接近t的索引位置
        let idx = binarySearch(this.index, t);
        let prevOffset = this.index[idx].offset;
        let nextOffset = this.index[idx+1].offset;
        // 从文件中读取两个状态块
        let prevState = this.readStateBlock(prevOffset);
        let nextState = this.readStateBlock(nextOffset);
        return { prevState, nextState };
    }
}
```

### 3.2 分页与预取
为避免频繁磁盘I/O，采用双缓冲区和预取策略：
- 维护当前时间前后若干秒的状态块在内存中。
- 当时间接近缓冲区边缘时，后台线程异步加载下一批数据。

## 4. 回放控制逻辑

### 4.1 时间轴控制器
```javascript
class TimelineController {
    constructor() {
        this.currentTime = 0;      // 当前回放时间 (秒)
        this.totalDuration = 3600; // 总时长
        this.speed = 1.0;          // 播放速度倍率
        this.paused = true;
        this.loop = false;
    }
    
    // 每帧更新 (由渲染循环调用)
    update(deltaRealTime) {
        if (this.paused) return;
        // 计算应推进的仿真时间
        let deltaSim = deltaRealTime * this.speed;
        this.currentTime += deltaSim;
        if (this.currentTime > this.totalDuration) {
            if (this.loop) {
                this.currentTime = 0;
            } else {
                this.currentTime = this.totalDuration;
                this.paused = true;
            }
        }
    }
    
    // 跳转到指定时间
    seek(targetTime) {
        this.currentTime = Math.max(0, Math.min(targetTime, this.totalDuration));
        // 触发数据重读
        this.onSeek(this.currentTime);
    }
}
```

### 4.2 播放速度控制
支持多级倍率：0.5x、1x、2x、4x、8x、16x、32x等。在高速播放时，可减少可视化更新频率（如每2帧更新一次模型位置）以保证帧率。

## 5. 状态插值与同步

### 5.1 线性插值算法
对于位置和姿态（四元数），分别采用线性插值和球面线性插值（SLERP）：

```javascript
function interpolateState(prev, next, t) {
    // t 是相对于prev的时间比例 (0~1)
    let result = {};
    result.position = new THREE.Vector3().lerpVectors(prev.position, next.position, t);
    result.quaternion = new THREE.Quaternion().copy(prev.quaternion).slerp(next.quaternion, t);
    // 离散状态（如mode）直接使用prev或根据事件决定
    result.mode = (t < 0.5) ? prev.mode : next.mode;
    return result;
}
```

### 5.2 处理离散事件
对于模式切换等离散状态，需要在事件触发时刻瞬间改变，而非插值。事件调度器负责在对应时间触发更新。

## 6. 事件标注

### 6.1 事件数据结构
```javascript
class ReplayEvent {
    constructor(time, type, entityId, params) {
        this.time = time;      // 发生时间
        this.type = type;      // "fire", "hit", "radar_on" 等
        this.entityId = entityId;
        this.params = params;  // 附加信息 {position, target, ...}
    }
}
```

### 6.2 事件调度
- 每帧检查当前时间前后的事件列表，若事件时间在当前帧范围内，则触发标注。
- 标注表现形式：
  - **文字提示**：在事件位置显示Billboard，显示事件描述。
  - **闪烁特效**：关联实体高亮闪烁。
  - **粒子效果**：如爆炸、烟雾等。
  - **轨迹标记**：在事件发生点留下标记点。

```javascript
function checkEvents(currentTime, lastTime, events) {
    events.forEach(event => {
        if (event.time >= lastTime && event.time <= currentTime) {
            triggerEventEffect(event);
        }
    });
}
```

### 6.3 事件列表界面
提供一个侧边栏，按时间顺序列出所有事件，点击可跳转到对应时间。

## 7. 交互设计

### 7.1 用户界面
- **进度条**：显示当前时间/总时长，支持拖拽跳转。
- **播放控制**：播放/暂停、加速、减速、单步前进/后退。
- **速度显示**：当前倍率。
- **时间显示**：当前时间 (HH:MM:SS.ms)。
- **事件列表**：可折叠面板，点击事件跳转。

### 7.2 快捷键支持
- 空格键：暂停/继续
- 方向键左/右：单步后退/前进（按帧）
- Ctrl + 方向键：跳转到上一个/下一个事件

## 8. 性能优化

| **优化项** | **实现方法** | **效果** |
|------------|--------------|----------|
| 数据分页 | 只加载当前时间附近的数据块 | 内存占用降低80% |
| 插值计算并行化 | 使用Web Worker或SIMD | 计算时间减少50% |
| LOD更新 | 远距离实体每2帧更新一次 | 渲染开销降低 |
| 事件标注实例化 | 同类型事件使用InstancedMesh | Draw Call减少 |
| 时间索引缓存 | 索引常驻内存 | 跳转延迟<1秒 |

### 8.1 跳转优化
跳转到任意时间点时，需要重新定位数据并快速更新场景：
- 使用时间索引二分查找定位数据块。
- 暂停回放，直接加载目标时间点的完整状态（不插值），然后恢复插值。
- 对于大量实体，可并行加载状态数据。

## 9. 与三维渲染引擎的集成

### 9.1 实体更新接口
三维渲染引擎提供批量更新实体的方法：

```javascript
class SceneManager {
    updateEntities(states) {
        states.forEach(state => {
            let obj = this.entityMap.get(state.id);
            if (obj) {
                obj.position.copy(state.position);
                obj.quaternion.copy(state.quaternion);
                // 更新材质/颜色等
            }
        });
    }
    
    triggerEvent(event) {
        // 根据事件类型创建特效
        if (event.type === 'explosion') {
            createExplosion(event.params.position);
        } else if (event.type === 'missile_launch') {
            showLaunchMarker(event.entityId);
        }
    }
}
```
