## 3. 分析流程

动态仿真的核心是混合仿真引擎的设计，需结合离散事件（如雷达探测、导弹发射、引信引爆）和时间步进（如连续运动、搜索扫描）。分析流程如下：

### 3.1 初始化阶段
- 解析输入的场景和规划文件，构建实体列表（无人机、目标、威胁等）。
- 为每个实体创建对应的模型实例，并设置初始位置、姿态、状态。
- 初始化全局事件队列（优先队列，按时间排序）。
- 设置当前仿真时间 `t = start_time`。
- 根据规划方案中的任务起始时间，生成初始事件（如“无人机起飞”、“开始搜索”等）并加入队列。

### 3.2 仿真循环（混合推进）
仿真循环采用**事件驱动与时间步进混合**机制：

1. **事件处理**：从事件队列中取出所有时间 ≤ 当前时间 `t` 的事件，按时间顺序（同时间按优先级）依次处理。每个事件回调对应的模型函数，该函数可能修改模型状态，并可能生成新事件（插入队列）。
2. **时间步进**：确定下一个步进目标时间 `next_step = t + dt`。若事件队列中下一个事件时间早于 `next_step`，则先推进到该事件时间，处理事件；否则推进到 `next_step`，执行所有模型的连续行为更新：
   - 对每个模型调用 `step(dt_actual)`，更新其连续状态（如位置、速度、姿态）。
   - 检查模型间的交互（如探测、碰撞），若满足条件则生成新事件（延迟或立即）。
3. **时间推进**：`t = min(next_step, next_event_time)`，重复步骤1-2直到 `t >= end_time` 或所有任务完成。

### 3.3 混合仿真机制的优势
- 对于高动态连续过程（如飞行、搜索），采用固定时间步进保证精度。
- 对于稀疏的离散事件（如武器发射、目标摧毁），采用事件驱动提高效率。

### 3.4 并行化考虑
当实体数量达240架无人机时，可对时间步进阶段进行并行处理：
- 将实体分组，每组由独立线程执行 `step()`。
- 事件处理阶段需串行，确保事件顺序正确，但可对事件回调进行优化。

### 3.5 仿真控制
- 支持实时仿真（按实际时间比例）、超实时仿真（加速）、欠实时仿真（减速或单步）。
- 提供暂停、继续、停止、断点保存/加载功能。

### 3.6 数据记录
- 每个时间步记录关键状态（位置、速度、事件）到内存缓冲区，定期写入磁盘。
- 支持事后回放和分析。

## 3. 关键术语解释

| **术语** | **解释** |
|----------|----------|
| **动态仿真** | 指仿真系统能够随时间推进而连续地模拟实体行为和交互，反映系统的动态变化过程。在本系统中，动态仿真意味着无人机、目标、威胁等实体在时间维度上不断运动、探测、交战。 |
| **离散事件仿真** | 一种仿真方法，系统状态仅在离散的时间点上发生变化（称为事件）。例如，导弹发射、雷达探测到目标、引信引爆等都属于离散事件。事件之间有明确的时间间隔，仿真引擎通过维护一个事件队列，按时间顺序处理事件。 |
| **时间步进仿真** | 一种仿真方法，将仿真时间划分为固定的时间步长（如0.1秒），在每个步长处更新所有实体的连续状态（如位置、速度）。适用于描述连续变化的过程，如飞行器的运动、传感器的持续扫描。 |
| **混合仿真机制** | 结合离散事件和时间步进两种方法，既能高效处理稀疏的离散事件，又能精确模拟连续的动态过程。通常采用时间步进作为主循环，在步进之间处理到期的事件，并在步进过程中检测事件触发条件。 |
| **仿真引擎** | 仿真系统的核心软件模块，负责管理仿真时间、调度事件、更新模型状态、处理交互、记录数据等。它按照设定的规则驱动仿真运行。 |
| **协同作战场景** | 包含作战背景、地理环境、初始兵力部署、敌方目标、威胁区域等信息的完整设定，是仿真的起点。 |
| **规划方案** | 由任务规划模块生成的详细计划，包括每架无人机的任务序列、飞行路径、工作参数（如导引头开机时刻、引炸高度）等。 |

## 4. 分析流程

### 4.1 混合仿真机制设计
动态仿真采用**混合驱动**策略，核心是时间步进循环与事件队列的协同工作。具体流程如下：

1. **初始化**：
   - 加载场景和规划文件，创建实体对象（无人机、目标、威胁等）。
   - 为每个实体分配唯一ID，设置初始位置、速度、状态。
   - 初始化优先队列（按时间排序）作为事件队列。
   - 根据规划方案生成初始事件（如“无人机起飞”、“进入搜索区域”等）并加入队列。
   - 设置当前仿真时间 `t = start_time`，步长 `dt`（用户可配置，默认0.1秒）。

2. **主循环（while t < end_time 且 未手动停止）**：
   - **处理到期事件**：从事件队列中取出所有时间 ≤ `t` 的事件，按时间顺序（同时间按优先级）处理。每个事件回调对应模型的处理函数，函数可能修改模型状态、产生新事件（插入队列）。
   - **计算下一步进目标**：`next_step = t + dt`。检查事件队列中下一个事件的时间 `next_event`，若 `next_event < next_step`，则将 `next_target = next_event`，否则 `next_target = next_step`。
   - **时间步进更新**：若 `next_target > t`，则实际步长 `delta = next_target - t`。对每个活动实体调用 `step(delta)`，更新其连续状态（如运动学积分、传感器累积）。
     - 运动学积分：采用欧拉法或四阶龙格库塔法更新位置、速度。
     - 传感器模拟：根据当前探测模型，计算是否探测到目标，若探测到则生成“探测事件”插入队列。
   - **检测瞬时交互**：步进后检查实体间的碰撞、引信触发等条件，若满足则生成相应事件（可立即处理或延迟）。
   - **时间推进**：`t = next_target`，回到循环开始。

3. **结束处理**：
   - 保存最终状态到结果文件。
   - 生成统计报告（任务完成率、消耗等）。
   - 清理资源。

### 4.2 并行化策略
为支持240架无人机的大规模仿真，对时间步进阶段进行并行化：
- 实体分组：将无人机等实体按区域或类型分组，每组由一个线程负责执行 `step()`。
- 同步点：每个时间步结束时，所有线程同步，确保全局一致性。
- 事件处理串行化：事件队列的访问需要加锁，或采用无锁队列优化。
- 使用OpenMP或QtConcurrent实现多线程。

## 6. 开发实现方案

### 6.1 总体架构
仿真引擎采用**模块化、插件化**设计，核心模块包括：
- **时间管理器**：负责时间推进、步长控制、同步。
- **事件调度器**：管理事件队列、事件分发。
- **实体管理器**：维护所有实体对象，支持快速查找（如空间索引）。
- **模型接口**：定义实体的行为，所有模型实现 `step(dt)` 和事件回调接口。
- **数据记录器**：异步记录仿真数据。
- **可视化适配器**：将状态数据推送给GUI显示。

### 6.2 核心技术选型
- **开发语言**：C++17（性能优先），结合Qt（界面、线程管理）。
- **并行计算**：OpenMP（多线程并行）、Intel TBB（并发队列）。
- **事件队列**：使用优先队列（`std::priority_queue`）或基于时间轮的调度器优化大量事件。
- **数值积分**：Eigen库处理向量运算，集成四阶龙格库塔法。
- **空间索引**：使用四叉树或网格法加速碰撞检测和探测查询。
- **日志记录**：使用spdlog异步日志库，减少仿真干扰。

struct Event {

### 6.4 混合仿真核心实现
```cpp
void SimulationEngine::mainLoop() {
    while (m_currentTime < m_endTime && m_running) {
        // 1. 处理所有到期事件
        processEvents(m_currentTime);
        
        // 2. 确定下一步目标时间
        double nextStep = m_currentTime + m_dt;
        double nextEventTime = m_eventQueue.empty() ? nextStep : m_eventQueue.top().time;
        double targetTime = std::min(nextStep, nextEventTime);
        
        // 3. 步进更新
        if (targetTime > m_currentTime) {
            double delta = targetTime - m_currentTime;
            stepAll(delta);  // 并行更新所有实体
            // 检查事件触发条件（如碰撞、探测）
            detectEvents();
        }
        
        // 4. 时间推进
        m_currentTime = targetTime;
        
        // 5. 推送状态给可视化（按需，如每帧）
        if (m_currentTime - lastVisualTime >= 0.1) { // 10Hz更新
            emit stateUpdated(collectStates());
            lastVisualTime = m_currentTime;
        }
    }
    emit simulationFinished(collectResults());
}

void SimulationEngine::stepAll(double delta) {
    #pragma omp parallel for
    for (int i = 0; i < m_entities.size(); ++i) {
        m_entities[i]->step(delta);
    }
}
```

### 6.5 大规模优化措施
- **事件队列优化**：采用多级时间轮，减少事件插入/删除的复杂度。
- **实体分组并行**：将实体按空间区域分组，每组独立线程更新，减少锁竞争。
- **数据记录异步化**：使用环形缓冲区，由独立线程写入磁盘。
- **内存池**：预分配实体内存，避免频繁分配/释放。
- **LOD（细节层次）**：对远距离实体简化更新计算。

- 支持单步调试。
