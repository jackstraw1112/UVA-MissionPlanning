## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 评估结果展示 |
| **需求标识**     | FR-EVAL-06 |
| **需求概述**     | 提供多种图表类型直观展示评估结果，包括但不限于柱状图、折线图、饼图、雷达图等；支持根据评估指标的特点和分析目的选择合适的图表类型，以增强评估结果的可视化分析能力。 |
| **需求来源**     | 系统总体设计要求 / 作战效能评估模块能力指标 |
| **前置条件**     | 1. 评估计算已完成，生成了评估结果数据。<br>2. 评估结果数据已结构化存储，可供前端调用。<br>3. 用户已进入评估结果展示界面。 |
| **触发条件**     | 1. 用户完成评估计算后自动跳转至结果展示界面。<br>2. 用户在历史评估结果列表中选择某次评估任务，点击“查看结果”。<br>3. 用户在展示界面切换图表类型或调整参数。 |
| **退出条件**     | 1. 用户关闭结果展示界面或返回上一级菜单。<br>2. 用户选择新的评估任务，重新加载结果。 |
| **功能参与者**   | 用户（评估分析人员）、评估计算模块（提供结果数据）、图表渲染引擎 |
| **输入**         | 评估结果数据（包含指标名称、得分、权重、排序等），用户选择的图表类型（柱状图、折线图、饼图、雷达图等），图表定制参数（如颜色、标题、坐标轴范围）。 |
| **输出**         | 生成的图表（可交互、可缩放、可保存）、图表对应的数据表格、导出图片/数据功能。 |
| **处理流程**     | 1. **数据准备**：从评估计算模块获取当前任务的评估结果数据，包括指标层次、得分、综合得分等。<br>2. **图表类型选择**：根据用户选择或系统推荐（如雷达图适用于多维度对比，饼图适用于占比分析）确定图表类型。<br>3. **数据映射**：将指标数据映射到图表的对应维度：<br>   - 柱状图：指标名称为X轴，得分为Y轴，可分组显示（如不同方案对比）。<br>   - 折线图：适用于时间序列指标，以时间为X轴，指标值为Y轴。<br>   - 饼图：用于显示各指标对总体的贡献占比，通常取权重或得分归一化。<br>   - 雷达图：多个指标的得分映射到各轴，形成多边形，便于多方案对比。<br>4. **图表渲染**：调用图表库（如QChart、ECharts）生成交互式图表，支持悬停显示数值、点击高亮等交互。<br>5. **辅助功能**：提供图表导出（PNG/SVG）、数据导出（CSV/Excel）、缩放、平移、重置视图等功能。<br>6. **动态更新**：当用户切换图表类型或调整参数时，重新执行映射和渲染。 |
| **成功判据**     | 1. 图表正确显示，数据与计算结果一致。<br>2. 图表类型切换后，数据映射合理，图形清晰可读。<br>3. 交互功能（悬停、点击、缩放）响应正常。<br>4. 导出功能生成的文件内容正确。 |
| **异常处理**     | 1. 数据缺失或格式错误：显示“数据异常”提示，并展示空白图表。<br>2. 图表渲染失败（如内存不足）：降级为静态图片或表格，并记录错误。<br>3. 用户选择的图表类型与数据结构不匹配（如饼图用于非占比数据）：给出提示并建议其他类型。 |
| **后置条件**     | 1. 用户可对展示结果进行保存、分享或进一步分析。<br>2. 图表交互状态保留，直至用户关闭界面。 |
| **显示约束**     | 1. 图表应支持全屏查看，适应不同分辨率。<br>2. 提供图例、坐标轴标签、标题等，且可自定义。<br>3. 支持多图表联动（如点击雷达图中的某个指标，柱状图中对应高亮）。<br>4. 图表颜色应采用色盲友好的配色方案。 |
| **其他约束**     | 1. 图表库应轻量级，加载速度快。<br>2. 支持导出为矢量图（SVG）和位图（PNG）。<br>3. 提供数据表格视图作为图表补充，便于精确查看。 |

## 4. 分析流程

### 4.1 数据映射规则
- **柱状图**：X轴为指标名称，Y轴为得分。若有多组方案（如不同任务或不同权重设置），可分组显示，每组用不同颜色。
- **折线图**：X轴为时间（如任务阶段），Y轴为指标值，每条线代表一个指标。若只有一个时间点，则不适合用折线图。
- **饼图**：通常用于展示权重占比，或归一化后的得分占比。输入数据需为各部分的数值（如权重），总和不为零。
- **雷达图**：指标数量不宜过多（一般不超过8个），每个指标归一化到0~1范围，雷达图每个轴最大值固定为1，便于对比。

### 4.2 图表渲染
- 前端采用 **ECharts** 或 **Chart.js** 等轻量级JavaScript库，后端采用 **Qt Charts**（如果桌面应用）。根据用户环境选择。
- 图表配置：包括标题、图例、工具提示、缩放平移等。
- 交互：悬停显示数值，点击可高亮对应指标或方案。

### 4.3 多图表联动
- 当存在多个图表时（如雷达图和柱状图），点击雷达图某个指标，柱状图中对应条高亮，实现联动分析。

# 反辐射无人机仿真系统——评估结果展示模块开发实现方案

## 1. 总体架构

评估结果展示模块采用**模型-视图-控制器（MVC）**模式，前端基于Qt Charts（或嵌入Web图表库）实现交互式可视化，后端通过标准化数据接口与评估计算模块通信。模块支持多种图表类型动态切换、用户自定义展示参数，并提供导出功能。

```plaintext
┌─────────────────────────────────────────────────────┐
│                   用户交互层                           │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │
│  │ 图表类型选择│ │ 参数配置面板│ │ 导出工具栏  │   │
│  └─────────────┘ └─────────────┘ └─────────────┘   │
├─────────────────────────────────────────────────────┤
│                   图表渲染层                           │
│  ┌─────────────────────────────────────────────┐   │
│  │   Qt Charts / ECharts 图表引擎              │   │
│  │   ├─ 柱状图 │ 折线图 │ 饼图 │ 雷达图        │   │
│  └─────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│                   数据适配层                           │
│  │  结果数据解析器 │ 数据映射器 │ 缓存管理器    │   │
├─────────────────────────────────────────────────────┤
│                   数据源层                             │
│  │  评估计算模块（结果JSON/数据库）                │   │
└─────────────────────────────────────────────────────┘
```

## 2. 技术选型

### 2.1 方案一：纯Qt Charts（桌面应用）
- **Qt Charts** 模块（`QChart`, `QChartView`, `QBarSeries`, `QLineSeries`, `QPieSeries`, `QRadarSeries`）提供原生图表支持，与Qt界面无缝集成。
- **优势**：无需额外依赖，性能好，与Qt信号槽完美配合。
- **适用**：本地桌面应用，实时性要求高。

### 2.2 方案二：Web图表嵌入（跨平台/富交互）
- **Qt WebEngine** 加载本地HTML页面，使用 **ECharts** 或 **Chart.js** 渲染。
- **优势**：图表样式丰富，交互性强，支持移动端适配。
- **适用**：需要更复杂交互或跨平台Web部署。

本文以**方案一**为主进行实现说明，同时提及Web方案的关键点。

## 3. 核心功能实现

### 3.1 数据准备与映射

#### 3.1.1 数据结构
评估计算模块输出的结果JSON格式示例：
```json
{
  "task_id": "anti_radar_001",
  "timestamp": "2025-03-18T15:30:00",
  "indicators": [
    { "name": "目标摧毁概率", "score": 0.82, "weight": 0.4 },
    { "name": "无人机生存率", "score": 0.75, "weight": 0.3 },
    { "name": "弹药消耗", "score": 0.60, "weight": 0.3 }
  ],
  "composite_score": 0.74
}
```

#### 3.1.2 数据解析器
```cpp
class EvaluationResult {
public:
    QString taskId;
    QDateTime timestamp;
    QList<Indicator> indicators;
    double compositeScore;
    
    static EvaluationResult fromJson(const QByteArray& json);
};

struct Indicator {
    QString name;
    double score;
    double weight;
};
```

#### 3.1.3 数据映射到图表系列
根据图表类型不同，将指标数据转换为对应的Qt系列：

- **柱状图**：`QBarSet` 每个指标一个集合。
```cpp
QBarSet* set = new QBarSet("得分");
for (auto& ind : result.indicators) {
    *set << ind.score;
}
QBarSeries* series = new QBarSeries();
series->append(set);
```

- **折线图**：`QLineSeries` 点集，可用于时间序列或多个方案对比。
```cpp
QLineSeries* series = new QLineSeries();
for (int i = 0; i < result.indicators.size(); ++i) {
    series->append(i, result.indicators[i].score);
}
```

- **饼图**：`QPieSeries` 每个指标一个切片。
```cpp
QPieSeries* series = new QPieSeries();
for (auto& ind : result.indicators) {
    series->append(ind.name, ind.score);
}
```

- **雷达图**：Qt Charts 5.7+ 支持 `QRadarSeries`。
```cpp
QRadarSeries* series = new QRadarSeries();
for (auto& ind : result.indicators) {
    series->append(ind.score);
}
series->setName("效能得分");
```

### 3.2 图表类型动态切换

- 界面提供一个 `QComboBox` 供用户选择图表类型。
- 当选择改变时，清除当前图表视图，创建新的图表并绑定数据。

```cpp
void MainWindow::onChartTypeChanged(int index) {
    // 清除旧图表
    m_chartView->chart()->removeAllSeries();
    
    // 根据选择创建新系列
    QAbstractSeries* series = nullptr;
    switch (index) {
    case 0: series = createBarSeries(); break;
    case 1: series = createLineSeries(); break;
    case 2: series = createPieSeries(); break;
    case 3: series = createRadarSeries(); break;
    }
    if (series) {
        m_chartView->chart()->addSeries(series);
        m_chartView->chart()->createDefaultAxes(); // 自动创建坐标轴
        // 对饼图特殊处理
        if (index == 2) {
            m_chartView->chart()->setAnimationOptions(QChart::SeriesAnimations);
        }
    }
}
```

### 3.3 图表定制化

- 提供侧边栏或右键菜单，允许用户调整：
  - **标题**：`chart->setTitle("自定义标题")`
  - **颜色**：通过 `series->setColor()` 或调色板。
  - **坐标轴范围**：`axisX->setRange(min, max)`
  - **图例位置**：`chart->legend()->setAlignment(Qt::AlignBottom)`

### 3.4 交互功能实现

#### 3.4.1 悬停提示
利用 `QChart` 的信号 `hovered()` 或 `QToolTip` 显示数值。

```cpp
connect(series, &QBarSeries::hovered, this, [](bool status, int index) {
    if (status) {
        QToolTip::showText(QCursor::pos(), QString("得分: %1").arg(m_data[index]));
    }
});
```

#### 3.4.2 点击高亮
点击某个柱/点时，可高亮并显示详细信息。

```cpp
connect(series, &QBarSeries::clicked, this, [this](int index) {
    // 高亮该柱（例如通过改变颜色）
    // 并在侧边栏显示详细指标信息
    showIndicatorDetail(index);
});
```

#### 3.4.3 缩放与平移
`QChartView` 默认支持鼠标拖拽平移和滚轮缩放，只需设置交互属性：

```cpp
m_chartView->setRubberBand(QChartView::RectangleRubberBand);
m_chartView->chart()->setZoomFactor(1.0);
```

### 3.5 多图表联动

当同时展示雷达图和柱状图时，点击雷达图某指标轴，柱状图中对应条高亮。可通过自定义信号实现：

```cpp
// 雷达图点击信号（需自定义，或使用QRadarSeries的clicked信号）
connect(radarSeries, &QRadarSeries::clicked, this, [this](int index) {
    barSeries->setSelectedBar(index); // 假设有自定义高亮方法
});
```

### 3.6 导出功能

- **导出图片**：`chartView->grab().save("chart.png")` 或使用 `QPdfWriter` 导出PDF。
- **导出数据**：将结果数据转换为CSV/Excel，使用 `QFile` 和 `QTextStream` 写入。

```cpp
void MainWindow::exportData() {
    QString fileName = QFileDialog::getSaveFileName(this, "导出数据", "", "CSV (*.csv)");
    QFile file(fileName);
    if (file.open(QIODevice::WriteOnly)) {
        QTextStream stream(&file);
        stream << "指标,得分,权重\n";
        for (auto& ind : m_result.indicators) {
            stream << ind.name << "," << ind.score << "," << ind.weight << "\n";
        }
        file.close();
    }
}
```

## 4. 性能优化

- **数据缓存**：切换图表类型时，避免重复解析数据，缓存已解析的 `QList<Indicator>`。
- **懒加载**：只有当用户切换到该图表类型时才创建对应的系列。
- **异步渲染**：如果数据量大，可在后台线程准备系列，然后通过信号槽更新UI。
- **降低刷新频率**：缩放/平移时禁用动画，操作结束后再开启。

## 5. 与评估计算模块的集成

- 评估计算完成后，发送信号 `evaluationFinished(const EvaluationResult&)`。
- 结果展示模块接收该信号，自动加载并显示默认图表（如雷达图）。
- 历史结果可通过任务ID从数据库查询，调用 `loadResult(taskId)` 重新展示。

## 6. Web方案关键点（备选）

若采用ECharts嵌入：
- 使用 `QWebEngineView` 加载本地HTML，通过 `QWebChannel` 将C++数据传递到JavaScript。
- HTML中编写ECharts初始化代码，并监听来自C++的数据更新。
- ECharts配置灵活，交互更丰富，但需要处理C++与JS的通信开销。
