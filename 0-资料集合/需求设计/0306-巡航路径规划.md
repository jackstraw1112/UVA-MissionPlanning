# 反辐射无人机仿真系统——巡航路径规划模块功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 巡航路径规划 |
| **需求标识**     | FR-PATH-01 |
| **需求概述**     | 提供路径规划算法，能够依据协同作战任务目标、无人机作战性能（最大航程、巡航速度、续航时间等），综合考虑敌方威胁、地形、气象等战场环境因素，自动生成优化的协同作战巡航路径，支持多机协同路径规划。 |
| **需求来源**     | 系统总体设计要求 / 协同任务规划管理CSCI能力指标 |
| **前置条件**     | 1. 作战任务分配已完成，明确每架无人机的任务序列（目标点顺序）。<br>2. 无人机资源信息已加载（包含每架无人机的性能参数：最大航程、巡航速度、续航时间、机动性能等）。<br>3. 战场环境信息已从环境参数设置模块获取（地形高程数据、障碍物分布、威胁区（敌方防空火力范围）、气象条件（风速、能见度等））。<br>4. 所有输入数据通过合法性校验。<br>5. 路径规划算法所需参数（如算法类型、优化目标权重、安全距离等）已配置。 |
| **触发条件**     | 1. 用户点击“开始路径规划”按钮。<br>2. 任务分配方案变更后自动触发重新规划（可选）。 |
| **退出条件**     | 1. 路径规划完成，所有无人机的巡航路径生成并显示。<br>2. 用户手动停止规划（若计算时间过长）。<br>3. 规划过程中发生严重错误，模块退出并报告错误。 |
| **功能参与者**   | 任务规划人员（用户）、环境参数设置模块（提供环境数据）、无人机资源数据库（提供性能参数）、任务分配模块（提供任务序列） |
| **输入**         | 无人机任务序列（每架无人机的目标点列表及顺序）、无人机性能参数（最大航程、巡航速度、续航时间、转弯半径等）、战场环境数据（地形栅格、障碍物列表、威胁区多边形、气象影响因子）、路径规划算法参数（如优化目标权重、搜索步长、迭代次数等） |
| **输出**         | 每架无人机的巡航路径（三维航点序列，包含经纬度、高度、预计到达时间）、总航程、预计总耗时、油耗/能量消耗估计、路径安全评估（是否规避威胁、地形碰撞等） |
| **处理流程**     | 1. 获取输入数据，构建综合环境模型（地形高程网格、威胁区代价场、气象影响修正）。<br>2. 对每架无人机（或协同编队）依次或并行进行路径规划：<br>   - 若为单机路径：采用A*、Dijkstra、RRT*等算法在代价地图中搜索从起始点到第一个任务点，再到后续任务点的最优路径，满足航程约束、规避威胁和障碍。<br>   - 若为多机协同路径：考虑机间避撞、时空协同（如同时到达），采用多智能体路径规划算法（如优先级规划、协同A*、分布式RRT、遗传算法等）。<br>3. 对生成的路径进行平滑处理（如B样条曲线），调整航点满足无人机机动性能（转弯半径、爬升率）。<br>4. 计算路径的详细参数：分段距离、预计飞行时间、累积威胁暴露风险、燃油消耗等。<br>5. 验证路径是否满足无人机性能约束（总航程≤最大航程，续航时间≤最大续航时间）。<br>6. 输出所有无人机的路径数据及评估指标。 |
| **成功判据**     | 1. 每架无人机的路径均满足自身性能约束（航程、时间、机动性）。<br>2. 路径有效规避所有已知威胁区和障碍物，且满足地形跟随/回避要求（如离地高度安全）。<br>3. 多机路径之间无时空冲突（满足安全间隔）。<br>4. 路径规划的代价（如总航程、暴露风险）在优化目标下达到可接受水平。 |
| **异常处理**     | 1. 输入数据缺失或格式错误：弹出提示框，高亮错误字段，禁止规划。<br>2. 某无人机无法找到可行路径（如所有路径被威胁阻断）：提示用户该无人机无可行路径，建议调整任务分配或放宽约束。<br>3. 规划算法超时：提示用户是否继续等待，允许手动终止。<br>4. 路径验证不通过（如超航程）：提示用户并建议调整任务序列或增加中继点。<br>5. 多机路径冲突无法解耦：提示用户尝试调整优先级或使用协同算法重试。 |
| **后置条件**     | 1. 生成的路径数据保存于内存中，可供后续仿真推演、飞行控制模块调用。<br>2. 用户可保存当前路径规划结果为项目文件（含路径点及评估数据）。<br>3. 用户可导出路径报告或KML文件用于外部查看。 |
| **显示约束**     | 1. 在二维/三维地图上以不同颜色绘制每架无人机的规划路径，并标注关键航点。<br>2. 支持路径分段显示（如巡航段、任务段）。<br>3. 鼠标悬停或点击路径可显示该点的高度、预计时间等信息。<br>4. 列表展示每架无人机的路径统计：总航程、总耗时、剩余燃油等。<br>5. 应提供威胁区和障碍物的可视化叠加。 |
| **其他约束**     | 1. 算法应支持在线重规划（如遇突发威胁，可局部调整）。<br>2. 需支持多种优化目标：最短路径、最低暴露风险、最省油等，并可配置权重。<br>3. 路径规划应考虑无人机编队保持（若需要）。<br>4. 算法应具备可扩展性，以适应未来更多无人机数量或更复杂环境。 |

## 3. 分析流程

巡航路径规划采用分层、混合算法，综合考虑环境约束和无人机性能，具体流程如下：

### 3.1 环境建模
1. **地形处理**：将地形高程网格转换为代价地图，高代价区域代表陡峭地形或禁飞区（如超过无人机爬升能力的坡度）。
2. **威胁区建模**：将威胁区（如敌方雷达覆盖范围）映射为代价场，路径穿过威胁区时增加额外代价，代价与暴露时间或距离成正比。
3. **障碍物处理**：将障碍物膨胀为安全缓冲区，禁止路径进入。
4. **气象影响**：根据风速、风向修正无人机地速，影响飞行时间；强风区可增加代价。

### 3.2 单机路径规划
对于每架无人机，采用以下步骤生成初步路径：
1. **全局路径搜索**：
   - 在栅格化的代价地图上，使用A*算法搜索从起点到第一个任务点的最低代价路径。
   - 若存在多个任务点，采用旅行商问题（TSP）启发式或顺序规划：先规划起点→任务点1，再以任务点1为新起点规划到任务点2，依此类推。或采用联合优化（如遗传算法同时优化顺序和路径）。
   - 对于复杂三维环境，可使用RRT*算法进行连续空间搜索，生成平滑路径。
2. **局部路径优化**：对生成的折线路径进行平滑处理，使用B样条或Dubins曲线，使其符合无人机转弯半径和爬升率约束。
3. **航程与时间校验**：计算路径总长，若超过最大航程，则返回重新搜索（可通过增加航程代价权重重新规划）或提示用户。
4. **威胁规避校验**：检查路径是否穿过威胁区，若有则提高威胁代价重新规划，直至满足要求或达到迭代上限。

### 3.3 多机协同路径规划
当多架无人机需协同作战（如同时到达、编队飞行）时，需进行协同规划：
1. **时空协同约束处理**：
   - **同时到达**：要求多架无人机在同一时间窗口内到达指定目标点。可采用速度调整或路径长度调整来实现。算法上可使用协同A*（Cooperative A*）在时空联合空间搜索，或采用优先级规划：先规划高优先级无人机，后续无人机在其路径基础上增加时空避撞。
   - **编队保持**：要求无人机以固定队形飞行，则需规划领航机路径，僚机通过相对位置生成路径。
2. **冲突检测与解脱**：
   - 对初步生成的各无人机路径，检查是否存在时空冲突（如同时同地小于安全距离）。
   - 若存在冲突，可采用调整速度（在时间维度错开）、局部路径重规划（在空间维度错开）或重新排序优先级等方法解决。
3. **迭代优化**：多机协同规划通常需要多次迭代，直到所有冲突解除且满足协同要求。

### 3.4 后处理与评估
1. 生成最终路径点序列，计算各航段的预计到达时间（ETA）。
2. 评估每架无人机的燃油消耗、剩余电量，确认满足续航要求。
3. 汇总所有无人机的路径数据，生成评估报告。

### 3.5 算法选型
针对240架无人机的大规模场景，采用分层分布式策略：
- **上层协同**：使用基于市场机制的合同网协议进行任务区域分配，减少耦合。
- **下层路径**：使用快速探索随机树（RRT*）或改进的A*并行计算每架无人机的路径，结合优先级规划处理冲突。
- **并行计算**：利用多线程或GPU加速，将无人机分组并行规划，组内协同，组间松耦合。

## 1. 总体技术架构与开发路线

开发一个支持240架无人机的仿真系统，需要采用**分层解耦、并行计算、算法插件化**的架构。

-   **开发语言与框架**：C++ (Qt 5.15+) 负责界面与主逻辑，Python 负责算法原型与数据科学计算（通过C++调用Python或嵌入解释器）。
-   **核心库依赖**：
    -   **地理处理**：**GDAL** (用于读取GeoTIFF/DEM地形数据)。
    -   **数值计算**：**Eigen** (矩阵运算)、**Boost** (图算法)。
    -   **并行计算**：OpenMP (CPU多线程)、CUDA (GPU加速，可选)。
    -   **可视化**：Qt Charts / QCustomPlot (甘特图)，QGIS C++ API 或 自定义OpenGL (三维地图)。
-   **开发步骤**：
    1.  **数据结构定义**：建立 `Task`, `UAVResource`, `Environment`, `WayPoint` 等核心数据类。
    2.  **算法原型验证**：在Python (Jupyter Notebook) 中编写并测试蒙特卡洛仿真、遗传算法、RRT路径规划等核心算法。
    3.  **C++核心引擎封装**：将验证通过的算法用C++重写或通过 `pybind11` 调用Python库，封装成高性能的动态库。
    4.  **Qt界面集成**：设计UI文件，连接信号槽，将算法引擎挂载到后台线程中执行，避免界面卡死。
    5.  **性能调优**：针对240架无人机的规模，进行内存优化、多线程并行化改造。

## 2. 兵力需求计算模块实现

### 2.1 核心算法实现

**解析模型**：直接实现公式 \( N = \lceil \frac{\ln(1-P_{req})}{\ln(1-P_s)} \rceil \)。`P_s` 由基础成功率乘以环境因子乘积得到。

**蒙特卡洛仿真（核心）**：
-   **并行化**：由于1000次迭代是相互独立的，非常适合并行。
    ```cpp
    // 使用 OpenMP 进行并行加速
    int successCount = 0;
    #pragma omp parallel for reduction(+:successCount)
    for (int i = 0; i < ITERATIONS; ++i) {
        if (simulateSingleMission(uavCount, targetList, envFactor)) {
            successCount++;
        }
    }
    double prob = static_cast<double>(successCount) / ITERATIONS;
    ```
-   **外层搜索**：采用**二分查找法**快速定位满足概率阈值的最小无人机数量，从 `min=1`，`max=availableCount` 开始搜索。

### 2.2 大规模支持（240架）

兵力计算模块本身不直接处理240架飞机的详细编组，它输出的是一个**总量**和**编组比例**（如：需要120架，其中侦察型：攻击型 = 1：2）。这为后续任务分配模块提供了宏观约束。

### 2.3 Qt集成
-   创建一个 `ForceCalculator` 类，继承自 `QObject`，包含 `calculate()` 方法。
-   在界面线程点击“计算”后，将 `ForceCalculator` 移动至 `QThread` 中运行，通过信号（signal）将进度和结果传回主界面更新UI。

## 3. 作战任务分配模块实现（240架）

这是实现难点，240架无人机对应的是超大规模的组合优化问题。

### 3.1 核心算法选型与实现
传统遗传算法直接编码240架无人机会导致解空间爆炸。我们采用**分层混合策略**：

| **层级** | **算法** | **目标** |
| :--- | :--- | :--- |
| **第一层：区域分割** | 聚类算法（K-means 或 谱聚类） | 根据任务点的空间分布，将240架无人机和任务点划分为若干个相对独立的子群（如10个区域，每个区域24架无人机）。 |
| **第二层：子群内分配** | 改进遗传算法 或 蚁群算法 | 在每个子群内部，进行详细的任务分配和排序。 |
| **第三层：协同调整** | 合同网协议 | 处理跨子群边界的任务冲突，或进行全局负载均衡。 |

**遗传算法实现细节**：
-   **编码**：采用“任务ID-无人机ID”的两段式编码，或用基于优先级的编码。
-   **适应度函数**：考虑多目标加权，必须包含**罚函数**来处理超航程、能力不匹配等约束。
-   **并行计算**：遗传算法的种群个体适应度评估可以并行。

### 3.2 大规模优化策略
-   **启发式初始化**：利用兵力需求计算模块给出的编组比例，以及空间就近原则，生成高质量的初始种群，加速收敛。
-   **GPU加速**：对于适应度计算这种可并行的操作，可以使用CUDA将计算任务卸载到GPU，大幅提升每代的计算速度 。

## 4. 巡航路径规划模块实现（240架）

这是技术难度最高的模块，需要在三维环境中为240架飞机规划出无冲突、满足性能、协同作战的路径。

### 4.1 核心算法选型

根据不同的协同要求，选择不同的算法实现：

| **算法名称** | **适用场景** | **实现要点** |
| :--- | :--- | :--- |
| **改进A* / RRT\*** | 单机或多机独立路径规划 | 在三维栅格地图上搜索，代价函数综合考虑距离、威胁场、地形代价。 |
| **Multi-RRT** | 多机同时到达（时间协同） | 在扩展随机树时，引入时间维度约束，确保各无人机的路径长度在速度可调范围内能满足同时到达要求 。 |
| **SIFDS + MATD3 (深度强化学习)** | **大规模、动态环境、智能协同** | 1. **SIFDS**：将复杂的多障碍物环境简化为流场，为无人机提供避障引导。 <br> 2. **MATD3**：多智能体强化学习算法，训练无人机在流场中决策最优反应系数，实现全局协同 。 |
| **多蚁群系统** | 有时间窗的协同路径规划 | 模拟蚂蚁觅食，通过信息素引导无人机寻找既满足任务顺序（时间窗）又无冲突的路径 。 |

### 4.2 针对240架规模的实现策略

1.  **环境模型预处理**：
    -   **构建代价地图**：提前将地形、威胁区、障碍物融合成一个**三维代价场**。这个代价场存储在内存中，供所有路径规划器快速查询。
    -   **空间哈希/八叉树**：对障碍物和威胁区建立空间索引，加速碰撞检测。

2.  **分层规划与优先级**：
    -   **优先级规划**：为无人机设定优先级（如根据任务重要性）。先规划高优先级无人机的路径，并将这些路径作为**时空障碍**加入到环境代价场中。后续无人机规划时必须规避这些时空障碍。
    -   **编队规划**：对于需要编队飞行的无人机，只规划长机路径，僚机路径通过编队几何关系动态生成。

3.  **路径平滑与验证**：
    -   生成的原始路径（如A*的折线）需要用**B样条曲线 (Bezier Curves)** 或 **Dubins曲线** 进行平滑，以满足无人机转弯半径等动力学约束 。
    -   平滑后需**批量验证**所有路径的航程约束、安全距离、无碰撞特性。

4.  **多线程并行计算**：
    -   将无人机分组到不同的线程中进行路径规划。由于优先级规划引入了依赖，可以采用**流水线并行**方式：线程1规划第1-10架，完成后线程2在此基础上规划第11-20架，以此类推。

## 2. 核心算法体系

### 2.1 多算法融合策略

根据不同的任务场景和约束条件，采用分层混合算法：

| 算法层 | 适用场景 | 核心算法 | 主要功能 |
|--------|----------|----------|----------|
| 全局规划层 | 大规模战场、多目标 | 改进型A*、RRT* | 快速生成可行路径走廊 |
| 局部优化层 | 威胁规避、地形跟随 | 人工势场法、DWA | 局部路径精细调整 |
| 协同调整层 | 多机编队、时间协同 | 一致性算法、遗传算法 | 多路径时空协同 |
| 路径平滑层 | 无人机动力学约束 | B样条曲线、Dubins曲线 | 满足机动性能 |

### 2.2 算法选择逻辑

```python
def select_algorithm(task_type, environment_complexity, uav_count):
    """
    根据任务特征自动选择最优算法组合
    """
    if uav_count == 1:
        if environment_complexity == "简单":
            return "A* + 人工势场法"
        elif environment_complexity == "复杂":
            return "RRT* + B样条平滑"
    else:

            return "协同A* + 一致性算法"
        elif task_type == "编队保持":

        else:
```

## 3. 环境建模思路

### 3.1 综合代价场构建

将多种环境因素融合为统一的代价场：

```
总代价 = 地形代价 × w1 + 威胁代价 × w2 + 气象代价 × w3 + 隐身代价 × w4
```

- **地形代价**：基于高程数据，陡峭区域代价高，平坦区域代价低
- **威胁代价**：采用高斯混合模型叠加多个威胁源
- **气象代价**：根据风速、能见度计算影响因子
- **隐身代价**：考虑雷达探测概率的空间分布

### 3.2 多分辨率网格策略

采用自适应网格划分：

- **远距离巡航**：低分辨率网格（快速搜索）
- **威胁区域周边**：中分辨率网格
- **目标区域附近**：高分辨率网格（精细规划）

## 4. 协同规划机制

### 4.1 多机协同规划流程

```
1. 任务分解：将整体任务分解为各无人机的子任务
2. 优先级排序：根据任务重要性确定规划顺序
3. 顺序规划：按优先级依次规划单机路径
4. 时空冲突检测：检查多机路径冲突
5. 协同调整：通过速度调节或局部重规划解决冲突
6. 迭代优化：重复3-5步直到无冲突
```

### 4.2 时间协同策略

- **同时到达**：通过路径长度调节 + 速度控制实现同步
- **顺序到达**：设定时间窗口，通过待机盘旋调整
- **时间间隔**：保持安全时间差，避免同时暴露

## 5. 约束处理机制

### 5.1 无人机性能约束

- **最大航程约束**：路径总长度 ≤ 最大航程 × 安全系数（0.7-0.8）
- **续航时间约束**：总飞行时间 ≤ 续航时间 × 安全系数
- **机动性能约束**：转弯半径、最大爬升率、最大俯冲角
- **速度约束**：最小速度 ≤ 巡航速度 ≤ 最大速度

### 5.2 战场环境约束

- **禁飞区**：严格禁止进入（代价无限大）
- **威胁区**：根据威胁等级赋予不同代价
- **地形约束**：最低安全高度、地形跟随要求
- **气象约束**：恶劣天气区域规避

## 6. 路径优化与平滑

### 6.1 多目标优化函数

```
F(路径) = w1 × 航程代价 + w2 × 威胁代价 + w3 × 时间代价 + w4 × 能耗代价
```

### 6.2 路径平滑算法

- **B样条曲线**：生成C²连续的光滑路径
- **Dubins曲线**：满足最小转弯半径约束
- **贝塞尔曲线**：局部调整，保持整体形状

## 7. 并行计算策略

针对240架无人机的协同规划，采用以下并行优化：

```
1. 区域分解：将战场划分为多个子区域
2. 分组规划：各子区域独立并行规划
3. 边界协调：处理跨区域路径的冲突
4. GPU加速：代价场计算、碰撞检测并行化
```

## 8. 评估与验证机制

### 8.1 路径质量评估指标

- **可行性**：是否满足所有硬约束
- **最优性**：与理论最优解的接近程度
- **鲁棒性**：对参数扰动的敏感性
- **实时性**：规划算法运行时间

### 8.2 验证流程

1. **单元测试**：验证各算法模块正确性
2. **场景测试**：在典型作战场景下验证
3. **压力测试**：测试240架无人机大规模场景
4. **对比验证**：与已知最优解对比
