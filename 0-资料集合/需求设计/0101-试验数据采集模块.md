## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 试验数据采集 |
| **需求标识**     | FR-DATA-01 |
| **需求概述**     | 支持处理不同编码格式的数据文件，能够从常见格式的文件中采集试验数据，将异构的原始试验数据转换为系统内部统一格式，为后续的数据处理、分析和评估提供可靠的数据基础。 |
| **需求来源**     | 系统总体设计要求 / 试验数据处理模块能力指标 |
| **前置条件**     | 1. 系统已具备相应文件格式的解析库（如CSV、Excel、JSON、XML、HDF5、MAT等）。<br>2. 文件系统可访问，磁盘空间充足。<br>3. 用户具有文件读取权限。 |
| **触发条件**     | 1. 用户在界面上点击“导入数据”按钮并选择文件。<br>2. 通过API调用触发数据导入。<br>3. （可选）系统自动监控指定文件夹，当新文件到达时自动触发采集。 |
| **退出条件**     | 1. 数据成功采集并存储至数据库，或导出至内部缓存。<br>2. 用户手动取消导入操作。<br>3. 发生不可恢复错误，模块退出并记录错误日志。 |
| **功能参与者**   | 用户（试验分析人员）、文件系统、数据存储模块、解析器库 |
| **输入**         | 试验数据文件（支持多种格式）、用户指定的解析参数（如分隔符、编码、数据映射规则）、文件路径。 |
| **输出**         | 解析后的内部数据表（内存/数据库）、导入统计信息（总行数、成功/失败行数）、错误日志。 |
| **处理流程**     | 1. **文件选择**：用户通过文件对话框选择一个或多个数据文件，或指定文件夹。<br>2. **格式识别**：根据文件扩展名（.csv, .xlsx, .json等）或用户指定格式，选择对应的解析器。若无法识别，允许用户手动指定。<br>3. **编码探测**：对文本文件自动探测编码（如UTF-8, GBK, GB2312, ISO-8859-1等），或由用户指定。<br>4. **数据解析**：调用解析器读取文件，按配置的分隔符、字段顺序等解析数据。对于二进制格式（如HDF5），按数据集路径读取。<br>5. **字段映射**：根据预定义或用户自定义的映射规则，将文件字段映射到系统内部标准字段。支持字段重命名、类型转换（如字符串转浮点）。<br>6. **数据清洗**：对解析后的数据进行基本校验，如剔除全空行、检测异常值（可选），记录错误数据。<br>7. **预览与确认**：显示前N行预览，用户确认无误后执行入库。<br>8. **数据存储**：将清洗后的数据批量插入数据库，并记录导入日志。<br>9. **完成反馈**：显示导入统计信息，包括成功条数、失败条数、消耗时间。 |
| **成功判据**     | 1. 文件中的所有有效数据均被正确解析，无数据丢失或错位。<br>2. 数据类型转换正确，字段映射关系准确。<br>3. 数据成功存入数据库，可通过查询验证。 |
| **异常处理**     | 1. 文件格式不支持：提示用户，并建议支持格式列表。<br>2. 编码解析错误：提示用户手动选择编码，并提供备选方案。<br>3. 字段缺失或类型错误：跳过错误行，记录日志，并继续解析剩余行，最后汇总错误信息。<br>4. 文件过大导致内存不足：采用分块读取策略，每处理一块即释放内存。<br>5. 权限不足：提示用户检查文件权限。 |
| **后置条件**     | 1. 数据已存入数据库，可供查询、分析、可视化等后续模块使用。<br>2. 导入日志已记录，便于审计和问题追溯。 |
| **显示约束**     | 1. 导入过程中显示进度条和当前操作状态（如“正在解析...”、“正在写入数据库...”）。<br>2. 完成后弹窗显示统计信息，并提供错误日志下载链接。<br>3. 预览区域以表格形式展示前20行数据，便于用户核对。 |
| **其他约束**     | 1. 支持的文件格式至少包括：CSV、Excel（.xls/.xlsx）、JSON、XML、HDF5、MAT（MATLAB）、TXT（自定义分隔符）。<br>2. 支持的文件编码包括：UTF-8、UTF-16、GBK、GB2312、ISO-8859-1、Windows-1252等。<br>3. 单个文件大小上限不低于2GB，且支持分块读取。<br>4. 支持多文件批量导入。 |

## 4. 分析流程

### 4.1 格式识别与解析器选择
- 首先根据文件扩展名初步判断格式，如`.csv`采用CSV解析器，`.xlsx`采用Excel解析器，`.json`采用JSON解析器，`.h5`或`.hdf5`采用HDF5解析器。
- 若扩展名缺失或通用（如`.txt`），则可通过读取文件头或用户指定来确认格式。
- 每种格式封装为独立的解析器插件，遵循统一接口，便于扩展新格式。

### 4.2 编码探测（针对文本文件）
- 对于CSV、TXT等文本格式，使用第三方库（如`uchardet`）自动检测编码。
- 若自动检测失败，回退到用户指定编码，并提供常用编码列表。

### 4.3 数据解析
- **CSV/文本**：按指定分隔符（逗号、制表符等）逐行解析，处理引号转义。支持跳过空行和注释行。
- **Excel**：使用`libxls`或`QtXlsx`读取，可选择指定工作表。
- **JSON**：解析为嵌套的键值对结构，支持扁平化处理。
- **XML**：使用DOM或SAX解析，根据XPath提取数据。
- **HDF5**：使用HDF5库读取指定数据集，支持多维数组。
- **MAT**：使用MAT文件库读取变量。

### 4.4 字段映射与类型转换
- 加载映射规则（JSON格式），如：
  ```json
  {
    "source": "time",
    "target": "timestamp",
    "type": "double",
    "format": "yyyy-MM-dd HH:mm:ss"
  }
  ```
- 对每个字段执行类型转换（字符串转整数/浮点/日期等），转换失败的行标记为错误。

### 4.5 数据校验
- 检查必填字段是否缺失。
- 检查数值范围是否合理（可选）。
- 记录错误行号及原因。

### 4.6 分块处理与存储
- 为避免内存溢出，设置每批次处理的行数（如10000行），边解析边批量插入数据库。
- 使用事务提高插入效率。

# 反辐射无人机仿真系统——试验数据采集模块开发实现方案

## 1. 总体架构

试验数据采集模块采用**插件化解析器 + 流水线处理**架构，支持多种文件格式、编码和映射规则。模块与系统其他部分（如数据库、分析模块）解耦，通过标准数据接口交互。

```plaintext
┌─────────────────┐      ┌─────────────────────────────────────┐      ┌─────────────────┐
│ 用户界面层      │ ──→ │        采集控制器                    │ ──→ │   数据存储层    │
└─────────────────┘      │  ┌──────────┐ ┌──────────┐        │      └─────────────────┘
                          │  │格式识别器│ │编码探测  │        │
                          │  └──────────┘ └──────────┘        │
                          │  ┌─────────────────────────────┐  │
                          │  │     解析器插件管理器         │  │
                          │  │  ┌────┐ ┌────┐ ┌────┐     │  │
                          │  │  │CSV │ │Excel│ │HDF5│ ... │  │
                          │  │  └────┘ └────┘ └────┘     │  │
                          │  └─────────────────────────────┘  │
                          │  ┌──────────┐ ┌──────────┐        │
                          │  │字段映射  │ │类型转换  │        │
                          │  └──────────┘ └──────────┘        │
                          └─────────────────────────────────────┘
```

## 2. 核心技术选型

| 功能 | 技术方案 | 说明 |
|------|----------|------|
| CSV/文本解析 | Qt `QTextStream` + 自定义状态机 | 支持自定义分隔符、引号转义 |
| Excel解析 | `QXlsx`（QtXlsxWriter）或 `libxlsxio` | 读取.xlsx文件，支持多工作表 |
| JSON解析 | Qt `QJsonDocument` | 支持对象和数组的扁平化 |
| XML解析 | Qt `QXmlStreamReader` | 基于SAX，内存友好 |
| HDF5解析 | HDF5 C++ 库 | 读取多维数据集 |
| MAT文件解析 | `matio` 库 | 支持MATLAB v7.3之前的格式 |
| 编码探测 | `uchardet`（Mozilla字符集探测库） | 自动检测文本编码 |
| 并发处理 | QtConcurrent / QThreadPool | 分块读取，批量入库 |
| 数据库 | SQLite / PostgreSQL（根据需求） | 存储清洗后数据 |

## 3. 核心功能实现

### 3.1 文件格式识别
- 优先使用文件扩展名：`.csv` -> CSV解析器，`.xlsx` -> Excel解析器，`.json` -> JSON解析器等。
- 若扩展名缺失或冲突，读取文件头部特征（magic number）判断：
  - HDF5：文件头包含“HDF”标识。
  - MAT：文件头包含“MATLAB”标识。
- 提供接口供用户手动指定格式。

### 3.2 编码探测（文本文件）
```cpp
QString detectEncoding(const QString& filePath) {
    QFile file(filePath);
    if (!file.open(QIODevice::ReadOnly)) return QString();
    QByteArray data = file.read(1024 * 10); // 读取前10KB
    file.close();

    uchardet_t ud = uchardet_new();
    uchardet_handle_data(ud, data.constData(), data.size());
    uchardet_data_end(ud);
    const char* charset = uchardet_get_charset(ud);
    QString encoding = QString::fromUtf8(charset);
    uchardet_delete(ud);
    return encoding;
}
```

### 3.3 解析器插件化
定义抽象基类 `DataParser`：
```cpp
class DataParser {
public:
    virtual ~DataParser() = default;
    virtual bool canParse(const QString& filePath) = 0; // 快速判断
    virtual ParseResult parse(const QString& filePath, 
                              const ParserOptions& options,
                              std::function<void(int)> progressCallback) = 0;
};
```
每种格式实现一个子类，在启动时注册到 `ParserManager`。

### 3.4 CSV/文本解析
- 使用 `QTextStream` 设置检测到的编码，逐行读取。
- 支持自定义分隔符（逗号、制表符、空格等），处理引号包裹的字段（`"a,b"`）。
- 使用状态机处理转义字符。

### 3.5 字段映射与类型转换
- 映射规则以JSON文件配置，示例：
```json
{
  "field_mappings": [
    { "source": "时间", "target": "timestamp", "type": "datetime", "format": "yyyy-MM-dd HH:mm:ss" },
    { "source": "经度", "target": "lon", "type": "double" },
    { "source": "纬度", "target": "lat", "type": "double" },
    { "source": "高度", "target": "alt", "type": "float" }
  ]
}
```
- 解析每一行时，按映射生成目标字段值，并使用类型转换器（`QVariant::toDouble()`、`QDateTime::fromString()`等）。
- 若转换失败，记录错误行，跳过该行继续处理。

### 3.6 大数据量分块处理
- 设置批处理大小（如5000行），每读满一批，启动一个异步任务进行类型转换和数据库插入。
- 使用 `QtConcurrent::run` 或 `QThreadPool` 实现并发，主线程继续读取下一批。
- 数据库插入使用事务，每批一个事务，显著提升性能。

```cpp
void processBatch(const QList<QStringList>& rawRows) {
    QSqlDatabase db = QSqlDatabase::database("import_conn");
    db.transaction();
    for (const auto& row : rawRows) {
        QSqlQuery query;
        // 插入数据...
    }
    db.commit();
}
```

### 3.7 进度反馈与预览
- 解析过程中通过信号发出进度（当前行/总行数），界面显示进度条。
- 完成第一批解析后，立即显示前20行预览，允许用户确认映射正确性。

## 4. 性能优化措施

| **优化项** | **实现方法** | **效果** |
|------------|--------------|----------|
| 分块读取 | 每次读固定大小到内存，避免一次性加载大文件 | 支持2GB以上文件，内存占用＜200MB |
| 批量插入 | 每批5000行开启一个事务 | 插入速度提升10倍以上 |
| 并发解析 | 读取线程与处理线程分离 | CPU利用率提升，总体时间缩短 |
| 缓存映射规则 | 预编译映射表达式，避免每行重复解析JSON | 减少30%时间 |
| 使用内存映射文件 | 对超大文件采用 `QFile::map` | 减少系统调用 |
| 编码探测缓存 | 同一文件只探测一次 | 减少开销 |

## 5. 异常处理与日志

- 每一条解析失败的行记录到日志文件，包含文件名、行号、原始数据和错误原因。
- 提供“下载错误报告”功能，供用户分析。
- 使用 `QMutex` 保护共享的日志队列，避免多线程竞争。
