## 4. 分析流程

三维态势显示的实现涉及数据组织、渲染优化和交互处理三大部分：

### 4.1 数据组织
- **地形数据**：将DEM和影像预处理为金字塔瓦片结构，便于动态加载不同细节层次。使用四叉树或八叉树管理瓦片，根据视点距离加载相应精度的瓦片。
- **实体模型**：模型库按类型索引，每个模型支持多级LOD，在渲染时根据距离选择合适细节。实例数据包括位置（经纬度转笛卡尔坐标）、姿态（四元数或欧拉角）、状态（用于材质变化）。

### 4.2 渲染流程
- **初始化**：创建OpenGL上下文，加载着色器，初始化场景图。
- **地形渲染**：根据视点动态构建地形网格，应用纹理，计算光照。
- **实体渲染**：对每个实体，根据其位置和姿态变换模型矩阵，绑定对应LOD模型，绘制。状态影响颜色或添加特效（如闪烁）。
- **标注渲染**：在屏幕上绘制实体标签、符号，采用公告牌技术使标签始终面向相机。
- **后处理**：添加雾效、太阳光晕等增强真实感。

### 4.3 交互处理
- **相机控制**：通过鼠标/键盘事件实现平移、旋转、缩放，支持第三人称跟随、自由飞行模式。
- **实体拾取**：通过射线检测或颜色缓冲区技术，识别用户点击的实体，高亮显示并展示详细信息。
- **量算工具**：计算两点间距离、多边形面积，并在场景中绘制辅助线。

### 4.4 性能优化
- **视锥体裁剪**：剔除视锥体外的地形瓦片和实体，减少绘制调用。
- **遮挡剔除**：利用硬件遮挡查询或预计算遮挡关系，剔除被遮挡的对象。
- **LOD动态切换**：根据距离选择合适的地形瓦片和实体模型细节。
- **实例化渲染**：对同种实体（如多架同型号无人机）使用实例化绘制，减少Draw Call。

## 2. 数据组织与预处理

### 2.1 地理信息数据处理

**地形数据**（DEM/高程）和**影像数据**（卫星图/航拍图）是三维场景的底座。

**实现方法**：
- **瓦片金字塔构建**：将原始GeoTIFF/HGT格式的地形数据预处理为多级分辨率的瓦片金字塔，采用四叉树索引。例如，将1000km×1000km的区域划分为层级0（全球）、层级1（大洲）到层级N（局部），每个瓦片大小为256×256像素。
- **格式转换**：地形瓦片转换为`terrain`格式（如Cesium的Quantized-Mesh），影像瓦片转换为PNG/JPEG格式并压缩。
- **坐标系转换**：实现WGS84经纬度坐标到WebGL世界坐标（笛卡尔坐标）的精准转换。

```javascript
// 使用Cesium加载地形和影像（示例）
const viewer = new Cesium.Viewer('cesiumContainer', {
    terrainProvider: Cesium.createWorldTerrain({
        requestWaterMask: true,
        requestVertexNormals: true
    }),
    imageryProvider: Cesium.createWorldImagery()
});

// 加载自定义本地地形
const terrainProvider = new Cesium.CesiumTerrainProvider({
    url: '/data/terrain/'
});
viewer.terrainProvider = terrainProvider;
```

### 2.2 三维模型资产处理

作战实体模型（无人机、雷达、防空系统等）需要经过**轻量化处理**以适应Web端加载。

**实现步骤**：
1. **建模与减面**：在Blender/3ds Max中建模，严格控制多边形数量。对于500个实体，单个实体面数建议控制在5000面以内。
2. **烘焙纹理**：将光影效果烘焙到纹理贴图中，减少实时计算压力。
3. **格式转换**：导出为glTF/GLB格式，并使用**DRACO压缩**进一步减小文件体积（可压缩至原大小的20-30%）。
4. **LOD模型生成**：为每个实体生成2-3个级别的LOD模型（高模、中模、低模），根据视距动态切换。

### 2.3 实体实例数据

仿真引擎实时输出的实体状态数据（位置、姿态、状态）通过WebSocket推送到前端。

## 3. 三维场景构建与渲染实现

### 3.1 地形场景构建

**实现流程**：
1. 初始化Cesium场景，设置光照、雾效等环境参数。
2. 根据仿真区域范围，动态加载对应区域的瓦片数据。
3. 支持多层影像叠加（如卫星底图+作战图层）。

**关键技术**：
- **视锥体裁剪**：只渲染当前视野内的瓦片。
- **遮挡剔除**：不渲染被地形遮挡的区域。
- **渐进式加载**：先显示低精度瓦片，逐步替换为高精度。

### 3.2 实体模型渲染

实体模型使用Three.js加载和渲染，并与Cesium场景融合。

**实现方法**：
```javascript
// 使用Three.js加载glTF模型
const loader = new THREE.GLTFLoader();
loader.load('/models/uav.glb', (gltf) => {
    const model = gltf.scene;
    // 设置LOD
    const lod = new THREE.LOD();
    lod.addLevel(model, 0);          // 近处使用原模型
    lod.addLevel(simpleModel, 200);  // 200米外使用简化模型
    
    // 将Three.js模型添加到Cesium场景
    const cesiumModel = viewer.entities.add({
        position: Cesium.Cartesian3.fromDegrees(125.3, 78.2, 3500),
        model: {
            gltf: '/models/uav.glb',
            scale: 1.0,
            minimumPixelSize: 16
        }
    });
});
```

**实例化渲染**：对于同类型多数量实体（如240架同型号无人机），使用**实例化渲染**大幅减少Draw Call。只需加载一次模型，通过矩阵变换批量绘制所有实例。

```javascript
// 实例化渲染示例
const instances = [];
for (let i = 0; i < 240; i++) {
    const matrix = new THREE.Matrix4();
    matrix.setPosition(x[i], y[i], z[i]);
    instances.push(matrix);
}
const instancedMesh = new THREE.InstancedMesh(geometry, material, 240);
instancedMesh.setMatrixAt(i, matrix);
```

### 3.3 态势符号与标注

在三维场景中绘制攻击箭头、侦察范围、雷达包络等态势符号。

**实现方法**：
- **攻击箭头**：采用直线箭头和攻击箭头生成算法，通过贝塞尔曲线或自定义几何体实现。
- **雷达包络**：使用八叉树+GPU加速策略，实时计算和绘制多雷达威力包络。
- **实体标签**：采用公告牌技术，使标签始终面向相机，显示实体ID、状态等信息。

### 3.4 状态驱动的视觉变化

根据实体状态实时改变显示效果：

| **状态** | **视觉表现** | **实现方式** |
|----------|--------------|--------------|
| 正常 | 默认材质 | 正常渲染 |
| 受损 | 红色闪烁/烟雾粒子 | 材质颜色插值 + 粒子系统 |
| 被选中 | 发光轮廓 | 后期处理（OutlinePass） |
| 发射武器 | 轨迹线 | 动态绘制贝塞尔曲线 |

## 4. 性能优化策略

针对500个实体、240架无人机的规模，必须采用多层次优化措施。

### 4.1 渲染优化

| **优化技术** | **实现方式** | **效果** |
|--------------|--------------|----------|
| **LOD（细节层次）** | 根据视距切换模型精细度 | 远距离实体面数减少90% |
| **实例化渲染** | 同类型实体一次绘制 | Draw Call从500降到10 |
| **视锥体裁剪** | 剔除视野外实体 | 同时渲染实体数减少70% |
| **遮挡剔除** | 不渲染被遮挡物体 | 进一步减少渲染负载 |
| **纹理压缩** | 使用KTX2/Basis格式 | 显存占用降低50% |
| **分块加载** | 3D Tiles流式加载 | 首屏加载时间≤3秒 |

### 4.2 数据优化

| **优化技术** | **实现方式** | **效果** |
|--------------|--------------|----------|
| **数据降采样** | 对实时数据流动态采样 | 数据传输量减少80% |
| **二进制传输** | 使用Protobuf/MessagePack代替JSON | 数据解析速度提升5倍 |
| **增量更新** | 只传输变化的数据 | 网络负载降低90% |
| **Web Workers** | 复杂数据解析放在后台线程 | 主线程不卡顿 |

### 4.3 内存管理

- **模型缓存**：同类型模型只加载一次，所有实例共享。
- **纹理共享**：公用纹理（如迷彩、金属）全局共享。
- **显存监控**：动态监测显存占用，超过阈值时自动降低纹理精度。

## 5. 交互功能实现

### 5.1 相机控制
- **自由漫游**：鼠标/键盘控制视角平移、旋转、缩放。

# 反辐射无人机仿真系统——效果显示模块开发实现方案

## 1. 总体架构

效果显示模块是三维态势显示的增强子模块，负责将仿真引擎产生的行动数据（轨迹、波束、毁伤、威力范围）实时转换为三维视觉元素。模块采用**数据驱动 + 特效系统**的架构，与三维渲染引擎紧密集成。

## 2. 各效果的具体实现技术

### 2.1 作战实体运行轨迹

**实现方法**：
- 使用Three.js的`Line`或`LineSegments`绘制轨迹线。每个实体维护一个环形缓冲区（如存储最近N个位置点），每帧更新几何体顶点。
- 轨迹支持颜色渐变（根据速度或时间），可使用`Line2`（支持宽线）实现平滑曲线。
- 可设置轨迹最大长度（如只显示最近60秒轨迹）或永久保存。

**代码示例**：
```javascript
class Trajectory {
    constructor(maxPoints = 100) {
        this.maxPoints = maxPoints;
        this.points = [];
        this.geometry = new THREE.BufferGeometry();
        this.material = new THREE.LineBasicMaterial({ color: 0x00ff00 });
        this.line = new THREE.Line(this.geometry, this.material);
    }
    
    addPoint(position, time) {
        this.points.push({ pos: position.clone(), time });
        if (this.points.length > this.maxPoints) this.points.shift();
        this.updateGeometry();
    }
    
    updateGeometry() {
        const positions = [];
        this.points.forEach(p => positions.push(p.pos.x, p.pos.y, p.pos.z));
        this.geometry.setAttribute('position', new THREE.Float32BufferAttribute(positions, 3));
        this.geometry.setDrawRange(0, this.points.length); // 避免显示不完整的线段
    }
}
```

**性能优化**：
- 对长轨迹进行抽稀（如每隔1秒取一个点）。
- 远距离轨迹简化或隐藏。
- 使用`LineSegments` + 索引减少重复顶点。

### 2.2 雷达扫描波束 / 导引头搜索波束

**实现方法**：
- 雷达扫描波束：使用扇形网格（`CylinderGeometry`或自定义形状）结合透明度动画，并绕垂直轴旋转模拟扫描。可用多个半透明层叠增加立体感。
- 导引头视场锥：使用圆锥几何体（`ConeGeometry`），半透明材质，指向目标方向。
- 波束位置和方向与父实体绑定，每帧更新矩阵。

**代码示例**：
```javascript
class RadarBeam {
    constructor() {
        const geometry = new THREE.ConeGeometry(5, 10, 32); // 半径、高度、分段
        const material = new THREE.MeshPhongMaterial({
            color: 0xff0000,
            transparent: true,
            opacity: 0.3,
            side: THREE.DoubleSide
        });
        this.mesh = new THREE.Mesh(geometry, material);
        // 波束尖端指向Z轴正方向，需旋转使圆锥指向正确
        this.mesh.rotation.x = Math.PI / 2; // 使圆锥指向Y轴？需根据实际坐标系调整
    }
    
    update(parentPosition, direction) {
        this.mesh.position.copy(parentPosition);
        this.mesh.quaternion.setFromUnitVectors(new THREE.Vector3(0, 1, 0), direction.clone().normalize());
    }
}
```

**动态扫描**：在`update`中改变波束的角度（如绕垂直轴旋转），实现扫描效果。

### 2.3 火力毁伤效果

**实现方法**：
- 使用粒子系统模拟爆炸瞬间的火光、碎片。爆炸时生成大量粒子，赋予初始速度、生命周期、颜色变化。
- 命中后残留火焰/烟雾：使用持续的粒子系统，位置固定在命中点，随时间消散。
- 目标外观变化：切换模型的材质（如增加烧灼纹理）或替换为受损模型。

**代码示例**：
```javascript
function createExplosion(position) {
    const particleCount = 200;
    const geometry = new THREE.BufferGeometry();
    const positions = new Float32Array(particleCount * 3);
    const velocities = [];
    for (let i = 0; i < particleCount; i++) {
        positions[i*3] = position.x;
        positions[i*3+1] = position.y;
        positions[i*3+2] = position.z;
        velocities.push(new THREE.Vector3(
            (Math.random() - 0.5) * 10,
            Math.random() * 15,
            (Math.random() - 0.5) * 10
        ));
    }
    geometry.setAttribute('position', new THREE.BufferAttribute(positions, 3));
    
    const material = new THREE.PointsMaterial({
        color: 0xff6600,
        size: 1.0,
        map: createParticleTexture() // 圆形渐变纹理
    });
    
    const particles = new THREE.Points(geometry, material);
    particles.userData = { velocities, life: 1.0 }; // 生命周期1秒
    return particles;
}
```

**烟雾特效**：使用公告牌（billboard）纹理，始终面向相机，配合透明度变化。

### 2.4 威力范围

**实现方法**：
- 球体范围（如导弹杀伤半径）：使用球体几何体（`SphereGeometry`），半透明材质，线框模式可选。
- 圆锥范围（如雷达探测扇区）：使用自定义网格生成扇形区域。
- 范围随实体移动而更新位置；静态范围（如基地防空圈）可预先放置。

**代码示例**：
```javascript
class PowerRange {
    constructor(radius, color = 0x0000ff) {
        this.geometry = new THREE.SphereGeometry(radius, 32, 16);
        this.material = new THREE.MeshPhongMaterial({
            color: color,
            transparent: true,
            opacity: 0.2,
            wireframe: false,
            side: THREE.DoubleSide
        });
        this.mesh = new THREE.Mesh(this.geometry, this.material);
    }
    
    setPosition(pos) {
        this.mesh.position.copy(pos);
    }
}
```

**优化**：低分辨率球体（减少分段数），远距离自动隐藏。
