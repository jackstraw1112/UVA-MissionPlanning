## 2. 关键术语解释

| **术语** | **解释** |
|----------|----------|
| **数据转换** | 将数据从原始形式转换为适合分析的形式，包括但不限于：规范化（标准化）、离散化、函数变换（如对数变换）、编码转换等。 |
| **标准化** | 指将数据按比例缩放，使其落入特定区间或具有特定统计分布，常用的有 Min-Max 归一化（将数据映射到[0,1]）和 Z-score 标准化（使数据均值为0，标准差为1）。 |
| **标准化算法** | 实现标准化的具体数学方法，如 Min-Max 归一化公式：`x' = (x - min) / (max - min)`，Z-score：`x' = (x - μ) / σ`。 |
| **多源数据整合** | 将来自不同来源、不同格式、不同粒度的数据合并为一个统一、一致的数据集。需要解决字段映射、实体对齐、冲突检测与解决等问题。 |
| **字段映射** | 确定不同数据源中哪些字段代表同一属性的过程，例如一个源中的“timestamp”和另一个源中的“time”映射到目标字段“time”。 |

## 4. 分析流程

### 4.1 数据标准化流程
1. 用户选择需要标准化的列，并指定标准化方法（Min-Max、Z-score、小数定标等）。
2. 系统扫描选定列，计算所需统计量（如 Min-Max 需要 min 和 max；Z-score 需要 mean 和 std）。
3. 对每一行数据应用转换公式，生成新值。
4. 可选地，将标准化后的列保存为新列，或替换原列。

### 4.2 多源数据整合流程
1. **数据加载**：从多个文件或数据库表读取数据。
2. **字段映射**：用户配置源字段到目标字段的映射表，例如：
   - 源1: `timestamp` → 目标 `time`
   - 源2: `record_time` → 目标 `time`
3. **实体对齐**：根据指定的键字段（如时间戳、ID）对记录进行匹配。支持内连接、外连接、左连接等。
4. **冲突解决**：当同一个键在多个源中出现且对应字段值不同时，按用户策略处理：
   - 取平均值（数值）
   - 取最新（时间戳）
   - 投票（分类数据）
   - 保留所有版本（标记源）
5. **去重**：整合后若有完全重复的行，可选择去重。
6. **输出**：生成统一的数据表。

# 反辐射无人机仿真系统——数据转换与整合模块开发实现方案

## 1. 总体架构

数据转换与整合模块采用**管道-过滤器**架构，支持多种标准化算法和灵活的多源整合策略。模块与数据预处理、数据管理模块解耦，通过标准数据接口交互。

```plaintext
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│ 数据加载   │ → │ 标准化    │ → │ 字段映射   │ → │ 对齐合并  │ → 整合数据
└────────────┘    └────────────┘    └────────────┘    └────────────┘
       ↑                  ↑                  ↑                  ↑
   多数据源          算法配置          映射规则          冲突策略
```

### 1.1 核心类设计
- **DataTransformer**：总控类，管理转换流程，支持标准化、编码转换。
- **Normalizer**：标准化算法基类，派生MinMaxNormalizer、ZScoreNormalizer等。
- **MultiSourceIntegrator**：多源整合引擎，负责字段映射、对齐、冲突解决。
- **FieldMapper**：管理源字段到目标字段的映射关系。
- **ConflictResolver**：处理数据冲突（平均、最新、投票等）。

## 3. 功能实现细节

### 3.1 数据标准化算法实现

#### 3.1.1 Min-Max 归一化
- 公式：`x_norm = (x - min) / (max - min)`
- 实现：先扫描列计算 min 和 max，再应用变换。
- 处理边界：当 max == min 时，所有值归一化为 0.5（或用户指定值）。

```cpp
void MinMaxNormalizer::normalize(QVector<double>& column) {
    double min = *std::min_element(column.begin(), column.end());
    double max = *std::max_element(column.begin(), column.end());
    double range = max - min;
    if (range < 1e-12) {
        std::fill(column.begin(), column.end(), 0.5);
    } else {
        for (double& val : column) {
            val = (val - min) / range;
        }
    }
}
```

#### 3.1.2 Z-score 标准化
- 公式：`x_std = (x - μ) / σ`
- 使用 Eigen 加速计算：
```cpp
void ZScoreNormalizer::normalize(QVector<double>& column) {
    Eigen::Map<Eigen::VectorXd> vec(column.data(), column.size());
    double mean = vec.mean();
    double std = std::sqrt((vec.array() - mean).square().sum() / (vec.size() - 1));
    if (std < 1e-12) {
        vec.setZero();
    } else {
        vec = (vec.array() - mean) / std;
    }
}
```

#### 3.1.3 其他标准化方法
- **小数定标**：除以 10^k，其中 k 为使所有值绝对值小于1的最小整数。
- **Log变换**：`x' = log(x + c)`，需处理负数。
- **Box-Cox变换**：可选实现。

### 3.2 多源数据整合

#### 3.2.1 字段映射配置
使用 JSON 定义映射关系：
```json
{
  "sources": [
    { "name": "radar_data", "type": "csv", "path": "radar.csv" },
    { "name": "uav_data", "type": "csv", "path": "uav.csv" }
  ],
  "mappings": [
    { "sourceField": "timestamp", "targetField": "time", "source": "radar_data" },
    { "sourceField": "time", "targetField": "time", "source": "uav_data" },
    { "sourceField": "position_x", "targetField": "x", "source": "radar_data" },
    { "sourceField": "pos_x", "targetField": "x", "source": "uav_data" }
  ],
  "joinKeys": ["time"],
  "conflictResolution": {
    "x": "average"  // 冲突时取平均
  }
}
```

#### 3.2.2 数据对齐
- 使用键字段进行连接，支持内连接、外连接等。
- 对于时间序列数据，可先排序后按时间窗口合并。
- 使用哈希表加速查找：将主数据源按键建立索引。

```cpp
void MultiSourceIntegrator::alignData() {
    // 构建主源索引
    QHash<QString, QVector<int>> index;
    for (int i = 0; i < m_primarySource.size(); ++i) {
        QString key = buildKey(m_primarySource[i], m_joinKeys);
        index[key].append(i);
    }
    
    // 遍历其他源，匹配键
    for (auto& otherSource : m_otherSources) {
        for (int i = 0; i < otherSource.size(); ++i) {
            QString key = buildKey(otherSource[i], m_joinKeys);
            if (index.contains(key)) {
                mergeRow(index[key], otherSource[i]);  // 合并到主源
            } else {
                handleUnmatched(otherSource[i]);       // 处理未匹配行
            }
        }
    }
}
```

#### 3.2.3 冲突解决策略
- **平均值**：适用于数值，多个源取平均。
- **最新值**：按时间戳取最新的记录。

- **保留所有**：增加一列标识源，保留多个版本。

## 4. 性能优化措施

| **优化项**         | **实现方法**                                                | **效果**                     |
|--------------------|-------------------------------------------------------------|------------------------------|
| 向量化计算         | 使用 Eigen 进行批量数值运算                                  | 比循环快 5~10 倍             |
| 并行列处理         | OpenMP 并行遍历各列                                          | 充分利用多核                 |
| 哈希索引           | 对连接键建立哈希表，查找 O(1)                                | 对齐速度提升 10 倍以上       |
| 分块处理           | 对超大数据集分块读取，每块独立处理                           | 内存可控                     |
| 列式存储           | 将数据按列存储在 `QVector` 中，提高缓存命中率                 | 统计分析加速 3 倍            |
| 惰性计算           | 只在必要时计算统计量（如标准化时的 min/max）                 | 减少不必要计算               |

## 6. 关键技术难点与解决方案

| **难点**                     | **解决方案**                                             |
|------------------------------|----------------------------------------------------------|
| 不同源数据格式不一致         | 统一转换为内部表结构（`DataTable`），屏蔽底层差异        |
