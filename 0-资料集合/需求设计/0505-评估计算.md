# 反辐射无人机仿真系统——作战效能评估模块评估计算功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 评估计算 |
| **需求标识**     | FR-EVAL-05 |
| **需求概述**     | 提供评估所需的核心计算功能，基于预设的指标体系、评估模型和仿真/试验数据，自动完成评估指标的计算和评估模型解算，生成评估结果。 |
| **需求来源**     | 系统总体设计要求 / 作战效能评估模块能力指标 |
| **前置条件**     | 1. 评估指标体系已构建，指标权重已设置。<br>2. 评估模型（如加权求和、层次分析综合、模糊综合评判等）已配置。<br>3. 评估指标数据采集模块已完成数据采集，提供了标准化的指标数据。<br>4. 仿真任务已完成或处于可计算状态。 |
| **触发条件**     | 1. 用户点击“开始评估”按钮。<br>2. 仿真任务结束后自动触发评估计算。<br>3. 定时任务或事件触发（如达到预设的评估时间点）。 |
| **退出条件**     | 1. 评估计算完成，生成评估结果并展示。<br>2. 用户手动取消计算。<br>3. 计算过程中发生严重错误，终止计算。 |
| **功能参与者**   | 评估计算引擎、指标体系库、数据采集模块、用户（查看结果） |
| **输入**         | 指标体系结构（包含指标节点、权重、计算公式）、评估模型参数（如聚合方式、阈值）、标准化后的指标数据（时间序列或统计值）。 |
| **输出**         | 评估结果报告，包含各指标得分、综合得分、置信区间（可选）、评估结论等。 |
| **处理流程**     | 1. **数据准备**：从数据采集模块获取当前评估任务所需的标准化指标数据，按时间范围或任务阶段筛选。<br>2. **指标计算**：对每个底层指标，根据其定义公式（可能涉及多个原始数据项）计算指标值。若已有采集数据直接可用，则直接读取。<br>3. **层次聚合**：按照指标体系层次结构，从底层向上逐层聚合计算。聚合方式由评估模型指定（如加权求和、几何平均、模糊算子等）。<br>4. **模型解算**：根据选定的评估模型（如多指标综合、TOPSIS、灰色关联等），对指标数据进行综合解算，得到最终的综合评估值或排序。<br>5. **结果生成**：将计算结果整理为评估报告，包含表格、图表、文字结论等。<br>6. **结果存储与展示**：将评估结果存入数据库，并在界面上以图表、仪表盘等形式展示。 |
| **成功判据**     | 1. 所有指标均能正确计算，无缺失或异常值。<br>2. 聚合过程符合指标体系层次关系和预设模型逻辑。<br>3. 生成的评估结果与手工验算（抽样）一致。<br>4. 计算结果合理，无数学错误。 |
| **异常处理**     | 1. 数据缺失：若某个指标所需数据缺失，可采用插值、默认值或标记为无效，并在结果中提示。<br>2. 除零错误：若计算公式中出现除零，捕获异常并记录，该指标结果置为无效。<br>3. 模型解算失败（如矩阵奇异）：回退到备用模型或提示用户检查模型参数。<br>4. 计算超时：增加进度显示，允许用户取消。 |
| **后置条件**     | 1. 评估结果持久化存储，可供后续查询、对比、导出。<br>2. 用户可查看详细的评估报告。 |
| **显示约束**     | 1. 评估结果应以可视化的方式呈现，如雷达图、柱状图、仪表盘、表格等。<br>2. 支持结果导出为PDF、Excel等格式。<br>3. 可展开查看每个指标的详细计算过程和数据来源。 |
| **其他约束**     | 1. 评估模型应支持插件式扩展，便于增加新算法。<br>2. 计算过程应记录日志，便于审计和调试。<br>3. 支持批处理多个任务的评估。 |

| **术语** | **解释** |

## 4. 分析流程

### 4.1 数据准备
- 从数据采集模块获取指定时间范围（或任务阶段）的标准化指标数据。
- 若指标体系包含动态指标（如随时间变化），需按时间窗口处理（如取均值、最大值等）。

### 4.2 指标计算（叶子节点）
- 对每个叶子指标，根据其公式计算。公式可能引用多个原始数据项。
- 使用表达式解析引擎（如 muParser）执行计算。
- 示例公式：`kill_prob = hit_count / launch_count`

### 4.3 层次聚合
- 从叶子节点向上，对每个父节点，根据其子节点的值和权重，按指定聚合方式计算。
- 聚合方式示例：
  - **加权求和**：`parent = sum(child_i * weight_i)`
  - **几何平均**：`parent = exp( sum( ln(child_i) * weight_i) )`
  - **取最大值/最小值**：用于某些约束指标。

### 4.4 模型解算
- 如果选用高级评估模型（如TOPSIS），则基于所有叶子指标值构造决策矩阵，计算理想解距离，得到综合排序。
- 若为模糊综合评判，则需根据隶属度函数和模糊算子计算。

### 4.5 结果生成
- 生成结构化的JSON结果，包含所有节点得分。
- 前端根据结果绘制雷达图、柱状图、仪表盘等。

# 反辐射无人机仿真系统——评估计算模块开发实现方案

## 1. 总体架构

评估计算模块采用**管道-过滤器架构**，将数据准备、指标计算、层次聚合、模型解算、结果生成串联为可配置的流水线。模块与指标体系管理、数据采集模块解耦，通过标准化接口交互。

```plaintext
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 数据准备     │ → │ 指标计算     │ → │ 层次聚合     │ → │ 模型解算     │ → 结果
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
       ↓                   ↓                   ↓                   ↓
   数据采集模块       表达式引擎         聚合策略工厂         算法库
```

### 1.1 核心类设计
- **EvaluationEngine**：总控类，管理计算流程。
- **DataProvider**：从数据采集模块获取标准化指标数据。
- **IndicatorCalculator**：解析指标公式，计算底层指标值。
- **AggregationStrategy**：抽象聚合策略接口，实现加权求和、几何平均等。
- **ModelSolver**：抽象模型解算接口，实现TOPSIS、模糊综合评判等。
- **ResultGenerator**：生成最终报告。

## 2. 核心数据流

### 2.1 输入数据
- **指标体系结构**：从数据库加载，表示为树形结构（节点ID、名称、权重、公式、子节点列表）。
- **标准化指标数据**：从数据采集模块获取，通常为时间序列或统计值，格式如：
```json
{
  "indicator_id": "kill_prob",
  "values": [0.82, 0.79, 0.85],
  "timestamps": [...]
}
```
- **评估模型参数**：用户在前端选择的模型类型及参数（如权重、阈值）。

### 2.2 计算流程
1. **数据准备**：根据评估任务的时间范围，从数据提供器获取所有叶子指标的数据，可能需进行聚合（如取均值、最大值）。
2. **指标计算**：对每个叶子节点，若其公式依赖多个原始数据项，则调用表达式引擎计算。
3. **层次聚合**：自底向上遍历指标树，对每个父节点调用聚合策略计算得分。
4. **模型解算**：若选择了高级评估模型，将叶子指标得分作为输入，运行相应算法得到综合得分或排序。
5. **结果生成**：将计算结果组装为JSON，供前端展示；同时生成报告文档。

## 3. 指标计算实现

### 3.1 表达式解析
- 采用 **muParser**（C++数学表达式解析库）执行公式计算。
- 公式示例：`hit_count / launch_count * 100`
- 变量值从数据采集模块动态获取。
- 支持自定义函数（如`if(cond, a, b)`、`max`、`min`）。

### 3.2 性能优化
- 预编译公式：每个指标首次计算时编译表达式，缓存AST，后续直接求值。
- 批量计算：一次获取所有所需变量值，避免多次查询数据源。

## 4. 层次聚合实现

### 4.1 聚合策略接口
```cpp
class IAggregationStrategy {
public:
    virtual double aggregate(const QVector<double>& childValues,
                             const QVector<double>& childWeights) = 0;
    virtual QString name() const = 0;
};
```

### 4.2 内置策略
- **加权求和**：`sum(child * weight)`
- **几何平均**：`exp(sum(ln(child) * weight))`
- **取最大值**：`max(child)`
- **取最小值**：`min(child)`
- **加权调和平均**：`sum(weight) / sum(weight/child)`

### 4.3 聚合过程
- 使用递归或栈遍历指标树。
- 对每个父节点，收集子节点得分和权重，调用当前策略计算。
- 若子节点尚未计算，先递归计算子节点。

### 4.4 并行化
- 同一层级的节点可并行计算，使用 `QtConcurrent::map` 或 OpenMP。
- 由于树结构依赖，只能对同层无依赖的节点并行。

## 5. 评估模型解算实现

### 5.1 模型解算器接口
```cpp
class IModelSolver {
public:
    virtual EvaluationResult solve(const QVector<IndicatorScore>& scores,
                                   const QVariantMap& params) = 0;
};
```

### 5.2 内置模型

#### 5.2.1 加权求和模型（已涵盖在聚合中，但也可作为最终综合）
- 直接使用层次聚合的顶层结果。

#### 5.2.2 TOPSIS（逼近理想解排序法）
- **步骤**：
  1. 构造决策矩阵（m个方案 × n个指标）。
  2. 向量规范化（归一化）。
  3. 确定正理想解和负理想解。
  4. 计算每个方案到理想解的距离。
  5. 计算相对贴近度。
- **实现**：使用Eigen库进行矩阵运算，加速计算。

#### 5.2.3 模糊综合评判
- 确定因素集、评语集。
- 构建隶属度矩阵（根据指标值计算隶属度）。
- 选择模糊算子（如加权平均型、主因素突出型）合成。
- 得到模糊向量，可进一步加权得到综合得分。

#### 5.2.4 灰色关联分析
- 计算关联系数矩阵。
- 求关联度，排序。

### 5.3 模型参数配置
- 前端传递模型类型和参数（如TOPSIS的权重、模糊评判的隶属度函数参数）。
- 后端根据参数实例化相应求解器。

## 6. 性能优化策略

| **优化项** | **实现方法** | **效果** |
|------------|--------------|----------|
| 缓存叶子指标值 | 避免重复计算同一指标 | 减少50%计算量 |
| 并行聚合 | 同层节点并行计算 | 加速2~4倍（多核） |
| 矩阵运算优化 | 使用Eigen库，开启编译器优化 | TOPSIS计算加速5倍 |
| 预编译公式 | 指标公式编译后缓存 | 表达式求值加速3倍 |
| 延迟计算 | 只在需要时计算（如展开树节点） | 提升UI响应 |
| 结果缓存 | 对相同输入缓存计算结果 | 多次查看无需重算 |

## 7. 与前后端的集成

### 7.1 前端调用
- 通过REST API或Qt信号槽触发评估计算。
- 请求参数包含：`task_id`、`model_type`、`model_params`。
- 返回计算结果（JSON），前端动态渲染雷达图、柱状图、表格。

### 7.2 数据存储
- 计算结果存入数据库表 `evaluation_result`，包含任务ID、指标树JSON快照、综合得分、生成时间。
- 便于历史对比和导出。
