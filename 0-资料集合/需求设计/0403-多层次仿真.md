## 3. 分析流程

### 3.1 多层次仿真的内涵与关系
- **交战级仿真**：以高保真度模拟每个作战单元的物理行为。例如，反辐射攻击中，导弹的六自由度运动、导引头探测模型、引信引爆逻辑均被详细建模；电子对抗中，干扰信号的传播、接收机烧穿效应等精确计算。步长通常在毫秒级，输出高密度轨迹数据。
- **任务级仿真**：聚焦于作战任务的宏观效果，如压制概率、目标毁伤数量、资源消耗等。实体行为用聚合模型描述（如兰彻斯特方程、蒙特卡洛法），步长在秒级，输出统计指标。

### 3.2 多层次调度机制
1. **模型选择与加载**：用户选择仿真层次后，模型管理器从库中检索对应粒度的模型。交战级使用详细模型（如六自由度导弹），任务级使用聚合模型（如基于概率的毁伤模型）。
2. **时间同步**：系统采用统一的主仿真时钟。交战级子步长可设为主步长的子倍数，在每个主步内进行多次迭代；任务级步长直接与主步长一致。若混合仿真（部分实体交战级，部分任务级），则所有实体按最小步长推进，但任务级实体可简化更新。
3. **聚合/解聚（可选）**：当需要从任务级聚焦到局部交战细节时，可将聚合实体解聚为多个详细实体；反之，当交战结束需宏观统计时，可将多个详细实体聚合成一个聚合实体。这要求模型具备多分辨率建模能力，且状态转换连续。

### 3.3 多场景综合仿真
- **场景定义**：每个场景包含特定的参战实体、规则和判定逻辑。例如，反辐射攻击场景包括无人机、目标雷达和防空火力；电子对抗场景包括干扰机、雷达和通信链路。
- **场景切换策略**：
  - **时间触发**：按预设时间轴依次激活场景（如0~300s电子对抗，300~600s空中打击）。
  - **事件触发**：基于仿真事件（如“雷达被摧毁”触发火力防空场景）。
  - **用户手动**：允许用户随时切换场景。
- **场景兼容性**：不同场景中的实体共享状态（如无人机在电子对抗后进入反辐射攻击），需确保状态一致。场景切换时，实体模型可保持不变，仅改变行为规则或交互对象。

### 3.4 数据输出与对比
- 交战级输出：高采样率的实体轨迹、事件日志，用于事后回放和详细分析。
- 任务级输出：统计图表（如任务完成百分比随时间变化）、资源消耗曲线。
- 对比报告：将交战级仿真结果聚合为任务级指标（如平均毁伤概率），与纯任务级仿真结果对比，评估聚合模型的精度，为模型校准提供依据。

## 5. 关键术语解释

| **术语** | **解释** |
|----------|----------|
| **交战级仿真** | 以高保真度模拟武器平台和目标的详细物理过程，包括运动学、传感器探测、武器制导、引信引爆等，通常步长很小，计算量大。 |
| **任务级仿真** | 聚焦于作战任务的宏观效果，使用聚合模型描述兵力损耗、任务完成概率等，步长较大，适合长时间、大范围的作战推演。 |
| **聚合模型** | 将多个详细实体的行为用统计或解析公式简化表示的模型，如兰彻斯特方程、概率分布函数。 |
| **解聚** | 从聚合模型还原为多个详细实体的过程，需保持状态一致（如兵力总数分解为个体位置）。 |
| **多场景综合** | 在一个仿真运行中，按时间或事件顺序切换多个不同性质的作战场景（如电子对抗、反辐射攻击），各场景间实体状态可连续。 |
| **反辐射攻击** | 利用反辐射无人机/导弹跟踪敌方雷达辐射并实施摧毁的作战样式。 |
| **火力防空** | 使用地空导弹、高炮等武器拦截来袭空中目标的作战。 |
| **电子对抗** | 使用电子干扰、欺骗等手段破坏敌方雷达、通信等电子设备的作战。 |

## 6. 开发实现方案

### 6.1 总体架构
采用**分层模块化**设计，核心模块包括：
- **仿真引擎**：负责时间推进、事件调度、实体管理。
- **多层次调度器**：管理仿真层次切换、模型粒度选择、场景切换。
- **模型管理器**：维护模型库，提供不同粒度的模型实例。
- **聚合/解聚器**：实现实体粒度的转换。
- **数据记录器**：异步记录仿真数据。

### 6.2 关键技术实现

#### 6.2.1 模型粒度支持
- 为每个实体类型定义两种模型：详细模型（`DetailedModel`）和聚合模型（`AggregateModel`）。它们继承自同一接口 `SimEntity`，实现 `step()` 方法。
- 模型管理器根据用户选择的层次，在创建实体时返回对应粒度的实例。例如，`EntityFactory::createEntity(type, level)`。

#### 6.2.2 时间同步与混合推进
- 采用主时钟驱动，基础步长设为最小步长（如0.001秒）。每个主步内，所有实体执行 `step(dt)`。
- 任务级实体可进行“聚合更新”：它们内部维护一个时间累计器，累计到其自身步长时才真正计算，否则直接返回，以减少计算量。
- 示例代码：
  ```cpp
  void AggregateEntity::step(double dt) {
      m_accumulated += dt;
      if (m_accumulated >= m_stepSize) {
          // 执行聚合计算，如更新兵力数量
          m_accumulated = 0;
      }
  }
  ```

#### 6.2.3 聚合/解聚算法
- **解聚**：将聚合实体（如一个防空营）分解为多个详细实体（如6部发射架）。解聚时需根据聚合状态（如剩余导弹数）合理分配个体状态，可采用空间分布算法（如随机位置、编队阵型）。
- **聚合**：将多个详细实体合并为一个聚合实体，需统计总兵力、平均位置、剩余资源等。聚合后保留原实体的统计信息，以备后续可能再次解聚。
- 实现时采用“聚合代理”模式：聚合实体内部维护一个详细实体列表，解聚时直接激活这些实体；聚合时则销毁详细实体，更新聚合状态。

#### 6.2.4 多场景综合调度
- 定义场景描述文件（XML/JSON），包含场景类型、参与实体、触发条件（时间/事件）、行为规则库。
- 场景调度器在仿真主循环中检查触发条件，当满足时，向仿真引擎发送 `SwitchSceneEvent`，引擎处理事件时更新实体行为规则（通过策略模式切换实体的 `Behavior` 对象）。

#### 6.2.5 数据记录与对比
- 交战级数据记录：使用环形缓冲区，由独立线程写入磁盘（如HDF5格式），避免阻塞仿真。
- 任务级数据记录：定时采样，记录关键指标（如剩余无人机数量、目标毁伤数）。
- 对比报告生成：仿真结束后，调用对比分析模块，将交战级数据聚合成任务级指标，计算两者误差，生成图表。

### 6.3 并行化策略
- 实体按空间区域分组，每组由独立线程更新（适用于交战级）。
- 任务级仿真本身计算量小，可采用单线程或简单并行。
- 聚合/解聚操作需全局同步，可放在主线程执行。

### 6.4 类设计（核心）

```cpp
// 实体基类
class SimEntity {
public:
    virtual void step(double dt) = 0;
    virtual void handleEvent(Event* e) = 0;
    // ... 属性
};

// 详细实体
class DetailedUAV : public SimEntity {
    // 六自由度运动、导引头模型...
};

// 聚合实体
class AggregateUAVSquadron : public SimEntity {
    int count;          // 无人机数量
    double avgPos;      // 平均位置
    // 按时间步更新count（如兰彻斯特方程）
};

// 场景调度器
class SceneScheduler {
    void checkAndSwitch(double currentTime, EventQueue& queue);
};

// 仿真引擎（扩展支持多层次）
class MultiLevelSimEngine {
    void run() {
        while (m_time < m_endTime) {
            // 处理事件
            processEvents();
            // 更新所有实体
            #pragma omp parallel for
            for (auto entity : m_entities) {
                entity->step(m_dt);
            }
            // 检查场景切换
            m_sceneScheduler.checkAndSwitch(m_time, m_eventQueue);
            m_time += m_dt;
        }
    }
};
```

### 6.5 性能优化措施
- 对聚合实体采用异步更新（当累计时间未达步长时不计算）。
- 使用空间索引（四叉树）加速交战级实体间的交互检测。
- 数据记录采用压缩二进制格式，减少I/O压力。
