# 需求描述
# 反辐射无人机仿真系统——工作参数规划模块功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 工作参数规划 |
| **需求标识**     | FR-WORKPARAM-01 |
| **需求概述**     | 具备作战行动及工作参数规划功能，能够根据目标特征、协同任务要求以及飞行路径等约束条件，自动规划反辐射无人机的作战行动及工作参数，包括导引头搜索模式及参数、引炸高度等，确保无人机在协同作战中有效执行任务。 |
| **需求来源**     | 系统总体设计要求 / 协同任务规划管理CSCI能力指标 |
| **前置条件**     | 1. 任务目标信息已确定，包含目标类型、辐射源特征（频率、重频、脉宽等）、威胁等级、防护能力等。<br>2. 协同任务要求已明确，如同时到达时间窗、压制持续时间、多机协同探测覆盖要求等。<br>3. 飞行路径已完成规划，包含每架无人机的三维航点序列及预计到达时间。<br>4. 无人机资源信息已知，包括型号、载荷类型（导引头类型、引信类型）、可配置参数范围（如导引头搜索模式、频率范围、角度范围、引炸高度范围等）。<br>5. 作战规则库已加载，包含不同目标特征与参数配置的映射规则。<br>6. 所有输入数据通过合法性校验。 |
| **触发条件**     | 1. 用户点击“工作参数规划”按钮。<br>2. 在任务分配或路径规划完成后自动触发（可选配置）。<br>3. 用户修改了目标特征、任务要求或飞行路径后手动触发重新规划。 |
| **退出条件**     | 1. 工作参数规划完成，结果保存并显示在界面。<br>2. 用户手动取消规划（若计算时间过长）。<br>3. 规划过程中发生严重错误，模块退出并报告错误。 |
| **功能参与者**   | 任务规划人员（用户）、目标数据库（提供目标特征）、路径规划模块（提供飞行路径）、协同任务管理模块（提供协同要求）、无人机资源数据库（提供载荷能力） |
| **输入**         | 目标特征参数（辐射源类型、工作频率、信号特征、威胁等级等）、协同任务要求（时间同步、覆盖要求、作战意图等）、飞行路径（每架无人机的三维航点序列及时间）、无人机载荷能力（导引头类型、引信类型、可配置参数范围）、作战规则库（不同情况下的参数配置规则） |
| **输出**         | 每架无人机的作战行动序列及工作参数配置（包括导引头搜索模式（如扫描、跟踪、静默）、搜索参数（扫描范围、中心频率、重频、开机时刻等）、引炸高度、引信工作方式（触发/近炸/延时）、其他载荷参数（如电子干扰样式、功率））、参数合理性验证报告 |
| **处理流程**     | 1. **数据解析与校验**：接收输入数据，检查完整性，构建目标-无人机映射关系。<br>2. **目标特征分析**：根据辐射源类型、频率、信号特征等，确定适用的导引头工作模式（如被动寻的、主动寻的、复合制导）。<br>3. **导引头参数规划**：结合飞行路径（相对目标的进入方向、距离、时间）优化导引头搜索参数，包括搜索角度范围、频率搜索范围、重频跟踪策略、开机时刻（确保在有效探测距离内开机）。<br>4. **引炸参数规划**：根据目标防护特性（硬防护/软防护）、协同任务要求（同时引爆、延时引爆、多次打击）计算最佳引炸高度、引信延时及工作方式。<br>5. **协同调整**：考虑多机协同约束（如多机同时开机的时间同步、覆盖重叠避免干扰、分时攻击），调整参数确保整体作战效能。<br>6. **规则校验**：利用作战规则库对生成的参数进行合理性检查，如是否符合载荷能力限制、是否满足安全要求（如引炸高度高于最低安全高度）。<br>7. **结果输出**：生成参数配置表，附带校验报告。 |
| **成功判据**     | 1. 每架无人机均获得一组合理的作战参数配置。<br>2. 所有参数均在无人机载荷能力范围内。<br>3. 参数配置符合目标特征和协同任务要求。<br>4. 通过作战规则库的合理性校验，无致命冲突。<br>5. 输出结果与输入约束一致，可被后续仿真或实装使用。 |
| **异常处理**     | 1. 输入数据缺失或格式错误：弹出提示框，高亮错误字段，禁止规划。<br>2. 目标特征与无人机载荷能力不匹配（如无对应导引头类型）：提示用户更换无人机型号或调整任务目标。<br>3. 无法在约束条件下找到可行参数（如引炸高度超出无人机性能）：给出警告，建议放宽约束或调整路径。<br>4. 规划算法超时：提示用户是否继续等待，允许手动终止。<br>5. 规则校验不通过时，输出具体原因和建议修改项。 |
| **后置条件**     | 1. 工作参数规划结果保存于任务规划方案中，可持久化存储（项目文件）。<br>2. 结果可供仿真推演、飞行控制模块调用。<br>3. 用户可导出参数表（如CSV、XML）用于外部系统加载。 |
| **显示约束**     | 1. 以表格形式展示每架无人机的详细参数，列包括：无人机ID、导引头模式、搜索中心频率、搜索带宽、扫描角度范围、开机时刻、引炸高度、引信延时、状态标识（正常/警告/错误）。<br>2. 支持按无人机ID、任务类型筛选查看。<br>3. 提供参数合理性可视化标识（如颜色标记异常项）。<br>4. 可在地图上叠加显示引炸点、导引头开机点等关键信息。<br>5. 汇总显示参数配置成功率、异常项数量等统计信息。 |
| **其他约束**     | 1. 支持多型号无人机混合编队的参数规划。<br>2. 参数规划应考虑时间协同，如多机同时到达时导引头开机时间同步。<br>3. 规划算法应具有可扩展性，可添加新的目标类型和载荷配置（基于规则库）。<br>4. 支持用户手动修改部分参数并锁定，后续重新规划时保持锁定值不变。<br>5. 需记录参数规划版本和规则库版本，便于追溯。 |

## 3. 分析流程

工作参数规划采用基于规则和优化相结合的方法，具体流程如下：

### 3.1 数据预处理与映射
1. **数据整合**：从任务管理模块获取目标列表，从路径规划模块获取每架无人机的航迹，建立无人机-目标对应关系（可能多架无人机攻击同一目标，或一架无人机攻击多个目标需按序处理）。
2. **参数范围提取**：根据无人机型号，从资源数据库获取导引头、引信的可配置参数范围（如频率下限/上限、扫描角度极限、引炸高度范围等）。

### 3.2 目标特征分析
1. **辐射源识别**：根据目标类型和工作频率，判断辐射源特性（如固定频率、频率捷变、重频参差等）。
2. **威胁等级评估**：结合目标威胁等级和防护能力，确定所需攻击的优先级和引信要求（如硬防护目标需近炸引信）。

### 3.3 导引头参数规划
1. **工作模式选择**：基于目标特征和无人机载荷类型，选择最合适的导引头工作模式（规则库驱动）。例如：
   - 固定频率辐射源 → 被动寻的，中心频率设为辐射源频率。
   - 频率捷变辐射源 → 宽带搜索或跟踪模式，搜索带宽覆盖捷变范围。
   - 需多机协同探测 → 部分无人机设为主动寻的，部分被动协同。
2. **搜索参数计算**：
   - **频率参数**：根据辐射源频率和捷变范围，设定导引头中心频率和搜索带宽。
   - **角度参数**：根据无人机进入目标的方位角和距离，计算导引头开机时的角度搜索范围。考虑目标位置误差和无人机导航误差，适当扩展搜索扇区。
   - **开机时刻**：根据无人机航迹和导引头有效作用距离，计算开机时刻，确保在有效距离内开机，同时考虑提前预热时间。公式：`t_on = t_arrival - (R_effective / v) - t_warmup`，其中 `R_effective` 为导引头最大作用距离，`v` 为接近速度。

### 3.4 引炸参数规划
1. **引炸高度确定**：
   - 对于硬防护目标，引炸高度应适当提高，使战斗部破片覆盖目标；对于软防护或无防护目标，可降低引炸高度。
   - 根据目标类型和防护能力，从规则库中查询推荐引炸高度，并约束在无人机允许范围内。
   - 若多机协同攻击同一目标，可设置不同引炸高度实现立体覆盖。
2. **引信工作方式**：
   - 触发引信：适用于直接命中，需考虑引信延时（确保穿入目标内部）。
   - 近炸引信：根据目标尺寸和引炸高度设定起爆距离。
   - 延时引信：用于同时引爆或多波次攻击，计算绝对延时时间。

### 3.5 协同调整
1. **时间同步**：若任务要求多机同时到达目标，需确保各无人机的导引头开机时刻、引信引爆时刻对齐。可通过调整速度或路径长度实现，但在参数规划阶段需输出精确的同步时间。
2. **电磁兼容**：若多机同时开机搜索，可能产生相互干扰，需错开工作频率或时间（如分时开机）。
3. **覆盖优化**：对于协同探测，调整各无人机的扫描角度范围，使覆盖区域重叠最小化，同时确保无盲区。

### 3.6 规则校验与输出
1. **合理性检查**：将生成的参数与载荷能力范围比较，若超出则发出警告并尝试修正（如降低性能要求或选择替代模式）。
2. **生成报告**：输出每架无人机的参数配置表，附带校验结果。
3. **可视化**：将关键参数（开机点、引炸点）叠加到地图显示。

## 4. 性能需求（支持240架无人机）

| **需求标识**        | **描述** | **指标** |

| FR-WORKPARAM-PERF-04 | 单架无人机参数规划平均时间（在并行环境下） | ≤0.1秒 |

| FR-WORKPARAM-PERF-06 | 内存占用（最大规模） | ≤2GB |
| FR-WORKPARAM-PERF-07 | 参数合理性校验时间 | ≤5秒（所有无人机） |
| FR-WORKPARAM-PERF-08 | 结果保存与加载时间（项目文件） | ≤3秒 |
| FR-WORKPARAM-PERF-09 | 支持并行计算（多线程） | 是，可利用OpenMP加速 |

### 补充说明
- **并行策略**：无人机之间的参数规划相对独立（除协同调整外），可采用多线程并行计算，显著提升效率。
- **规则库优化**：使用哈希表或内存数据库存储规则，实现快速查询。
- **可扩展性**：规划算法应支持动态添加新目标类型和载荷配置，无需修改核心代码。
- **硬件基准**：以上指标基于现代高性能服务器（如8核以上CPU，16GB内存）实现。

# 实现思路
# 反辐射无人机仿真系统——工作参数规划模块开发实现方案

## 1. 总体技术架构

针对240架反辐射无人机协同作战的工作参数规划需求，采用**分层解耦、规则驱动、并行计算**的架构。系统整体采用C++/Qt开发，核心算法库集成规则引擎、优化求解器和并行计算框架。

### 1.1 系统分层架构

| **层级** | **功能描述** | **技术选型** |
|----------|--------------|--------------|
| **应用层** | 用户交互、参数配置、结果可视化 | Qt Widgets, QCustomPlot |
| **业务逻辑层** | 目标特征分析、导引头参数规划、引炸参数规划、协同调整 | C++核心算法库 |
| **规则引擎层** | 基于规则的参数配置、合理性校验 | Drools规则引擎/Rete算法、JSON规则库 |
| **数据层** | 目标特征库、无人机载荷能力库、作战规则库、飞行路径数据 | SQLite, JSON, 自定义数据结构 |
| **并行计算层** | 多线程并行参数规划 | OpenMP, QtConcurrent |

### 1.2 开发路线

1. **数据结构定义**：建立 `TargetFeature`, `UAVPayload`, `FlightPath`, `WorkingParameters` 等核心数据类
2. **规则库构建**：收集整理目标特征与参数配置的映射规则，构建JSON格式规则库
3. **核心算法开发**：实现导引头参数计算、引炸高度解算等核心算法
4. **规则引擎集成**：集成规则引擎进行参数合理性校验

```cpp
public:
    ScanAngleRange calculate(const FlightPath& path, 
                             const TargetPosition& target,
                             double positionError) {
        // 获取无人机进入目标的方向角
        double bearing = calculateBearing(path.lastPoint(), target);
        
        // 计算目标位置不确定性引起的角度误差
        double angularError = atan2(positionError, 
                                     path.distanceToTarget());

        ScanAngleRange range;

        range.azimuthEnd = bearing + angularError + SAFETY_MARGIN;

        double elevation = calculateElevation(path.altitude(), target);
        range.elevationStart = elevation - ELEVATION_MARGIN;
        range.elevationEnd = elevation + ELEVATION_MARGIN;
        
        return range;
    }
};
```

### 2.4 多机协同调整算法

针对240架无人机的大规模协同，采用**分层协同优化**策略：

```cpp
class CooperativeAdjustment {
public:
    void adjustParameters(QVector<UAVParams>& allParams,
                         const MissionRequirements& mission) {
        // 阶段1：时间同步调整（一致性算法）
        synchronizeTiming(allParams, mission.simultaneousWindow);
        
        // 阶段2：电磁兼容调整
        mitigateInterference(allParams);
        
        // 阶段3：覆盖优化
        optimizeCoverage(allParams, mission.coverageRequirement);
    }
    
private:
    // 一阶一致性算法实现时间同步
    void synchronizeTiming(QVector<UAVParams>& params, double window) {
        // 建立通信拓扑（基于空间邻近性）
        Graph commGraph = buildProximityGraph(params);
        
        // 迭代一致性协议
        for (int iter = 0; iter < MAX_ITER; ++iter) {
            #pragma omp parallel for
            for (int i = 0; i < params.size(); ++i) {
                double sum = 0;
                int neighborCount = 0;
                
                for (int neighbor : commGraph.neighbors(i)) {
                    sum += params[neighbor].activationTime;
                    neighborCount++;
                }
                
                if (neighborCount > 0) {
                    // 一致性更新
                    params[i].activationTime = 
                        (1 - ALPHA) * params[i].activationTime + 
                        ALPHA * (sum / neighborCount);
                }
            }
            
            // 检查收敛
            if (checkConvergence(params)) break;
        }
    }
    
    // 电磁兼容调整（分频/分时）
    void mitigateInterference(QVector<UAVParams>& params) {
        // 检测频率冲突
        auto conflicts = detectFrequencyConflicts(params);
        
        // 对冲突组进行频率错开
        for (auto& group : conflicts) {
            assignFrequencyHops(group);
        }
        
        // 若频率资源不足，采用分时工作
        if (hasResidualConflicts(conflicts)) {
            scheduleTimeDivision(params);
        }
    }
};
```

## 三、导引头参数规划

### 3.1 导引头工作模式选择

根据目标特征和任务类型，选择导引头工作模式：

| 目标特征 | 推荐工作模式 | 适用场景 |
|---------|-------------|---------|
| 固定频率雷达 | 单频点跟踪 | 精确打击 |
| 频率捷变雷达 | 宽带搜索+跟踪 | 抗干扰打击 |
| 间歇工作雷达 | 周期扫描+锁定 | 截获概率优化 |
| 多目标环境 | 边搜索边跟踪 | 区域压制 |
| 未知目标 | 全频段搜索 | 侦察任务 |

### 3.2 频率参数规划

频率参数规划基于目标雷达信号特征和导引头性能：

- **中心频率**：设置为目标雷达工作频率
- **搜索带宽**：根据目标频率捷变范围确定（通常为捷变范围×1.2）
- **频率步进**：根据扫描速度和截获概率要求确定

### 3.3 角度搜索范围规划

根据飞行路径和目标位置计算导引头开机时的角度搜索范围：

- **水平搜索范围**：根据无人机进入方向和目标位置不确定性确定
- **垂直搜索范围**：根据目标高度和无人机高度确定
- **搜索扇区**：结合雷达主瓣方向和盲区优化

### 3.4 开机时刻计算

开机时刻是导引头开始工作的关键参数，需确保在有效探测距离内开机：

```
t_on = t_arrival - (R_effective / v_closure) - t_warmup

其中：
- t_arrival：预计到达目标时间
- R_effective：导引头有效作用距离
- v_closure：接近速度
- t_warmup：导引头预热时间
```

### 3.5 导引头参数约束

| 参数 | 约束条件 | 优化目标 |
|------|---------|---------|
| 搜索带宽 | ≤ 导引头最大带宽 | 覆盖目标频率范围 |
| 扫描角度 | ≤ 导引头最大扫描角 | 覆盖目标可能位置 |
| 开机距离 | ≥ 最小截获距离 | 确保可靠截获 |
| 预热时间 | ≥ 导引头预热要求 | 提前准备 |

## 四、引信参数规划

### 4.1 引信工作方式选择

根据目标类型、防护能力和攻击方式选择引信工作方式：

| 目标类型 | 防护能力 | 攻击方式 | 推荐引信方式 |
|---------|---------|---------|-------------|
| 雷达天线 | 无防护 | 俯冲攻击 | 近炸引信 (5-8m) |
| 雷达方舱 | 硬防护 | 俯冲攻击 | 碰炸+延时 (20ms) |
| 指挥中心 | 硬防护 | 水平攻击 | 碰炸+延时 (30ms) |
| 电源车 | 软防护 | 俯冲攻击 | 碰炸 |
| 通信天线 | 无防护 | 俯冲攻击 | 近炸引信 (3-5m) |
| 机动目标 | 软防护 | 迎面攻击 | 近炸引信 (8-10m) |

### 4.2 引炸高度计算

引炸高度根据目标特性、攻击角度和战斗部威力确定：

```
h_burst = f(target_type, attack_angle, warhead_type)

对于硬防护目标：
h_burst = 0 (碰炸) + penetration_depth

对于软防护目标：
h_burst = optimal_height(warhead) × sin(attack_angle)

对于无防护目标：
h_burst = 5-15m (视战斗部而定)
```

### 4.3 延时设置

延时引信用于穿透目标壳体后爆炸：

```
t_delay = penetration_depth / v_remaining + fuzing_delay

其中：
- penetration_depth：预期穿透深度
- v_remaining：剩余速度
- fuzing_delay：引信固有延迟
```

### 4.4 引信参数约束

| 参数 | 约束条件 | 优化目标 |
|------|---------|---------|
| 炸高 | 在引信可调范围内 | 最大化杀伤效果 |
| 延时 | 在引信可调范围内 | 确保穿透后爆炸 |
| 灵敏度 | 根据目标硬度调整 | 可靠起爆 |

## 五、作战行动序列规划

### 5.1 行动阶段划分

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 静默   │ → │ 搜索   │ → │ 跟踪   │ → │ 攻击   │ → │ 评估   │
│ 巡航   │    │ 阶段   │    │ 阶段   │    │ 阶段   │    │ 阶段   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### 5.2 各阶段工作参数

| 阶段 | 导引头状态 | 发射状态 | 机动方式 | 持续时间 |
|------|-----------|---------|---------|---------|
| **静默巡航** | 关机 | 无 | 经济巡航 | 路径规划决定 |
| **搜索阶段** | 搜索模式 | 无 | 搜索机动 | 直至截获 |
| **跟踪阶段** | 跟踪模式 | 待命 | 跟踪机动 | 3-10秒 |
| **攻击阶段** | 锁定模式 | 发射 | 俯冲攻击 | 10-30秒 |
| **评估阶段** | 可选开机 | 无 | 侦察机动 | 5-15秒 |

### 5.3 状态转移条件

```
静默巡航 → 搜索阶段: 到达搜索区域或接收到搜索指令
搜索阶段 → 跟踪阶段: 信号强度 > 截获阈值
跟踪阶段 → 攻击阶段: 进入攻击窗口且目标确认
攻击阶段 → 评估阶段: 引信引爆后
评估阶段 → 返航: 评估完成或续航不足
```

### 5.4 行动时序生成

行动序列规划器根据飞行路径和目标位置，生成精确的行动时间点：

```
T0: 导引头开机 (进入搜索区域前30秒)
T1: 进入搜索区域
T2: 首次截获目标
T3: 转入跟踪模式
T4: 开始俯冲攻击
T5: 引信引爆
T6: 评估侦察开始
T7: 评估结束
```

### 7.1 规则库结构

```
规则库:
├── 目标类型规则

│   ├── IF 目标类型 = 预警雷达 THEN 导引头模式 = 宽带搜索

├── 防护能力规则
│   ├── IF 防护能力 = 硬防护 THEN 引信方式 = 碰炸+延时
│   ├── IF 防护能力 = 软防护 THEN 引信方式 = 近炸
│   └── IF 防护能力 = 无防护 THEN 引信方式 = 近炸/碰炸
│
├── 攻击角度规则
│   ├── IF 攻击角度 < 30° THEN 引炸高度 = 低
│   ├── IF 攻击角度 30-60° THEN 引炸高度 = 中
│   └── IF 攻击角度 > 60° THEN 引炸高度 = 高
│
└── 协同角色规则
    ├── IF 角色 = 主攻 THEN 发射功率 = 高
    ├── IF 角色 = 诱饵 THEN 发射功率 = 中
    └── IF 角色 = 干扰 THEN 发射功率 = 连续
```

```cpp
class AngleSearchRangeCalculator {
public:
    struct ScanAngles {
        double azStart, azEnd;
        double elStart, elEnd;
        double scanRate;  // 扫描速率 (deg/s)
    };
    
    ScanAngles calculate(const FlightPath& path,
                         const Point& targetPos,
                         double positionError,
                         const SeekerCapability& seeker) {
        ScanAngles angles;
        
        // 获取无人机进入目标的方向
        double bearing = calculateBearing(path.waypoints.back(), targetPos);
        
        // 计算目标位置不确定性引起的角度误差
        double angularError = atan2(positionError,
                                     path.distanceToTarget);
        angularError = angularError * 180 / M_PI;  // 转为度
        
        // 水平搜索范围 = 进入方向 ± 误差 ± 安全裕量
        double safetyMargin = 5.0;  // 5度安全裕量
        angles.azStart = bearing - angularError - safetyMargin;
        angles.azEnd = bearing + angularError + safetyMargin;
        
        // 归一化到0-360度
        angles.azStart = normalizeAngle(angles.azStart);
        angles.azEnd = normalizeAngle(angles.azEnd);
        
        // 如果跨过0度，调整
        if (angles.azStart > angles.azEnd) {
            angles.azEnd += 360;
        }
        
        // 水平扫描角约束
        double scanWidth = angles.azEnd - angles.azStart;
        if (scanWidth > seeker.maxScanAzimuth) {
            // 超出最大扫描角，以进入方向为中心缩小
            double excess = scanWidth - seeker.maxScanAzimuth;
            angles.azStart += excess / 2;
            angles.azEnd -= excess / 2;
        }
        
        // 俯仰角范围
        double elevation = calculateElevation(path.altitude(),
                                             targetPos.z);
        double elevError = angularError * 0.5;  // 俯仰误差较小
        
        angles.elStart = elevation - elevError - safetyMargin;
        angles.elEnd = elevation + elevError + safetyMargin;
        
        // 俯仰角约束
        if (angles.elStart < -90) angles.elStart = -90;
        if (angles.elEnd > 90) angles.elEnd = 90;
        
        // 扫描速率 (确保在一个扫描周期内覆盖)
        double timeToTarget = path.remainingTime;
        angles.scanRate = scanWidth / min(timeToTarget, 10.0);
        
        return angles;
    }
    
private:
    double normalizeAngle(double angle) {
        while (angle < 0) angle += 360;
        while (angle >= 360) angle -= 360;
        return angle;
    }
    
    double calculateElevation(double uavAlt, double targetAlt) {
        double deltaAlt = targetAlt - uavAlt;
        double deltaDist = 1000;  // 简化的水平距离
        return atan2(deltaAlt, deltaDist) * 180 / M_PI;
    }
};
```

```cpp
class ActivationTimeCalculator {
public:
    double calculateActivationTime(const FlightPath& path,
                                   double seekerMaxRange,
                                   double warmupTime,
                                   double closureSpeed) {
        // 寻找导引头进入有效作用距离的时刻
        double activationTime = path.arrivalTime;
        
        // 从后向前遍历航点
        for (int i = path.waypoints.size() - 1; i >= 0; i--) {
            double distanceToTarget = distance(path.waypoints[i],
                                              path.targetPosition);
            
            if (distanceToTarget <= seekerMaxRange) {
                // 计算从该点到目标的飞行时间
                double timeToTarget = distanceToTarget / closureSpeed;
                
                // 开机时刻 = 到达时间 - 飞行时间 - 预热时间
                activationTime = path.arrivalTime - timeToTarget - warmupTime;
                
                // 确保开机不早于路径起点
                if (activationTime < path.startTime) {
                    activationTime = path.startTime;
                }
                
                break;
            }
        }
        
        return activationTime;
    }
    
    // 考虑截获概率的优化版本
    double optimizeWithCaptureProbability(
        const FlightPath& path,
        const TargetFeature& target,
        const SeekerCapability& seeker) {
        
        double bestTime = path.arrivalTime;
        double bestProb = 0;
        
        // 在时间窗口内搜索最优开机时刻
        double searchStart = path.startTime;
        double searchEnd = path.arrivalTime;
        
        for (double t = searchStart; t <= searchEnd; t += 1.0) {
            // 计算在t时刻开机的截获概率
            double prob = estimateCaptureProbability(t, path, target, seeker);
            
            if (prob > bestProb) {
                bestProb = prob;
                bestTime = t;
            }
        }
        
        return bestTime;
    }
    
private:
    double estimateCaptureProbability(double time,
                                      const FlightPath& path,
                                      const TargetFeature& target,
                                      const SeekerCapability& seeker) {
        // 简化的截获概率模型
        Point posAtTime = path.getPositionAtTime(time);
        double distance = distance(posAtTime, path.targetPosition);
        double angleDiff = calculateAngleDiff(posAtTime, path);
        
        double prob = 1.0;
        
        // 距离因素
        if (distance > seeker.maxDetectionRange) {
            prob *= exp(-(distance - seeker.maxDetectionRange) / 1000);
        }
        
        // 角度因素
        prob *= exp(-angleDiff * angleDiff / (15 * 15));
        
        // 目标信号因素
        prob *= target.rcs / 10.0;
        
        return min(prob, 1.0);
    }
};
```

```cpp
class TimelineOptimizer {
public:
    void optimizeTimeline(vector<ActionPoint>& actions,
                          const FlightPath& path) {
        // 确保时间顺序
        sort(actions.begin(), actions.end(),
             [](auto& a, auto& b) { return a.time < b.time; });
        
        // 调整动态时间点
        double lastFixedTime = path.startTime;
        
        for (auto& action : actions) {
            if (action.time < 0) {
                // 动态时间点，需要根据条件设置
                action.time = lastFixedTime + estimateDuration(action);

            lastFixedTime = action.time;
        }
        
        // 冲突检测
        for (int i = 0; i < actions.size() - 1; i++) {
            if (actions[i+1].time - actions[i].time < 0.1) {
                // 时间太近，需要错开
                actions[i+1].time += 0.1;
            }
        }
    }
```

```cpp
                if (freqOverlap(params[i].seeker, params[j].seeker)) {
                    conflicts.push_back({i, j});
                }
            }
        }
        
        // 解决冲突
        for (auto& conf : conflicts) {
            resolveFrequencyConflict(params[conf.first],
                                     params[conf.second]);
        }
    }
    
private:
    bool freqOverlap(const SeekerParams& a, const SeekerParams& b) {
        double a_start = a.centerFreq - a.bandwidth/2;
        double a_end = a.centerFreq + a.bandwidth/2;
        double b_start = b.centerFreq - b.bandwidth/2;
        double b_end = b.centerFreq + b.bandwidth/2;
        
        return !(a_end < b_start || a_start > b_end);
    }
    
    void resolveFrequencyConflict(SeekerParams& a, SeekerParams& b) {
        // 策略1: 频率偏移
        if (a.bandwidth + b.bandwidth < 200) {
            double shift = (a.bandwidth + b.bandwidth) / 2 + 10;
            a.centerFreq -= shift / 2;
            b.centerFreq += shift / 2;
        }
        // 策略2: 时分工作
        else {
            a.timeShare = 0.5;
            b.timeShare = 0.5;
            a.timeShareStart = 0;
            b.timeShareStart = 0.5;
        }
    }
};
```

```cpp
class AngleCoordinator {
public:
    void coordinateAngles(vector<UavWorkParams>& params,
                          const Point& targetPos) {
        // 确保攻击角度分散
        vector<double> angles;
        for (auto& p : params) {
            angles.push_back(p.approachAngle);
        }
        
        // 检查角度间隔
        sort(angles.begin(), angles.end());
        
        for (int i = 0; i < angles.size() - 1; i++) {
            double interval = angles[i+1] - angles[i];
            if (interval < 30) {
                // 间隔太小，调整
                double adjust = (30 - interval) / 2;
                angles[i] -= adjust;
                angles[i+1] += adjust;
            }
        }
        
        // 写回参数
        for (int i = 0; i < params.size(); i++) {
            params[i].approachAngle = normalizeAngle(angles[i]);
        }
    }
    
private:
    double normalizeAngle(double angle) {
        while (angle < 0) angle += 360;
        while (angle >= 360) angle -= 360;
        return angle;
    }
};
```

```cpp
class TimeCoordinator {
public:
    void coordinateTimes(vector<UavWorkParams>& params,
                         const MissionRequirements& mission) {
        if (mission.timeSyncType == SIMULTANEOUS) {
            coordinateSimultaneous(params, mission.targetTime);
        }
        else if (mission.timeSyncType == SEQUENTIAL) {
            coordinateSequential(params, mission.interval);
        }
        else if (mission.timeSyncType == WAVE) {
            coordinateWaves(params, mission.waveCount, mission.waveInterval);
        }
    }
    
private:
    void coordinateSimultaneous(vector<UavWorkParams>& params,
                                double targetTime) {
        // 所有无人机调整速度，使攻击时间一致
        double maxETA = 0;
        for (auto& p : params) {
            maxETA = max(maxETA, p.eta);
        }
        
        for (auto& p : params) {
            double timeDiff = maxETA - p.eta;
            if (timeDiff > 0) {
                // 需要减速
                p.cruiseSpeed *= p.eta / maxETA;
            }
        }
    }
    
    void coordinateSequential(vector<UavWorkParams>& params,
                              double interval) {
        // 按优先级排序
        sort(params.begin(), params.end(),
             [](auto& a, auto& b) { return a.priority > b.priority; });
        
        double baseTime = params[0].eta;
        for (int i = 1; i < params.size(); i++) {
            params[i].eta = baseTime + i * interval;
        }
    }
};
```

```cpp
class Rule {
public:
    struct Condition {
        string parameter;
        string op;  // "eq", "gt", "lt", "in", etc.
        Value value;
    };
    
    struct Action {
        string targetParameter;
        Value value;
    };
    
    string name;
    vector<Condition> conditions;
    vector<Action> actions;
    double confidence;  // 规则置信度 0-1
    
    bool evaluate(const Context& ctx) {
        for (auto& cond : conditions) {
            Value paramValue = ctx.get(cond.parameter);
            if (!compare(paramValue, cond.op, cond.value)) {
                return false;
            }
        }
        return true;
    }
};

class RuleBase {
private:
    vector<Rule> rules;
    
public:
    void loadRules(const string& ruleFile) {
        // 从JSON加载规则
        QJsonDocument doc = loadJson(ruleFile);
        for (auto ruleJson : doc.array()) {
            Rule r;
            r.name = ruleJson["name"].toString();
            // 解析conditions和actions
            rules.push_back(r);
        }
    }
    
    vector<Action> infer(const Context& ctx) {
        vector<Action> results;
        
        for (auto& rule : rules) {
            if (rule.evaluate(ctx)) {
                // 规则触发
                for (auto& action : rule.actions) {
                    action.confidence = rule.confidence;
                    results.push_back(action);
                }
            }
        }
        
        // 冲突解决
        return resolveConflicts(results);
    }
    
private:
    vector<Action> resolveConflicts(vector<Action>& actions) {
        map<string, Action> bestActions;
        
        for (auto& action : actions) {
            string key = action.targetParameter;
            if (!bestActions.count(key) ||
                action.confidence > bestActions[key].confidence) {
                bestActions[key] = action;
            }
        }
        
        vector<Action> result;
        for (auto& pair : bestActions) {
            result.push_back(pair.second);
        }
        return result;
    }
};
```
