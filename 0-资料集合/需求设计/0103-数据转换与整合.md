| **术语** | **解释** |
|----------|----------|
| **重复数据消除** | 检测并删除数据集中完全重复或根据关键字段判定为重复的记录，以避免统计偏差和冗余存储。 |

## 4. 分析流程

### 4.1 重复数据消除
- 用户指定关键字段组合（如 `[时间, 无人机ID]`）。
- 使用SQL `GROUP BY` 或内存中的哈希表检测重复。
- 根据用户选择保留策略（第一条/最后一条/合并）执行删除。

### 4.2 缺失值处理
- 逐列检测空值（`NULL` 或特定占位符如 `-9999`）。
- 对数值型列，可选择均值、中位数、固定值填充。
- 对时间序列数据，可使用前向填充（`ffill`）或后向填充（`bfill`）。
- 对类别型列，可使用众数填充或“未知”类填充。

### 4.3 异常值检测与处理
- **标准差法**：计算均值 `μ` 和标准差 `σ`，标记 `|x - μ| > kσ` 为异常（k通常取3）。
- **四分位距法**：计算第一四分位数 Q1 和第三四分位数 Q3，IQR = Q3 - Q1，标记 `x < Q1 - 1.5IQR` 或 `x > Q3 + 1.5IQR` 为异常。
- **处理**：删除、用边界值替换（如 `Q1 - 1.5IQR` 或 `Q3 + 1.5IQR`）、或保留并标记。

### 4.4 数据格式标准化
- 日期时间：统一为 `YYYY-MM-DD HH:MM:SS` 格式。
- 单位转换：预定义转换规则（如 `英尺 * 0.3048 -> 米`）。
- 文本编码：统一转换为UTF-8。

# 反辐射无人机仿真系统——数据预处理模块开发实现方案

## 1. 总体架构

数据预处理模块作为试验数据处理的核心组件，采用**管道-过滤器**架构，将重复数据消除、缺失值处理、异常值检测、格式标准化等步骤串联为可配置的数据清洗流水线。模块与数据采集、数据管理模块解耦，通过标准数据接口交互。

```plaintext
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│ 数据加载   │ → │ 重复消除   │ → │ 缺失值处理 │ → │ 异常值处理 │ → │ 标准化    │ → 清洗后数据
└────────────┘    └────────────┘    └────────────┘    └────────────┘    └────────────┘
       ↓                                                                         ↓
   原始数据表                                                             清洗后数据表
```

### 1.1 核心类设计
- **DataPreprocessor**：总控类，管理预处理流水线，支持配置各步骤参数。
- **DuplicateRemover**：重复数据检测与消除。
- **MissingHandler**：缺失值处理（删除、填充等）。
- **OutlierDetector**：异常值检测（标准差法、IQR法）。
- **OutlierHandler**：异常值处理（删除、替换、标记）。
- **Normalizer**：数据格式标准化（日期、单位、编码）。
- **ProgressReporter**：进度反馈与日志记录。

## 3. 功能实现细节

### 3.1 数据加载
- 从数据库读取数据时，使用分页或流式读取，避免一次性加载全部数据到内存。
- 数据以`QVector<QVector<QVariant>>`或自定义结构存储于内存，支持随机访问。
- 对于超大文件（如CSV），可边读边处理，减少内存占用。

### 3.2 重复数据消除
- **检测**：用户指定关键字段索引，构建哈希表（`QHash`）记录已出现的键值。遍历数据行，若键已存在，则根据保留策略决定是否删除当前行。
- **保留策略**：保留第一条、最后一条、或合并（需用户定义合并规则，如取均值）。
- **性能**：哈希表查找O(1)，整体O(n)。

```cpp
QHash<QString, bool> seenKeys;
for (int i = 0; i < data.size(); ++i) {
    QString key = buildKey(data[i], keyColumns);
    if (seenKeys.contains(key)) {
        if (keepStrategy == KeepFirst) {
            markForDeletion(i);  // 标记删除当前行
        } else if (keepStrategy == KeepLast) {
            // 需要后续处理，此处先标记之前保留的行
        }
    } else {
        seenKeys.insert(key, true);
    }
}
```

### 3.3 缺失值处理
- **检测**：遍历每列，检查是否为`NULL`或用户指定的缺失标记（如`-9999`）。
- **处理方法**：
  - **删除行**：收集所有含缺失值的行号，批量删除（倒序遍历避免索引变化）。
  - **统计填充**：先计算非缺失值的均值/中位数/众数（需扫描一次），再填充。
  - **前向/后向填充**：适用于时间序列数据，需按时间排序后处理。
  - **线性插值**：对连续数值列，用前后有效值线性插值。

```cpp
// 均值填充示例
double sum = 0; int count = 0;
for (auto& row : data) {
    if (!row[col].isNull()) {
        sum += row[col].toDouble();
        count++;
    }
}
double mean = sum / count;
for (auto& row : data) {
    if (row[col].isNull()) row[col] = mean;
}
```

### 3.4 异常值检测与处理

#### 3.4.1 标准差法
- 计算均值和标准差，使用Eigen加速：
```cpp
Eigen::VectorXd values = extractColumn(data, col);
double mean = values.mean();
double std = std::sqrt((values.array() - mean).square().sum() / (values.size() - 1));
double threshold = k * std;
for (int i = 0; i < values.size(); ++i) {
    if (std::abs(values[i] - mean) > threshold) {
        markOutlier(i);
    }
}
```

#### 3.4.2 四分位距法
- 使用快速选择算法（`std::nth_element`）求分位数。
- IQR = Q3 - Q1，异常值界限为 `[Q1 - 1.5*IQR, Q3 + 1.5*IQR]`。

#### 3.4.3 处理
- **删除**：标记行删除。
- **替换**：用边界值（如Q1-1.5IQR或Q3+1.5IQR）替换。
- **标记**：增加一列标记字段（如 `outlier_flag`）。

### 3.5 数据格式标准化
- **日期时间**：使用`QDateTime::fromString()`解析多种格式，再统一输出为`yyyy-MM-dd HH:mm:ss`。
- **单位转换**：预定义映射表，如 `{"ft": 0.3048, "mile": 1609.34}`。
- **文本编码**：使用`QTextCodec`将字符串转换为UTF-8。

## 4. 性能优化措施

| **优化项**         | **实现方法**                                                | **效果**                     |
|--------------------|-------------------------------------------------------------|------------------------------|
| 批量处理           | 使用Eigen的向量化操作代替逐元素循环                         | 数值计算加速5~10倍           |
| 多线程并行         | 不同列的处理可并行（如各列独立计算统计值）                   | 充分利用多核CPU              |
| 内存池             | 使用`QVector`预分配内存，避免频繁扩容                        | 减少内存碎片                 |
| 索引加速           | 对关键字段建立临时哈希表                                     | 重复检测O(n)                 |
| 延迟删除           | 先标记待删除行，最后一次性删除（倒序遍历）                   | 避免频繁移动数据             |
| 数据库内计算       | 部分操作可直接用SQL实现（如去重、删除缺失行），减少数据传输 | 利用数据库引擎优化           |

## 7. 关键技术难点与解决方案

| **难点**                     | **解决方案**                                             |
|------------------------------|----------------------------------------------------------|
| 超大文件内存溢出             | 流式读取 + 分块处理，每块处理完释放内存                  |
| 异常值检测对非正态分布不适用 | 提供多种算法（IQR、聚类、孤立森林）可选，支持扩展        |
| 缺失值插值对不规则时间序列   | 先按时间排序，再使用线性插值或样条插值                  |
| 多步操作的事务一致性         | 使用SQL事务或内存中的版本控制，支持回滚                   |
| 用户自定义规则               | 支持通过脚本（Lua/Python）扩展预处理逻辑                  |
