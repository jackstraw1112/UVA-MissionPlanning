# 反辐射无人机仿真系统——仿真推演模块实体状态显示功能需求描述

## 1. 功能需求描述

| **项目**         | **描述** |
|------------------|----------|
| **需求名称**     | 实体状态显示 |
| **需求标识**     | FR-SIM-07 |
| **需求概述**     | 能够在三维场景中实时显示作战实体的状态变化，通过颜色、图形或文字等效果给出实体状态变化的视觉提醒，如雷达开关机、导引头开机等，以增强态势感知能力。 |
| **需求来源**     | 系统总体设计要求 / 仿真推演模块能力指标 |
| **前置条件**     | 1. 三维场景已初始化并正常渲染。<br>2. 实体状态数据流已建立，能够实时接收状态更新。<br>3. 实体模型已加载且支持状态变化显示（如材质可动态修改、附加特效组件等）。<br>4. 状态与视觉表现的映射规则已配置。 |
| **触发条件**     | 1. 实体状态发生变化（如雷达从关机转为开机、导引头激活等）时触发。<br>2. 用户手动刷新视图（可选）。 |
| **退出条件**     | 1. 仿真结束或用户关闭三维视图。<br>2. 实体被摧毁或从场景中移除。 |
| **功能参与者**   | 仿真引擎（状态数据提供者）、三维渲染引擎（显示实现者）、用户（观察者） |
| **输入**         | 实体ID、实体类型、状态名称、状态值、时间戳（可选） |
| **输出**         | 三维场景中对应实体的视觉变化（颜色改变、图标闪烁、文字标签更新、特效显示） |
| **处理流程**     | 1. 接收仿真引擎发送的实体状态更新消息。<br>2. 根据实体ID在场景图中查找对应的三维模型实例。<br>3. 根据实体类型、状态名称和状态值，查询预设的映射规则，确定视觉表现方式（颜色、文字、特效）。<br>4. 应用视觉变化：更新模型材质颜色、修改标签文本、启用/禁用特效组件。<br>5. 在下一渲染帧中呈现更新后的外观。 |
| **成功判据**     | 1. 状态变化后，三维场景中对应实体的视觉表现能正确反映新状态。<br>2. 变化响应时间在用户可感知的延迟内（≤100ms）。<br>3. 不同状态对应的视觉差异明显，易于区分。 |
| **异常处理**     | 1. 状态映射规则缺失时，使用默认显示（如高亮边框）并记录日志。<br>2. 模型实例查找失败时，忽略该更新并记录错误。<br>3. 状态数据异常（如未知状态码）时，显示问号标签或默认状态。 |
| **后置条件**     | 1. 实体状态显示持续更新，直至仿真结束。<br>2. 视觉表现随状态变化实时刷新。 |
| **显示约束**     | 1. 状态提醒不应过度干扰主要场景，采用半透明、边缘高亮等适度方式。<br>2. 支持用户自定义状态显示样式或关闭某些提醒。<br>3. 文字标签应清晰可读，避免重叠，可随相机距离自动缩放。 |
| **其他约束**     | 1. 状态显示应与仿真时间同步，避免延迟或超前。<br>2. 支持批量状态更新（如多架无人机同时改变状态）。<br>3. 状态映射规则应可配置（如通过JSON文件），便于扩展新状态。 |

## 4. 分析流程

### 4.1 状态-视觉映射配置
预先为每种实体类型定义状态映射表，例如（JSON格式）：
```json
{
  "UAV": {
    "states": {
      "seeker_active": { "color": [1.0, 0.5, 0.0], "text": "导引头开", "effect": "scan_cone" },
      "radar_on": { "color": [1.0, 0.0, 0.0], "text": "雷达开", "effect": "pulse" },
      "idle": { "color": [0.0, 1.0, 0.0], "text": "待机" }
    }
  },
  "RADAR": {
    "states": {
      "on": { "color": [1.0, 0.2, 0.2], "text": "开机", "effect": "scan_beam" },
      "off": { "color": [0.3, 0.3, 0.3], "text": "关机" }
    }
  }
}
```

### 4.2 实时数据处理流程
1. **数据接收**：三维渲染引擎通过WebSocket或共享内存订阅仿真引擎的状态更新流。
2. **解析消息**：每帧处理缓冲区中的状态更新消息，提取实体ID、状态名和值。
3. **模型查找**：使用空间索引或哈希表快速定位实体ID对应的三维模型实例。
4. **规则匹配**：根据实体类型和状态名查找映射表，获得颜色、文字、特效参数。若状态值为数值，可进一步调整表现（如颜色渐变）。
5. **视觉更新**：
   - **颜色**：修改模型材质的emissive或base color，或添加边缘高光。
   - **文字**：更新模型头顶的标签（Billboard）内容，并设置颜色。
   - **特效**：若状态关联特效（如扫描锥），则启用对应的粒子系统或网格，并调整位置/旋转以匹配实体。
6. **渲染**：所有更新在下一帧统一提交渲染。

### 4.3 批量更新优化
对于大量实体同时更新（如240架无人机），可采用以下优化：
- 将状态更新分组，同一帧内批量处理，减少查找开销。
- 使用实例化渲染时，状态颜色可通过uniform buffer批量更新。

### 4.4 用户交互
- 用户可点击实体查看详细状态信息面板。
- 提供图层开关，允许用户关闭某些类型的状态提醒（如只显示颜色，不显示文字）。

# 反辐射无人机仿真系统——实体状态显示模块开发实现方案

## 1. 总体架构

实体状态显示模块是三维态势显示的增强子模块，负责将仿真引擎传来的实体状态变化实时转换为视觉表现。模块采用**数据驱动+规则映射**的架构，与三维渲染引擎紧密集成。

### 1.1 模块层次结构

```plaintext
┌─────────────────────────────────────────────────────┐
│                  仿真引擎 (状态数据源)               │
│  └─ 通过 WebSocket / ZeroMQ 发布状态更新            │
├─────────────────────────────────────────────────────┤
│               三维态势显示前端                        │
│  ┌─────────────────────────────────────────────┐   │
│  │       实体状态订阅器 (WebSocket客户端)       │   │
│  └─────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────┐   │
│  │       状态队列 & 解析器                      │   │
│  └─────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────┐   │
│  │       状态-视觉映射规则引擎                  │   │
│  └─────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────┐   │
│  │       模型实例管理 & LOD                      │   │
│  └─────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────┐   │
│  │       Three.js / Cesium 渲染器               │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 1.2 数据流
- **输入**：仿真引擎实时推送状态更新（JSON格式，包含实体ID、状态名、状态值、时间戳）。
- **处理**：前端解析消息，查找实体对应的三维模型，根据映射规则改变外观。
- **输出**：三维场景中实体的颜色、文字标签、特效实时更新。

## 2. 状态到视觉的映射机制

### 2.1 可配置的映射规则库
采用JSON文件定义状态与视觉表现的映射，支持动态加载和热更新。

```json
{
  "version": "1.0",
  "entityTypes": {
    "UAV": {
      "default": { "color": "#00ff00", "text": "待机" },
      "states": {
        "seeker_active": {
          "color": "#ffaa00",
          "text": "导引头开",
          "effect": "scan_cone",
          "duration": 0
        },
        "radar_on": {
          "color": "#ff0000",
          "text": "雷达开",
          "effect": "pulse",
          "intensity": 1.5
        },
        "damaged": {
          "color": "#880000",
          "text": "受损",
          "effect": "smoke",
          "duration": -1  // 永久持续直到状态变更
        }
      }
    },
    "RADAR": {
      "default": { "color": "#888888", "text": "关机" },
      "states": {
        "on": {
          "color": "#ff4444",
          "text": "开机",
          "effect": "scan_beam"
        }
      }
    }
  }
}
```

### 2.2 规则引擎实现
- 加载规则库后，构建内存中的映射表（以实体类型+状态名为键，视觉表现为值）。
- 支持状态值为数值范围的情况（如高度状态可通过插值得到颜色渐变）。

## 3. 视觉表现的具体实现技术

### 3.1 颜色变化
**方法**：
- 使用Three.js的`Mesh`材质，修改`emissive`或`color`属性。对于实例化渲染，使用uniform数组或修改每个实例的颜色属性。
- 对于模型不同部位（如机头灯）可单独控制。

**代码示例**：
```javascript
function setEntityColor(entity, colorHex) {
    if (entity.isInstanced) {
        // 实例化渲染：更新实例颜色uniform
        entity.material.uniforms.instanceColor.value[entity.instanceIndex] = new THREE.Color(colorHex);
        entity.material.uniforms.instanceColor.needsUpdate = true;
    } else {
        // 单例模型：直接修改材质
        entity.mesh.material.emissive.set(colorHex);
    }
}
```

### 3.2 文字标签
**实现**：
- 使用Three.js的`CSS2DRenderer`或`CSS3DRenderer`创建始终面向相机的标签。
- 标签内容动态更新，支持多行，背景半透明。

**代码示例**：
```javascript
const labelDiv = document.createElement('div');
labelDiv.className = 'entity-label';
labelDiv.style.color = 'white';
labelDiv.style.backgroundColor = 'rgba(0,0,0,0.6)';
labelDiv.style.padding = '2px 6px';
labelDiv.style.borderRadius = '4px';
labelDiv.style.fontSize = '14px';

const label = new CSS2DObject(labelDiv);
label.position.copy(entity.position);
label.name = `label-${entity.id}`;
```

### 3.3 特效实现
- **扫描锥/波束**：使用自定义几何体（圆锥、圆环）结合透明度动画，位置跟随实体。
- **脉冲光环**：使用ShaderMaterial实现周期性缩放和透明度变化。
- **烟雾/火焰**：粒子系统（PointsMaterial + 纹理），动态更新粒子位置和颜色。

**特效管理**：为每个实体维护一个特效列表，状态变化时添加/移除特效。特效的播放和生命周期由特效管理器统一控制。

## 4. 大规模场景优化（针对240+架无人机）

### 4.1 实例化渲染与状态批量更新
- 对同型号无人机使用`InstancedMesh`，每个实例的位置、颜色、状态等作为属性缓冲区。
- 状态变化时，只更新对应实例的缓冲区数据，而非重建整个网格。

```javascript
// 实例化无人机群
const geometry = new THREE.BoxGeometry(1, 0.2, 0.5); // 简化模型
const material = new THREE.MeshPhongMaterial();
const instancedMesh = new THREE.InstancedMesh(geometry, material, 240);
// 设置每个实例的矩阵、颜色等
```

### 4.2 状态更新批处理
- 将一帧内收到的所有状态更新合并处理，避免每一条消息都触发一次渲染。
- 使用双缓冲队列：接收线程写入队列，渲染线程每帧开始前处理所有待处理更新。

### 4.3 LOD与细节层次
- 远距离实体简化状态显示：例如只显示颜色，不显示文字标签和复杂特效。
- 根据距离动态切换模型细节和特效精度。

### 4.4 空间索引加速实体查找
- 使用`Map`数据结构（实体ID -> 模型对象）快速定位，查找时间复杂度O(1)。

## 5. 与仿真引擎的通信

### 5.1 WebSocket实时数据流
```javascript
const ws = new WebSocket('ws://localhost:8080/state');
ws.onmessage = (event) => {
    const updates = JSON.parse(event.data);
    stateQueue.push(...updates);
};

// 每帧处理队列
function processStateUpdates() {
    while (stateQueue.length > 0) {
        const update = stateQueue.shift();
        applyStateChange(update);
    }
}
```

### 5.2 消息格式示例
```json
{
  "timestamp": 12345.6,
  "updates": [
    { "id": "UAV-001", "state": "seeker_active", "value": true },
    { "id": "UAV-002", "state": "damaged", "value": 0.3 }
  ]
}
```

### 5.3 数据压缩
- 使用MessagePack或Protocol Buffers替代JSON，减少传输体积。
- 增量更新：只发送变化的状态，而非全量。

## 7. 关键技术难点与解决方案

| **难点** | **解决方案** |
|----------|--------------|
| 240架无人机同时改变状态导致性能骤降 | 实例化渲染 + 批量属性更新 + 分帧处理 |
