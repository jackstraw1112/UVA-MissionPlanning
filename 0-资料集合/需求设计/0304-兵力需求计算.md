## 1. 引言

### 1.1 背景
在现代防空压制（SEAD）作战中，反辐射无人机已成为摧毁或压制敌方雷达系统的关键装备。为高效运用反辐射无人机资源，实现协同作战任务规划，需要科学计算所需兵力数量，确保任务成功概率满足要求，同时避免资源浪费。兵力需求计算模块作为协同任务规划管理CSCI的核心组件，为指挥员提供决策支持。

### 1.2 目的
本文档旨在完整定义反辐射无人机协同任务规划中兵力需求计算模块的能力需求、实现思路及界面设计，为后续软件开发、测试和验收提供依据。

### 1.3 适用范围
本文档适用于JWS-01型反辐射无人机协同作战仿真系统及实际任务规划系统的开发。

### 1.4 术语与缩略语
| 术语 | 定义 |
|------|------|
| SEAD | Suppression of Enemy Air Defenses，压制敌防空 |
| DEAD | Destruction of Enemy Air Defenses，摧毁敌防空 |
| CSCI | 计算机软件配置项 |
| 单机任务成功率 | 单架反辐射无人机在给定战场环境下完成预定任务的概率 |
| 蒙特卡洛仿真 | 通过大量随机模拟计算统计概率的方法 |

## 2. 能力需求

### 2.1 功能需求

#### 2.1.1 任务类型解析（F-01）
- 支持用户选择任务类型，包括：区域侦察、目标定位、电子干扰、硬杀伤、协同压制（SEAD/DEAD混合）。
- 不同任务类型影响后续计算模型（如探测概率、毁伤概率、留空时间等）。

#### 2.1.2 任务目标信息获取（F-02）
- 支持手动输入或从文件（CSV/XML）导入目标信息，包括：
  - 目标数量
  - 目标类型（预警雷达、火控雷达、通信节点等）
  - 威胁等级（低/中/高）
  - 防护能力（无防护/软防护/硬防护）
  - 目标位置（可选，用于航程约束）
- 提供表格界面，支持增删改目标条目。

#### 2.1.3 战场环境信息接入（F-03）
- 从环境参数设置模块获取地形复杂度、气象影响、电磁干扰强度等数据。
- 将环境因素量化为修正系数，影响单机任务成功率。

#### 2.1.4 反辐射无人机资源信息管理（F-04）
- 维护资源数据库，包含以下信息：
  - 无人机型号（如JWS-01）
  - 可用数量
  - 基础单机任务成功率（标准条件下）
  - 航程、巡航速度、留空时间
  - 载荷类型（侦察/干扰/攻击）
  - 抗干扰等级
- 支持从配置文件初始化、手动编辑、保存修改。

#### 2.1.5 兵力计算与优化（F-05）
- 提供两种计算模型：
  - **解析模型**：基于概率论快速计算所需数量，适用于单目标或同质目标场景。
  - **蒙特卡洛仿真模型**：通过多次随机仿真，精确评估多目标、多类型无人机协同场景下的成功概率，并迭代寻找最小数量。
- 考虑因素：目标数量与类型、单机任务成功率（经环境修正）、任务成功率要求、资源可用数量。
- 输出结果：
  - 所需无人机数量（整数）
  - 推荐编组方案（如X架侦察型 + Y架攻击型）
  - 计算依据描述（关键参数、置信区间等）

#### 2.1.6 结果输出与报告生成（F-06）
- 界面实时显示计算结果。
- 支持生成Word/PDF格式报告，包含输入参数、计算过程、结果和建议。

## 3. 实现思路

### 3.1 算法模型

#### 3.1.1 解析模型（基于概率）
- **单目标场景**：设单机任务成功率为 \(P_s\)（经环境修正），任务要求成功概率为 \(P_{req}\)。所需无人机数量 \(N\) 满足：
  \[
  1 - (1 - P_s)^N \geq P_{req}
  \]
  解得：
  \[
  N = \left\lceil \frac{\ln(1 - P_{req})}{\ln(1 - P_s)} \right\rceil
  \]
- **多目标场景**（简化为各目标独立，一机可对付多目标）：引入覆盖因子 \(C\)（每架无人机可覆盖的目标数），则所需架数 \(N = \lceil N_t / C \rceil\)，再按概率修正。复杂情况建议采用蒙特卡洛仿真。

#### 3.1.2 蒙特卡洛仿真模型
- 设定仿真次数 \(M\)（如1000）。
- 对每次仿真：
  - 根据战场环境随机抽样生成无人机突防损耗（生存概率）。
  - 对每架无人机模拟其对目标的探测/攻击结果（基于单机能力随机化）。
  - 统计成功摧毁/压制的目标数量。
  - 判断任务是否成功（如目标摧毁数≥阈值）。
- 统计成功概率 \(P_{est}\)。
- 若 \(P_{est} < P_{req}\)，增加无人机数量，重复仿真；反之减少，用二分法寻找最小 \(N\)。

#### 3.1.3 环境修正因子
建立环境因素与单机成功率的关系（示例）：
| 地形复杂度 | 修正系数 |
|------------|----------|
| 平坦 | 1.0 |
| 丘陵 | 0.9 |
| 山地 | 0.7 |
| 复杂 | 0.5 |

| 气象影响 | 修正系数 |
|----------|----------|
| 良好 | 1.0 |
| 轻度 | 0.9 |
| 中度 | 0.7 |
| 重度 | 0.4 |

| 电磁干扰 | 修正系数 |
|----------|----------|
| 无 | 1.0 |
| 弱 | 0.8 |
| 中 | 0.5 |
| 强 | 0.2 |

最终单机成功率 = 基础成功率 × 地形系数 × 气象系数 × 电磁系数。

### 3.2 计算流程

1. **初始化**：加载资源数据、环境参数。
2. **用户输入**：选择任务类型、输入目标信息、设置成功率要求。
3. **环境修正**：根据当前环境计算综合修正系数，得到单机成功率（可按无人机型号差异化）。
4. **模型选择**：
   - 若目标数≤3且类型单一，默认采用解析模型快速计算。
   - 若目标数较多或类型复杂，提供蒙特卡洛仿真选项。
5. **计算**：
   - 解析模型直接计算。
   - 蒙特卡洛仿真迭代计算。
6. **结果显示**：更新界面，展示所需数量、编组方案和计算依据。
7. **报告生成**：按需导出。

### 3.3 关键技术
- **Qt框架**：用于跨平台界面开发。
- **多线程**：蒙特卡洛仿真在后台线程执行，避免界面卡顿。
- **数据持久化**：使用JSON/XML保存资源配置和任务配置。
- **报告生成**：利用QAxObject（Windows）或QtPdf生成文档。

## 4. 界面设计

### 4.1 整体布局
采用主窗口（`QMainWindow`）结构，包含菜单栏、工具栏、中央部件和状态栏。中央部件使用垂直分割，上部为输入区域（分页），下部为输出区域。

```
+---------------------------------------------------+
| 菜单栏 (文件、编辑、工具、帮助)                    |
+---------------------------------------------------+
| 工具栏 (保存配置、加载配置、开始计算、生成报告)     |
+---------------------------------------------------+
| 输入区域 (QTabWidget)                             |
| +-----------------------------------------------+ |
| | 任务信息 | 目标信息 | 战场环境 | 资源信息      | |
| +-----------------------------------------------+ |
| 输出区域 (QGroupBox)                              |
| +-----------------------------------------------+ |
| | 所需数量: XX   [生成报告]                       |
| | 编组方案: ...                                   |
| | 计算依据: ...                                   |
| +-----------------------------------------------+ |
+---------------------------------------------------+
| 状态栏 (当前资源总数、计算进度)                    |
+---------------------------------------------------+
```

### 4.2 输入区域详细设计

#### 4.2.1 任务信息页
| 控件 | 类型 | 属性 |
|------|------|------|
| 任务类型 | `QComboBox` | 区域侦察、目标定位、电子干扰、硬杀伤、协同压制 |
| 成功率要求 | `QDoubleSpinBox` | 范围50%~100%，步长1%，默认95% |

#### 4.2.2 目标信息页
- **目标数量**：`QSpinBox`，1~100，默认1。
- **目标列表**：`QTableWidget`，列包括：类型（下拉框）、威胁等级（下拉框）、防护能力（下拉框）、经度（可选）、纬度（可选）。
- **按钮**：“添加行”、“删除行”、“从文件导入”。

#### 4.2.3 战场环境页
| 控件 | 类型 | 说明 |
|------|------|------|
| 地形复杂度 | `QComboBox` | 平坦/丘陵/山地/复杂 |
| 气象条件 | `QComboBox` | 良好/轻度/中度/重度 |
| 电磁干扰强度 | `QComboBox` | 无/弱/中/强 |
| 综合修正系数 | `QLabel` | 实时显示当前组合的乘积 |

#### 4.2.4 资源信息页
- **资源列表**：`QTableWidget`，列包括：型号、可用数量、基础成功率、航程(km)、抗干扰等级。
- **按钮**：“添加资源”、“删除资源”、“从配置文件加载”。
- **可用总数**：自动统计，显示在底部。

### 4.3 输出区域
- **所需数量**：大字体显示整数。
- **编组方案**：`QTextEdit`（只读），如“4架（2架干扰型+2架攻击型）”。
- **计算依据**：`QTextEdit`（只读），显示关键参数、中间结果、置信区间等。
- **生成报告按钮**：`QPushButton`，点击后弹出保存对话框，生成Word/PDF报告。

### 4.4 交互逻辑
- 任何输入参数变化（除表格编辑外）立即触发重新计算（使用`QTimer`延迟0.5s防抖）。
- 表格编辑完成后点击空白处或按回车触发重新计算。
- 计算过程较长时间（蒙特卡洛）显示进度条，禁用部分控件，计算完成后恢复。
- 资源信息修改后立即统计可用总数，并检查是否小于所需数量（若小于，提示警告）。

## 3. 分析流程

兵力需求计算采用混合模型，根据用户选择的算法（解析模型或蒙特卡洛仿真）执行，具体流程如下：

### 3.1 通用预处理
1. **数据获取**：从界面控件或数据接口读取所有输入参数。
2. **参数校验**：
   - 检查目标数量是否≥1，目标类型是否有效。
   - 检查成功率要求在0~1之间。
   - 检查无人机资源可用数量是否≥0，基础成功率在0~1之间。
   - 若提供目标位置，检查经纬度合法性。
3. **环境修正**：根据地形、气象、电磁因子，查找映射表获得各修正系数，计算综合修正因子 \( K = K_{terrain} \times K_{weather} \times K_{jamming} \)。
4. **单机成功率修正**：每架无人机的实际成功率 \( P_{actual} = P_{base} \times K \)。若无人机有不同基础成功率（因型号或载荷），分别计算。

### 3.2 解析模型计算流程（适用于单目标或同质多目标简化）
1. **输入**：目标数量 \( N_t \)，单机成功率 \( P_s \)（经修正），要求成功概率 \( P_{req} \)。
2. **若 \( N_t = 1 \)**：所需数量 \( N_u = \lceil \frac{\ln(1-P_{req})}{\ln(1-P_s)} \rceil \)。
3. **若 \( N_t > 1 \)**：
   - 假设各目标独立，每架无人机只能攻击一个目标（简单情况），则问题转化为每个目标需至少一架无人机攻击，且每个目标的成功率要求为 \( P_{req}^{1/N_t} \)（近似），则总数量 \( N_u = N_t \times \lceil \frac{\ln(1-P_{req}^{1/N_t})}{\ln(1-P_s)} \rceil \)。
   - 若考虑一架无人机可攻击多个目标（如侦察或干扰可覆盖多目标），则引入覆盖因子 \( C \)（用户可设），所需数量 \( N_u = \lceil N_t / C \rceil \)，再按概率校验，若不满足则增加。
4. **输出**：\( N_u \)，并生成编组建议（按比例分配可用资源）。

### 3.3 蒙特卡洛仿真流程
1. **设定**：仿真次数 \( M \)（默认1000），迭代搜索步长。
2. **外层循环**：从 \( N_{min} \)（1）到 \( N_{max} \)（可用总数）进行搜索，用二分法寻找最小 \( N \) 满足成功率≥ \( P_{req} \)。
3. **对于每个候选 \( N \)**：
   - 初始化成功计数 \( S = 0 \)。
   - 内层循环 \( i = 1 \) 到 \( M \)：
     - 生成随机种子，模拟 \( N \) 架无人机的作战过程：
       - 根据环境修正后的单机成功率 \( P_s \) 随机判定每架无人机是否成功突防/探测/攻击（可按阶段细化）。
       - 统计成功摧毁的目标数量（需考虑目标数量、多机协同命中逻辑）。
       - 若成功摧毁目标数≥目标数量（或按任务要求），则本次仿真成功。
     - 若成功，\( S++ \)。
   - 计算估计成功率 \( \hat{P} = S / M \)。
   - 若 \( \hat{P} \geq P_{req} \)，则 \( N \) 可行，向下搜索；否则向上搜索。
4. **结果**：找到最小 \( N \)，并计算置信区间（如用正态近似）。
5. **生成编组建议**：根据仿真中成功案例的典型配置（如载荷类型比例）推荐编组。

### 3.4 后处理
- 格式化输出文本，包括关键参数、所用算法、中间计算结果。
- 若资源可用数量小于计算所需数量，发出警告，并输出最大可行数量及对应的成功率。
