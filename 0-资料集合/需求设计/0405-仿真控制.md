## 2. 关键术语解释

| **术语** | **解释** |
|----------|----------|
| **开始** | 启动仿真运行，仿真引擎从当前时间开始推进，实体开始按照模型规则行动。 |
| **暂停** | 临时中止仿真运行，冻结所有实体状态和仿真时间，等待用户后续指令。 |
| **继续** | 从暂停状态恢复仿真运行，时间继续推进，实体从冻结状态继续行动。 |
| **快进** | 以高于正常速度（如2倍、10倍）运行仿真，用于快速跳过无关紧要的阶段，缩短等待时间。 |
| **停止** | 完全终止仿真运行，释放资源，保存当前结果，仿真回到初始配置状态。 |

## 4. 分析流程

### 4.1 控制状态机
仿真引擎内部维护一个状态机，状态包括：`IDLE`（待命）、`RUNNING`（运行）、`PAUSED`（暂停）、`FAST_FORWARD`（快进）、`STOPPING`（停止中）。用户命令触发状态转移：

- 在 `IDLE` 状态，点击“开始” → `RUNNING`。
- 在 `RUNNING` 状态，点击“暂停” → `PAUSED`；点击“快进” → `FAST_FORWARD`；点击“停止” → `STOPPING`。
- 在 `PAUSED` 状态，点击“继续” → `RUNNING`；点击“停止” → `STOPPING`。
- 在 `FAST_FORWARD` 状态，点击“暂停” → `PAUSED`；点击“停止” → `STOPPING`；点击“继续”可能回到 `RUNNING`（恢复常速）。
- `STOPPING` 状态执行清理后回到 `IDLE`。

### 4.2 时间推进机制
- **正常运行**：按设定的时间步长 `dt` 推进，每步更新实体状态，实时渲染（若开启）。
- **快进模式**：仿真引擎仍按相同步长计算，但每次更新后不等待实际时间，连续计算，同时可选择降低可视化刷新频率（如每秒更新画面一次），以达到加速效果。快进倍率 `k` 表示仿真时间推进速度是实际时间的 `k` 倍，即实际等待 `dt` 时间才推进一步变为不等待直接计算下一步。

### 4.3 暂停/继续的数据一致性
暂停时，引擎停止时间推进，所有实体状态冻结。继续时，直接从当前时间开始运行，无需特殊处理。需注意事件队列中的事件时间应保持不变，不会因暂停而改变触发时间。

# 实现思路
# 反辐射无人机仿真系统——仿真控制模块开发实现方案（扩展版）

## 1. 总体架构

仿真控制模块采用**状态机驱动 + 倍率/步长实时调节**的设计，与仿真引擎主循环紧密集成。核心组件包括：

- **仿真控制器（`SimulationController`）**：接收用户命令，管理状态机，控制引擎生命周期，维护倍率和步长。
- **仿真引擎（`SimulationEngine`）**：执行主循环，支持动态调整时间步长和运行倍率。
- **状态同步器**：将引擎状态、当前倍率、步长实时反馈给UI。

## 2. 状态机设计

使用枚举定义仿真状态，并在控制器中维护：

```cpp
enum class SimulationState {
    IDLE,           // 待命（未启动）
    RUNNING,        // 正常运行（当前倍率下）
    PAUSED,         // 暂停
    STOPPING        // 停止中（过渡态）
};
```

**加速（快进）**和**减速**实际上是在 `RUNNING` 状态下改变速度倍率，不引入新状态。倍率值从预设集合中选取：1、2、4、8、16、32、64、128。

状态转移图如下：
```
IDLE --(开始)--> RUNNING
RUNNING --(暂停)--> PAUSED
RUNNING --(停止)--> STOPPING
PAUSED --(继续)--> RUNNING
PAUSED --(停止)--> STOPPING
STOPPING --(清理完成)--> IDLE
```

## 3. 仿真引擎主循环的实现

### 3.1 基本结构
引擎主循环在独立线程中运行，使用 `QThread`。循环体根据当前状态决定行为：

```cpp
void SimulationEngine::run() {
    m_running = true;
    while (m_running) {
        switch (m_state) {
        case SimulationState::RUNNING:
            // 执行一步仿真（步长可能变化）
            stepSimulation();
            // 控制实际时间等待（基于倍率调整等待时间）
            if (m_speedFactor > 0) {
                int waitMs = qRound(m_stepMs / m_speedFactor);
                if (waitMs > 0) {
                    QThread::msleep(waitMs);
                }
            }
            break;
        case SimulationState::PAUSED:
            // 暂停：不执行仿真，等待信号
            m_pauseMutex.lock();
            m_pauseCondition.wait(&m_pauseMutex);
            m_pauseMutex.unlock();
            break;
        case SimulationState::STOPPING:
            cleanup();
            return;
        default:
            break;
        }
    }
}
```

### 3.2 步长与倍率管理
步长和倍率作为引擎的成员变量，提供线程安全的修改方法：

```cpp
class SimulationEngine : public QThread {
    Q_OBJECT
public:
    // 步长支持：1000,500,250,125,100,50,25,10 (ms)
    void setStepMs(int ms) {
        QMutexLocker locker(&m_paramMutex);
        m_stepMs = ms;
    }
    
    // 倍率支持：1,2,4,8,16,32,64,128
    void setSpeedFactor(int factor) {
        QMutexLocker locker(&m_paramMutex);
        m_speedFactor = factor;
    }
    
private:
    int m_stepMs;        // 当前步长（毫秒）
    int m_speedFactor;   // 当前倍率
    QMutex m_paramMutex;
};
```

### 3.3 暂停/继续实现
与之前类似，使用条件变量：

```cpp
void SimulationEngine::pause() {
    QMutexLocker locker(&m_mutex);
    if (m_state == SimulationState::RUNNING) {
        m_state = SimulationState::PAUSED;
    }
}

void SimulationEngine::resume() {
    QMutexLocker locker(&m_mutex);
    if (m_state == SimulationState::PAUSED) {
        m_state = SimulationState::RUNNING;
        m_pauseCondition.wakeOne();
    }
}
```

### 3.4 停止实现
```cpp
void SimulationEngine::stop() {
    {
        QMutexLocker locker(&m_mutex);
        m_state = SimulationState::STOPPING;
        m_pauseCondition.wakeOne();
    }
    wait();
    emit simulationStopped();
}
```

## 4. 步长调整的实现

步长调整可以随时进行，不需要停止仿真。引擎主循环每次迭代时读取最新的步长值，因此调整后下一次步进即生效。

**步长预定义列表**：为了确保数值精确且符合用户预期，可以提供一个下拉选择框，直接映射到对应的毫秒值。

```cpp
// 步长选项映射
const QMap<QString, int> stepOptions = {
    {"1s", 1000},
    {"500ms", 500},
    {"250ms", 250},
    {"125ms", 125},
    {"100ms", 100},
    {"50ms", 50},
    {"25ms", 25},
    {"10ms", 10}
};
```

**界面交互**：用户从下拉框选择步长，控制器调用 `engine->setStepMs(selectedMs)`。

**注意事项**：步长过小可能导致计算负担加重，需确保引擎在最小步长（10ms）下仍能实时运行（即每步计算时间<10ms）。若无法满足，可限制最小步长或采用跳过帧策略。

## 5. 倍率调整的实现

倍率调整同样实时生效。倍率影响主循环中的等待时间：`waitMs = stepMs / speedFactor`。当倍率大于1时，等待时间缩短，实现快进；倍率为1时正常实时；倍率小于1？但用户要求只有大于等于1的倍率，因此不需要减速（慢放），但用户要求中有“减速”功能，这可能意味着倍率可以小于1（慢放）。注意用户描述：“加速减速支持（1倍速、2倍速、4倍速、8倍速、16倍速、32倍速、64倍速、128倍速）”，只列出了加速选项，没有列出小于1的减速选项。但用户又说有“减速”功能，可能意味着减速是将倍率降低到1以下？需要澄清。但在实际仿真中，减速（慢放）也是常见需求，比如0.5倍速。但用户给定的列表只有≥1的数值，可能是笔误？我们可假设减速也是通过降低倍率实现，但倍率集合只给了1,2,4,...，那么减速只能降到1，无法慢放。为了全面，可以增加小于1的倍率如0.5,0.25等，但用户未要求。根据用户明确列出的值，我们只实现所列倍率。

实现时，倍率切换通过下拉框或按钮循环选择。控制器将选中的倍率值传给引擎。

```cpp
// 倍率选项列表
const QList<int> speedFactors = {1,2,4,8,16,32,64,128};
```

**加速/减速按钮逻辑**：点击“加速”切换到下一个更大的倍率（循环或限制在最大值）；点击“减速”切换到下一个更小的倍率（循环或限制在1）。

## 6. 加步长/减步长功能

加步长/减步长指增加或减少当前步长值（按预设列表循环）。需要维护当前步长在列表中的索引。

```cpp
void SimulationController::onStepIncrease() {
    int currentStep = m_engine->stepMs();
    // 找到下一个更大的步长值（注意列表是从大到小还是从小到大？）
    // 假设列表从大到小：1000,500,250,...,10
    // 增加步长意味着数值变大，但通常“加步长”是让仿真变粗糙，步长变大，所以应往列表头部移动。
    int idx = stepOptions.values().indexOf(currentStep);
    if (idx > 0) {
        idx--;
        m_engine->setStepMs(stepOptions.values()[idx]);
    }
}

void SimulationController::onStepDecrease() {
    int currentStep = m_engine->stepMs();
    int idx = stepOptions.values().indexOf(currentStep);
    if (idx < stepOptions.size()-1) {
        idx++;
        m_engine->setStepMs(stepOptions.values()[idx]);
    }
}
```

注意：步长列表需排序，确保索引正确。

## 7. 控制器与UI的交互

采用Qt信号槽机制，确保线程安全：

```cpp
class SimulationController : public QObject {
    Q_OBJECT
public:
    SimulationController(SimulationEngine* engine) : m_engine(engine) {}
    
public slots:
    void onStartClicked();
    void onPauseClicked();
    void onResumeClicked();
    void onStopClicked();
    void onSpeedUp();      // 加速
    void onSpeedDown();    // 减速
    void onStepIncrease();
    void onStepDecrease();
    void onSpeedFactorSelected(int factor);
    void onStepSelected(int ms);
    
signals:
    void stateChanged(SimulationState state);
    void speedFactorChanged(int factor);
    void stepChanged(int ms);
};

// 在引擎中发射状态变化信号
void SimulationEngine::setState(SimulationState newState) {
    if (m_state != newState) {
        m_state = newState;
        emit stateChanged(newState);
    }
}
```

UI按钮的启用/禁用根据状态自动更新，并显示当前倍率和步长。

## 8. 界面交互设计

- **开始/暂停/继续/停止**：按钮随状态变化。
- **加速/减速**：两个按钮，点击分别切换到下一个更高/更低倍率。也可提供下拉框直接选择倍率。
- **加步长/减步长**：两个按钮，调整步长。
- **当前倍率显示**：如“倍率: 8x”。
- **当前步长显示**：如“步长: 100ms”。

## 9. 性能优化措施

- **步长下限保证**：最小步长10ms时，每步计算必须在10ms内完成，否则无法实时。可采用动态调整：若计算超时，则自动降低倍率或发出警告。
- **快进模式优化**：当倍率很高（如≥64）且步长较小时，可暂时关闭可视化更新，仅进行后台计算，并适当合并数据记录。
- **多线程渲染**：可视化更新放在GUI线程，仿真计算在后台线程，两者解耦。

## 10. 测试与验证

- **单元测试**：验证状态转移、步长/倍率设置后的时间计算。
- **性能测试**：测量在不同步长和倍率组合下的实际仿真速度，确保满足实时性要求。
- **边界测试**：倍率最大值128，步长最小值10ms，测试引擎稳定性。
