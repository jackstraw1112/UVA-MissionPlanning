# 辐射源压制效益评估与仿真设计方案（完整合并版）

## 一、 方案概述

本方案聚焦**辐射源压制效益**的核心评估，在原有多指标体系的基础上，精准剥离出两项黄金指标（P1、P2），用于量化反辐射无人机对敌方雷达体系的压制效果。

**适用场景**：红方（解放军空军）出动反辐射无人机压制蓝方（台军）防空雷达，掩护 J-16 战斗机执行对地/对海打击任务。

**核心建模思想（时空耦合）**：彻底摒弃硬编码的计划时间窗口。本方案中的所有时间维度（特别是攻击窗口和突防窗口），均不是人为设定的，而是由实体在空间中的动态交互（无人机与雷达的截获、战机与雷达探测边界的交会）实时解算得出的物理时间。这种设计排除了战机在盲区飞行等“水分”时间，还原了最真实的战术生存概率。

## 二、 评估指标体系

本评估体系仅保留两项必选核心指标，互为补充，共同构成压制效益的完整图景。

| 指标代号   | 指标名称          | 回答的核心问题              | 单位  |
| :----- | :------------ | :------------------- | :-- |
| **P1** | **雷达关机覆盖率**   | 敌方雷达“瞎了”多久？（压得怎么样）   | %   |
| **P2** | **区域防空压制覆盖率** | 我方的安全通道打开了多久？（掩护够不够） | %   |

### 2.1 P1：雷达关机覆盖率

*   **定义**：在动态攻击窗口 $[T_{start}, T_{end}]$ 内，所有参与评估的敌方雷达处于关机状态的平均累计时长占比。
    *(注：$T_{start}$ 为第一架无人机进入截获线时刻；$T_{end}$ 为最后一架无人机脱离/命中/自毁时刻)*
*   **计算公式**：
    $$
    P1 = \frac{1}{N} \sum_{i=1}^{N} \frac{T_{off\_i}}{T_{attack}} \times 100\%
    $$
    *(其中 $N$ 为有效威胁雷达数量，$T_{off\_i}$ 为第 $i$ 部雷达累计关机秒数，$T_{attack} = T_{end} - T_{start}$)*
*   **达标阈值**：优秀(≥70%)，合格(50%~70%)，不合格(<50%)。

### 2.2 P2：区域防空压制覆盖率（时空耦合核心）

*   **定义**：在 J-16 实际暴露于雷达探测区的时间内，完全不受敌方雷达有效覆盖的时长占比。
*   **核心机制**：突防窗口总时长（$T_{escort}$）绝不能提前人为设定，必须由 J-16 与“有效威胁雷达子集”探测边界在空间上的交会动态解算得出：**$T_{escort}$ = J-16 编队处于有效威胁雷达子集内，任意一部雷达有效探测范围内的累计物理时间。**
*   **计算公式**：
    $$
    P2 = \frac{T_{safe}}{T_{escort}} \times 100\%
    $$
    *(其中分母 $T_{escort}$ 为动态累计暴露时间；分子 $T_{safe}$ 为在暴露时间内，同时满足所有覆盖自身的威胁雷达均处于关机状态的累计时间)*
*   **达标阈值**：优秀(≥80%)，合格(60%~80%)，不合格(<60%)。

## 四、 核心仿真设计解答

### 4.1 多雷达环境下的任务规划：引入“有效威胁雷达子集”
**问题**：实际作战可能有几十部雷达，是否都要考虑？无人机不够分怎么办？
**解答**：仿真系统必须分层解耦。
1. **战前过滤**：系统根据 J-16 预选航路拉一条空间走廊，遍历蓝方全量雷达库，剔除无火控能力或探测距离无法覆盖航路的雷达，提取出**“有效威胁雷达子集”**。
2. **任务聚焦**：反辐射无人机的任务分配，**仅针对这个“子集”进行**。
3. **评估边界**：计算 P2 时，引擎**只遍历该子集的状态**。无论蓝方总共有多少部雷达，任务规划模块永远只处理“走廊上的核心威胁”。

### 4.2 攻击窗口（$T_{attack}$）的动态解算
攻击窗口由仿真引擎内的**任务执行事件**动态解算：
*   **任务包定义**：如 `Mission_A`: 目标=Radar_A, 执行者=[UAV-1, UAV-3]。
*   **触发器设计**：事件 `E_enter`（UAV进入雷达截获距离触发）；事件 `E_exit`（UAV命中/坠毁/脱离触发）。
*   **聚合算法**：$T_{start} = MIN(所有UAV的 E\_enter)$，$T_{end} = MAX(所有UAV的 E\_exit)$，$T_{attack} = T_{end} - T_{start}$。

### 4.3 突防窗口（$T_{escort}$）的时空耦合解算
**问题**：如果硬编码“突防窗口300秒”，而 J-16 实际在雷达盲区里飞了100秒，会把盲区时间算进 P2 分母，严重稀释指标。
**解答**：突防窗口必须由 J-16 与雷达探测边界在空间上的交会动态解算。引擎在每个时间步 $t$ 实时计算 J-16 是否处于任意威胁雷达的物理探测范围内，只有当 `Is_Exposed=True` 时，才将该秒累加进 $T_{escort}$。排除了盲区时间，P2 算出的是纯粹的“战机一旦进入死地，存活的物理概率”。

## 五、 仿真系统数据采集与底层实现要求

### 5.1 协同关联机制：必须在“仿真运行时”解决
P2 判断的是“J-16在某一秒是否被雷达照到”。绝对不能在事后评估时才去算，必须在仿真引擎内形成闭环：
1. UAV 飞入截获线 -> 引擎修改 `Radar_Status = 0`。
2. J-16 飞到航路点 -> 引擎读取 `Radar_Status`，计算经纬度距离。
3. 如果距离 < 探测半径 且 状态为1 -> 记录 `Is_Safe = False`。
4. 事后评估脚本**只读取**记录好的状态数组进行统计，完全不参与战术协同计算。

### 5.2 必须记录的参数表（最终交付物）
由于时间全部改为动态解算，不再需要记录预设的起止时间戳，只需以下三张表：

**表1：任务执行与时间解算表 (Mission_Execution.csv)**
*作用：用于动态解算攻击窗口 $T_{attack}$。*

| 字段名 | 数据类型 | 物理含义 | 评估脚本用法 |
| :--- | :--- | :--- | :--- |
| UAV_ID | String | 无人机标识 | 分组聚合 |
| Target_Radar_ID | String | 分配的威胁雷达标识 | 过滤关联 |
| Time_Enter_Threat | Float | 进入截获距离时刻(秒) | 取MIN得到 $T_{start}$ |
| Time_Exit_or_Killed | Float | 脱离/命中/自毁时刻(秒) | 取MAX得到 $T_{end}$ |

**表2：有效威胁雷达状态时序表 (Radar_Status_Log.csv)**
*作用：用于计算 P1，以及作为 P2 判定的前提。*

| 字段名 | 数据类型 | 物理含义 | 评估脚本用法 |
| :--- | :--- | :--- | :--- |
| Timestamp | Float | 仿真当前时间戳 | 基础时间轴 |
| Radar_A_Status | Int (0/1) | 威胁雷达A状态 | 累加0得到 $T_{off\_A}$ |
| Radar_B_Status | Int (0/1) | 威胁雷达B状态 | 累加0得到 $T_{off\_B}$ |

**表3：突防走廊时空状态表 (Escort_Spatiotemporal_Log.csv)**
*作用：同时提供 P2 的分子、分母计算依据。由仿真引擎调用距离算法实时写入。*

| 字段名 | 数据类型 | 物理含义 | 评估脚本用法 |
| :--- | :--- | :--- | :--- |
| Timestamp | Float | 仿真当前时间戳 | 基础轴 |
| J16_Lon | Float | J-16当前经度 | 空间回放/验证 |
| J16_Lat | Float | J-16当前纬度 | 空间回放/验证 |
| Is_Exposed | Boolean | J-16是否处于**任意**威胁雷达探测范围内？ | **用于累加得到分母 $T_{escort}$** |
| Is_Safe | Boolean | 暴露状态下，是否**所有**威胁雷达均关机？ | **用于累加得到分子 $T_{safe}$** |

### 5.3 覆盖判定算法（Qt C++ 仿真引擎底层实现）

在 C++ 仿真引擎中，我们通过定义数据结构和使用 `<cmath>` 库来实现高精度的时空判定。以下是推荐的 Qt C++ 类与方法设计：

```cpp
#include <QObject>
#include <QVector>
#include <QMap>
#include <cmath>
#include <limits>

#ifndef M_PI
#define M_PI 3.14159265358979323846
#endif

// 定义威胁雷达实体结构
struct ThreatRadar {
    QString id;
    double lon;
    double lat;
    double detectRadiusKm;
};

// 定义时空判定返回结果结构
struct J16StatusResult {
    bool isExposed;
    bool isSafe;
};

class RadarSuppressionEngine : public QObject {
    Q_OBJECT
public:
    explicit RadarSuppressionEngine(QObject *parent = nullptr) : QObject(parent) {}

    // 计算两点间球面距离 (Haversine公式)
    static double calcHaversineDistance(double lon1, double lat1, double lon2, double lat2) {
        const double earthRadiusKm = 6371.0;
        
        double dLat = qDegreesToRadians(lat2 - lat1);
        double dLon = qDegreesToRadians(lon2 - lon1);
        
        double a = qSin(dLat / 2) * qSin(dLat / 2) +
                   qCos(qDegreesToRadians(lat1)) * qCos(qDegreesToRadians(lat2)) *
                   qSin(dLon / 2) * qSin(dLon / 2);
                   
        double c = 2 * qAtan2(qSqrt(a), qSqrt(1 - a));
        
        return earthRadiusKm * c;
    }

    // 评估J-16在时刻t的时空状态 (时空耦合核心函数)
    // radarStatusHistory: 传入雷达的历史状态映射 Map<雷达ID, Map<时间戳, 状态0/1>>
    static J16StatusResult evaluateJ16StatusAtT(
        int t, 
        double j16Lon, 
        double j16Lat, 
        const QVector<ThreatRadar>& threatRadars,
        const QMap<QString, QMap<int, int>>& radarStatusHistory) 
    {
        J16StatusResult result = {false, false};
        bool allRadarOffInZone = true;

        for (const auto& radar : threatRadars) {
            double distance = calcHaversineDistance(j16Lon, j16Lat, radar.lon, radar.lat);
            
            // 判断1: 是否在该雷达的物理探测范围内？
            if (distance <= radar.detectRadiusKm) {
                result.isExposed = true; // 只要有一部雷达在物理范围内，即标记为暴露
                
                // 判断2: 在范围内的雷达，当前是否开机？
                // 优先获取t时刻状态，若不存在则获取最近的过去状态
                int currentStatus = 1; // 默认开机
                if (radarStatusHistory.contains(radar.id)) {
                    const auto& timeMap = radarStatusHistory[radar.id];
                    auto it = std::upper_bound(timeMap.constBegin(), timeMap.constEnd(), t);
                    if (it != timeMap.constBegin()) {
                        --it;
                        currentStatus = it.value();
                    }
                }
                
                if (currentStatus == 1) {
                    allRadarOffInZone = false; // 只要有一部在范围内且开机，就不安全
                    break; // 性能优化：已经判定不安全，无需继续检查其他雷达
                }
            }
        }
        
        // 注意：如果isExposed为False(在盲区)，isSafe也为False，因为不参与P2的分母统计
        result.isSafe = result.isExposed && allRadarOffInZone;
        return result;
    }
};
```

## 六、 仿真推演案例演示

### 6.1 案例态势
红方出动 6 架反辐射无人机压制蓝方防空体系，掩护 4 架 J-16。蓝方纵深有数十部雷达，经仿真战前空间解算，提取出**3部天弓/爱国者火控雷达**作为“有效威胁雷达子集”。
*   **RADAR_A**：新竹阵地，探测半径 120km，保守操作员（关机25秒后重启）。
*   **RADAR_B**：台中阵地，探测半径 110km，激进操作员（关机15秒后重启）。
*   **RADAR_C**：嘉义阵地，探测半径 115km，备用雷达。
J-16 总飞行剖面为 400 秒，前 120 秒在西侧盲区集结，脱离航线也有一段盲区。

### 6.2 关键时空事件时序

| 相对时刻 | 时空事件描述 |
| :--- | :--- |
| T+0s | 第一波无人机进入截获线，**($T_{start}$ 产生)**，RADAR_A告警关机 |
| T+15s | 第二波接近，RADAR_B告警关机 |
| T+120s | J-16编队飞行至航路点1，**首次进入 RADAR_B 探测范围 (Is_Exposed=True，开始累计 $T_{escort}$)** |
| T+140s | UAV-1命中 RADAR_A(K-Kill) |
| T+155s | RADAR_B激进重启，此时J-16刚好处于其覆盖内 (Is_Exposed=True, Is_Safe=False) |
| T+220s | UAV-5命中 RADAR_B附近，操作员心理崩溃不敢开机 |
| T+250s | RADAR_C紧急开机补位，J-16进入其探测范围 |
| T+340s | J-16投弹后大过载脱离，**飞出所有雷达探测范围 (Is_Exposed=False，停止累计 $T_{escort}$)** |
| T+480s | 最后一架无人机失效，**($T_{end}$ 产生)** |

### 6.3 指标动态计算过程

#### P1 计算（基于表1和表2）
通过表1解算：$T_{start} = 0s$，$T_{end} = 480s$，得出 $T_{attack} = 480$ 秒。
通过表2统计状态为0的秒数：
*   RADAR_A 累计关机 475 秒（早期关机+被毁）
*   RADAR_B 累计关机 288 秒（激进策略导致部分时间开机）
*   RADAR_C 累计关机 130 秒（后期才介入）
$$P1 = \frac{475 + 288 + 130}{3 \times 480} = 62.0\% \quad \text{(评级：合格)}$$

#### P2 计算（基于表3的 Qt C++ 时空耦合聚合逻辑）
不再使用假设的 300 秒，而是使用 C++ 对 `Escort_Spatiotemporal_Log.csv` 解析后的内存对象进行高效遍历统计：

```cpp
#include <QFile>
#include <QTextStream>
#include <QStringList>

// 对应表3的数据结构
struct EscortLogEntry {
    int timestamp;
    double j16Lon;
    double j16Lat;
    bool isExposed;
    bool isSafe;
};

class SuppressionEvaluator {
public:
    // 从CSV加载数据到内存 (省略具体流读取细节，假设已解析为列表)
    static QVector<EscortLogEntry> loadEscortLog(const QString& filePath) {
        QVector<EscortLogEntry> logs;
        // ... 使用 QFile 和 QTextStream 逐行读取解析 ...
        return logs;
    }

    // 动态解算 P2 的核心算法
    static double calculateP2(const QVector<EscortLogEntry>& logs) {
        int tEscort = 0; // 分母：J-16真正面临威胁的累计秒数
        int tSafe = 0;   // 分子：面临威胁且雷达全关机的秒数

        for (const auto& entry : logs) {
            if (entry.isExposed == true) {
                tEscort++; // 累加分母
                
                if (entry.isSafe == true) {
                    tSafe++; // 累加分子
                }
            }
        }

        if (tEscort == 0) {
            return 100.0; // 避免除以0，如果全程未暴露则视为绝对安全
        }

        return (static_cast<double>(tSafe) / tEscort) * 100.0;
    }
};

// 调用示例：
// auto logs = SuppressionEvaluator::loadEscortLog("Escort_Spatiotemporal_Log.csv");
// double P2 = SuppressionEvaluator::calculateP2(logs);
// 假设引擎算出 tEscort 结果是 195 秒，tSafe 是 178 秒
// 最终 P2 = (178.0 / 195.0) * 100.0 = 91.3%
```
**P2 = 91.3%，评级：优秀**
