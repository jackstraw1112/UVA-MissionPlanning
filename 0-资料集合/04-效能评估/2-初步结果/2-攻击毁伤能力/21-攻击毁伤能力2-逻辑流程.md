# 总体流程
1. 设置战斗机主打击任务与目标体系（关联蓝方雷达实体与高价值目标）。
2. 添加反辐射无人机协同打击任务，并为每个雷达目标配置攻击批次。
3. 任务分配时，按“高威胁优先”原则给每个核心雷达分配无人机（建议每目标不少于2架，主攻+补刀）。
4. 对无人机进行部署并设计航路，确保能够进入导引头截获距离。
5. 初始化攻击毁伤能力评估参数：`D_acq`、`T_dive`、`T_react`、`T_memo`、`P_hit_base`、`eta_fuze`、`V_k`。
6. 进入仿真主循环，按时间步执行 S1 -> S2 -> S3 状态机判定并记录日志。

# 单次交战状态机（L1）
1. S1：目标截获状态判定
	1. 若目标雷达已被摧毁（`RADAR_DEAD`），输出 `Target_Already_Killed`，当前交战链终止。
	2. 若无人机与目标距离 `D > D_acq`，输出 `Out_of_Range`，维持巡航/机动。
	3. 若 `D <= D_acq` 且满足辐射几何条件，输出 `Acquired`，进入 S2。
2. S2：末段攻击状态判定（时间线博弈）
	1. 初始化俯冲时间线：`t=0`，总时长 `T_dive`。
	2. 每秒推进循环（`while t < T_dive`）：
		1. 若 `t >= T_react`，目标进入关机状态，触发我方记忆倒计时。
		2. 若目标已关机且 `(t - T_react) >= T_memo`，输出 `Lost_Shutdown`（记忆耗尽脱靶）。
		3. 若目标未关机且制导稳定性失败，输出 `Miss_Guide_Error`（信号断连脱靶）。
		4. 若诱饵生效且偏置超出杀伤半径，输出 `Lost_Decoy`。
	3. 若循环结束仍未脱靶，执行基础命中判定：
		1. 生成随机数 `R1`。
		2. 若 `R1 <= P_hit_base`，输出 `Hit`，进入 S3。
		3. 否则输出 `Miss_Guide_Error`。
3. S3：最终毁伤等级判定（起爆质量映射）
	1. 前提：S2=`Hit`。
	2. 生成引战随机数 `R_fuse`：
		1. 若 `R_fuse <= eta_fuze`（优质起爆区）：
			1. 生成穿透随机数 `R_vk`。
			2. 若 `R_vk <= V_k`，输出 `K_Kill`（硬毁伤/彻底摧毁）。
			3. 若 `R_vk > V_k`，输出 `F_Disable`（功能瘫痪）。
		2. 若 `R_fuse > eta_fuze`（非优质起爆区）：
			1. 对外露阵列目标，优先输出 `F_Disable`。
			2. 对方舱化目标，执行边缘毁伤判定，输出 `F_Disable` 或 `Light_Damage`。

# 多次统计评估（L2）
1. 执行多轮蒙特卡洛仿真（建议 1000 次及以上）。
2. 聚合 S2 状态，统计攻击链路达成率（E1）：
	1. `E1 = Hit次数 / 总交战次数`。
3. 聚合 S3 状态，统计硬毁伤达成率（E2）：
	1. `E2 = K_Kill次数 / S2命中次数`（或按总次数口径另行统计）。
4. 输出 K/F/Light 各等级占比，形成目标抗毁画像。

# 仿真数据保存要求
1. 交战事件日志（`Engage_Event_Log`）
	1. 记录字段：`Run_ID`、`UAV_ID`、`Target_ID`、`S1`、`S2`、`S3`、关键时间戳、关键随机数。
2. 时间线过程日志（`Dive_Timeline_Log`）
	1. 记录字段：`t`、目标开关机状态、记忆倒计时状态、诱饵状态、节点判定结果。
3. 毁伤结果日志（`Damage_Result_Log`）
	1. 记录字段：`R_fuse`、`R_vk`、目标类型、最终毁伤等级、恢复时间参数（如适用）。

# 核心计算口径
1. `P_hit_base` 由 CEP 与有效杀伤半径溯源计算，禁止人工拍脑袋赋值。
2. `eta_fuze` 由引信-目标几何匹配能力设定，区分目标类型（外露阵列/方舱化/小型机动）。
3. `V_k` 由目标结构易损性设定，反映“可穿透性”而非“是否被命中”。
4. S2 负责“能否打进去”，S3 负责“打进去后能否打死”，两者严格解耦。

# 与辐射源压制效益（P1/P2）联动流程
1. 若 S3=`K_Kill`：
	1. 雷达状态置为 `RADAR_DEAD`，长期记为关机/失能。
2. 若 S3=`F_Disable`：
	1. 雷达状态置为 `RADAR_DISABLED`，按维修时间窗口记录关机段。
3. 若 S3=`Light_Damage`：
	1. 仅记录轻伤，不默认改变雷达核心开机能力。
4. 若 S2=`Lost_Shutdown`：
	1. 记录敌方主动关机形成的短时软压制窗口（可用于 P1/P2 统计）。
5. 联动输出：
	1. 将 S2/S3结果降维映射到雷达状态时序表，供后续 P1（关机覆盖率）与 P2（突防安全覆盖率）统一统计。

# 结果解释流程
1. 优先判断 S2：若 E1 偏低，说明瓶颈在“末段时间线博弈”（关机反制/记忆不足）。
2. 再判断 S3：若 E1 尚可但 E2 偏低，说明瓶颈在“起爆质量与结构穿透”。
3. 输出战术建议：
	1. S2瓶颈：优化航路、缩短 `T_dive`、提升 `T_memo`、增加波次压制。
	2. S3瓶颈：优化引信策略、提高战斗部破片效能、采用多机齐射提升毁伤概率。
