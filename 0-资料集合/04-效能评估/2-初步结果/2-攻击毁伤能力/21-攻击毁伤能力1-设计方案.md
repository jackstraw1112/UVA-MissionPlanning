# 反辐射无人机攻击毁伤能力评估与仿真设计方案（参数溯源与推演实践完整版）

## 一、 方案概述

### 1.1 推演系统痛点剖析
在基于蒙特卡洛的作战推演系统中，打击效果并非通过解算物理弹道和破片飞散生成，而是通过**“概率阈值对比随机数”**直接裁决（如 `rand() < P_hit` 则判定命中）。
在这种底层逻辑下，传统的“末端制导精度(CEP)”指标在推演中面临**不可评估**的困境：推演引擎并没有生成真实的落点坐标，强行通过随机数反推CEP不仅违背逻辑，还会产生伪精度误差。

### 1.2 解决思路：状态机跃迁与参数溯源
本方案彻底摒弃对中间物理量（CEP、炸高、脱靶量）的提取，将评估指标直接锚定推演引擎原生的**“实体状态跃迁”**。
同时，彻底解决推演参数“无源之水”的问题：引擎配置表中的每一个概率阈值，严禁凭空赋值，必须能通过严密的物理公式和战术逻辑，**溯源到无人机的CEP、杀伤半径，以及目标的易损性结构**上。

## 二、 评估指标体系：双层解耦架构

指标分为“单次交战层（看过程）”和“多次统计层（看能力）”。单次指标是推演日志；多次指标是基于海量单次日志的统计学聚合。

| 层级 | 指标代号 | 指标名称 | 评估维度 | 数据来源 |
| :--- | :--- | :--- | :--- | :--- |
| **L1: 单次交战<br>(必选主推)** | **S1** | **目标截获状态** | 无人机找到目标了吗？ | 推演引擎空间距离判定 |
| | **S2** | **末段攻击状态** | 无人机成功打进去了吗？ | 引擎随机数与阈值比对 |
| | **S3** | **最终毁伤等级** | 目标雷达最后变成啥样了？ | 引擎随机数与阈值比对 |
| **L2: 多次统计<br>(扩充论证)** | **E1** | **攻击链路达成率** | 综合命中能力到底如何？ | 1000次S2状态的聚合统计 |
| | **E2** | **硬毁伤达成率** | 命中后的杀伤力到底如何？ | 1000次S3状态的聚合统计 |

## 三、 核心参数字典与设定规范（底层Config依据）

推演系统在执行状态跃迁时，必须依赖底层的物理参数和概率阈值。本章为参数设定的强制规范字典。

### 3.1 L1层：单次交战状态机触发参数（非概率判定基础）

| 参数名称           | 符号          | 物理含义                              | 设定依据与获取途径                                        | 典型值范围                 |
| :------------- | :---------- | :-------------------------------- | :----------------------------------------------- | :-------------------- |
| **导引头截获距离**    | $D_{acq}$   | 无人机被动雷达导引头能稳定识别目标信号的物理距离。         | 装备说明书/靶场测试数据。受目标雷达发射功率制约。                        | 15km ~ 25km           |
| **目标雷达旁瓣电平**   | $L_{sl}$    | 目标雷达主瓣之外的信号泄露强度。                  | 情报侦察数据。决定了无人机能否从侧向或尾向逼近而不丢失目标。                   | -20dB ~ -30dB         |
| **目标雷达关机响应时间** | $T_{react}$ | 敌雷达操作员从收到告警到完成关机动作的物理耗时。          | 结合《台军雷达手册》评估：固定大型雷达流程长；轻型机动雷达自动化高。               | 固定: 8~15s<br>机动: 3~8s |
| **无人机末制导记忆时间** | $T_{memo}$  | 目标突然关机后，导引头依靠惯性导航维持正确弹道而不发散的最大时长。 | 装备设计指标。若 $T_{react} < T_{memo}$，则关机无效。           | 5s ~ 12s              |
| **诱饵诱惑偏置距离**   | $D_{decoy}$ | 敌方有源诱饵开启后，在导引头视野中产生的角度偏移等效距离。     | 电子战侦察数据。若 $D_{decoy} > R_{lethal}$ (杀伤半径)，则诱饵成功。 | 50m ~ 200m            |

### 3.2 L2层：多次统计概率阈值参数溯源规范

推演引擎在进行蒙特卡洛循环时，需要读取以下拆解公式计算得出的概率阈值。

#### 3.2.1 攻击链路达成率阈值 ($P_{engage\_config}$) 拆解字典
此参数决定引擎是否输出 `S2 = Hit`。
$$ P_{engage\_config} = P_{hit\_base} \times P_{track\_stable} \times P_{overcome\_counter} $$

*   **参数A：基础命中概率 ($P_{hit\_base}$) —— 核心物理计算**
    *   **物理意义**：在无对抗、完美跟踪下，仅由飞控误差和战斗部散布决定的命中概率。
    *   **计算公式**：$P_{hit\_base} = 1 - 0.5^{\left(\frac{R_{lethal}}{CEP}\right)^2}$
    *   **易混淆参数详解**：$R_{lethal}$ (战斗部有效杀伤半径) 不是爆炸冲击波半径，而是针对特定目标的“破片穿透临界半径”（破片密度 $\ge 1$ 枚/$m^2$，且比动能 $\ge 100~J/cm^2$）。
*   **参数B：稳定跟踪概率 ($P_{track\_stable}$)**
    *   **设定标准**：相控阵雷达(持续照射) **0.95~0.98**；机械扫描雷达(波束扫过) **0.70~0.85**。
*   **参数C：抗关机/诱饵成功概率 ($P_{overcome\_counter}$)**
    *   **设定标准**：若无记忆跟踪能力，设为 **0.1**；若有，$P_{overcome\_counter} = \frac{T_{memo} - T_{react}}{T_{memo}}$（若为负，取 **1.0**）。

#### 3.2.2 硬毁伤达成率阈值 ($P_{K\_Kill\_config}$) 拆解字典
此参数在引擎判定 `S2=Hit` 后，决定输出 `K_Kill` 还是 `F_Disable`。
$$ P_{K\_Kill\_config} = V_k \times \eta_{fuze} $$

*   **参数D：目标易损性系数 ($V_k$) —— 决定性战术参数**
    *   **第一类：外露阵列式**（如长白雷达、铺路爪）：阵面直接暴露。设定 $V_k =$ **0.85 ~ 0.95**。
    *   **第二类：背罩式/方舱式**（如爱国者MPQ-65）：核心组件被金属方舱包裹。设定 $V_k =$ **0.30 ~ 0.50**。
    *   **第三类：小型全机动式**（如AN/MPQ-64）：天线极小且隐蔽。设定 $V_k =$ **< 0.20**。
*   **参数E：引战配合效率 ($\eta_{fuze}$)**
    *   **设定标准**：大型固定面(易触发近炸) **0.90~0.95**；小型机动目标 **0.70~0.80**。

### 3.3 联动接口参数：功能瘫痪(F级)持续时间映射规范
当输出 `S3 = F_Disable` 时，转化为压制效益模块(P1/P2)中的“雷达关机时长”。

| 目标类型 | 物理抢修逻辑 | $T_{restore}$ 分布模型 | 赋值示例 (引擎随机数) |
| :--- | :--- | :--- | :--- |
| **大型固定雷达** | 无备件、需吊车拆卸。 | 正态分布 $N(\mu=3600s, \sigma=600s)$ | `random_normal(3600, 600)` |
| **中型方舱雷达** | 美军标备件箱，模块化拔插。 | 均匀分布 $U(600s, 1800s)$ | `random_uniform(600, 1800)` |
| **轻型伴随雷达** | 备用车替换，或重启备机。 | 均匀分布 $U(120s, 600s)$ | `random_uniform(120, 600)` |

## 四、 L1层：单次交战状态机跃迁判定流程（引擎核心计算逻辑）

本章节详细规定推演引擎在每一个仿真步长中，如何依据第三章节的参数，一步步计算并输出 S1、S2、S3 的具体枚举值。

### 4.1 S1：目标截获状态判定流程
S1 判定完全基于**空间几何与实体状态**，不涉及概率掷骰子。

```text
开始 S1 判定
  │
  ├─ 步骤1：读取目标雷达当前全局状态
  │    └─ 若 radar.status == RADAR_DEAD (已被摧毁)
  │       └─ 输出：S1 = Target_Already_Killed (目标已死，终止交战)
  │
  ├─ 步骤2：空间距离计算
  │    └─ 计算无人机与雷达的空间距离 D
  │       └─ 若 D > 导引头截获距离 D_acq
  │          └─ 输出：S1 = Out_of_Range (未进入截获区，保持巡航)
  │
  └─ 步骤3：辐射区几何判定
       └─ 若 D ≤ D_acq，计算无人机是否处于雷达主瓣/高旁瓣辐射锥内
          ├─ 若在辐射锥内
          │   └─ 输出：S1 = Acquired (成功截获，触发 S2 判定流程)
          └─ 若不在辐射锥内 (处于雷达盲区飞行)
              └─ 输出：S1 = No_Signal_Blind (信号被地形遮挡或处于旁瓣盲区)
```

### 4.2 S2：末段攻击状态判定流程
**前置条件**：仅当 `S1 == Acquired` 时触发。S2 判定引入**蒙特卡洛随机数**与**战术对抗逻辑**。

```text
开始 S2 判定 (已获得 S1 = Acquired)
  │
  ├─ 步骤1：生成系统随机数 R1 (0.0 ~ 1.0)
  │
  ├─ 步骤2：与攻击链路达成阈值比对
  │    └─ 读取配置表中的 P_engage_config
  │
  ├─ 分支A：若 R1 ≤ P_engage_config (判定命中成功)
  │    └─ 输出：S2 = Hit (成功进入引爆区，触发 S3 判定流程)
  │
  └─ 分支B：若 R1 > P_engage_config (判定未命中，需解析失败原因)
       │
       ├─ 对抗解析1：检查目标关机逻辑
       │    └─ 生成敌方反应随机数，判定敌是否采取关机对抗
       │       ├─ 若敌采取关机，且计算 T_react < T_memo (我方记忆成功)
       │       │   └─ 跳过此解析，进入对抗解析2
       │       └─ 若敌采取关机，且 T_react ≥ T_memo (我方记忆失败)
       │          └─ 输出：S2 = Lost_Shutdown (因目标关机导致脱靶)
       │
       ├─ 对抗解析2：检查有源诱饵逻辑
       │    └─ 判定敌是否开启诱饵
       │       └─ 若开启，且计算偏置距离 D_decoy > R_lethal
       │          └─ 输出：S2 = Lost_Decoy (被诱饵欺骗导致脱靶)
       │
       └─ 兜底解析：若未触发上述对抗，或对抗未生效
            └─ 输出：S2 = Miss_Guide_Error (基础制导误差导致脱靶)
```

### 4.3 S3：最终毁伤等级判定流程
**前置条件**：仅当 `S2 == Hit` 时触发。S3 判定决定目标的最终物理形态。

```text
开始 S3 判定 (已获得 S2 = Hit)
  │
  ├─ 步骤1：生成系统随机数 R2 (0.0 ~ 1.0)
  │
  ├─ 步骤2：与硬毁伤达成阈值比对
  │    └─ 读取配置表中的 P_K_Kill_config
  │
  ├─ 分支A：若 R2 ≤ P_K_Kill_config (判定彻底摧毁)
  │    └─ 输出：S3 = K_Kill (物理摧毁/报废)
  │
  └─ 分支B：若 R2 > P_K_Kill_config (判定未彻底摧毁，解析毁伤程度)
       │
       ├─ 逻辑说明：由于 S2=Hit 意味着破片场已覆盖目标，在此逻辑下，
       │           目标不可能毫发无损，必然停留在功能受损状态。
       │
       ├─ 细分判定 (可选)：生成随机数 R3 判定受损部位
       │    ├─ 若受损部位为天线面、馈源等外露部件
       │    │   └─ 输出：S3 = F_Disable (功能瘫痪，无法探测但可修复)
       │    └─ 若受损部位为非核心附属设施(如空调、电缆外露段)
       │        └─ 输出：S3 = Light_Damage (轻伤，不影响核心作战功能)
       │
       └─ 简化判定 (推荐默认)：
            └─ 直接输出：S3 = F_Disable 
               (注：在战术推演中，只要打中而没炸碎，一律按瘫痪处理，
               这是最稳妥的效能评估底线，可大幅降低引擎计算复杂度)
```

## 六、 仿真推演与案例深度剖析

### 6.1 场景一：单次战术推演复盘（L1单次指标应用）
**态势**：红方出动 4 架无人机，分别打击上述两部雷达。指挥员查看的是单次状态跃迁日志。

| Run_ID | UAV_ID | Target_ID | S1_截获状态 | S2_攻击状态 | S3_毁伤等级 | 战术备注（底层判定逻辑解析） |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| R-001 | UAV-01 | Radar-A | **Acquired** | **Hit** | **K_Kill** | 随机数0.65 < 0.721(S2命中)；0.4 < 0.855(S3硬毁)。彻底炸毁。 |
| R-001 | UAV-02 | Radar-A | **Target_Already_Killed** | - | - | 触发S1步骤1判定，因UAV-01已摧毁目标，直接改航。 |
| R-001 | UAV-03 | Radar-B | **Acquired** | **Lost_Shutdown** | - | 随机数本可命中，但触发S2分支B对抗解析1：敌关机5s<我方记忆10s...（注：若遇更先进雷达则脱靶）*此处假设为复杂电磁环境下记忆失效*，脱靶。 |
| R-001 | UAV-04 | Radar-B | **Acquired** | **Hit** | **F_Disable** | 随机数0.28 < 0.313(S2命中)；但0.5 > 0.340(进入S3分支B)，方舱挡住破片，按简化逻辑输出F级瘫痪。 |

### 6.2 场景二：装备战法论证推演（L2多次指标应用）
**态势**：系统后台分别对两部雷达各发起 **1000次独立的蒙特卡洛仿真**（每次重置目标状态，仅派1架）。

**1000次推演聚合统计结果：**

| 统计项 | Target_A (长白雷达) | Target_B (爱国者MPQ-65) |
| :--- | :--- | :--- |
| 推演总架次 | 1000 次 | 1000 次 |
| 判定命中次数 (S2=Hit) | **719 次** | **316 次** |
| 判定 K_Kill 次数 | **615 次** | **107 次** |
| 判定 F_Disable 次数 | 104 次 | 209 次 |

**指标计算与深度溯源分析：**

**1. 攻击链路达成率 (E1)** = $Count(S2=Hit) / 1000$
*   **长白雷达**：$E1 = \mathbf{71.9\%}$ (吻合配置表 $P_{engage}=0.721$)
*   **爱国者雷达**：$E1 = \mathbf{31.6\%}$ (吻合配置表 $P_{engage}=0.313$)
*   **[溯源诊断]**：打爱国者命中率暴跌的核心原因，**不是因为我方无人机精度变差了**（基础命中概率都是0.736），而是因为**敌雷达关机反应太快(5s)**，我方记忆时间(10s)余量不足，导致抗关机概率仅50%，直接腰斩命中链路。

**2. 硬毁伤达成率 (E2)** = $Count(S3=K\_Kill) / Count(S2=Hit)$
*   **长白雷达**：$E2 = 615 / 719 = \mathbf{85.5\%}$ (吻合配置表 $P_{K\_Kill}=0.855$)
*   **爱国者雷达**：$E2 = 107 / 316 = \mathbf{33.9\%}$ (吻合配置表 $P_{K\_Kill}=0.340$)
*   **[溯源诊断]**：打爱国者毁伤率低的核心原因，**不是因为战斗部威力小**，而是因为**目标采用方舱防护设计($V_k$仅0.4)**，破片无法穿透核心舱段。

**3. 终极指标：完整摧毁概率 ($P_k$)** = $E1 \times E2$
*   **长白雷达**：$P_k = \mathbf{61.5\%}$ $\rightarrow$ **单发具备较高摧毁把握。**
*   **爱国者雷达**：$P_k = \mathbf{10.7\%}$ $\rightarrow$ **单发基本无法摧毁，大量停留在F级瘫痪状态。**

## 七、 与“辐射源压制效益(P1/P2)”的接口联动

上述毁伤评估不是孤立的，L1层的 `S3_毁伤等级` 将直接转化为时间轴上的 0/1 状态，驱动《辐射源压制效益评估》中的核心指标计算。

**底层联动逻辑（C++伪代码规范）：**

```cpp
void updateStrikeAndSuppression(int currentTick, UAV& uav, Radar& radar) {
    // 1. 执行第四章的 S1 状态机判定
    string s1_status = evaluateS1Status(uav, radar);
    
    if (s1_status == "Target_Already_Killed") {
        MissionPlanner::reassignTarget(uav); // 触发的改变航逻辑
        return;
    }
    else if (s1_status != "Acquired") {
        return; // 未截获或不在辐射区，继续飞行
    }

    // 2. 执行第四章的 S2、S3 状态机判定
    EngageResult result = evaluateS2AndS3(uav, radar); 

    // 3. 将 S3 毁伤结果映射为 P1(雷达关机覆盖率) 需要的状态流
    if (result.S3 == "K_Kill") {
        radar.status = RADAR_DEAD;
        // 接口输出：从当前Tick开始，该雷达永远为 0 (关机)
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    } 
    else if (result.S3 == "F_Disable") {
        radar.status = RADAR_DISABLED;
        // 接口输出：调用第三章规定的 T_restore 分布模型生成恢复时间
        int disableTime = random_uniform(600, 1800); // 爱国者方舱抢修时间
        radar.restartTick = currentTick + disableTime;
        // 在这段时间内，状态强制为 0 (关机)，计入 P1 的分子
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
    else if (result.S2 == "Lost_Shutdown") {
        // 逻辑陷阱：敌为躲避攻击主动关机，无人机脱靶，但雷达客观上处于关机状态！
        // 接口输出：同样记录关机状态，体现“硬杀伤未成，但软压制生效”
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
}
```
