# 反辐射无人机攻击毁伤能力评估与仿真设计方案（时间线博弈与起爆映射完整版）

## 一、 方案概述

### 1.1 推演系统痛点剖析
在基于蒙特卡洛的作战推演系统中，打击效果并非通过解算物理弹道和破片飞散生成，而是通过**“概率阈值对比随机数”**直接裁决。
在这种底层逻辑下，传统的“末端制导精度(CEP)”指标在推演中面临**不可评估**的困境。同时，传统的“静态单次掷骰子”判定模式（截获后瞬间算出命中和毁伤），完全抹杀了反辐射作战中最重要的**“俯冲倒计时博弈”**与**“引信起爆时机对抗”**过程。

### 1.2 解决思路：时间线博弈与起爆质量映射
本方案彻底摒弃对中间物理坐标的提取，将评估指标直接锚定推演引擎原生的**“实体状态跃迁”**。
同时，对状态判定逻辑进行深度战术化重构：
1.  **S2优化**：引入“俯冲时间线”模型，将抗关机转化为飞行秒级的倒计时博弈。
2.  **S3优化**：引入“起爆质量区间映射”模型，将引战配合与目标易损性转化为清晰的物理逻辑分支。
引擎配置表中的每一个概率阈值，严禁凭空赋值，必须能通过严密的物理公式溯源到装备的底层参数上。

## 三、 核心参数字典与设定规范（底层Config依据）

### 3.1 基础物理参数与时间参数

| 参数名称 | 符号 | 物理含义 | 典型值范围/设定标准 |
| :--- | :--- | :--- | :--- |
| **导引头截获距离** | $D_{acq}$ | 被动雷达稳定识别信号的物理距离。 | 15km ~ 25km |
| **俯冲攻击总时间** | $T_{dive}$ | 从进入截获距离开始俯冲，到抵达目标上方引爆的飞行时间。受速度和俯冲角影响。 | **10秒 ~ 20秒** |
| **目标雷达关机响应时间** | $T_{react}$ | 敌方从告警到完成关机的耗时。固定雷达慢，机动雷达快。先进雷达可在俯冲前预判关机(值为负或极小)。 | 固定: 8~15s<br>先进机动: <5s |
| **无人机末制导记忆时间** | $T_{memo}$ | 目标关机后，导引头依靠惯导维持正确弹道的最大时长。 | 5s ~ 12s |
| **诱饵偏置距离** | $D_{decoy}$ | 有源诱饵产生的角度偏移等效距离。 | 50m ~ 200m |

### 3.2 概率阈值参数溯源规范

#### 3.2.1 基础命中概率 ($P_{hit\_base}$)
*   **物理意义**：无对抗下，纯飞控误差导致的散布覆盖概率。
*   **计算公式**：$P_{hit\_base} = 1 - 0.5^{\left(\frac{R_{lethal}}{CEP}\right)^2}$
*   **注**：$R_{lethal}$ 为破片穿透临界半径（破片密度 $\ge 1$枚/$m^2$ 且比动能 $\ge 100~J/cm^2$ 的距离）。

#### 3.2.2 引战配合效率 ($\eta_{fuze}$) —— S3核心前置参数
*   **物理意义**：无人机飞抵目标上方时，近炸/触发引信在**最佳炸高区间**成功起爆的概率。
*   **设定标准**：大型固定面(易触发近炸) **0.90~0.95**；小型机动目标(反射截面小易早炸/晚炸) **0.70~0.80**。

#### 3.2.3 目标易损性系数 ($V_k$) —— S3核心穿透参数
*   **物理意义**：在**最佳炸高起爆**的前提下，破片场摧毁核心部件的概率。
*   **设定标准**：
    *   **外露阵列式**（长白、铺路爪）：无装甲，打断单元即烧毁。$V_k =$ **0.85 ~ 0.95**
    *   **方舱化设计**（爱国者MPQ-65）：核心在金属壳内，仅靠破片极难穿透。$V_k =$ **0.30 ~ 0.50**
    *   **小型全机动**（哨兵雷达）：天线极小且隐蔽。$V_k =$ **< 0.20**

### 3.3 联动接口参数：功能瘫痪(F级)持续时间 ($T_{restore}$)
*   **大型固定雷达**：正态分布 $N(\mu=3600s, \sigma=600s)$ (无备件需吊装)
*   **中型方舱雷达**：均匀分布 $U(600s, 1800s)$ (美军标模块化拔插)

## 四、 L1层：单次交战状态机跃迁判定流程（引擎核心计算逻辑）

### 4.1 S1：目标截获状态判定流程
纯空间几何与实体状态判定，无概率掷骰子。

```text
开始 S1 判定
  ├─ 若 radar.status == RADAR_DEAD
  │   └─ 输出：S1 = Target_Already_Killed (终止交战，触发改航)
  ├─ 若 空间距离 D > D_acq
  │   └─ 输出：S1 = Out_of_Range (保持巡航)
  └─ 若 D ≤ D_acq 且 处于雷达主瓣/高旁瓣辐射锥内
      └─ 输出：S1 = Acquired (成功截获，启动 S2 俯冲计时器)
```

### 4.2 S2：末段攻击状态判定流程（基于时间线的俯冲博弈模型）
**优化核心**：引入“俯冲总时间($T_{dive}$)”，模拟无人机与雷达操作员的“生死倒计时”。

```text
开始 S2 判定 (已获得 S1 = Acquired)
  │
  ├─ 步骤1：初始化俯冲时间线 (t = 0，总时长 T_dive = 15秒)
  │
  ├─ 步骤2：进入按秒推进的俯冲博弈循环 (while t < T_dive)
  │    │
  │    ├─ [敌方节点] 若 t ≥ T_react (如第5秒)
  │    │   └─ 敌方状态切换为：Radar_Shutdown，触发我方记忆倒计时
  │    │
  │    ├─ [我方节点] 若敌已关机，且经过时间 (t - T_react) ≥ T_memo
  │    │   └─ 跳出循环，输出：S2 = Lost_Shutdown (记忆耗尽，脱靶)
  │    │
  │    ├─ [环境节点] 若敌未关机，且随机数 > P_track_stable
  │    │   └─ 跳出循环，输出：S2 = Miss_Guide_Error (信号断连脱靶)
  │    │
  │    └─ [诱饵节点] 若敌开启诱饵，且 D_decoy > R_lethal
  │        └─ 跳出循环，输出：S2 = Lost_Decoy (被诱饵欺骗脱靶)
  │
  ├─ 步骤3：循环正常结束 (t 达到 T_dive，成功挺过对抗抵达引爆区)
  │
  └─ 步骤4：最终基础散布判定 (纯物理散布，与对抗无关)
       ├─ 生成随机数 R1
       └─ 若 R1 ≤ P_hit_base (基于CEP和R_lethal计算得出)
           ├─ 成立：输出 S2 = Hit (进入引爆区，触发 S3 判定)
           └─ 不成立：输出 S2 = Miss_Guide_Error (飞到了但散布偏了一点)
```

### 4.3 S3：最终毁伤等级判定流程（基于起爆质量区间的映射模型）
**优化核心**：摒弃直接用 $V_k \times \eta_{fuze}$ 掷骰子。先用随机数模拟“引信起爆质量”，再根据质量分支结合“目标防护结构”进行判定，完美还原“打得准不如炸得好，炸得好不如穿得透”的物理逻辑。

```text
开始 S3 判定 (已获得 S2 = Hit，说明破片场已覆盖目标区域)
  │
  ├─ 步骤1：引战配合判定 (模拟炸点相对于目标核心的位置)
  │    └─ 生成随机数 R_fuse (0.0 ~ 1.0)
  │
  ├─ 分支A：优质起爆区 (R_fuse ≤ η_fuze，如 ≤ 0.90)
  │    │   物理意义：触发了最佳近炸高度，或直接撞击天线阵面，破片场完美覆盖目标要害。
  │    │
  │    └─ 步骤2：核心结构抗毁判定
  │         └─ 生成随机数 R_vk (0.0 ~ 1.0)
  │             ├─ 若 R_vk ≤ V_k (如外露雷达 0.90)
  │             │   └─ 输出：S3 = K_Kill (破片轻易穿透薄弱结构，彻底报废)
  │             └─ 若 R_vk > V_k (如方舱雷达 0.40)
  │                 └─ 输出：S3 = F_Disable (炸得极好，但被方舱装甲硬扛下来，仅外露件受损瘫痪)
  │
  └─ 分支B：非优质起爆区 (R_fuse > η_fuze，如 0.10 的概率区间)
      │   物理意义：引信早炸/晚炸，导致破片仅扫过目标边缘，或未正对核心舱段。
      │
      └─ 步骤3：边缘毁伤逻辑判定
           ├─ 若目标为 外露阵列式 (如长白)
           │   └─ 输出：S3 = F_Disable (外露设备极其脆弱，边缘扫过也足以打坏馈源导致瘫痪)
           └─ 若目标为 方舱化设计 (如爱国者)
               ├─ 生成随机数 R_edge
               ├─ 若 R_edge ≤ 0.5
               │   └─ 输出：S3 = F_Disable (边缘破片偶然打断了外露的光缆/天线驱动马达)
               └─ 若 R_edge > 0.5
                   └─ 输出：S3 = Light_Damage (破片连方舱外壳都没打透，核心设备毫发无损，雷达继续运转)
```

**深度战术溯源诊断：**

1.  **关于 S2 (链路达成率 E1) 的倒计时博弈真相**：
    *   **长白雷达**为什么命中率(73.8%)几乎等于基础命中概率？因为 $T_{react}(12s) + T_{memo}(10s) = 22s$，远大于俯冲时间 $T_{dive}(15s)$。**长白雷达在物理上根本不具备对抗俯冲的能力**，只要被截获就等于被命中。
    *   **爱国者雷达**为什么命中率暴跌至21.5%？因为其 $T_{react}(4s) + T_{memo}(10s) = 14s$，小于 $T_{dive}(15s)$。在推演时间线上，爱国者操作员在第4秒关机后，无人机刚好在第14秒记忆耗尽迷失方向，仅差1秒就能撞上！这种“极限拉扯”在静态概率模型中是完全体现不出来的。

2.  **关于 S3 (硬毁伤达成率 E2) 的起爆映射真相**：
    *   **长白雷达** E2 高达 85.2%，是因为一旦命中，引信极大概率工作正常(0.95)，而外露阵面又极易被穿透(0.90)。
    *   **爱国者雷达** E2 仅为 33.5%，且出现了 **7.5% 的 Light_Damage (轻伤)**。这完美解释了战术困境：即便我方无人机运气爆棚，突破了时间线博弈成功打入引爆区，由于方舱的装甲防护($V_k$仅0.4)和复杂的反射特性导致的引信偏差，仍然有高达 66.5% 的概率无法将其彻底摧毁，甚至有极小概率“挠痒痒”(Light_Damage)。

## 七、 与“辐射源压制效益(P1/P2)”的接口联动

S3的优化使得状态输出更加丰富，与压制效益模块的联动逻辑也需相应细化。

**底层联动逻辑（C++伪代码规范）：**

```cpp
void updateStrikeAndSuppression(int currentTick, UAV& uav, Radar& radar) {
    // 执行完整的 S1 -> S2时间线 -> S3起爆映射 判定链
    EngageResult result = evaluateFullEngageChain(uav, radar); 

    // 将复杂的 S3 毁伤结果，降维映射为 P1(雷达关机覆盖率) 需要的 0/1 状态流
    if (result.S3 == "K_Kill") {
        radar.status = RADAR_DEAD;
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); // 永久关机
    } 
    else if (result.S3 == "F_Disable") {
        radar.status = RADAR_DISABLED;
        int disableTime = random_uniform(600, 1800); // 方舱抢修时间
        radar.restartTick = currentTick + disableTime;
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); // 瘫痪期关机
    }
    else if (result.S3 == "Light_Damage") {
        // 逻辑关键点：轻伤状态，雷达核心功能完好，不触发关机状态！
        // 此时该雷达依然对 J-16 构成致命威胁，P2 突防安全率不会因此提升。
        // 推演日志仅记录受损，不改变 P1 状态流。
        radar.status = RADAR_LIGHT_DAMAGED; 
    }
    else if (result.S2 == "Lost_Shutdown") {
        // 敌主动关机导致我方脱靶，客观上形成软压制
        // 假设敌雷达在确认安全后会重新开机，设定一个短暂的被迫关机时间(如30-60秒)
        int forcedOffTime = random_uniform(30, 60);
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
        SuppressionLogger::logRadarStatus(radar.id, currentTick + forcedOffTime, 1); // 恢复开机
    }
}
```
