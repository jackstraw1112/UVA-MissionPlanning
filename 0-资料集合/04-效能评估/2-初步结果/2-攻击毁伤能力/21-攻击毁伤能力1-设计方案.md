# 反辐射无人机攻击毁伤能力评估与仿真设计方案（参数溯源与推演实践完整版）

## 一、 方案概述

### 1.1 推演系统痛点剖析
在基于蒙特卡洛的作战推演系统中，打击效果并非通过解算物理弹道和破片飞散生成，而是通过**“概率阈值对比随机数”**直接裁决（如 `rand() < P_hit` 则判定命中）。
在这种底层逻辑下，传统的“末端制导精度(CEP)”指标在推演中面临**不可评估**的困境：推演引擎并没有生成真实的落点坐标，强行通过随机数反推CEP不仅违背逻辑，还会产生伪精度误差。

### 1.2 解决思路：状态机跃迁与参数溯源
本方案彻底摒弃对中间物理量（CEP、炸高、脱靶量）的提取，将评估指标直接锚定推演引擎原生的**“实体状态跃迁”**。
同时，彻底解决推演参数“无源之水”的问题：引擎配置表中的每一个概率阈值，严禁凭空赋值，必须能通过严密的物理公式和战术逻辑，**溯源到无人机的CEP、杀伤半径，以及目标的易损性结构**上。

## 二、 评估指标体系：双层解耦架构

指标分为“单次交战层（看过程）”和“多次统计层（看能力）”。单次指标是推演日志；多次指标是基于海量单次日志的统计学聚合。

| 层级 | 指标代号 | 指标名称 | 评估维度 | 数据来源 |
| :--- | :--- | :--- | :--- | :--- |
| **L1: 单次交战<br>(必选主推)** | **S1** | **目标截获状态** | 无人机找到目标了吗？ | 推演引擎空间距离判定 |
| | **S2** | **末段攻击状态** | 无人机成功打进去了吗？ | 引擎随机数与阈值比对 |
| | **S3** | **最终毁伤等级** | 目标雷达最后变成啥样了？ | 引擎随机数与阈值比对 |
| **L2: 多次统计<br>(扩充论证)** | **E1** | **攻击链路达成率** | 综合命中能力到底如何？ | 1000次S2状态的聚合统计 |
| | **E2** | **硬毁伤达成率** | 命中后的杀伤力到底如何？ | 1000次S3状态的聚合统计 |

## 三、 核心参数字典与设定规范（底层Config依据）

推演系统在执行状态跃迁时，必须依赖底层的物理参数和概率阈值。本章为参数设定的强制规范字典。

### 3.1 L1层：单次交战状态机触发参数（非概率判定）

L1层的 S1/S2 状态跃迁，在底层是由物理量直接判定的。

| 参数名称 | 符号 | 物理含义 | 设定依据与获取途径 | 典型值范围 |
| :--- | :--- | :--- | :--- | :--- |
| **导引头截获距离** | $D_{acq}$ | 无人机被动雷达导引头能稳定识别目标信号的物理距离。 | 装备说明书/靶场测试数据。受目标雷达发射功率制约。 | 15km ~ 25km |
| **截获判定逻辑** | - | **非概率**。当推演引擎计算实体空间距离 $D \le D_{acq}$，且无人机处于目标雷达主瓣/高旁瓣辐射区内时，`S1 = Acquired`。 | 空间几何计算 | - |
| **目标雷达关机响应时间** | $T_{react}$ | 敌雷达操作员从收到告警到完成关机动作的物理耗时。 | 结合《台军雷达手册》评估：固定大型雷达流程长；轻型机动雷达自动化高。 | 固定: 8~15s<br>机动: 3~8s |
| **无人机末制导记忆时间** | $T_{memo}$ | 目标突然关机后，导引头依靠惯性导航维持正确弹道而不发散的最大时长。 | 装备设计指标。若 $T_{react} < T_{memo}$，则关机无效，`S2=Hit`。 | 5s ~ 12s |
| **诱饵诱惑偏置距离** | $D_{decoy}$ | 敌方有源诱饵开启后，在导引头视野中产生的角度偏移等效距离。 | 电子战侦察数据。若 $D_{decoy} > R_{lethal}$ (杀伤半径)，则`S2=Lost_Decoy`。 | 50m ~ 200m |

### 3.2 L2层：多次统计概率阈值参数溯源规范

L2层的 E1、E2 指标是统计结果。推演引擎在进行蒙特卡洛循环时，需要读取以下拆解公式计算得出的概率阈值。

#### 3.2.1 攻击链路达成率阈值 ($P_{engage\_config}$) 拆解字典
此参数决定引擎是否输出 `S2 = Hit`。
$$ P_{engage\_config} = P_{hit\_base} \times P_{track\_stable} \times P_{overcome\_counter} $$

*   **参数A：基础命中概率 ($P_{hit\_base}$) —— 核心物理计算**
    *   **物理意义**：在无对抗、完美跟踪下，仅由飞控误差和战斗部散布决定的命中概率。
    *   **计算公式（基于圆正态分布推导）**：$P_{hit\_base} = 1 - 0.5^{\left(\frac{R_{lethal}}{CEP}\right)^2}$
    *   **易混淆参数详解**：
        1.  **CEP**：无人机设计指标（飞控+导引头测角误差），反辐射无人机典型值 **3m ~ 8m**。
        2.  **$R_{lethal}$ (战斗部有效杀伤半径)**：**极易被误解**。它不是爆炸冲击波半径，而是针对特定目标的“破片穿透临界半径”。定义为：在该距离上，战斗部破片密度 $\ge 1$ 枚/$m^2$，且单枚破片比动能 $\ge 100~J/cm^2$（足以穿透2mm铝装甲或损坏雷达波导管）。

*   **参数B：稳定跟踪概率 ($P_{track\_stable}$)**
    *   **物理意义**：末段俯冲过程中信号不断连的概率。取决于目标雷达扫描周期和天线类型。
    *   **设定标准**：相控阵雷达(持续照射) **0.95~0.98**；机械扫描雷达(波束扫过) **0.70~0.85**。

*   **参数C：抗关机/诱饵成功概率 ($P_{overcome\_counter}$)**
    *   **物理意义**：综合反映对抗胜利的概率。
    *   **设定标准**：若无记忆跟踪能力，设为 **0.1**；若有记忆能力，$P_{overcome\_counter} = \frac{T_{memo} - T_{react}}{T_{memo}}$（若结果为负数，说明敌来不及关机，直接取 **1.0**）。

#### 3.2.2 硬毁伤达成率阈值 ($P_{K\_Kill\_config}$) 拆解字典
此参数在引擎判定 `S2=Hit` 后，决定输出 `K_Kill` 还是 `F_Disable`。
$$ P_{K\_Kill\_config} = V_k \times \eta_{fuze} $$

*   **参数D：目标易损性系数 ($V_k$) —— 决定性战术参数**
    *   **物理意义**：被破片场覆盖后，关键部件被摧毁导致整机报废的概率。与雷达结构设计强相关。
    *   **设定标准（基于《台军雷达手册》结构分析）**：
        *   **第一类：外露阵列式**（如长白雷达、铺路爪）：阵面直接暴露无装甲，打断单元易烧毁发射机。设定 $V_k =$ **0.85 ~ 0.95** (极度脆弱)。
        *   **第二类：背罩式/方舱式**（如爱国者MPQ-65/53、天弓三）：核心组件被金属方舱包裹。设定 $V_k =$ **0.30 ~ 0.50** (仅能打烂外露天线造成F级，极难打穿方舱造成K级)。
        *   **第三类：小型全机动式**（如AN/MPQ-64哨兵）：天线极小且隐蔽。设定 $V_k =$ **< 0.20**。

*   **参数E：引战配合效率 ($\eta_{fuze}$)**
    *   **物理意义**：引信在最佳位置起爆的概率。
    *   **设定标准**：大型固定面(易触发近炸) **0.90~0.95**；小型机动目标(反射面小) **0.70~0.80**。

### 3.3 联动接口参数：功能瘫痪(F级)持续时间映射规范
当引擎输出 `S3 = F_Disable` 时，该状态将直接传递给《辐射源压制效益评估(P1/P2)》模块，作为雷达“关机时长”计入。该时长必须符合后勤抢修逻辑。

| 目标类型 | 物理抢修逻辑 | $T_{restore}$ 分布模型 | 赋值示例 (推演引擎随机数) |
| :--- | :--- | :--- | :--- |
| **大型固定雷达**<br>*(长白、铺路爪)* | 无备件、需吊车拆卸、联系原厂。 | 正态分布 $N(\mu=3600s, \sigma=600s)$ (约1小时+) | `random_normal(3600, 600)` |
| **中型方舱雷达**<br>*(爱国者、天弓三)* | 美军标备件箱随队，模块化拔插更换。 | 均匀分布 $U(600s, 1800s)$ (10~30分钟) | `random_uniform(600, 1800)` |
| **轻型伴随雷达**<br>*(哨兵、蜂眼)* | 有备用雷达车直接替换，或重启备机。 | 均匀分布 $U(120s, 600s)$ (2~10分钟) | `random_uniform(120, 600)` |

## 五、 仿真推演与案例深度剖析

### 5.1 场景一：单次战术推演复盘（L1单次指标应用）
**态势**：红方出动 4 架无人机，分别打击上述两部雷达。指挥员查看的是单次状态跃迁日志。

| Run_ID | UAV_ID | Target_ID | S1_截获状态 | S2_攻击状态 | S3_毁伤等级 | 战术备注（底层逻辑解析） |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| R-001 | UAV-01 | Radar-A (长白) | Acquired | **Hit** | **K_Kill** | 随机数0.65 < 0.721(命中)；0.4 < 0.855(硬毁)。彻底炸毁。 |
| R-001 | UAV-02 | Radar-A (长白) | **Target_Already_Killed** | - | - | **关键逻辑**：因UAV-01已摧毁目标信号消失，UAV-02判定目标已死，自动改航。 |
| R-001 | UAV-03 | Radar-B (爱国者) | Acquired | **Lost_Shutdown** | - | 随机数本可命中，但触发敌关机对抗，且计算抗关机失败，脱靶。 |
| R-001 | UAV-04 | Radar-B (爱国者) | Acquired | **Hit** | **F_Disable** | 随机数0.28 < 0.313(命中)；但0.5 > 0.340(未硬毁)。方舱挡住破片，仅天线受损瘫痪。 |

### 5.2 场景二：装备战法论证推演（L2多次指标应用）
**态势**：系统后台分别对两部雷达各发起 **1000次独立的蒙特卡洛仿真**（每次重置目标状态，仅派1架）。

**1000次推演聚合统计结果：**

| 统计项 | Target_A (长白雷达) | Target_B (爱国者MPQ-65) |
| :--- | :--- | :--- |
| 推演总架次 | 1000 次 | 1000 次 |
| 判定命中次数 (S2=Hit) | **719 次** | **316 次** |
| 判定 K_Kill 次数 | **615 次** | **107 次** |
| 判定 F_Disable 次数 | 104 次 | 209 次 |

**指标计算与深度溯源分析：**

**1. 攻击链路达成率 (E1)**
*   **长白雷达**：$E1 = 719 / 1000 = \mathbf{71.9\%}$ (吻合配置表 $P_{engage}=0.721$)
*   **爱国者雷达**：$E1 = 316 / 1000 = \mathbf{31.6\%}$ (吻合配置表 $P_{engage}=0.313$)
*   **[溯源诊断]**：打爱国者命中率暴跌的核心原因，**不是因为我方无人机精度变差了**（基础命中概率都是0.736），而是因为**敌雷达关机反应太快(5s)**，我方记忆时间(10s)余量不足，导致抗关机概率仅50%，直接腰斩命中链路。

**2. 硬毁伤达成率 (E2)**
*   **长白雷达**：$E2 = 615 / 719 = \mathbf{85.5\%}$ (吻合配置表 $P_{K\_Kill}=0.855$)
*   **爱国者雷达**：$E2 = 107 / 316 = \mathbf{33.9\%}$ (吻合配置表 $P_{K\_Kill}=0.340$)
*   **[溯源诊断]**：打爱国者毁伤率低的核心原因，**不是因为战斗部威力小**，而是因为**目标采用方舱防护设计($V_k$仅0.4)**，破片无法穿透核心舱段。

**3. 终极指标：完整摧毁概率 ($P_k$)**
*   **长白雷达**：$P_k = 71.9\% \times 85.5\% = \mathbf{61.5\%}$ $\rightarrow$ **单发具备较高摧毁把握。**
*   **爱国者雷达**：$P_k = 31.6\% \times 33.9\% = \mathbf{10.7\%}$ $\rightarrow$ **单发基本无法摧毁，大量停留在F级瘫痪状态。**

## 六、 与“辐射源压制效益(P1/P2)”的接口联动

上述毁伤评估不是孤立的，L1层的 `S3_毁伤等级` 将直接转化为时间轴上的 0/1 状态，驱动《辐射源压制效益评估》中的核心指标计算。

**底层联动逻辑（C++伪代码规范）：**

```cpp
void updateStrikeAndSuppression(int currentTick, UAV& uav, Radar& radar) {
    // 1. 检查目标状态 (利用S1逻辑)
    if (radar.status == RADAR_DEAD) {
        uav.logStatus(S1_CAPTURE, "Target_Already_Killed");
        MissionPlanner::reassignTarget(uav); // 触发自动改航逻辑
        return;
    }

    // 2. 执行交战判定，读取第四章生成的Config表阈值
    EngageResult result = evaluateStrike(uav, radar); 

    // 3. 将 S3 毁伤结果映射为 P1(雷达关机覆盖率) 需要的状态流
    if (result.S3 == "K_Kill") {
        radar.status = RADAR_DEAD;
        // 接口输出：从当前Tick开始，该雷达永远为 0 (关机)
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    } 
    else if (result.S3 == "F_Disable") {
        radar.status = RADAR_DISABLED;
        // 接口输出：调用第三章规定的 T_restore 分布模型生成恢复时间
        int disableTime = random_uniform(600, 1800); // 爱国者方舱抢修时间
        radar.restartTick = currentTick + disableTime;
        // 在这段时间内，状态强制为 0 (关机)，计入 P1 的分子
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
    else if (result.S2 == "Lost_Shutdown") {
        // 逻辑陷阱：敌为躲避攻击主动关机，无人机脱靶，但雷达客观上处于关机状态！
        // 接口输出：同样记录关机状态，体现“硬杀伤未成，但软压制生效”
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
}
```

## 七、 战术诊断与结论矩阵

通过上述将物理参数彻底透明化的评估方案，指挥员和装备论证人员可依据以下矩阵快速得出结论：

| $P_k$ 区间 | E1(链路)表现 | E2(毁伤)表现 | 综合评价与战术指导 |
| :--- | :--- | :--- | :--- |
| **≥ 60%**<br>*(如长白雷达)* | 正常 | 极高 | **定点清除目标**。单发或双发齐射即可确保摧毁，无需后续压制。 |
| **15% ~ 40%** | 偏低 | 正常 | **对抗瓶颈目标**。失败多因抗关机/抗诱饵能力弱。战术：需配合电子战机压制敌雷达操作员反应时间。 |
| **< 15%**<br>*(如爱国者)* | 极低 | 极低 | **持续压制目标**。物理特性决定单发无法摧毁且极难命中。战术：放弃“摧毁”企图，改为**多波次轮替攻击**，利用 `F_Disable` 状态累计瘫痪时间（10~30分钟/次），为攻击编队强行撕开 P2（突防安全率）时间窗口。 |
