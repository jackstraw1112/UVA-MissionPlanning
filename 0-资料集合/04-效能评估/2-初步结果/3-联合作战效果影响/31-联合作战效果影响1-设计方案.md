# 联合作战效果影响评估方案（台海实战化定制版）

## 一、 评估指标体系的“去冗余”重塑

### 1.1 解决指标重复问题：坚决砍掉“安全走廊时长(J3)”
您指出得非常准确：“区域防空压制覆盖率(P2)”和“安全走廊连续有效时长(J3)”在底层计算逻辑上高度同构（都在算“没被雷达照到的时间”），且存在致命缺陷：**绝对时长（秒）无法跨编队通用**（慢速无人机需要300秒，快速导弹只需30秒）。

**优化决策**：
1.  **彻底删除 J3 指标**。
2.  **将原 P2（区域防空压制覆盖率，基于动态时空解算的百分比）降级为“底层诊断参数”**。当联合作战效果不达标时，复盘去看 P2 即可知道是不是走廊没压好。
3.  联合作战层面，**绝对只看两个结果**：我方任务完成了没？（J1），敌方打不准了没？（J2）。

### 1.2 重塑后的极简评估架构

| 指标代号   | 指标名称              | 核心逻辑                                     | 达标阈值      |
| :----- | :---------------- | :--------------------------------------- | :-------- |
| **J1** | **关联任务突防成功率提升幅度** | 【结果导向】受特定无人机群保护的特定打击编队，其存活/完成任务的比例提升了多少？ | ≥15%为有效   |
| **J2** | **关联任务防空拦截效能下降率** | 【原因导向】在上述特定任务的交战过程中，敌方发射导弹的命中率下降了多少？     | ≥50%为显著   |
| *(辅助)* | **P1/P2 底层时空参数**  | 【诊断导向】雷达关机率多少？战机实际暴露时长的安全占比多少？           | 仅用于不达标时复盘 |

## 二、 引入“任务关联映射”机制（解决计算对象混乱问题）

现代空战中，几十架飞机同时出击，如果不做关联，把“没受保护的侦察机”算进“受保护的攻击编队”里，J1将毫无意义。

### 2.1 任务映射数据结构设计
在仿真系统初始化阶段，必须建立**【压制任务】与【突防任务】的强绑定关系**。

| 压制任务ID (UAV_Task) | 执行实体 | 压制目标 (蓝方雷达) | 关联保护的突防任务ID (Atk_Task) | 突防任务执行实体 |
| :--- | :--- | :--- | :--- | :--- |
| **SUP_01** | UAV-1, UAV-2 | 万里阵地 AN/MPQ-53 | **ATK_01** | J-16 × 4架 (松山方向) |
| **SUP_02** | UAV-3, UAV-4 | 三芝阵地 CS/MPN-45 | **ATK_01** | J-16 × 4架 (松山方向) |
| **SUP_03** | UAV-5, UAV-6 | 清水阵地 AN/MPQ-65 | **ATK_02** | JH-7A × 4架 (清泉岗方向) |

### 2.2 J1 指标的参数重构与计算边界
基于上述映射，J1 的计算公式被严格限定在**同一个 `Atk_Task_ID` 内部**：

*   **参数定义**：
    *   $N_{total}(ATK\_x)$：某突防任务编队的总平台数（如ATK_01为4架）。
    *   $N_{success}(ATK\_x)$：该任务编队中，成功到达投弹线并完成攻击的平台数。
    *   $P_{base}$：无压制时，该型号战机突防该类型阵地历史基准成功率。
*   **计算公式**：
    $$J1(ATK\_x) = \left( \frac{N_{success}(ATK\_x)}{N_{total}(ATK\_x)} - P_{base} \right) \times 100\%$$
*   **防混乱机制**：如果 UAV-5/6 在压制清水阵地时失败，导致 JH-7A 被击落，**绝对不会影响** J-16 编队（ATK_01）的 J1 计算结果。各算各的账。

## 三、 J2 敌方防空拦截效能下降率：深度解析与案例化说明

很多指挥员对 J2 不理解，核心疑惑是：**“敌人都把导弹打出去了，跟我的无人机还有啥关系？”**

### 3.1 J2 的物理本质：打乱敌方的“OODA闭环”
防空拦截不是“看到就打中”的，它需要：雷达发现→锁定跟踪→火控解算→导弹发射→**雷达持续照射修正（最关键）**→命中。
反辐射无人机压制的目的，**不是不让敌人开火，而是让敌人的导弹变成“瞎子”**。J2 衡量的就是：**无人机让敌人的导弹“变瞎”的程度。**

### 3.2 J2 参数严格绑定任务空间
与 J1 类似，J2 的统计也必须框定在特定任务的时空范围内：
*   **$N_{missile\_fired}(ATK\_x)$**：在 ATK_01 任务期间，蓝方雷达（受 UAV 压制状态）针对该突防编队发射的拦截弹总数。
*   **$N_{missile\_hit}(ATK\_x)$：**上述拦截弹中，实际命中的数量。
*   **$P_{intercept\_base}$**：蓝方在正常状态下的标称命中概率。

### 3.3 结合台海想定的 J2 案例推演拆解

**【场景设定】**
红方 J-16 编队（ATK_01）突防。蓝方新北万里阵地爱国者雷达（受 UAV 压制中）发射了拦截弹。

**【情况A：没有无人机压制时（计算基准）】**
爱国者雷达正常工作，持续照射 J-16。蓝方发射 **5枚** PAC-2 导弹，其中 **2枚** 命中。
*   基准命中率 $P_{intercept\_base} = 2 / 5 = 40\%$。

**【情况B：有无人机压制时（仿真实际发生）】**
UAV 迫使万里雷达处于“间歇关机”或“被迫频闪”状态。
*   T+10秒：雷达短暂开机，抓到 J-16，发射第1枚导弹。雷达随即被 UAV 胁迫关机。
*   **结果**：第1枚导弹在飞行中**丢失中段修正指令**，偏离目标未命中。
*   T+30秒：雷达再次尝试开机，发射第2、3枚导弹。但因受到 UAV 电子干扰，解算距离错误。
*   **结果**：第2枚近炸未毁伤，第3枚脱靶。
*   整个交战过程中，蓝方针对 ATk_01 任务共发射了 **5枚** 导弹，但最终 **0枚** 有效命中。

**【J2 指标计算】**
$$J2 = \frac{P_{intercept\_base} - P_{intercept\_with}}{P_{intercept\_base}} \times 100\%$$
$$J2 = \frac{40\% - (0/5)}{40\%} = \frac{40\%}{40\%} = \mathbf{100\%}$$

**【通俗解读】**
J2 = 100% 意味着：**虽然蓝方操作员拼命发射了5枚导弹（消耗了价值上千万的弹药），但在反辐射无人机的压制下，这5枚导弹全部变成了废铁，拦截效能被彻底抹平（下降100%）。**
这就是 J2 的价值——它反映了无人机对敌方**“开火后有效性”**的致命削弱。
