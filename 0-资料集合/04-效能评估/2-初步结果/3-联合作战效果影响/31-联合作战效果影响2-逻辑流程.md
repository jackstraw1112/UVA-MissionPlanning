# 总体流程
1. 明确联合作战评估边界：仅保留 J1、J2 两个联合作战结果指标，P1/P2仅作为底层诊断参数。
2. 设置红方任务体系：战斗机突防任务（ATK）与反辐射无人机压制任务（SUP）。
3. 建立“任务关联映射表”：每个 SUP 任务必须绑定其保护的唯一或指定 ATK 任务。
4. 进行兵力分配与航路部署：无人机围绕威胁雷达部署，战斗机按突防航线执行。
5. 运行仿真并按任务包记录交战事件、导弹发射命中、平台生存与投弹结果。
6. 按任务包输出 J1/J2 结果，不跨任务混算。

# 任务关联映射流程（防混算核心）
1. 战前从全量雷达库提取“有效威胁雷达子集”。
2. 为每个突防任务建立关联压制任务：
	1. `ATK_x -> {SUP_a, SUP_b, ...}`。
	2. 每个 `SUP` 明确压制目标雷达与执行无人机集合。
3. 仿真记录时携带 `Task_ID`：
	1. 无人机交战事件绑定 `SUP_ID`。
	2. 战斗机突防、投弹、受击事件绑定 `ATK_ID`。
	3. 雷达发射导弹事件同时标注对应被拦截对象所属 `ATK_ID`。
4. 评估时仅筛选目标 `ATK_ID` 的数据进行 J1/J2 统计。

# J1 指标逻辑流程（关联任务突防成功率提升幅度）
1. 确定评估对象：选择一个突防任务包 `ATK_x`。
2. 统计任务包内平台数量：
	1. `N_total(ATK_x)`：编队总平台数。
	2. `N_success(ATK_x)`：成功到达投弹线并完成任务的平台数。
3. 计算压制条件下成功率：
	1. `P_with = N_success / N_total`。
4. 读取该任务历史基准成功率 `P_base`（无压制条件）。
5. 计算提升幅度：
	1. `J1 = (P_with - P_base) * 100%`。
6. 输出判定：
	1. `J1 >= 15%` 视为联合作战收益有效。

# J2 指标逻辑流程（关联任务防空拦截效能下降率）
1. 确定评估对象：与 J1 相同的 `ATK_x` 任务包。
2. 在该任务暴露窗口内，统计敌方拦截行为：
	1. `N_missile_fired(ATK_x)`：针对该任务发射的导弹数。
	2. `N_missile_hit(ATK_x)`：上述导弹中实际命中的数量。
3. 计算压制条件下拦截命中率：
	1. `P_intercept_with = N_missile_hit / N_missile_fired`（若分母为0则记为0）。
4. 读取敌方标称基准命中率 `P_intercept_base`（正常火控条件）。
5. 计算拦截效能下降率：
	1. `J2 = (P_intercept_base - P_intercept_with) / P_intercept_base * 100%`。
6. 输出判定：
	1. `J2 >= 50%` 视为敌防空效能被显著削弱。

# 仿真运行流程（按时间线）
1. 仿真初始化
	1. 加载雷达部署、平台参数、航路与任务映射表。
2. 运行阶段
	1. 无人机进入威胁区触发雷达反应（关机/重启/频闪）并记录。
	2. 战斗机进入暴露区，记录是否被雷达稳定跟踪。
	3. 敌方在可发射窗口内发射拦截弹，记录发射与命中结果。
	4. 战斗机完成投弹或退出任务，记录任务完成状态。
3. 结束阶段
	1. 汇总 `ATK` 与 `SUP` 关联数据，进入指标计算。

# 数据保存要求
1. 任务映射表（`Task_Mapping_Log`）
	1. 字段：`SUP_ID`、`ATK_ID`、`UAV_List`、`ThreatRadar_List`、`Start_Time`、`End_Time`。
2. 突防任务结果表（`Atk_Task_Result_Log`）
	1. 字段：`ATK_ID`、`Platform_ID`、`Reached_Strike_Line`、`Strike_Completed`、`Survival_Status`。
3. 敌拦截事件表（`Intercept_Event_Log`）
	1. 字段：`ATK_ID`、`Radar_ID`、`Missile_ID`、`Fire_Time`、`Hit_Result`、`Guidance_Status`。
4. 底层诊断表（可选）
	1. `Radar_Status_Log`（用于P1复盘），`Escort_Spatiotemporal_Log`（用于P2复盘）。

# 核心计算与判定顺序
1. 先算 J1：确认“友军任务是否完成得更好”。
2. 再算 J2：确认“敌方拦截是否被压低”。
3. 结果合成规则：
	1. J1达标且J2达标：联合作战效果显著。
	2. J1达标但J2不达标：可能依赖航路/地形而非稳定压制，需要复盘 P2。
	3. J1不达标但J2达标：敌方拦截虽下降，但友军任务组织或打击链仍存在瓶颈。
	4. J1/J2均不达标：当前协同战法失效，需重构任务分配与节奏。

# 结果输出流程（面向指挥与复盘）
1. 指挥简报层（对外）
	1. 输出 J1、J2 两个最终结论与达标判定。
2. 技术复盘层（对内）
	1. 若 J1/J2 不达标，调用 P1/P2 与事件日志定位问题环节：
		1. 压制不足（雷达重启过频/关机覆盖不足）。
		2. 关联错误（SUP 与 ATK 绑定不合理）。
		3. 节奏失配（压制峰值与突防暴露窗口未重叠）。
