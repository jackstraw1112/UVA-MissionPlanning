好的，理解您的需求。在保持评估方案有效性的前提下，将指标体系从10项精简至2-3项核心指标，需要抓住 **“辐射源压制效益”最本质的两个问题**：

1. **时间维度：敌方雷达有多长时间无法正常工作？**（这是压制效益的直接体现）
2. **交换维度：我方付出的代价是否值得？**（这是作战效率的核心判据）

基于此，我推荐保留以下 **三项黄金指标**（可进一步压缩至前两项）：

| :--- | :--- | :--- |
| **I. 雷达关机覆盖率 (M4)** | 敌方雷达“瞎了”多久？ | 最直接、最通用的压制效益度量，是所有软压制行为的最终落脚点。 |
| **II. 区域防空压制覆盖率 (M5)** | 我方的安全通道打开了多久？ | 将单部雷达的压制效果转化为对突防编队的实际保护价值，是联合作战视角下的关键指标。 |
| **III. 等效战损比 (M7)** | 这笔买卖划不划算？ | 衡量经济性和可持续性。若只需最简方案，此项可作为次要参考。 |

## 精简版辐射源压制效益评估方案（2+1指标）

### 一、 评估指标体系

| 指标代号 | 指标名称 | 定义 | 单位 |
| :--- | :--- | :--- | :--- |
| **P1** | **雷达关机覆盖率** | 在攻击窗口 \( T_{attack} \) 内，目标雷达处于关机状态的累计时长占比。 | % |
| **P2** | **区域防空压制覆盖率** | 在指定保护空域（突防走廊）内，完全不受敌方雷达有效覆盖的时长占突防窗口的比例。 | % |
| **P3** | **等效战损比 (LER)** | 我方损失无人机数量与敌方等效压制雷达数量的比值。 | 比值 |

*说明：P1 和 P2 为必选核心指标，P3 为可选辅助指标。*

---

### 二、 仿真推演所需记录的原始数据（极简清单）

仅需记录以下三类数据，即可支撑上述2-3项指标的计算。

#### 2.1 全局时间参数

| 数据项      | 符号                                               | 说明                    |
| :------- | :----------------------------------------------- | :-------------------- |
| 攻击窗口起始时刻 | \( T_{start} \)                                  | 第一架无人机进入敌方雷达截获距离的时刻。  |
| 攻击窗口结束时刻 | \( T_{end} \)                                    | 最后一架无人机命中目标/脱离/自毁的时刻。 |
| 突防窗口起始时刻 | \( T_{esc\_start} \)                             | 后续突防编队进入危险空域的时刻。      |
| 突防窗口结束时刻 | \( T_{esc\_end} \)                               | 后续突防编队离开危险空域的时刻。      |
| 攻击窗口总时长  | \( T_{attack} = T_{end} - T_{start} \)           |                       |
| 突防窗口总时长  | \( T_{escort} = T_{esc\_end} - T_{esc\_start} \) |                       |

#### 2.2 雷达状态时序数据（每部雷达）

| 记录项 | 格式要求 | 用途 |
| :--- | :--- | :--- |
| 雷达开关机状态时序 | 时间序列 (每秒)：1=开机，0=关机 | 计算 P1 (关机覆盖率) |
| 雷达物理摧毁时刻 | 事件时间戳 | 计算 P3 (等效压制数量) |
| 雷达有效探测范围 | 多边形坐标 (预设参数) | 计算 P2 (区域覆盖判断) |

#### 2.3 无人机损耗与战果数据

| 记录项 | 格式要求 | 用途 |
| :--- | :--- | :--- |
| 我方无人机总损失数 | 整数 \( N_{lost} \) | 计算 P3 |
| 无人机命中目标雷达记录 | (时刻，目标雷达ID) | 判定物理摧毁 (K-Kill) |
| 雷达持续关机超30分钟记录 | (雷达ID，关机起止时刻) | 判定功能压制 (F-Kill) |

---

### 三、 核心指标计算方法

#### P1. 雷达关机覆盖率

**计算公式：**
\[
P1 = \frac{1}{N} \sum_{i=1}^{N} \frac{T_{off\_i}}{T_{attack}} \times 100\%
\]

- \( N \)：参与评估的敌方雷达总数（可取全部雷达，或仅取对我方突防有威胁的关键雷达）。
- \( T_{off\_i} \)：雷达 \( i \) 在 \( [T_{start}, T_{end}] \) 内的累计关机时长（从状态时序数据中累加“0”状态的秒数）。

**计算示例（沿用前例）：**
- Radar A：关机 230 秒
- Radar B：关机 230 秒
- Radar C：关机 0 秒（始终保持开机）
- \( T_{attack} = 425 \) 秒
- \( P1 = \frac{230 + 230 + 0}{3 \times 425} = \frac{460}{1275} \approx 36.1\% \)

---

#### P2. 区域防空压制覆盖率

**前提**：需预先定义 **保护空域**（例如一条宽10km、长50km的走廊）和 **雷达有效覆盖判据**（通常以雷达位置为圆心、最大探测距离为半径的圆，并考虑地形遮蔽）。

**计算步骤**：
1. 将突防窗口 \( [T_{esc\_start}, T_{esc\_end}] \) 离散为时间步（每秒）。
2. 对于每个时间步 \( t \)，检查保护空域内的任意一点是否被 **任意一部处于开机状态的雷达** 有效覆盖。
3. 若 **全空域均未被覆盖**，则该时间步标记为“安全”。
4. 统计“安全”时间步数量 \( N_{safe} \)。
5. 计算覆盖率：
\[
P2 = \frac{N_{safe}}{T_{escort}} \times 100\%
\]

**简化版（针对单条航线）**：
若保护对象为一条固定航线，则只需判断该航线在 \( t \) 时刻是否被开机雷达覆盖。若连续 \( T_{safe} \) 秒未被覆盖，则 \( P2 = T_{safe} / T_{escort} \)。

**计算示例**：
- 突防窗口 300 秒。
- 其中 120 秒内，航线上无任何开机雷达覆盖。
- \( P2 = 120 / 300 = 40\% \)。

---

#### P3. 等效战损比 (LER)（可选）

**计算公式：**
\[
P3 = \frac{N_{lost}}{N_{K\_Kill} \times 1.0 + N_{F\_Kill} \times 0.8}
\]

- \( N_{lost} \)：我方无人机总损失数。
- \( N_{K\_Kill} \)：被物理摧毁的雷达数量。
- \( N_{F\_Kill} \)：未摧毁但单次连续关机超过30分钟的雷达数量。

**判定阈值建议**：LER < 1.5 为可接受，LER < 1.0 为高效，LER > 2.0 为低效。

---

### 四、 综合评估结论输出

由于指标已精简至2-3项，建议不再进行加权综合，而是采用 **“雷达图”或“达标判定表”** 进行直观呈现。

#### 4.1 评估结果汇总表（示例）

| 评估指标 | 实测值 | 期望阈值 | 是否达标 |
| :--- | :--- | :--- | :--- |
| P1 雷达关机覆盖率 | 36.1% | ≥ 50% | ❌ 未达标 |
| P2 区域防空压制覆盖率 | 40.0% | ≥ 60% | ❌ 未达标 |
| P3 等效战损比 (LER) | 1.36 | ≤ 1.5 | ✅ 达标 |

#### 4.2 综合结论生成逻辑

| 达标情况组合 | 综合结论 | 建议行动 |
| :--- | :--- | :--- |
| P1、P2 均达标 | **压制有效** | 方案可行，可投入实战或转入下一阶段推演。 |
| 仅 P1 达标，P2 未达标 | **点压制有效，面压制不足** | 需增加无人机数量或优化航路规划，扩大压制覆盖范围。 |
| P1、P2 均未达标，P3 达标 | **效费比合格但战术效果不佳** | 无人机成本控制得当，但未能完成掩护任务，需重新设计协同策略。 |
| 三项均未达标 | **压制失败** | 方案需全面重新设计。 |

### 五、 精简前后对比总结

| 对比项 | 原方案（10项指标） | 精简方案（2+1项） |
| :--- | :--- | :--- |
| **数据采集复杂度** | 需记录雷达开关机频次、单次时长分布、导弹发射记录、战机保全价值等多项明细 | **仅需雷达开关机状态时序、无人机损失数、命中记录** |
| **计算耗时** | 需编写多个后处理脚本 | **可在 Excel 或简单 Python 脚本中 10 分钟内完成** |
| **评估结论清晰度** | 综合指数抽象，需解释权重 | **直接对照阈值，红绿灯式判断** |
| **适用场景** | 深度学术研究、装备论证 | **日常推演、快速方案比选、指挥员决策支持** |

这个精简方案既保留了评估的核心逻辑（时间和交换比），又大幅降低了仿真推演的数据记录和计算负担，适合快速迭代的作战方案论证场景。
