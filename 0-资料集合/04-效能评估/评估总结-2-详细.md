# 反辐射无人机作战效能仿真评估流程文档
## 第二章 辐射源压制效益评估模块

### 2.1 模块评估目标与核心问题

辐射源压制效益评估模块旨在回答一个核心问题：反辐射无人机对敌方雷达的压制行动是否有效？这个问题的回答需要从两个维度展开：一是时间维度，即敌方雷达在压制行动下有多长时间无法正常工作；二是效率维度，即我方为达成压制效果所付出的代价是否在可接受范围内。

在实际作战中，反辐射无人机对敌方雷达的压制效果最终要体现在为后续突防编队创造安全通过条件上。如果压制行动只是短暂致盲了敌方雷达，但持续时间不足以让友军完成突防，那么这种压制在战术上就是失败的。反之，即使压制行动没能完全摧毁敌方雷达，但只要在关键时间段内有效阻止了敌方雷达的正常工作，保障了友军的顺畅突防，压制行动就是成功的。

### 2.2 评估指标体系

本模块采用三项精简指标构成评估体系，其中前两项为必选核心指标，gi。

**指标一：雷达关机覆盖率（P1）**

雷达关机覆盖率定义为在设定的攻击窗口内，目标雷达处于关机状态的累计时长占攻击窗口总时长的百分比。这个指标直接衡量了压制行动对敌方雷达系统的时间占优程度。从作战效能角度理解，如果敌方雷达在我方攻击窗口内有80%以上的时间处于关机状态，那么敌方在这个时间段内就无法有效探测和跟踪我方突防编队，压制效果显著。

计算该指标时需要注意攻击窗口的起止时刻定义。攻击窗口起始时刻应为第一架反辐射无人机进入敌方雷达有效截获距离的时刻，而非仿真开始时刻。攻击窗口结束时刻应为最后一架参与攻击的无人机命中目标、脱离战场或自毁的时刻，即敌方雷达已无需继续保持关机状态的时刻。

**指标二：区域防空压制覆盖率（P2）**

区域防空压制覆盖率是雷达关机覆盖率在空间维度上的延伸和深化。该指标定义为在友军突防窗口内，预定义的突防航线上完全不受敌方任何开机雷达有效覆盖的时长占突防窗口总时长的百分比。

这个指标的设计考虑到了实际作战中可能出现的情况：某部雷达虽然关机了，但其他雷达仍在工作，如果突防航线处于其他雷达的覆盖范围内，友军仍然面临被探测和拦截的风险。因此，区域防空压制覆盖率的计算需要考虑敌方所有相关雷达的部署位置和覆盖范围，只有当突防航线在某个时间段内完全脱离所有开机雷达的有效覆盖时，该时间段才被计为安全时间。

**指标三：等效战损比（P3）**

等效战损比衡量的是我方作战资源消耗与敌方压制效果之间的交换关系，是评估作战经济性和可持续性的关键指标。该指标定义为：我方损失的反辐射无人机数量与敌方被等效压制的雷达数量之比。

在计算等效压制雷达数量时，需要区分不同类型的压制效果。被物理摧毁的雷达（K级毁伤）计为1.0个等效压制单位，因为这意味着敌方永久失去了该雷达的作战能力。被功能压制的雷达（F级毁伤）计为0.8个等效压制单位，系数小于1.0是因为功能压制通常有时间限制，敌方可能在修复后恢复雷达的部分功能。尽管如此，功能压制在突防窗口内的效果与物理摧毁相当，因此在等效计算时给予较高的权重。

### 2.3 指标计算方法详解

**P1指标计算公式：**

$$P1 = \frac{1}{N} \sum_{i=1}^{N} \frac{T_{off\_i}}{T_{attack}} \times 100\%$$

其中，$N$表示参与评估的敌方雷达总数，$T_{off\_i}$表示第$i$部雷达在攻击窗口内的累计关机时长，$T_{attack}$表示攻击窗口总时长。累计关机时长的计算方法是从雷达开关机状态时序数据中，统计所有“关机”状态的时间段并求和。特别需要注意的是，被物理摧毁的雷达从被摧毁时刻起至攻击窗口结束，全部计为关机状态。

**P2指标计算步骤：**

区域防空压制覆盖率的计算需要更复杂的空间判据。首先，需要在仿真系统中预设突防航线的几何坐标描述，通常表示为一系列航迹点的集合。其次，需要根据敌方各雷达的位置坐标、最大探测距离和波束指向，计算每部雷达在任意时刻对突防航线各点的覆盖情况。覆盖判断可采用简化的圆盘模型（以雷达为圆心、最大探测距离为半径的圆）或更精确的探测包络模型。

计算时，将突防窗口离散为固定时间步长（通常取1秒），对每个时间步判断突防航线上是否存在任何点被开机雷达覆盖。如果航线上所有点均未被覆盖，则该时间步标记为“安全”。最后统计“安全”时间步的数量即可得出P2指标值。

**P3指标计算公式：**

$$P3 = \frac{N_{lost}}{N_{K\_Kill} \times 1.0 + N_{F\_Kill} \times 0.8}$$

其中，$N_{lost}$表示我方损失的无人机总数，$N_{K\_Kill}$表示被物理摧毁的雷达数量，$N_{F\_Kill}$表示被功能压制的雷达数量。功能压制（F级毁伤）的判定标准通常为：雷达单次连续关机时长超过30分钟，或在突防窗口内持续关机。

### 2.4 评估阈值与判定标准

| 评估指标         | 计算公式                                                  | 优秀标准 | 合格标准 | 不合格标准 |
| ------------ | ----------------------------------------------------- | ---- | ---- | ----- |
| P1 雷达关机覆盖率   | $\frac{1}{N} \sum \frac{T_{off}}{T_{attack}}$         | ≥80% | ≥50% | <50%  |
| P2 区域防空压制覆盖率 | $\frac{T_{safe}}{T_{escort}}$                         | ≥80% | ≥60% | <60%  |
| P3 等效战损比     | $\frac{N_{lost}}{N_{K\_Kill}+0.8 \times N_{F\_Kill}}$ | ≤1.0 | ≤1.5 | >1.5  |

### 2.5 评估结论判定矩阵

综合评估结论不采用加权综合的单一得分方式，而是采用直观的达标判定矩阵，具体逻辑如下表所示：

| P1达标 | P2达标 | P3达标 | 综合结论            | 建议行动           |
| ---- | ---- | ---- | --------------- | -------------- |
| 是    | 是    | 是    | **压制高效**        | 方案可立即投入实战应用    |
| 是    | 是    | 否    | **压制有效但成本偏高**   | 可接受，但需优化资源使用效率 |
| 是    | 否    | 是    | **单点压制有效但整体不足** | 需扩大压制覆盖范围      |
| 是    | 否    | 否    | **压制效果与成本不匹配**  | 需重新规划压制策略      |
| 否    | 是    | 是    | **压制时间不足但效率高**  | 可接受，但需延长压制窗口   |
| 否    | 是    | 否    | **压制时间严重不足**    | 需大幅增加压制力量      |
| 否    | 否    | 是    | **效费比合理但战术失效**  | 需从根本上重新设计方案    |
| 否    | 否    | 否    | **压制全面失败**      | 方案需全面重新设计      |

## 第三章 攻击毁伤能力评估模块

### 3.1 模块评估目标与核心问题

攻击毁伤能力评估模块旨在回答关于反辐射无人机武器系统的两个根本问题：一是“能不能打上”，即无人机能否准确命中目标；二是“打上能不能毁”，即命中后是否能够有效摧毁目标雷达。这两个问题构成了反辐射无人机作战效能的物理基础：如果单机无法有效命中和摧毁目标，那么任何压制策略都将是空中楼阁。

从指挥决策角度，理解攻击毁伤能力的内涵同样重要。如果单发无人机的摧毁概率较低，指挥员就需要考虑采用多发齐射的战术来提升整体摧毁置信度。不同摧毁概率下的齐射数量需求差异巨大，直接影响作战资源规划和任务成本预算。因此，准确评估攻击毁伤能力是作战方案设计的重要前置工作。

### 3.2 评估指标体系

本模块采用“2+1”指标结构，即两项必选核心指标加一项可选诊断指标，以最精简的方式回答攻击毁伤能力的核心问题。

**指标一：单发摧毁概率（A1）**

单发摧毁概率（$P_k$）是攻击毁伤能力的终极综合指标，它将命中概率和毁伤条件概率聚合为一个单一数值，直接回答“发射一枚无人机能摧毁目标的概率是多少”这个指挥员最关心的问题。该指标定义为：单架反辐射无人机在典型作战环境下，对目标雷达造成K级（物理摧毁）或F级（功能瘫痪超过30分钟）毁伤的全任务链路概率。

从概率分解角度看，$P_k$实际上是多个子概率的乘积：$P_k = P_{launch} \times P_{hit} \times P_{kill|hit}$。其中，$P_{launch}$是攻击发起概率（无人机能够成功转入末制导俯冲的比例），$P_{hit}$是命中概率（无人机能够命中目标并在杀伤半径内的比例），$P_{kill|hit}$是命中条件下的毁伤概率（考虑了引战配合效率和战斗部威力）。通过这种分解，可以识别出影响最终摧毁概率的各个环节短板。

**指标二：末端制导精度（A2）**

末端制导精度以圆概率误差（CEP）作为量化度量，定义为以目标雷达天线阵面中心为瞄准点，包含50%命中弹着点（或爆炸投影点）的圆半径。CEP的单位为米，数值越小表示精度越高。

CEP指标的核心价值在于其诊断功能。当单发摧毁概率$ P_k $不达标时，CEP可以帮助快速定位问题根源：如果CEP远大于战斗部有效杀伤半径，说明命中概率是主要短板，问题出在导引头精度或飞控系统；如果CEP很小但$ P_k $仍然偏低，说明命中后的毁伤概率是短板，问题出在战斗部威力或引信设计。这种诊断能力对于武器系统的改进方向具有重要指导意义。

**指标三：攻击发起率（A3，可选）**

攻击发起率（$P_{launch}$）定义为无人机在截获目标信号后，能够成功建立攻击航路并转入末制导俯冲阶段的比例。这个指标主要用于区分两类问题：“平台可靠性”问题（无人机未能进入攻击位置）和“武器精度”问题（无人机已进入攻击位置但未能有效命中）。

在简化评估方案中，如果评估重点是武器系统本身的末端效能，可以将$P_{launch}$设为1.0，单独分析末制导后的毁伤能力。但在实战体系综合评估中，$P_{launch}$是不可忽视的因素，尤其是在复杂电磁环境或强对抗条件下，攻击发起失败可能是制约整体效能的主要瓶颈。

### 3.3 指标计算方法详解

**A1指标计算方法：**

单发摧毁概率支持两种计算方法，根据数据条件和精度要求选择使用。

方法一为直接统计法，适用于蒙特卡洛仿真次数足够多的情况。该方法直接统计目标被成功摧毁的架次比例：

$$P_k = \frac{N_{\text{目标被K/F级毁伤}}}{N_{\text{总仿真次数}}}$$

这种方法的优势是直观明了，无需理解中间的物理过程，适合快速评估和方案比选。

方法二为链路计算法，适用于需要分析各环节贡献度或仅有部分数据的情况。该方法按照物理链路逐步计算：

$$P_k = P_{launch} \times P_{hit} \times P_{kill|hit}$$

其中，命中概率可根据CEP和战斗部有效杀伤半径$R_{lethal}$计算：

$$P_{hit} = 1 - 0.5^{\left(\frac{R_{lethal}}{CEP}\right)^2}$$

这个公式基于正态分布假设推导，当落点偏差服从瑞利分布时，命中概率与CEP和杀伤半径的比值呈指数关系。

**A2指标计算方法：**

CEP的计算基于落点偏差数据的统计分析。具体步骤如下：首先，收集所有成功转入末制导架次的落点偏差值（从目标瞄准点到实际落点的距离）；其次，将所有偏差值按从小到大排序；最后，取第50%分位值作为CEP（若样本量为偶数，取中间两个数的平均值）。

在计算落点偏差时需要注意距离的定义方式。通常采用二维平面距离（目标中心的水平距离），但如果战斗部威力分布具有显著的高度维度特征，可能需要使用三维空间距离。具体的偏差类型应在评估方案开始前明确定义，并在整个评估过程中保持一致。

**A3指标计算方法：**

攻击发起率的计算相对简单：

$$P_{launch} = \frac{N_{\text{转入末制导}}}{N_{\text{截获目标信号}}}$$

在仿真记录中，需要区分“未截获目标信号”“截获后未能建立跟踪”“成功转入末制导”三种状态。只有成功转入末制导的架次才计入分子。此外，如果需要分析发起失败的原因，应在仿真数据中记录失败原因的分类标签，如“数据链中断”“目标信号丢失”“载机平台故障”等。

### 3.4 评估阈值与判定标准

| 评估指标 | 计算公式 | 优秀标准 | 合格标准 | 不合格标准 |
|---------|---------|---------|---------|-----------|
| A1 单发摧毁概率 $P_k$ | 直接统计或链路计算 | ≥60% | 40%-60% | <40% |
| A2 末端制导精度 CEP | 第50%分位落点偏差 | ≤5.0m | 5.0-8.0m | >8.0m |
| A3 攻击发起率（可选） | $\frac{N_{末制导}}{N_{总次数}}$ | ≥95% | 85%-95% | <85% |

### 3.5 齐射需求计算

当单发摧毁概率$ P_k $不足以满足作战任务要求时，需要通过多发齐射来提升整体摧毁置信度。齐射策略下的综合摧毁概率计算公式为：

$$P_{multi}(n) = 1 - (1 - P_k)^n$$

其中，$n$为齐射无人机数量。根据作战任务对摧毁置信度的要求，可以反推所需的齐射数量：

| 任务要求摧毁置信度 | $n = 1$ | $n = 2$ | $n = 3$ | $n = 4$ |
|-------------------|---------|---------|---------|---------|
| $P_k = 40\%$时的齐射效果 | 40.0% | 64.0% | 78.4% | 87.0% |
| $P_k = 50\%$时的齐射效果 | 50.0% | 75.0% | 87.5% | 93.8% |
| $P_k = 60\%$时的齐射效果 | 60.0% | 84.0% | 93.6% | 97.4% |
| $P_k = 70\%$时的齐射效果 | 70.0% | 91.0% | 97.3% | 99.2% |

## 第四章 联合作战效果影响评估模块

### 4.1 模块评估目标与核心问题

联合作战效果影响评估模块从更高的作战层面审视反辐射无人机压制行动的价值和贡献。该模块回答的核心问题是：压制行动对联合作战的整体效果产生了什么影响？这个问题的答案直接关系到压制行动在作战体系中的地位和价值认定。

从作战指挥的角度，压制行动从来不是目的，而是为后续作战行动创造有利条件的手段。因此，评估压制行动的成功与否，最终要落实到友军是否因此获得了实际的作战收益。这种收益可能表现为友军突防成功率的提升、敌方拦截效能的下降、作战损失的减少等多个维度。本模块通过精简的指标体系，将这些维度的效果量化为可比较的数值，便于指挥员进行方案决策和效果评估。

### 4.2 评估指标体系

本模块同样采用“2+1”指标结构，聚焦于联合作战效果评估的最核心问题。

**指标一：友军突防成功率提升幅度（J1）**

友军突防成功率提升幅度（J1）是本模块的终极效果指标，直接回答“压制行动让友军打进去变得更容易了吗”这个根本问题。该指标定义为：有反辐射无人机压制时，友军突防编队成功完成任务的平台比例，相对于无压制基准的绝对提升量。

这个指标的设计包含了基准对比的思想。通过将有无压制两种情况下的友军突防成功率进行对比，可以剔除掉环境因素、敌方其他防御力量等干扰因素的影响，精确量化压制行动本身的贡献。提升幅度越大，说明压制行动的作战价值越高。

**指标二：敌方防空拦截效能下降率（J2）**

敌方防空拦截效能下降率（J2）从敌方视角量化压制行动的效果，解释友军成功率提升的原因。该指标定义为：有压制时敌方每次拦截尝试的命中概率，相对于无压制基准的相对下降比例。

拦截效能的下降是压制行动产生效果的直接机制。当敌方雷达被压制后，拦截导弹无法获得准确的目标信息，导致拦截效率显著降低。这种降低可能表现为拦截弹脱靶率上升、拦截窗口缩短等多个方面。J2指标将这些因素综合为敌方“命中率”这一单一数值，便于直观理解压制效果。

**指标三：安全走廊连续有效时长（J3，可选）**

安全走廊连续有效时长（J3）是一个过程保障指标，用于诊断J1或J2不达标时的原因。该指标定义为：在友军突防窗口内，预定义突防航线上连续无雷达覆盖的最长时间段长度。

这个指标的诊断逻辑是：如果J3大于友军通过走廊所需的最小时间，但J1仍不理想，说明问题不在压制效果本身，而在于友军可能遭遇了其他未压制的威胁源（如敌方战斗机、地空导弹等）。如果J3小于友军所需最小时间，说明压制窗口太短，需要增加压制力量或优化压制时序。

### 4.3 指标计算方法详解

**J1指标计算公式：**

$$J1 = P_{success\_with} - P_{success\_base}$$

其中，$P_{success\_with} = \frac{N_{\text{成功突防平台（有压制）}}}{N_{\text{参战平台总数}}}$，$P_{success\_base}$为无压制时的基准成功率（来自历史数据或平行仿真）。

J1的单位通常用百分点表示，例如J1=+25%表示压制行动使友军突防成功率提升了25个百分点。如果提升幅度为负值或零，说明压制行动未产生正面效果甚至产生了负面影响。

**J2指标计算公式：**

$$J2 = \frac{P_{intercept\_base} - P_{intercept\_with}}{P_{intercept\_base}} \times 100\%$$

其中，$P_{intercept\_with} = \frac{N_{\text{友军被命中次数（有压制）}}}{N_{\text{敌方发射拦截弹总数（有压制）}}}$，$P_{intercept\_base}$为无压制时敌方的基准拦截成功率。

需要特别说明的是，$P_{intercept\_with}$的分母应包含敌方发射的所有拦截弹数量，无论其原定目标是无人机还是友军作战平台。这是因为在实战中，敌方发射拦截弹即消耗了防空资源，即使拦截弹未能命中任何目标，压制行动也实现了消耗敌方弹药的效果。

**J3指标计算方法：**

安全走廊连续有效时长的计算需要记录突防航线在每个时刻的雷达覆盖状态。具体的计算逻辑与P2指标类似，但J3仅关注“连续”安全时段的时长分布，取最长的那段连续安全时间作为指标值。

诊断逻辑如下：

- 若$J3 \geq T_{required}$（友军通过走廊所需最小时间），且$J1$达标，说明压制效果充足，问题可能出在其他环节。
- 若$J3 < T_{required}$，且$J1$未达标，说明压制窗口不足是主要问题，需增加压制力量。
- 若$J3 \geq T_{required}$，但$J1$仍未达标，说明友军可能遭遇了压制行动未覆盖的其他威胁。

### 4.4 评估阈值与判定标准

| 评估指标 | 计算公式 | 优秀标准 | 有效标准 | 不足标准 |
|---------|---------|---------|---------|---------|
| J1 友军突防成功率提升 | $P_{success\_with} - P_{success\_base}$ | ≥25% | ≥15% | <15% |
| J2 敌方拦截效能下降率 | $\frac{P_{base} - P_{with}}{P_{base}}$ | ≥70% | ≥50% | <50% |
| J3 安全走廊连续时长 | $\max(\text{连续安全时段})$ | $\geq 1.5 \times T_{req}$ | $\geq 1.0 \times T_{req}$ | $< T_{req}$ |

### 4.5 综合评估结论生成

联合作战效果评估的最终结论应综合三个指标的表现，给出定性评价和改进建议：

| J1达标 | J2达标 | J3达标 | 综合评价 |
|-------|-------|-------|---------|
| 是 | 是 | 是 | **压制行动对联合作战贡献显著，建议推广** |
| 是 | 是 | 否 | **压制有效但窗口余量不足，建议增加压制力量** |
| 是 | 否 | 是/否 | **友军收益明显但敌方拦截受影响有限，需深入分析** |
| 否 | 是 | 是 | **压制效果转化率低，需排查友军损失的其他原因** |
| 否 | 否 | 是/否 | **压制行动未能达成联合作战目的，需全面重新规划** |

## 第五章 综合仿真推演流程

### 5.1 推演准备阶段

在启动仿真推演之前，需要完成以下准备工作：

**作战场景参数设定**是推演准备的首要任务。这包括明确红蓝双方的作战力量配置、初始战场态势、关键时间节点等。对于反辐射无人机压制行动，核心参数包括：反辐射无人机的出动数量和编成、目标雷达的型号和部署位置、突防编队的规模和行动时刻、敌方可能的对抗策略（如雷达关机频率、拦截优先级等）。

**仿真环境边界定义**需要明确仿真系统的时间和空间范围。时间边界应覆盖从压制行动开始到友军突防结束的全过程，建议适当延长以捕捉后续影响。空间边界应包含所有相关作战区域的完整态势，确保能够记录到所有关键事件。

**基准数据准备**对于需要进行对比评估的指标（如J1、J2）至关重要。无压制情况下的基准数据可以通过历史推演数据、理论计算或专门的基准仿真获得。基准数据的准确性直接影响J1、J2等指标的可靠性。

### 5.2 仿真执行阶段

仿真执行阶段的核心任务是按照预设的场景参数和仿真逻辑，驱动整个作战过程并记录所有必要数据。

**实时数据记录**应按照第二章规定的统一数据规范进行。建议仿真系统采用事件驱动的记录方式，只在状态发生变化时记录数据，而非每帧全量输出。这种方式可以有效控制数据量，同时确保不遗漏关键事件。

**异常情况标注**对于后续的结果分析非常重要。当仿真过程中出现异常事件（如无人机故障、敌方雷达异常行为等）时，应在数据中明确标注，便于后续分析这些事件对评估结果的影响。

**蒙特卡洛批量执行**是获得统计可靠结果的必要手段。对于涉及概率的评估指标（如A1、P_k等），建议单个场景执行不少于200次仿真，以获得具有统计意义的均值和方差估计。批量执行可以采用并行计算方式以提高效率。

### 5.3 后处理与评估阶段

仿真执行完成后，进入后处理与评估阶段。这一阶段的主要任务是将原始仿真数据转化为评估指标值，并生成综合评估结论。

**数据清洗与校验**是后处理的第一步。需要检查仿真数据的完整性和一致性，剔除明显错误或异常的数据记录，补充缺失的关键事件标记。数据校验的重点包括：时间戳的逻辑一致性（事件顺序应符合因果关系）、关键状态转换的完整性（每个平台应有明确的起始和结束状态）、数值范围的合理性（坐标、速度等物理量应在合理范围内）。

**指标批量计算**根据各评估模块的公式，利用清洗后的数据进行批量统计。对于蒙特卡洛仿真，需要计算各指标的平均值和置信区间。对于单次确定性仿真，直接输出指标值即可。

**结论文档生成**将计算结果整理为可读性强的评估报告。报告应包含：场景概述（简要描述推演背景和关键参数）、评估指标汇总表（列出各指标实测值、对应的阈值和达标判定）、综合评估结论（根据判定矩阵给出定性评价）、问题诊断与改进建议（针对不达标指标分析可能原因并提出改进方向）。

## 第六章 仿真数据采集清单

### 6.1 辐射源压制效益评估数据需求

为支撑辐射源压制效益评估模块的数据需求，仿真系统需要记录以下数据：

**全局时间参数**记录推演过程的关键时间节点，包括攻击窗口起始时刻（第一架无人机进入雷达截获距离）、攻击窗口结束时刻（最后一架无人机任务结束）、突防窗口起始时刻（友军编队进入危险空域）、突防窗口结束时刻（友军编队脱离危险空域）。所有时间统一采用相对于仿真起点的秒数表示。

**雷达状态时序数据**记录每部参与评估的敌方雷达的开关机状态变化。建议以固定时间间隔（建议1秒或5秒）采样记录，或仅在状态发生变化时记录变化时刻和新的状态值。每条记录应包含：雷达唯一标识、采样时刻、开关机状态（1=开机，0=关机）、雷达物理状态（正常/被摧毁）。

**无人机损耗与战果数据**记录我方反辐射无人机的损失情况和命中记录。每架无人机的记录应包含：无人机唯一标识、任务结果状态（成功返航/任务中损失/命中目标/未命中目标）、命中目标时刻（如有命中）、被命中目标的雷达标识（如有命中）。

**功能压制判定数据**用于识别和判定F级毁伤事件。记录每次雷达关机事件的发生时刻、持续时长、恢复工作时刻。对于持续关机超过30分钟的情况，在评估计算时应将其计入F级毁伤。

### 6.2 攻击毁伤能力评估数据需求

为支撑攻击毁伤能力评估模块的数据需求，仿真系统需要记录以下数据：

**仿真架次基础信息**记录每次打靶仿真的基本信息，包括：架次唯一标识（Run_ID）、仿真总次数、该架次是否为有效统计样本（如因系统故障导致的异常结果应标记排除）。

**目标状态数据**记录每次架次中目标雷达的响应行为，包括：目标是否在无人机进入末制导前采取关机对抗（0=未关机，1=关机）、关机持续时长、是否进入记忆跟踪模式。这些数据用于分析对抗条件对毁伤效能的影响。

**末制导执行数据**记录无人机末制导阶段的执行情况，包括：是否成功转入末制导（0=未转入，1=成功）、末制导开始时刻、末制导结束时刻（命中目标或脱离）。

**命中与毁伤数据**记录每次架次的命中和毁伤结果，包括：落点偏差（从目标瞄准点到实际落点的距离，单位米）、是否在战斗部杀伤半径内（1=是，0=否）、目标最终状态（K_Kill/F_Kill/Light_Damage/Miss）。

### 6.3 联合作战效果影响评估数据需求

为支撑联合作战效果影响评估模块的数据需求，仿真系统需要记录以下数据：

**友军平台状态数据**记录所有参与突防的友军作战平台的情况，包括：平台总数、每架平台的任务执行结果（成功突防/被击落/中途返航/其他原因损失）、平台损失的时刻和原因。

**敌方拦截行动数据**记录敌方防空系统的拦截行为和结果，包括：拦截弹发射总数、针对不同目标的发射分布、拦截弹命中次数（区分命中无人机和友军平台）、拦截弹发射时刻。

**突防航线覆盖数据**记录突防航线在每个时刻的雷达覆盖状态。这一数据可以通过对敌方各雷达的覆盖范围计算获得，也可以通过在航线上设置虚拟传感器直接采样记录。数据格式建议为时间序列，每条记录包含：时刻、航线是否被完全覆盖（0=未覆盖/安全，1=被覆盖/危险）。

**基准对比数据**记录无压制情况下的基准评估参数，包括：无压制时友军突防成功率（来自历史数据或基准仿真）、无压制时敌方拦截成功率。这些数据在推演开始前确定，作为J1、J2计算的基准参照。

## 第七章 评估结果应用指南

### 7.1 方案比选应用

在作战方案论证阶段，评估体系可用于不同压制方案的效能对比。通过对多个备选方案进行仿真推演和评估计算，可以量化各方案的优势和不足，为指挥员提供科学的决策依据。

方案比选时应重点关注以下维度：核心指标的达标情况（是否满足阈值要求）、指标之间的平衡性（是否存在明显短板）、综合评估结论的等级、效费比的合理性。同时，应关注不同方案的敏感性分析，即当关键假设参数（如敌方对抗强度、气象条件等）发生变化时，评估结论的稳健性。

### 7.2 系统优化应用

评估体系不仅可以用于方案评估，还可以指导武器系统和作战策略的优化改进。通过对评估结果的深入分析，可以识别出制约作战效能的关键瓶颈，进而有针对性地进行优化。

以攻击毁伤能力为例，如果评估发现单发摧毁概率$ P_k $偏低，通过CEP诊断可以区分是“精度不足”还是“威力不足”的问题。如果是精度问题，应着重改进导引头性能或飞控算法；如果是威力问题，则需要考虑更换更大威力的战斗部或优化引战配合设计。这种诊断能力对于有限资源的优化配置具有重要指导价值。

### 7.3 人员培训应用

评估体系及其背后的评估方法论可以用于作战人员的能力培训。通过学习评估指标的内涵和计算方法，作战指挥人员能够更深刻地理解压制行动与作战效果之间的关联机制，提升战术决策的科学性和准确性。

培训内容可以包括：各评估指标的含义和作用、指标计算的数据来源和方法、不同评估结论对应的战术含义、如何根据评估结果调整作战方案等。培训形式可以采用理论讲解结合仿真推演实操的方式，使参训人员能够在实践中掌握评估方法的应用。
