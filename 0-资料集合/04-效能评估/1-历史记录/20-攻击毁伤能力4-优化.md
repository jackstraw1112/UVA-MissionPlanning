# 反辐射无人机攻击毁伤能力评估与仿真设计方案（完整实践版）

## 一、 方案概述

### 1.1 推演系统痛点剖析
在基于蒙特卡洛的作战推演系统中，打击效果并非通过解算物理弹道和破片飞散生成，而是通过**“概率阈值对比随机数”**直接裁决（如 `rand() < P_hit` 则判定命中）。
在这种底层逻辑下，传统的“末端制导精度(CEP)”指标在推演中面临**不可评估**的困境：推演引擎并没有生成真实的落点坐标，强行通过随机数反推CEP不仅违背逻辑，还会产生伪精度误差。

### 1.2 解决思路：状态机跃迁与参数溯源
本方案彻底摒弃对中间物理量（CEP、炸高、脱靶量）的提取，将评估指标直接锚定推演引擎原生的**“实体状态跃迁”**。
同时，彻底解决推演参数“无源之水”的问题：引擎配置表中的每一个概率阈值，必须能通过严密的物理公式，**溯源到无人机的CEP、杀伤半径，以及目标的易损性系数**上。

## 二、 评估指标体系：双层解耦架构

指标分为“单次交战层（看过程）”和“多次统计层（看能力）”。单次指标是推演日志；多次指标是基于海量单次日志的统计学聚合。

| 层级 | 指标代号 | 指标名称 | 评估维度 | 数据来源 |
| :--- | :--- | :--- | :--- | :--- |
| **L1: 单次交战<br>(必选主推)** | **S1** | **目标截获状态** | 无人机找到目标了吗？ | 推演引擎空间距离判定 |
| | **S2** | **末段攻击状态** | 无人机成功打进去了吗？ | 引擎随机数与阈值比对 |
| | **S3** | **最终毁伤等级** | 目标雷达最后变成啥样了？ | 引擎随机数与阈值比对 |
| **L2: 多次统计<br>(扩充论证)** | **E1** | **攻击链路达成率** | 综合命中能力到底如何？ | 1000次S2状态的聚合统计 |
| | **E2** | **硬毁伤达成率** | 命中后的杀伤力到底如何？ | 1000次S3状态的聚合统计 |

## 三、 核心参数溯源与设定规范（底层Config依据）

推演系统在进行蒙特卡洛循环时，需要读取概率阈值。**严禁凭空赋值，必须使用以下拆解公式计算得出。**

### 3.1 攻击链路达成率 ($P_{engage\_config}$) 拆解
此参数决定引擎是否输出 `S2 = Hit`。
$$ P_{engage\_config} = P_{hit\_base} \times P_{track\_stable} \times P_{overcome\_counter} $$

| 参数项 | 物理含义 | 设定依据与推导标准 |
| :--- | :--- | :--- |
| **$P_{hit\_base}$**<br>基础命中概率 | 无对抗下，由飞行误差决定的命中概率。 | **核心物理计算**：$1 - 0.5^{(R_{lethal}/CEP)^2}$。<br>*(注：$R_{lethal}$为破片穿透临界半径，非冲击波半径)* |
| **$P_{track\_stable}$**<br>稳定跟踪概率 | 末段俯冲不丢失目标的概率。 | 相控阵雷达(持续照射)：**0.95~0.98**<br>机械扫描雷达(波束扫过)：**0.70~0.85** |
| **$P_{overcome\_counter}$**<br>抗关机概率 | 目标关机时，记忆跟踪成功的概率。 | 公式：$(T_{memo} - T_{react}) / T_{memo}$<br>*(若无记忆功能则直接设为0.1)* |

### 3.2 硬毁伤达成率 ($P_{K\_Kill\_config}$) 拆解
此参数在引擎判定 `S2=Hit` 后，决定输出 `K_Kill` 还是 `F_Disable`。
$$ P_{K\_Kill\_config} = V_k \times \eta_{fuze} $$

| 参数项 | 物理含义 | 设定依据与推导标准 |
| :--- | :--- | :--- |
| **$V_k$**<br>目标易损性系数 | 命中后导致整机报废的概率。 | **第一类：外露阵列式**(如长白、铺路爪)：**0.85~0.95**<br>**第二类：方舱化设计**(如爱国者MPQ-65)：**0.30~0.50**<br>**第三类：小型全机动**(如哨兵雷达)：**< 0.20** |
| **$\eta_{fuze}$**<br>引战配合效率 | 引信在最佳位置起爆的概率。 | 大型固定面(易触发近炸)：**0.90~0.95**<br>小型机动目标(反射面小)：**0.70~0.80** |

### 3.3 联动接口参数：F级瘫痪恢复时间 ($T_{restore}$)
当输出 `S3 = F_Disable` 时，该状态将转化为压制效益模块(P1/P2)中的“雷达关机时长”。
*   **大型固定雷达**：无备件需吊装，`正态分布 N(3600s, 600s)` (约1小时+)
*   **中型方舱雷达**：美军标模块化拔插更换，`均匀分布 U(600s, 1800s)` (10~30分钟)

## 四、 想定背景与实体参数配置

基于《台湾地区军队雷达预警体系完整数据手册》，选取两类物理特性差异巨大的雷达作为打击目标，验证指标的敏感性。

### 4.1 红方攻击力量
*   **平台**：某型反辐射无人机（被动雷达导引头 + 高爆破片战斗部）。
*   **固有物理参数**：设计CEP = **5米**，战斗部有效杀伤半径 $R_{lethal}$ = **6米**，导引头记忆时间 $T_{memo}$ = **10秒**。

### 4.2 蓝方目标参数溯源配置表（推演引擎输入）

| 目标实体 | 手册依据及物理特性分析 | $T_{react}$<br>(关机反应) | $P_{hit\_base}$<br>(基础命中) | $P_{track}$<br>(稳定跟踪) | $P_{overcome}$<br>(抗关机) | $V_k$<br>(易损性) | $\eta_{fuze}$<br>(引战) | **最终阈值$P_{engage}$** | **最终阈值$P_{K\_Kill}$** |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Target_A：<br>天弓 长白雷达** | 三芝区阵地，固定部署。S波段，单面被动相控阵。**阵面巨大且外露无防护**，极易被破片切断波导管。 | 12s<br>(固定笨重) | $1 - 0.5^{(6/5)^2}$<br>= **0.736** | **0.98**<br>(相控阵) | $(10-12)/10$<br>负数取1.0<br>**敌来不及关** | **0.90**<br>(极度脆弱) | **0.95** | 0.736×0.98×1.0<br>= **0.721** | 0.90×0.95<br>= **0.855** |
| **Target_B：<br>爱国者 MPQ-65** | 万里区阵地，机动伴随。C波段，核心组件收纳于**具备装甲防护的方舱内**，仅天线外露。 | 5s<br>(自动化高) | 同左<br>= **0.736** | **0.85**<br>(机动干扰) | $(10-5)/10$<br>= **0.50** | **0.40**<br>(方舱防护) | **0.85** | 0.736×0.85×0.5<br>= **0.313** | 0.40×0.85<br>= **0.340** |

## 五、 仿真推演案例演示

### 5.1 场景一：单次战术推演复盘（L1单次指标应用）
**态势**：红方在同一波次出动 4 架无人机，分别打击上述两部雷达。指挥员查看的是单次状态跃迁日志。

**推演输出日志（节选）：**

| Run_ID | UAV_ID | Target_ID | S1_截获状态 | S2_攻击状态 | S3_毁伤等级 | 战术备注（底层逻辑解析） |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| R-001 | UAV-01 | Radar-A (长白) | Acquired | **Hit** | **K_Kill** | 随机数0.65 < 0.721(命中)；0.4 < 0.855(硬毁)。长白雷达被彻底炸毁。 |
| R-001 | UAV-02 | Radar-A (长白) | **Target_Already_Killed** | - | - | **关键逻辑**：因UAV-01已将其摧毁，信号消失，UAV-02判定目标已死，自动任务改航。 |
| R-001 | UAV-03 | Radar-B (爱国者) | Acquired | **Lost_Shutdown** | - | 随机数0.2 < 0.313(命中)...但触发敌关机对抗，且计算抗关机失败，脱靶。 |
| R-001 | UAV-04 | Radar-B (爱国者) | Acquired | **Hit** | **F_Disable** | 随机数0.28 < 0.313(命中)；但随机数0.5 > 0.340(未硬毁)。爱国者方舱挡住破片，仅外露天线受损，瘫痪。 |

**单次评估结论**：长白雷达已被物理铲除（K级）；爱国者雷达虽然被打中，但仅造成功能瘫痪（F级），敌阵地仍在运作，但雷达静默。

### 5.2 场景二：装备战法论证推演（L2多次指标应用）
**态势**：为论证该型无人机对不同目标的真实摧毁能力，系统后台分别对长白雷达和爱国者雷达各发起 **1000次独立的蒙特卡洛仿真**（每次仿真重置目标状态，仅派1架无人机）。

**1000次推演聚合统计结果：**

| 统计项 | Target_A (长白雷达) | Target_B (爱国者MPQ-65) |
| :--- | :--- | :--- |
| 推演总架次 | 1000 次 | 1000 次 |
| 判定命中次数 (S2=Hit) | **719 次** | **316 次** |
| 判定 K_Kill 次数 | **615 次** | **107 次** |
| 判定 F_Disable 次数 | 104 次 | 209 次 |

**指标计算与深度溯源分析：**

**1. 攻击链路达成率 (E1)**
*   **长白雷达**：$E1 = 719 / 1000 = \mathbf{71.9\%}$ (与配置表 $P_{engage}=0.721$ 吻合)
*   **爱国者雷达**：$E1 = 316 / 1000 = \mathbf{31.6\%}$ (与配置表 $P_{engage}=0.313$ 吻合)
*   **[溯源诊断]**：打爱国者命中率暴跌的核心原因，**不是因为我方无人机精度变差了**（基础命中概率都是0.736），而是因为**敌雷达关机反应太快(5s)**，而我方记忆时间(10s)余量不足，导致抗关机概率仅有50%，直接将命中链路腰斩。

**2. 硬毁伤达成率 (E2)**
*   **长白雷达**：$E2 = 615 / 719 = \mathbf{85.5\%}$ (与配置表 $P_{K\_Kill}=0.855$ 吻合)
*   **爱国者雷达**：$E2 = 107 / 316 = \mathbf{33.9\%}$ (与配置表 $P_{K\_Kill}=0.340$ 吻合)
*   **[溯源诊断]**：打爱国者毁伤率低的核心原因，**不是因为战斗部威力小**，而是因为**目标采用方舱防护设计($V_k$仅0.4)**，破片无法穿透核心舱段。

**3. 终极指标：完整摧毁概率 ($P_k$)**
*   **长白雷达**：$P_k = E1 \times E2 = 71.9\% \times 85.5\% = \mathbf{61.5\%}$ $\rightarrow$ **单发具备较高摧毁把握。**
*   **爱国者雷达**：$P_k = E1 \times E2 = 31.6\% \times 33.9\% = \mathbf{10.7\%}$ $\rightarrow$ **单发基本无法摧毁，大量停留在F级瘫痪状态。**

## 六、 与“辐射源压制效益(P1/P2)”的接口联动

上述毁伤评估不是孤立的，L1层的 `S3_毁伤等级` 将直接转化为时间轴上的 0/1 状态，驱动《辐射源压制效益评估》中的核心指标计算。

**底层联动逻辑（C++伪代码）：**

```cpp
void updateStrikeAndSuppression(int currentTick, UAV& uav, Radar& radar) {
    // 1. 检查目标状态 (利用S1逻辑)
    if (radar.status == RADAR_DEAD) {
        uav.logStatus(S1_CAPTURE, "Target_Already_Killed");
        MissionPlanner::reassignTarget(uav); // 触发改航逻辑
        return;
    }

    // 2. 执行交战判定，读取Config表中的 P_engage 和 P_Kill 阈值
    EngageResult result = evaluateStrike(uav, radar); 

    // 3. 将 S3 毁伤结果映射为 P1(雷达关机覆盖率) 需要的状态流
    if (result.S3 == "K_Kill") {
        radar.status = RADAR_DEAD;
        // 接口输出：从 currentTick 开始，该雷达永远为 0 (关机)
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    } 
    else if (result.S3 == "F_Disable") {
        radar.status = RADAR_DISABLED;
        // 接口输出：调用前文规定的 T_restore 分布模型生成恢复时间
        int disableTime = random_uniform(600, 1800); // 爱国者方舱抢修时间
        radar.restartTick = currentTick + disableTime;
        // 在这段时间内，该雷达状态强制为 0 (关机)，计入 P1 的分子
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
    else if (result.S2 == "Lost_Shutdown") {
        // 逻辑陷阱：敌为了躲避攻击主动关机，虽然无人机脱靶，但雷达客观上处于关机状态！
        // 接口输出：同样记录关机状态，体现“硬杀伤未成，但软压制生效”
        SuppressionLogger::logRadarStatus(radar.id, currentTick, 0); 
    }
}
```

## 七、 战术诊断与结论矩阵

通过上述将物理参数彻底透明化的评估方案，指挥员和装备论证人员可依据以下矩阵快速得出结论：

| $P_k$ 区间 | E1(链路)表现 | E2(毁伤)表现 | 综合评价与战术指导 |
| :--- | :--- | :--- | :--- |
| **≥ 60%**<br>*(如长白雷达)* | 正常 | 极高 | **定点清除目标**。单发或双发齐射即可确保摧毁，无需后续压制。 |
| **15% ~ 40%** | 偏低 | 正常 | **对抗瓶颈目标**。失败多因抗关机/抗诱饵能力弱。战术：需配合电子战机压制敌雷达操作员反应时间。 |
| **< 15%**<br>*(如爱国者)* | 极低 | 极低 | **持续压制目标**。物理特性决定单发无法摧毁且极难命中。战术：放弃“摧毁”企图，改为**多波次轮替攻击**，利用 `F_Disable` 状态累计瘫痪时间（10~30分钟/次），为攻击编队强行撕开 P2（突防安全率）时间窗口。 |
