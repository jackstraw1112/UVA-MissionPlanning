好的，以下为您构建一个面向仿真推演的 **辐射源压制效益综合评估方案**。方案将您提出的新增指标与之前讨论的内容有机整合，明确推演中需要记录的原始数据、计算步骤以及最终结果的呈现方式。

## 一、 评估指标体系总览

我们将评估指标分为 **行为层**（雷达反应）、**效能层**（压制结果）和 **效益层**（经济与体系价值）三个维度，共 10 项核心指标。

| 维度 | 指标代号 | 指标名称 | 指标性质 |
| :--- | :--- | :--- | :--- |
| **行为层** | M1 | 雷达主动关机频次 | 正向指标（越高越好） |
| | M2 | 雷达平均开机时长缩减率 | 正向指标 |
| | M3 | 雷达工作时间占比下降值 | 正向指标 |
| **效能层** | M4 | 雷达关机覆盖率 | 正向指标 |
| | M5 | 区域防空压制覆盖率 | 正向指标 |
| | M6 | 防空火力无法发挥率 | 正向指标 |
| | M7 | 战损比（LER） | 负向指标（越低越好，取倒数归一化） |
| **效益层** | M8 | 直接经济效费比 | 正向指标 |
| | M9 | 间接消耗诱导比 | 正向指标 |
| | M10 | 高价值资产保全贡献度 | 正向指标 |

## 二、 仿真推演所需记录的原始数据清单

为支撑上述 10 项指标的计算，仿真系统需在推演过程中以 **时间序列日志** 形式记录以下基础数据。

### 2.1 全局基础参数（推演前预设）

| 数据项 | 符号 | 单位 | 说明 |
| :--- | :--- | :--- | :--- |
| 仿真总时长 | \( T_{sim} \) | 秒 | 整个推演场景的持续时间 |
| 我方无人机单价 | \( C_{uav} \) | 万美元 | 单架反辐射无人机采购成本 |
| 敌方雷达单价 | \( C_{radar} \) | 万美元 | 被攻击型号雷达的采购成本 |
| 敌方防空导弹单价 | \( C_{missile} \) | 万美元 | 敌方用于拦截的导弹型号单价 |
| 我方突防战机单价 | \( C_{aircraft} \) | 万美元 | 后续突防编队中主要战机的单价 |
| 雷达正常运行基准 | \( T_{base\_on} \) | 秒 | 无压制时，雷达在 \( T_{sim} \) 内的正常开机时长（通常等于 \( T_{sim} \)） |

### 2.2 雷达侧实时状态数据（需逐秒或逐事件记录）

针对每一部参与推演的敌方雷达（\( i = 1, 2, ..., N \)）：

| 记录项 | 数据类型 | 记录内容 |
| :--- | :--- | :--- |
| 雷达开关机状态 | 布尔型时序 | 每秒钟雷达处于“开机(1)”或“关机(0)”的状态 |
| 雷达关机事件日志 | 事件型 | 每次关机的时刻、原因（被截获/主动规避/故障） |
| 雷达开机事件日志 | 事件型 | 每次开机的时刻 |
| 雷达是否被物理摧毁 | 事件型 | 被摧毁的时刻（若发生） |
| 雷达拦截导弹发射记录 | 事件型 | 每次发射导弹的时刻、导弹型号 |

### 2.3 我方反辐射无人机侧数据

| 记录项 | 数据类型 | 记录内容 |
| :--- | :--- | :--- |
| 无人机出动总数量 | 标量 | \( N_{uav\_total} \) |
| 无人机损失记录 | 事件型 | 每架无人机被击落/故障坠毁的时刻 |
| 无人机命中记录 | 事件型 | 每架无人机命中目标雷达的时刻及目标ID |

### 2.4 区域压制与联合作战数据

| 记录项 | 数据类型 | 记录内容 |
| :--- | :--- | :--- |
| 区域栅格安全状态 | 布尔型时序 | 将作战空域栅格化，逐秒记录每个栅格是否被至少一部敌方雷达有效覆盖（即“不安全”） |
| 后续突防编队通过情况 | 事件型 | 编队进入/离开危险空域的时刻、机型、数量 |

## 三、 各指标详细计算方法

以下基于仿真记录的原始数据，给出各指标的计算模型。

### 【行为层指标】

#### M1. 雷达主动关机频次

**定义**：在 \( T_{sim} \) 内，因感知到反辐射威胁而主动执行关机操作的总次数（所有雷达之和）。

**计算**：
\[
M1 = \sum_{i=1}^{N} Count_i(\text{关机事件})
\]
*说明：若仿真系统能区分“主动规避关机”与“被摧毁导致的停机”，则仅统计前者。*

---

#### M2. 雷达平均开机时长缩减率

**定义**：对比无压制基准，敌方雷达单次连续开机时长的平均缩短程度。

**计算步骤**：
1. 对于每部雷达 \( i \)，统计其所有连续开机段的时间长度序列 \( \{ t_{on\_k} \} \)。
2. 计算该雷达的平均单次开机时长 \( \bar{t}_{on\_i} \)。
3. 设定基准值：无压制时，雷达平均单次开机时长 \( \bar{t}_{on\_base} \)（通常等于 \( T_{sim} \)，或根据历史训练数据取值）。
4. 计算缩减率：
\[
M2 = \frac{1}{N} \sum_{i=1}^{N} \frac{\bar{t}_{on\_base} - \bar{t}_{on\_i}}{\bar{t}_{on\_base}} \times 100\%
\]

---

#### M3. 雷达工作时间占比下降值

**定义**：敌方雷达在总仿真时长内的实际开机时长占比，相对于基准值的绝对下降幅度。

**计算**：
1. 统计每部雷达 \( i \) 的总开机时长 \( T_{on\_i} = \sum t_{on\_k} \)。
2. 计算平均工作时间占比：
\[
R_{on} = \frac{1}{N} \sum_{i=1}^{N} \frac{T_{on\_i}}{T_{sim}}
\]
3. 基准工作时间占比 \( R_{base} \)（通常为 1.0，即全时工作）。
4. 计算下降值：
\[
M3 = R_{base} - R_{on}
\]
*取值范围：0 ~ 1。*

---

### 【效能层指标】

#### M4. 雷达关机覆盖率（沿用前文定义）

**定义**：在攻击窗口 \( T_{attack} \) 内，目标雷达处于关机状态的总时长占比。

**计算**（针对重点压制目标雷达或所有雷达取平均）：
\[
M4 = \frac{1}{N} \sum_{i=1}^{N} \frac{\sum T_{off\_i}}{T_{attack}}
\]
*\( T_{attack} \) 定义为从第一架无人机进入截获距离到最后一架命中/脱离的时间段。*

---

#### M5. 区域防空压制覆盖率

**定义**：在指定保护空域（如突防走廊）内，**完全没有敌方雷达有效覆盖**的时间占比。

**计算步骤**：
1. 将关注空域离散化为 \( G \) 个栅格。
2. 对于每个时间步 \( t \)，判定各栅格是否被至少一部处于“开机”状态的雷达有效探测（考虑雷达威力范围）。
3. 统计 **全区域安全**（即所有栅格均未被覆盖）的时间步数量 \( N_{safe} \)。
4. 计算覆盖率：
\[
M5 = \frac{N_{safe}}{N_{total\_steps}} \times 100\%
\]
*若只关注某条特定航线，可简化为该航线在雷达盲区内的通过时长占比。*

---

#### M6. 防空火力无法发挥率

**定义**：敌方防空导弹因缺乏雷达引导而 **无法发射** 或 **发射后无法命中** 的比例。

**计算逻辑**：
- 条件 A：敌方雷达处于关机状态。
- 条件 B：敌方雷达虽开机，但处于被压制状态（如频繁重启、数据率极低），无法形成稳定火控解算。
- **无法发挥时段**：满足 A 或 B 的时间段。

**计算**：
\[
M6 = \frac{\sum T_{\text{无法发挥时段}}}{T_{\text{突防编队暴露窗口}}}
\]
*\( T_{\text{突防编队暴露窗口}} \) 指后续编队处于敌方理论射程内的时间段。*

---

#### M7. 战损比（LER）

**定义**：我方消耗无人机数与敌方等效压制雷达数之比。

**计算**（沿用前文加权模型）：
\[
M7 = \frac{N_{\text{UAV\_lost}}}{\sum (N_{\text{K-Kill}} \times 1.0 + N_{\text{F-Kill}} \times 0.8 + N_{\text{D-Kill}} \times 0.4)}
\]
*注：由于 LER 为负向指标（越小越好），在综合评价时需将其转换为正向指标，例如取 \( 1 / M7 \) 或进行 min-max 归一化。*

---

### 【效益层指标】

#### M8. 直接经济效费比

**计算**：
\[
M8 = \frac{N_{\text{K-Kill}} \times C_{radar}}{N_{\text{UAV\_lost}} \times C_{uav}}
\]

---

#### M9. 间接消耗诱导比

**计算**：
\[
M9 = \frac{N_{\text{missile\_fired}} \times C_{missile}}{N_{\text{UAV\_lost}} \times C_{uav}}
\]
*注：\( N_{\text{missile\_fired}} \) 为敌方为拦截我方无人机而发射的防空导弹总数。*

---

#### M10. 高价值资产保全贡献度

**定义**：因反辐射无人机压制作用，我方突防编队避免的预期战损价值。

**计算**：
1. 无压制时，突防编队的预期战损数量 \( E_{\text{loss\_wo}} \)（根据历史或仿真数据）。
2. 有压制时，实际战损数量 \( L_{\text{actual}} \)。
3. 保全价值：
\[
M10 = (E_{\text{loss\_wo}} - L_{\text{actual}}) \times C_{aircraft}
\]
*若为正值，表示成功保全资产；若为负值或零，说明压制未达到预期保护效果。*

## 四、 综合评估结果计算流程

为了将多个指标融合为一个直观的 **辐射源压制效益综合指数（Suppression Benefit Index, SBI）**，建议采用以下流程：

### 步骤 1：指标归一化处理

由于各指标量纲和量级差异巨大，需先归一化到 \([0, 1]\) 区间。

| 指标 | 归一化方法建议 |
| :--- | :--- |
| M1, M4, M5, M6 | 本身即为比率（0~1），直接使用。若超过 1 则截断为 1。 |
| M2 | 缩减率本身在 0~1 之间，直接使用。 |
| M3 | 下降值本身在 0~1 之间，直接使用。 |
| M7 (LER) | 负向指标转换：\( M7' = \max(0, \min(1, \frac{2 - M7}{2})) \) *（假设 LER 合理上限为 2，2 以上归一化为 0 分）* |
| M8, M9 | 采用对数压缩归一化：\( \text{Score} = \min(1, \frac{\log_{10}(Ratio)}{\log_{10}(R_{max})}) \)，其中 \( R_{max} \) 为设定的期望最大比值（如 100）。 |
| M10 | 采用阈值法：\( \text{Score} = \min(1, \frac{M10}{\text{编队总价值} \times 0.5}) \) |

### 步骤 2：权重分配（建议采用层次分析法 AHP 确定）

提供一组参考权重（可根据实战侧重调整）：

| 维度 | 权重 | 指标 | 子权重 | 综合权重 |
| :--- | :--- | :--- | :--- | :--- |
| 行为层 | 0.20 | M1 关机频次 | 0.25 | 0.050 |
| | | M2 开机时长缩减率 | 0.40 | 0.080 |
| | | M3 工作时间占比下降 | 0.35 | 0.070 |
| 效能层 | 0.50 | M4 关机覆盖率 | 0.25 | 0.125 |
| | | M5 区域压制覆盖率 | 0.30 | 0.150 |
| | | M6 火力无法发挥率 | 0.25 | 0.125 |
| | | M7 战损比（转正） | 0.20 | 0.100 |
| 效益层 | 0.30 | M8 直接效费比 | 0.35 | 0.105 |
| | | M9 间接诱导比 | 0.35 | 0.105 |
| | | M10 资产保全度 | 0.30 | 0.090 |

### 步骤 3：计算综合指数

\[
SBI = \sum_{j=1}^{10} w_j \cdot \text{Score}_j \times 100
\]
*结果转换为百分制，便于直观比较不同作战方案的压制效益。*

## 六、 方案使用说明

1. **数据采集接口**：仿真平台需按第二章所列清单输出结构化的 `.csv` 或 `.json` 日志文件。
2. **后处理工具**：建议使用 Python（Pandas + NumPy）或 MATLAB 编写指标计算脚本，实现自动化评估。
3. **权重调整**：实际使用时，指挥员可根据任务侧重（如“以摧毁为主”还是“以掩护突防为主”）调整三维度权重分配。
