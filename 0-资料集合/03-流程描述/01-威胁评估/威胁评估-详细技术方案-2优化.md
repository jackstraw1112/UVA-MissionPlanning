# 反辐射无人机航路规划威胁评估系统

## 详细技术方案（优化版 — 六因子修正型）

---

## 第一章 总体设计

### 1.1 优化背景与变更说明

基于精简五维方案的实际使用反馈，进行两项核心优化：

**变更一：移除辐射状态因子，改为雷达类型威胁度**

| 项目 | 原方案 | 优化方案 | 变更理由 |
|------|:------:|:-------:|---------|
| 原F1 | 雷达辐射状态 | **移除** | 规划阶段无法预知目标实时开关机状态，该因子在规划场景下不可用 |
| 新F1 | — | **雷达类型威胁度** | 不同类型雷达对ARAD的固有威胁本质不同，是独立的评估维度 |
| 处理方式 | — | 关机目标作为**前置过滤条件**排除出候选列表，不参与评分 | 规划阶段默认候选列表中的目标都是"可攻击状态"，否则不应出现在列表中 |

**设计逻辑变化：**

```
原方案逻辑链：
  该雷达能不能打？──→ F1 辐射状态（规划阶段无法回答！）
  能不能飞到？────→ F2 距离
  导引头截不截得到？→ F3 频段匹配
  打它有没有风险？──→ F4 武器协同
  值不值得打？────→ F5 战略价值

优化方案逻辑链：
  该雷达有多危险？──→ F1 雷达类型（固有能力，可预知）
  能不能飞到？────→ F2 距离
  导引头截不截得到？→ F3 频段匹配
  打它有没有风险？──→ F4 武器协同
  值不值得打？────→ F5 战略价值
```

**变更二：关机目标的处理方式重新定义**

```
规划阶段的目标生命周期：

情报系统                        威胁评估系统                     航路规划器
   │                               │                               │
   ├─ 发现目标(全功率) ────────→ 纳入候选列表 ─────────────→ 纳入航路规划
   ├─ 发现目标(状态未知) ──────→ 纳入候选列表 ─────────────→ 纳入航路规划
   ├─ 确认关机(持续3个周期) ───→ 从候选列表移除 ──────────→ 不规划
   │                               │                               │
   │                          （不进入评分流程）                  │
   │                          （不输出威胁分数）                  │
   │                          （不参与排序）                      │
```

**核心原则：** 威胁评估系统的输入是"已知存在的、可攻击候选目标"。如果一个目标被确认关机，它应该在进入评估系统之前就被情报过滤层移除。评估系统内部不做开关机判断。

### 1.2 为什么保留五维而非六维

去掉"辐射状态"后加入"雷达类型"，仍然是五个因子。不进一步增加维度的原因：

```
Q: 雷达型号/名称不也是一种信息吗？为什么不需要单独列一个因子？

A: "雷达型号"不是评估维度，而是"雷达类型"的载体。
   - FPS-117 和 TPS-117 是不同型号，但都属于"预警雷达"类型 → 同一威胁等级
   - MPQ-65 和 MPQ-53 是不同型号，但都属于"火控雷达"类型 → 同一威胁等级
   - F1评估的是"类型"而非"型号"，型号差异已被类型抽象吸收
```

### 1.3 五因子独立性验证

|  | F1类型 | F2距离 | F3频段 | F4武器 | F5价值 |
|--|:-----:|:-----:|:-----:|:-----:|:-----:|
| **F1类型** | — | 独立 | 独立 | 独立 | 独立 |
| **F2距离** | 独立 | — | 独立 | 独立 | 独立 |
| **F3频段** | 独立 | 独立 | — | 独立 | 独立 |
| **F4武器** | 独立 | 独立 | 独立 | — | 独立 |
| **F5价值** | 独立 | 独立 | 独立 | 独立 | — |

验证要点：
- **F1 vs F5**：火控雷达(F1高)不一定在核心位置(F5不一定高)，反之亦然。例如：前沿补盲的火控雷达(F1=1.0)可能有替代(F5=0.3)，而后方的预警雷达(F1=0.55)无替代(F5=1.0)。**通过**
- **F3 vs F1**：同一类型雷达(F1相同)可能工作在不同频段(F3不同)。例如：标准C波段火控雷达(F1=1.0, F3=1.0) vs 变频火控雷达(F1=1.0, F3=0.5)。**通过**
- **F1 vs F4**：火控雷达(F1高)一定有武器协同吗？不一定。前沿独立部署的火控雷达可能武器未就绪(F4低)，而预警站旁的导弹阵地可能武器就绪(F4高)但雷达本身威胁不高(F1低)。**通过**

## 第二章 五维因子详细规格

### 2.1 F1 — 雷达类型威胁度

| 属性 | 规格 |
|------|------|
| 因子ID | F1 |
| 因子名称 | 雷达类型威胁度 |
| 物理含义 | 目标雷达类型对反辐射无人机的固有威胁程度——不同类型雷达对ARAD的威胁本质不同 |
| 数据来源 | 情报库中的目标类型标注（已知信息，规划阶段可用） |
| 更新频率 | 规划阶段为静态值（目标类型不会在规划期间变化） |
| 默认权重 | 0.25 |
| 设计地位 | 取代原"辐射状态"的核心位置，回答"这个目标对我有多危险" |

**核心区分逻辑：**

```
为什么火控雷达比预警雷达威胁更大？

火控雷达 (如MPQ-65):
  → 直接引导防空导弹的末段拦截
  → 被其照射意味着导弹可能已经在飞来
  → 对ARAD构成直接的、即时的生存威胁
  → 即使只开机几秒截获，也完成了致命使命

预警雷达 (如FPS-117):
  → 提供远程空情，间接威胁
  → 被其截获不会导致导弹来袭
  → 对ARAD构成的是"被发现"风险而非"被击落"风险
  → 威胁存在但层级不同
```

**分级取值表：**

| 等级 | 取值 | 包含类型 | 判定标准 | 威胁本质 |
|:----:|:----:|---------|---------|---------|
| T5 | 1.00 | 火控雷达 | 直接引导防空导弹末段拦截的雷达 | **直接威胁**：被照射=可能被击落 |
| T4 | 0.80 | 三维相控阵 | 具备目标指示和引导能力的多功能雷达 | **半直接威胁**：可引导拦截但有间接性 |
| T3 | 0.55 | 预警雷达 | 远程搜索预警，不直接引导武器 | **间接威胁**：被截获后可能引导其他力量 |
| T2 | 0.30 | 对空监视雷达 | 中近程搜索跟踪，无引导能力 | **信息威胁**：仅提供空情，无直接引导 |
| T1 | 0.15 | 超视距/天波雷达 | 远程但不精确，无法引导精确打击 | **预警威胁**：能发现但精度不足以引导攻击 |
| T0 | 0.40 | 未知类型 | 侦察发现但未能识别类型的辐射源 | **不确定威胁**：按保守偏中等处理 |

**常见雷达类型-等级速查表：**

| 雷达 | 类型 | F1取值 | 说明 |
|------|------|:------:|------|
| AN/MPQ-65 (PAC-3火控) | 火控雷达 | 1.00 | 直接引导PAC-3，最高威胁 |
| AN/MPQ-53 (PAC-2火控) | 火控雷达 | 1.00 | 直接引导PAC-2 |
| CS/MPN-45 (天弓火控) | 火控雷达 | 1.00 | 直接引导天弓导弹 |
| HR-2000S | 三维相控阵 | 0.80 | S波段多功能，可引导拦截 |
| AN/SPS-48E (基德舰) | 三维相控阵 | 0.80 | 舰载三维，引导标准/SM-2 |
| AN/FPS-117 | 预警雷达 | 0.55 | L波段远程预警 |
| AN/TPS-117 | 预警雷达 | 0.55 | 机动型FPS-117 |
| AN/TPS-75 | 对空监视 | 0.30 | C波段中程监视 |
| AN/SPS-49 | 对空监视 | 0.30 | 舰载二坐标搜索 |
| JYL-1 | 超视距雷达 | 0.15 | HF天波，精度极低 |
| FPS-120 | 超视距雷达 | 0.15 | 同上 |
| 未知辐射源 | 未知类型 | 0.40 | 无法判断类型，保守估计 |

**工程判定规则：**

```
IF 情报库中有该目标的类型标注:
    F1 = lookup(类型标注) → 直接取值

ELSE IF 有足够信号特征进行分析:
    IF 检测到精确测距脉冲(窄脉冲+高PRF):
        → 判定为 T5 (火控雷达)
    ELIF 检测到长脉冲+低PRF+机械扫描调制:
        → 判定为 T3 (预警雷达)
    ELIF 检测到连续波或特殊调制:
        → 判定为 T1 (超视距)
    ELSE:
        → 判定为 T0 (未知)

ELSE:
    → 默认 T0 (未知类型) = 0.40
```

### 2.2 F2 — 距离可达性

| 属性 | 规格 |
|------|------|
| 因子ID | F2 |
| 因子名称 | 距离可达性 |
| 物理含义 | 目标到发射点的大地距离，反映航程消耗和攻击窗口时间 |
| 数据来源 | 目标坐标 + 无人机位置（GPS/INS） |
| 更新频率 | 连续 |
| 默认权重 | 0.25 |

**计算公式：**

$$F_2(d) = \text{clamp}\left(1 - \frac{d}{d_{\max}},\ 0,\ 1\right)$$

| 参数 | 取值 | 说明 |
|------|:----:|------|
| $d$ | 实测值 | 单位：km |
| $d_{max}$ | **120** | JWS01满载攻击半径（可配置） |

**与原方案一致，无变化。**

## 第三章 量化计算模型

### 3.1 计算公式

$$S = \hat{W}_1 \cdot \hat{F}_1 + \hat{W}_2 \cdot \hat{F}_2 + \hat{W}_3 \cdot \hat{F}_3 + \hat{W}_4 \cdot \hat{F}_4 + \hat{W}_5 \cdot \hat{F}_5$$

展开（默认权重）：

$$S = 0.250 \cdot F_1 + 0.250 \cdot F_2 + 0.200 \cdot F_3 + 0.150 \cdot F_4 + 0.150 \cdot F_5$$

### 3.2 硬约束检查（前置过滤，不在评分流程内）

```
前置过滤（在目标进入评估系统之前由情报层执行）：

FILTER_1: 目标已被确认持续关机（连续3个侦察周期无信号）
    → 从候选列表中移除
    → 不进入威胁评估系统
    → 标记状态："已排除-确认关机"

FILTER_2: 目标距离超出最大航程且无接力方案
    → 从候选列表中移除
    → 不进入威胁评估系统
    → 标记状态："已排除-超出航程"

FILTER_3: 目标频段确认不匹配（通过信号分析）
    → 从候选列表中移除
    → 不进入威胁评估系统
    → 标记状态："已排除-频段不匹配"

其余候选目标 → 进入五维评分流程
```

**关键区别：** 硬约束不再是评分流程内部的检查步骤，而是系统输入的前置过滤。这保证了进入评分的每个目标都是"理论上可攻击的"，评分结果才能正确反映"谁更值得优先打"。

### 3.3 强制上浮规则（保留一条）

```
IF F4 = 1.00 (导弹就绪+已锁定) AND S < 0.80:
    → S' = max(S, 0.80)
    → 生存优先原则：被锁定时无论其他因子如何，必须优先处理
```

### 3.4 等级映射

| 等级 | 分数区间 | 含义 | 航路动作 |
|:----:|:--------:|------|---------|
| LEVEL_5 | [0.75, 1.00] | **紧急** | 立即规划攻击航路 |
| LEVEL_4 | [0.55, 0.75) | **高** | 优先纳入本轮航路 |
| LEVEL_3 | [0.35, 0.55) | **中** | 弹药有余量时纳入 |
| LEVEL_2 | [0.15, 0.35) | **低** | 持续监视，暂不规划 |
| LEVEL_1 | [0.00, 0.15) | **排除** | 不分配资源（理论上不应出现在候选列表中） |

### 3.5 排序规则

```
PRIMARY:  按 S 降序
SECONDARY: 分差 < 0.05 时，按 F2(距离) 升序（近处优先）
TERTIARY:  同等级内按 F1(类型) 降序（火控优先于预警）
```

## 第四章 完整计算流程

```
┌─────────────────────────────────────────────────────┐
│  PHASE 0: 前置过滤 (情报层执行)                      │
│  ├─ 确认关机 → 移除候选                              │
│  ├─ 超出航程 → 移除候选                              │
│  └─ 频段不匹配 → 移除候选                           │
└──────────────────────┬──────────────────────────────┘
                       ▼ (候选目标列表)
┌─────────────────────────────────────────────────────┐
│  PHASE 1: 因子提取                                  │
│  ├─ F1 = lookup(雷达类型)     → [0.15, 1.00]      │
│  ├─ F2 = clamp(1-d/120, 0, 1) → [0.00, 1.00]      │
│  ├─ F3 = lookup(频段匹配)     → [0.00, 1.00]      │
│  ├─ F4 = lookup(武器协同)     → [0.00, 1.00]      │
│  └─ F5 = lookup(战略价值)     → [0.10, 1.00]      │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  PHASE 2: 权重归一化 + 加权求和                    │
│  S = Σ(Ŵ_i × F_i)                                 │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  PHASE 3: 强制上浮检查                             │
│  IF F4=1.00 AND S<0.80 → S=max(S, 0.80)          │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  PHASE 4: 等级映射 + 排序                         │
│  Level = map(S) → L1~L5                           │
│  Sort → 降序 → 同分按距离升序 → 同级按类型降序       │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  OUTPUT: 攻击优先级队列                              │
│  每个目标: (S, Level, 各因子明细, 排名)             │
└─────────────────────────────────────────────────────┘
```

## 第五章 计算示例

### 5.1 示例一：高威胁目标

**T02: MPQ-65 清水 — 距离8km, 火控雷达, C波段, 导弹就绪+已锁定, 核心节点**

```
F1 = lookup("火控雷达") = 1.00
F2 = clamp(1 - 8/120, 0, 1) = 0.933
F3 = lookup("C波段→完全匹配") = 1.00
F4 = lookup("导弹就绪+已锁定") = 1.00
F5 = lookup("核心节点") = 1.00

S = 0.250×1.00 + 0.250×0.933 + 0.200×1.00 + 0.150×1.00 + 0.150×1.00
  = 0.250 + 0.233 + 0.200 + 0.150 + 0.150
  = 0.983

强制上浮: F4=1.00 但 S>0.80 → 不触发

等级映射: 0.983 ∈ [0.75, 1.00] → LEVEL_5 紧急
```

### 5.2 示例二：中等威胁目标

**T08: MPN-45 三芝 — 距离25km, 三维相控阵, C/D部分匹配, 导弹搜索中, 一般节点**

```
F1 = lookup("三维相控阵") = 0.80
F2 = clamp(1 - 25/120, 0, 1) = 0.792
F3 = lookup("部分匹配") = 0.50
F4 = lookup("导弹就绪+搜索中") = 0.65
F5 = lookup("一般节点") = 0.30

S = 0.250×0.80 + 0.250×0.792 + 0.200×0.50 + 0.150×0.65 + 0.150×0.30
  = 0.200 + 0.198 + 0.100 + 0.098 + 0.045
  = 0.641

强制上浮: F4=0.65 (未达W4标准) → 不触发

等级映射: 0.641 ∈ [0.55, 0.75) → LEVEL_4 高
```

### 5.3 示例三：低价值但近距离目标

**T05: TPS-75 基隆 — 距离15km, 对空监视, C波段完全匹配, 无武器, 一般节点**

```
F1 = lookup("对空监视") = 0.30
F2 = clamp(1 - 15/120, 0, 1) = 0.875
F3 = lookup("完全匹配") = 1.00
F4 = lookup("无武器协同") = 0.00
F5 = lookup("一般节点") = 0.30

S = 0.250×0.30 + 0.250×0.875 + 0.200×1.00 + 0.150×0.00 + 0.150×0.30
  = 0.075 + 0.219 + 0.200 + 0.000 + 0.045
  = 0.539

等级映射: 0.539 ∈ [0.35, 0.55) → LEVEL_3 中
```

**分析：** 虽然距离很近(F2高)、频段完全匹配(F3高)，但雷达类型威胁低(F1=0.30)且无武器风险(F4=0)，整体仅中等。说明"容易打的"不等于"值得打的"。

### 5.4 示例四：高价值但远距离目标

**T09: FPS-117 大岗山 — 距离78km, 预警雷达, L波段完全匹配, 无武器, 核心节点**

```
F1 = lookup("预警雷达") = 0.55
F2 = clamp(1 - 78/120, 0, 1) = 0.350
F3 = lookup("完全匹配") = 1.00
F4 = lookup("无武器协同") = 0.00
F5 = lookup("核心节点") = 1.00

S = 0.250×0.55 + 0.250×0.350 + 0.200×1.00 + 0.150×0.00 + 0.150×1.00
  = 0.138 + 0.088 + 0.200 + 0.000 + 0.150
  = 0.575

等级映射: 0.575 ∈ [0.55, 0.75) → LEVEL_4 高
```

**分析：** 预警雷达本身直接威胁不高(F1=0.55)，且距离较远(F2=0.35)，无武器风险(F4=0)。但战略价值极高(F5=1.00)，且频段完全匹配(F3=1.00)，综合仍为高优先级。说明"重要的目标即使难打也要优先规划"。

### 5.5 对比：原方案vs优化方案

以**T05 TPS-75 基隆(15km, 无武器)**为例对比两种方案的评分差异：

| 维度 | 原方案(含辐射状态) | 优化方案(含雷达类型) |
|------|:-----------------:|:-----------------:|
| F1(原:辐射/新:类型) | 假设"正常开机"=0.85 | 对空监视=0.30 |
| F2(距离) | 0.875 | 0.875 |
| F3(频段) | 1.00 | 1.00 |
| F4(武器) | 0.00 | 0.00 |
| F5(价值) | 0.30 | 0.30 |
| **总分** | **0.656 (LEVEL_4 高)** | **0.539 (LEVEL_3 中)** |

**差异分析：**
- 原方案将"正常开机"算作高威胁(0.85)，导致一个无武器的监视雷达因"开着机"被评为高级
- 优化方案正确识别为"对空监视"类型(0.30)，反映出其本质威胁低
- 优化方案的排序更符合指挥员直觉：火控雷达 > 预警雷达 > 监视雷达，而不是"谁开机谁就危险"

## 第八章 新旧方案对照总表

| 维度 | 原五维方案 | 优化五维方案 |
|------|-----------|-----------|
| **F1** | 雷达辐射状态(关机/开机/LPI...) | **雷达类型威胁度**(火控/预警/监视/超视距/未知) |
| **F2** | 距离可达性 | 距离可达性（不变） |
| **F3** | 频段匹配度 | 频段匹配度（不变） |
| **F4** | 武器协同威胁 | 武器协同威胁（不变） |
| **F5** | 战略节点价值(含类型信息) | **战略节点价值(纯体系位置)** |
| **关机处理** | F1=0时在评分内归零 | **前置过滤移除候选，不进入评分** |
| **规划适用性** | 存在不可预知的实时量 | **全部因子在规划阶段均可预知** |
| **核心问题** | "能不能打"（无法回答） | **"有多危险"（可准确回答） |

**一句话总结变化：** 将"实时状态判断"从前端评估中剥离为情报前置过滤，将"雷达固有威胁属性"(类型)独立为一个评估维度，使系统在任务规划阶段的所有输入均为已知确定量，消除了不确定性。
