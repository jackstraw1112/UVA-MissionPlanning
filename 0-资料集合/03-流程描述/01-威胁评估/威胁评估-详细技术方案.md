这个改动方向非常对——仿真系统里应该从客观物理参数推导威胁，而不是依赖人工标注的"类型名称"。频率、脉宽、PRF、探测范围这四个参数的组合，本质上就是雷达体制的指纹。

---

## 设计思路

四个参数的物理含义与威胁指向：

```
频率    → 越高，距离分辨率越高，越可能是火控/跟踪体制
脉宽    → 越窄，距离分辨率越高，越可能是跟踪/火控体制
PRF     → 越高，多普勒处理能力越强，越可能是PD体制火控
探测范围 → 同等体制下，覆盖越大的威胁越广
```

四个参数两两之间存在物理耦合（比如高频雷达通常配窄脉冲和高PRF），所以不能简单查表，需要用连续函数分别评估"火控倾向"，然后加权合成。

---

## 参数预处理

输入是范围（如"频率 4~8 GHz"），需要先提取代表值。对数空间的几何中心最合理，因为雷达参数跨度可达数个数量级：

$$f_{\text{center}} = \sqrt{f_{\min} \times f_{\max}}$$

$$\tau_{\text{center}} = \sqrt{\tau_{\min} \times \tau_{\max}}$$

$$\text{PRF}_{\text{center}} = \sqrt{\text{PRF}_{\min} \times \text{PRF}_{\max}}$$

如果某个参数只给了一个值，直接用该值。

---

## 三个子因子

### 子因子1：频率威胁倾向 f_freq

```
输入：频率中心值（GHz）

f_freq(f):
  f ≤ 0.1                    → 0.05    (天波超视距)
  0.1 < f ≤ 1.0              → 0.15    (UHF预警)
  1.0 < f ≤ 4.0              → 0.15 + 0.30 × (f-1.0)/3.0    (L/S波段)
  4.0 < f ≤ 8.0              → 0.45 + 0.25 × (f-4.0)/4.0    (C波段)
  8.0 < f ≤ 12.0             → 0.70 + 0.20 × (f-8.0)/4.0    (X波段，峰值区)
  12.0 < f ≤ 18.0            → 0.90 - 0.30 × (f-12.0)/6.0   (Ku波段，下降)
  f > 18.0                   → 0.40    (mm波，近程导引头)
```

**逻辑：** X波段(8-12GHz)是精密跟踪/火控的经典频段，取峰值；Ku开始下降因为更多用于近程；UHF/L/S主要是搜索预警。

### 子因子2：脉宽威胁倾向 f_pw

对数映射，因为脉宽跨度从0.01μs到100μs以上：

$$f_{\text{pw}}(\tau) = \text{clamp}\left(1.0 - \frac{\log_{10}(\tau) + 2}{4},\ 0,\ 1\right)$$

```
τ = 0.01μs  →  log=-2.0  →  f=1.00    (极窄，高精度火控)
τ = 0.1μs   →  log=-1.0  →  f=0.75    (窄脉冲，跟踪)
τ = 1.0μs   →  log= 0.0  →  f=0.50    (中等)
τ = 10μs    →  log= 1.0  →  f=0.25    (搜索模式)
τ = 100μs   →  log= 2.0  →  f=0.00    (远程搜索)
```

### 子因子3：PRF威胁倾向 f_prf

对数映射，PRF跨度从几十Hz到上百kHz：

$$f_{\text{prf}}(p) = \text{clamp}\left(\frac{\log_{10}(p) - 1.5}{3.5},\ 0,\ 1\right)$$

```
PRF = 30Hz     →  log=1.48  →  f=0.00    (天波超视距)
PRF = 100Hz    →  log=2.00  →  f=0.14    (远程搜索)
PRF = 300Hz    →  log=2.48  →  f=0.28    (中程搜索)
PRF = 1kHz     →  log=3.00  →  f=0.43    (搜索/监视)
PRF = 3kHz     →  log=3.48  →  f=0.57    (多功能)
PRF = 10kHz    →  log=4.00  →  f=0.71    (PD体制)
PRF = 30kHz    →  log=4.48  →  f=0.86    (高PRF跟踪)
PRF = 100kHz   →  log=5.00  →  f=1.00    (高PRF火控)
```

---

## 合成公式

$$F_{1,\text{raw}} = 0.40 \times f_{\text{freq}} + 0.30 \times f_{\text{prf}} + 0.30 \times f_{\text{pw}}$$

**权重逻辑：** 频率权重最高(0.40)，因为频率是区分雷达用途最稳定的特征；PRF和脉宽同等重要(各0.30)，它们共同描述雷达的工作体制。

---

## 探测范围修正

探测范围不独立贡献威胁分数，而是作为**条件修正因子**——同等体制下，覆盖越大的威胁越广：

$$\text{rangeMod}(R) = \begin{cases} 1.0 + 0.08 \times F_{1,\text{raw}} & R \ge 200\text{km} \\ 1.0 & 50 < R < 200\text{km} \\ 1.0 - 0.10 \times F_{1,\text{raw}} & R \le 50\text{km} \end{cases}$$

**逻辑：**
- 远程(≥200km)对已被判定为高威胁体制的雷达小幅放大——远程火控比近程火控覆盖范围更大，战斗机更难绕开
- 近程(≤50km)对高威胁体制小幅缩减——覆盖有限，不经过就无威胁
- 低威胁体制（预警/搜索）的远程探测不额外加分，因为探测远不等于能引导武器

$$F_1 = \text{clamp}(F_{1,\text{raw}} \times \text{rangeMod}(R),\ 0,\ 1)$$

- 火控雷达三型（MPQ-65/53、MPN-45）稳定在 0.61~0.69，明显高出其他类型，排序正确

```
目标编号 / 名称
├─ 频率范围：___ ~ ___ GHz（或单值）
├─ 脉宽范围：___ ~ ___ μs（或单值）
├─ PRF范围：___ ~ ___ Hz（或单值）
├─ 探测范围：___ km（单值即可）
├─ 阵地火力：（强防护 / 中防护 / 弱防护 / 无防护）
├─ 辐射模式：（全功率 / 正常 / LPI / 闪避 / 关机）
├─ 辐射功率：___ kW（可选）
└─ 额定功率：___ kW（可选）
```

雷达类型字段去掉了，完全由物理参数推导。你可以给出目标参数了。
