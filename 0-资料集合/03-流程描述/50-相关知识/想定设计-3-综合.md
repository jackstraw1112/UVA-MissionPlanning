### 第三部分：详细作战计划与仿真触发逻辑

本计划将文档3中的“四大协同模式”转化为仿真系统中的**事件触发器**，供受训的红方指挥员（或AI自动执行）操作。

#### 阶段零：隐蔽接敌与态势塑造 (H-4:00 至 H-0:00)
*   **仿真设定目标**：建立初始阵位，不被蓝方E-2K发现。
*   **行动指令**：
    1.  R-A01(空警-500)保持在大陆沿岸150km处，高度8000m，雷达开启但使用窄波束背对海峡。
    2.  R-S01~04(歼-20)超低空(200m)出海，开启被动ESM，不发射任何电磁信号。
    3.  R-U01~24(JWS01)分三波次（间隔8分钟）发射，高度分别为2000m、500m、3500m。导引头**强制关闭**（仿真参数：`Receiver = OFF`）。
*   **蓝方AI触发器**：若E-2K未侦测到异常，蓝方雷达网保持`DEFCON 3`（LPI低功率模式）。

#### 阶段一：协同模式一 ——「闪屏诱猎」(H+0:00 至 H+0:15)
*   **仿真设定目标**：利用电子战逼出蓝方雷达最大功率，为JWS01提供初始锁定。
*   **触发条件**：红方歼-16D进入干扰阵位（距大汉山200km）。
*   **执行逻辑链**：
    1.  R-E编队开启大功率瞄准式干扰（仿真内设定干扰噪声电平压制FPS-117信噪比至0dB以下）。
    2.  **蓝方AI强制响应**：大汉山FPS-117从M2(LPI 30%功率)强制切换至M4(抗干扰 100%功率，固定频点)。
    3.  **红方判定触发**：R-A01(空警-500)ESM侦测到FPS-117功率跃升，系统弹窗提示：“目标Alpha暴露，功率100%”。
    4.  R-A01通过数据链向JWS01集群广播目标坐标。
    5.  JWS01集群导引头开机（`Receiver = ON`），截获信号。
    6.  **关键操作**：受训者必须在JWS01截获后**3秒内**指令R-E编队“瞬时静默”（否则JWS01会被己方干扰杂波误导偏航）。

#### 阶段二：协同模式二 ——「盲弓引矢」(H+0:15 至 H+0:45)
*   **仿真设定目标**：应对蓝方最熟练的“关机+闪避+诱饵”战术。
*   **触发条件**：蓝方FPS-117执行紧急关机（M5模式），且TLQ-32诱饵开机。
*   **执行逻辑链**：
    1.  蓝方关机，JWS01丢失目标，转入盘旋搜索（仿真参数：`Flight_Mode = Orbit_Search`，半径3km）。
    2.  部分JWS01被TLQ-32诱饵吸引（根据设定的70%识别率，仿真系统进行掷骰判定，约30%的JWS01会偏离航线攻击诱饵）。
    3.  **红方关键动作**：指令R-A01(空警-500)对大汉山阵地开启**SAR/红外凝视模式**。
    4.  **仿真判定**：空警-500生成高精度坐标（误差<5m），通过Link-16发送给未受诱骗的JWS01。
    5.  JWS01切换飞行模式：`Guidance = INS/GPS_DataLink`（不再依赖自身雷达导引头）。
    6.  多架JWS01从不同方位角（设定间隔60度）同时俯冲，摧毁概率提升至75%以上。

#### 阶段三：协同模式三 ——「暗卫猎鹰」(贯穿全阶段，重点H+0:20)
*   **仿真设定目标**：保护JWS01免受蓝方F-16V硬杀伤。
*   **触发条件**：蓝方F-16V起飞，且开启APG-83雷达。
*   **执行逻辑链**：
    1.  F-16V雷达开机瞬间，被R-A01或R-S(歼-20)被动截获。
    2.  **仿真限制**：严格禁止受训者控制歼-20开启火控雷达（若开机，F-16V会提前规避，且暴露歼-20位置，判定扣分）。
    3.  歼-20完全依赖数据链提供的坐标，在F-16V侧后方进入射程（PL-15射程145km）。
    4.  实施“A射B导”或静默发射，击落F-16V。若F-16V发射AGM-88H反辐射导弹，JWS01需执行程序规避机动（`Evasive_Manuever = ON`）。

#### 阶段四：协同模式四 ——「双模补刀」与主战突击 (H+1:00 至 H+2:00)
*   **仿真设定目标**：彻底摧毁掩体化雷达，并利用打开的走廊打击高价值目标。
*   **触发条件**：蓝方雷达进入全面静默（天线降入掩体，M10模式），JWS01盘旋至燃油耗尽前15分钟。
*   **执行逻辑链**：
    1.  指令HS-010光学巡飞弹进入阵地正上方。
    2.  **仿真AI视觉判定**：HS-010执行红外扫描，识别掩体通风口/升降机构热信号（`Target_Type = Ventilation_Shaft`）。
    3.  HS-010俯冲攻击掩体薄弱处。若命中，大汉山FPS-117状态变为**“永久功能性摧毁”**（无法修复）。
    4.  **走廊打开判定**：系统计算防空杀伤区重叠度，当北部/中部杀伤区出现宽度>50km的缺口时，屏幕高亮提示：“防空走廊已建立，允许B1编队突防”。
    5.  指令R-B01~06(轰-6K)进入走廊，在250km外发射KD-20打击第一部分设计的**T-C01至T-M01**等衍生高价值目标。

### 仿真系统开发建议（给程序员的Tip）：
1.  **弹药经济系统**：一定要在UI上醒目显示蓝方PAC-3的剩余枚数（48->42->36...），让受训者直观感受到“用400万的导弹打40万的无人机”的交换比痛苦，这是该想定的核心体验。
2.  **时间窗口可视化**：蓝方雷达的“开/关机周期”（如15s开/45s关）应在态势图上用倒计时环显示，训练红方指挥官把握“3秒协同攻击”的手感。
3.  **数据链链路状态**：增加一个“电子战态势面板”，实时显示红方空警-500到JWS01的UHF数据链是否被蓝方通信对抗车压制，一旦链路断开，JWS01立刻变回“瞎子”单机模式。
