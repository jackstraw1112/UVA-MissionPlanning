## 2. 想定总览

### 2.1 想定代号
- **「碎盾-利刃」联合反辐射作战想定**

### 2.2 作战背景
- 红方目标：通过反辐射无人机与多平台协同压制蓝方雷达网，开辟突防窗口，保障后续打击任务。
- 蓝方目标：在遭遇饱和压制、诱骗和电子干扰条件下维持雷达网完整与拦截效能。

### 2.3 地理与环境边界
- 空域范围：经度 `118.5E~121.8E`，纬度 `23.5N~25.6N`。
- 典型气象：春季，海面风3-4级，能见度15-20km，降水弱，电磁环境复杂。
- 仿真步长建议：1秒（关键交战阶段可降至0.5秒）。

## 5. 核心战术机制（融合版）

### 5.1 协同机制A：闪屏诱猎（E1 + U1）
1. 歼-16D实施瞄准式干扰，迫使蓝方雷达提升功率并收窄波束。
2. 暴露窗口出现后，空警-500广播目标方位与置信度。
3. JWS01导引头并发截获并锁定。
4. 截获后3秒内歼-16D减干扰，避免己方导引头受污染。

### 5.2 协同机制B：盲弓引矢（A1 + U1）
1. 蓝方雷达关机/闪照时，空警-500以SAR/EO/IR生成精确目标坐标。
2. JWS01切入 `INS/GNSS+数据链坐标修正` 模式。
3. 多机多方位同时进入末段，提升关机状态下命中概率。

### 5.3 协同机制C：暗卫猎鹰（S1 + U1）
1. 蓝方F-16V起飞拦截时，歼-20保持静默，依托数据链实施超视距压制。
2. 目标是保护JWS01存活，避免低速无人机被拦截机清空。

### 5.4 协同机制D：双模补刀（U1 + D1）
1. 对“全面关机+诱饵+掩体”的目标，JWS01负责持续压制。
2. HS-010进行光学识别并攻击掩体弱点（如通风口/升降机构）。
3. 达成“功能性摧毁”，即便非直接炸毁天线亦可判定失能。

## 6. 任务映射与分配规则（防混算）

### 6.1 任务结构
- `SUP_x`：压制任务（UAV主导）
- `ATK_x`：突防/打击任务（战机或导弹主导）
- 强绑定关系：`ATK_x <- {SUP_a, SUP_b, ...}`

### 6.2 分配建议
1. 每个核心雷达至少分配2架JWS01（主攻+补刀）。
2. 对机动/高价值节点建议3-4架，含预备队。
3. 保留不少于总量15%的机动预备架次用于再攻击与补位。

### 6.3 评估边界
- 所有J1/J2统计必须按 `ATK_ID` 聚合，不得跨任务混算。

## 7. 仿真执行主流程（可落地）

### 7.1 阶段划分
1. **阶段0（H-4:00 ~ H-0:00）**：隐蔽部署、装订、接敌塑形。
2. **阶段1（H+0:00 ~ H+0:15）**：电子压制与首批截获。
3. **阶段2（H+0:15 ~ H+0:45）**：护航与第一轮雷达压制。
4. **阶段3（H+0:45 ~ H+1:30）**：高强度攻防博弈（关机、诱饵、硬杀伤、协同反制）。
5. **阶段4（H+1:30 ~ H+2:00）**：走廊利用与后续打击。

### 7.2 关键状态机（U1核心）
1. `PreLaunch` -> `Cruise` -> `Loiter/Search` -> `Lock` -> `Attack` -> `BDA/End`
2. 关键分支：
   - 目标关机：进入 `Memory/INS Attack` 或 `Orbit Search`
   - 诱饵判定：通过指纹识别阈值决定攻击/放弃
   - 链路中断：降级为单机策略
   - 燃油告警：触发安全自毁或终止

### 7.3 蓝方关键反制逻辑
1. 雷达模式切换：`LPI -> 抗干扰 -> 闪照 -> 关机/静默`
2. 诱饵策略：地面诱饵与升空诱饵组合。
3. 硬杀伤策略：PAC-3快速开机拦截，受弹药上限约束。
4. 机动接替：主站受损时机动站接替覆盖。

## 10. 评估指标体系（统一到三项评估）

### 10.1 压制效益（P）
- `P1` 雷达关机覆盖率
- `P2` 区域防空压制覆盖率

### 10.2 攻击毁伤能力（S/E）
- 单次：`S1`截获、`S2`打入、`S3`毁伤等级
- 统计：`E1`链路达成率、`E2`硬毁伤达成率

### 10.3 联合作战效果（J）
- `J1` 关联任务突防成功率提升幅度（建议阈值：>=15%）
- `J2` 关联任务防空拦截效能下降率（建议阈值：>=50%）

### 10.4 结论判定规则
1. P达标但J不达标：说明压制动作有效但任务协同节奏失配。
2. S/E不达标且J不达标：说明攻击链路本身不足，需先修复战术基本面。
3. J达标优先于局部战损结果，体现任务导向。

## 12. 最终结论（融合版）

1. JWS01单平台效能有限，但在“预警-电子战-隐身护航-光学补刀”网络中可形成体系级压制。
2. 蓝方“关机+诱饵+硬杀伤”在初期有效，但受成本与时窗约束，难以长期维持。
3. 决定胜负的不是单次命中，而是协同链路的完整性、反应速度与抗干扰韧性。
4. 本想定可直接转入仿真系统实施，建议先开展“小规模参数标定跑批”，再进行全量战役级推演。

1. 条件A：若RWR告警等级高且导弹库存充足，优先“短开机+快发射+关机”。
2. 条件B：若库存下降且诱饵可用，优先“关机+诱饵接替”。
3. 条件C：若核心节点暴露且接替站可用，执行“主站静默+机动站轮替闪照”。
4. 条件D：若多节点同时受压且空中掩护不足，进入“全面静默保网”。

## 16. 双方改进建议矩阵（补充版）

### 16.1 红方改进方向

| 问题点 | 改进措施 | 预期提升 |
|---|---|---|
| 诱饵识别不稳定 | 扩充辐射源指纹库，增加相位噪声/天线调制特征 | 诱饵识别率提升 |
| 末段偏差较大 | 增强末端光学/红外融合修正 | 命中精度提升 |
| 链路易受压制 | 跳频扩频+中继备链 | 协同稳定性提升 |
| 对掩体毁伤不足 | 战斗部组合优化（功能失能优先） | 对重防护目标效果提升 |

### 16.2 蓝方改进方向

| 问题点 | 改进措施 | 预期提升 |
|---|---|---|
| 成本交换比不利 | 增加低成本近防/激光拦截层 | 单次拦截成本下降 |
| 闪照窗口被突破 | 压缩火控链路时延，预编程发射流程 | 抢占反应时间 |
| 静默期间感知缺口 | 多源被动感知与外部数据融合 | 降低静默盲区风险 |
| 机动接替效率低 | 提升机动站隐蔽部署与快速展开能力 | 战损后恢复时间缩短 |
