# 反辐射无人机航路规划威胁评估系统

## 详细技术方案（精简五维版）

---

## 第一章 总体设计

### 1.1 设计定位

本系统是反辐射无人机（ARAD）**作战方案规划阶段**的辅助决策工具，用于在任务规划时对候选目标集合进行威胁排序，输出攻击优先级队列，驱动航路规划器生成最优飞行路径。

**明确不覆盖的能力边界：**
- 不负责飞行中的实时威胁响应（由机载告警系统处理）
- 不负责目标识别和分类（由情报系统提供输入）
- 不负责航路生成本身（由航路规划器消费本系统输出）
- 不负责武器投放决策（由火控系统处理）

### 1.2 设计原则

| 原则 | 含义 |
|------|------|
| **正交完备** | 每个因子对应一个独立的规划决策维度，因子之间无信息冗余 |
| **硬约束优先** | 不可行目标（关机/超距/不匹配）在评分前即被排除，不参与排序 |
| **单调递增** | 每个因子取值越大威胁越高，加权求和后总分严格单调 |
| **参数可调** | 所有权重和阈值可由指挥员根据态势调整，无需修改代码 |
| **可追溯性** | 每个目标的评分结果可反向追溯到每个因子的原始取值 |

### 1.3 为什么是五个因子

航路规划本质上是回答五个串行决策问题，每个问题对应一个因子：

```
问题1: 该雷达当前有没有辐射信号可以攻击？
  └── F1 雷达辐射状态
        ├─ 回答"没有" → 硬约束排除，后续问题不再讨论
        └─ 回答"有" → 继续问题2

问题2: 无人机在燃料限制内能否飞到目标位置？
  └── F2 距离可达性
        ├─ 回答"飞不到" → 硬约束排除
        └─ 回答"飞得到" → 继续问题3

问题3: 到了之后，导引头能不能截获目标信号？
  └── F3 频段匹配度
        ├─ 回答"截不到" → 硬约束排除
        └─ 回答"截得到" → 继续问题4

问题4: 攻击路线上有没有被防空系统拦截的风险？
  └── F4 武器协同威胁
        ├─ 回答"有高风险" → 航路需规避或提高优先级
        └─ 回答"风险可控" → 继续问题5

问题5: 这个目标值不值得消耗一架JWS01？
  └── F5 战略节点价值
        └─ 决定最终的攻击优先级排序

任意一个因子都不可以被其他因子推导出来，五个因子构成最小完备集。
```

### 1.4 被排除因子的独立性论证

| 被排除因子 | 排除理由 | 信息去向 |
|-----------|---------|---------|
| 辐射功率等级 | 功率信息已被F1"辐射状态"隐含：全功率=高功率，LPI=低功率，关机=零功率 | → 已包含在F1中 |
| 锁定状态 | 属于飞行阶段实时告警，规划阶段无法预知；且"已锁定"本质上是F4"武器协同"的最高等级 | → 已包含在F4中 |
| 打击窗口评估 | 窗口长短是F1(状态)×F2(距离)的函数，不独立；规划阶段估算精度极低 | → F1×F2的衍生量 |
| 战略价值(原F9) | 与F5完全重复 | → 合并到F5 |

## 第二章 五维因子详细规格

### 2.1 F1 — 雷达辐射状态

| 属性 | 规格 |
|------|------|
| 因子ID | F1 |
| 因子名称 | 雷达辐射状态 |
| 物理含义 | 目标雷达当前是否处于辐射状态，以及辐射信号的稳定性 |
| 数据来源 | 电子侦察系统（ESM）/ 信号情报（SIGINT）/ 态势通报 |
| 更新频率 | 规划阶段为快照值；仿真推演中可按规则变化 |
| 默认权重 | 0.30（最高权重，因为这是能否攻击的前提） |

**分级取值表：**

| 等级 | 取值 | 判定条件 | 物理解释 |
|:----:|:----:|---------|---------|
| S5 | 1.00 | 全功率连续辐射，无频率捷变 | 发射机满功率运行，信号稳定，导引头远距可靠截获 |
| S4 | 0.85 | 正常功率辐射，可能有频率捷变 | 标准工作模式，导引头可截获但需一定处理增益 |
| S3 | 0.40 | LPI低截获概率模式：低功率 + 宽带捷变 + 波形复杂 | 功率降低至额定值20-40%，频率快速跳变，截获距离大幅缩短 |
| S2 | 0.25 | 闪避照射：短脉冲间歇发射（脉冲宽度<1μs，占空比<5%） | 每次仅发射数个脉冲后即关机，导引头截获窗口极窄 |
| S1 | 0.00 | 完全关机，无任何电磁辐射 | **硬约束：导引头无法截获，Score强制归零** |

**工程判定规则：**

```
IF 侦察系统在目标方位检测到电磁信号:
    IF 信号功率 > 额定功率的70%:
        判定为 S5 (全功率)
    ELIF 信号功率 > 额定功率的20%:
        判定为 S4 (正常)
    ELIF 信号功率 > 额定功率的5%:
        判定为 S3 (LPI)
    ELIF 信号功率 > 噪声底噪10dB:
        判定为 S2 (闪避)
    ELSE:
        判定为 S1 (关机)
ELSE:
    连续3次扫描周期(每次30s)未检测到信号:
        判定为 S1 (关机)
    ELSE:
        维持上一周期判定结果
```

### 2.2 F2 — 距离可达性

| 属性 | 规格 |
|------|------|
| 因子ID | F2 |
| 因子名称 | 距离可达性 |
| 物理含义 | 目标到JWS01发射阵地（或当前位置）的大地距离，反映航程消耗 |
| 数据来源 | 目标坐标 + 无人机位置（GPS/INS） |
| 更新频率 | 连续 |
| 默认权重 | 0.25 |

**计算公式：**

$$F_2(d) = \text{clamp}\left(1 - \frac{d}{d_{\max}},\ 0,\ 1\right)$$

| 参数 | 符号 | 取值 | 说明 |
|------|------|:----:|------|
| 目标距离 | $d$ | 实测值 | 单位：km |
| 最大航程 | $d_{max}$ | **120** | JWS01满载状态下可靠攻击距离(含巡航+攻击段) |
| 下限 | — | 0 | 距离不超过航程时取0 |

**特殊说明：**
- $d_{max}=120\text{km}$ 是JWS01典型作战半径（200km/h巡航45分钟+攻击段5分钟，留30%余量）
- 如果采用多架接力攻击，$d_{max}$ 可由指挥员手动调整（如2架接力设为200km）
- 距离为0-120km时线性映射；超过120km触发硬约束排除

**距离分段与航路含义：**

| 距离范围 | F2值 | 航路含义 |
|:-------:|:----:|---------|
| 0-20 km | 0.83-1.00 | 近距直飞，燃油充足，多次修正机会 |
| 20-50 km | 0.58-0.83 | 中距，标准攻击剖面 |
| 50-80 km | 0.33-0.58 | 远距，需优化航路节省燃油 |
| 80-120 km | 0-0.33 | 极远距，需精确规划，余量紧张 |
| >120 km | 0 | **硬约束排除** |

### 2.5 F5 — 战略节点价值

| 属性 | 规格 |
|------|------|
| 因子ID | F5 |
| 因子名称 | 战略节点价值 |
| 物理含义 | 目标在敌方防空体系中的节点重要性，毁伤后对体系完整性的影响程度 |
| 数据来源 | 目标体系结构分析（人工标注或AI辅助评估） |
| 更新频率 | 战役级别为静态值；战术级别可动态调整 |
| 默认权重 | 0.10（最低权重，因为这是"加分项"而非"门槛项"） |

**分级取值表：**

| 等级 | 取值 | 判定标准 | 典型目标 |
|:----:|:----:|---------|---------|
| V4 | 1.00 | 核心节点：毁伤后导致防区预警能力完全丧失，且相邻防区无法有效弥补 | 大汉山FPS-117（北部唯一远程预警） |
| V3 | 0.65 | 重要节点：毁伤后导致局部降级，但可通过相邻节点部分弥补或机动雷达替补 | 乐山FPS-117（中部预警，大岗山可部分覆盖） |
| V2 | 0.30 | 一般节点：毁伤后有一定影响，但体系有充足的冗余手段 | TPS-75补盲雷达（有其他手段覆盖其责任区） |
| V1 | 0.10 | 次要目标：战术价值有限，毁伤收益不明确 | 未知雷达、诱饵辐射源 |

**节点价值评估辅助判据：**

```
评估维度（非独立计算，作为人工标注参考）：

1. 不可替代性
   该防区内是否有同类型同能力的替代雷达？
   → 无替代 = 加分，有替代 = 减分

2. 覆盖范围
   该雷达覆盖的空域是否与其他雷达覆盖重叠度低？
   → 重叠度低 = 独特覆盖区 = 加分

3. 体系位置
   该雷达在"预警→引导→火控"链条中的位置
   → 处于链条前端(预警) > 中端(引导) > 末端(火控，已被攻击时可跳过)

4. 物理防护
   该雷达是否有掩体/地下工事？（影响的是攻击难度，已由F1/F4覆盖）
   → 不作为F5的输入，避免与F4重复
```

## 第三章 量化计算模型

### 3.1 因子归一化

所有因子在进入计算前必须归一化到 $[0, 1]$ 区间：

$$\hat{F}_i = \text{normalize}(F_i^{\text{raw}}) \in [0, 1]$$

| 因子 | 归一化方法 | 原始值范围 | 归一化后范围 |
|:----:|-----------|:----------:|:----------:|
| F1 | 查表映射 | 离散等级 {0, 0.25, 0.40, 0.85, 1.00} | [0, 1] |
| F2 | 线性反比截断 | [0, +∞) km | [0, 1] |
| F3 | 查表映射 | 离散等级 {0, 0.25, 0.50, 1.00} | [0, 1] |
| F4 | 查表映射 | 离散等级 {0, 0.35, 0.65, 1.00} | [0, 1] |
| F5 | 查表映射 | 离散等级 {0.10, 0.30, 0.65, 1.00} | [0, 1] |

### 3.2 权重归一化

用户设置的原始权重 $W_i^{\text{raw}}$ 在计算前自动归一化：

$$\hat{W}_i = \frac{W_i^{\text{raw}}}{\sum_{j=1}^{5} W_j^{\text{raw}}}$$

**归一化保证：** $\sum_{i=1}^{5} \hat{W}_i = 1.000$，总分 $S \in [0, 1]$。

**默认权重配置：**

| 因子 | $W_i^{\text{raw}}$ | $\hat{W}_i$ | 设计依据 |
|:----:|:----------:|:----------:|---------|
| F1 辐射状态 | 0.30 | 0.300 | 能不能打的前提，权重最高 |
| F2 距离可达 | 0.25 | 0.250 | 能不能飞到的硬约束 |
| F3 频段匹配 | 0.20 | 0.200 | 技术可行性，较稳定的属性 |
| F4 武器威胁 | 0.15 | 0.150 | 航路风险，规划时重要但非决定性 |
| F5 战略价值 | 0.10 | 0.100 | 弹药分配参考，锦上添花 |
| **合计** | **1.00** | **1.000** | |

### 3.3 加权求和

$$S = \sum_{i=1}^{5} \hat{W}_i \times \hat{F}_i$$

展开：

$$S = 0.300 \cdot \hat{F}_1 + 0.250 \cdot \hat{F}_2 + 0.200 \cdot \hat{F}_3 + 0.150 \cdot \hat{F}_4 + 0.100 \cdot \hat{F}_5$$

### 3.4 硬约束检查（在加权求和之前执行）

```
INPUT: 目标t的因子取值 (F1, F2, F3)
OUTPUT: 是否排除 (isExcluded), 排除原因

RULE_1: IF F1 = 0 (关机)
    → isExcluded = TRUE
    → reason = "雷达关机，无辐射信号"
    → RETURN (isExcluded, reason)

RULE_2: IF F3 = 0 (频段不匹配)
    → isExcluded = TRUE
    → reason = "频段不匹配，导引头无法截获"
    → RETURN (isExcluded, reason)

RULE_3: IF F2 = 0 (距离超航程)
    → isExcluded = TRUE
    → reason = "距离超出JWS01航程(120km)"
    → RETURN (isExcluded, reason)

IF isExcluded:
    → S = 0 (强制归零)
    → 跳过加权求和和后续步骤
    → 直接标记为 LEVEL_1(排除)
ELSE:
    → 继续执行加权求和
```

### 3.5 强制上浮（在加权求和之后执行）

```
INPUT: 加权求和结果 S, 目标t的F4取值
OUTPUT: 修正后分数 S'

IF F4 = 1.00 (导弹就绪+已锁定) AND S < 0.80:
    → S' = max(S, 0.80)
    → overrideFlag = TRUE
    → overrideReason = "火力通道已锁定，强制提升至紧急级"
ELSE:
    → S' = S
    → overrideFlag = FALSE

S_final = clamp(S', 0, 1)
```

**设计意图：** 被导弹锁定意味着即使其他因子评分不高（如距离远、价值低），也必须优先处理，因为不处理就会被击落。这体现了"生存优先于效率"的规划原则。

### 3.6 等级映射

$$\text{Level} = \text{map}(S_{\text{final}})$$

| 等级 | 分数区间 | 含义 | 航路规划动作 |
|:----:|:--------:|------|------------|
| LEVEL_5 | [0.75, 1.00] | **紧急** | 立即规划攻击航路，分配JWS01，最高优先级 |
| LEVEL_4 | [0.55, 0.75) | **高** | 纳入本轮攻击航路规划，次优先级 |
| LEVEL_3 | [0.35, 0.55) | **中** | 作为备选目标，弹药有余量时纳入 |
| LEVEL_2 | [0.15, 0.35) | **低** | 暂不规划航路，持续监视，记录在案 |
| LEVEL_1 | [0.00, 0.15) | **排除** | 关机/不可达/不匹配，不分配资源 |

### 3.7 排序规则

```
对所有目标按以下规则排序：

PRIMARY: 按 S_final 降序排列
    → 分数高的排在前面

SECONDARY: 当 S_final 差值 < 0.05 时，按 F2(距离) 升序排列
    → 分数接近时，近处优先

TERTIARY: 排除目标(LEVEL_1)始终排在最后
    → 不论分数如何，排除目标不参与攻击队列
```

## 第四章 完整计算流程

### 4.1 单目标计算流程

```
┌─────────────────────────────────────────────┐
│  STEP 0: 数据输入                            │
│  ├─ 目标属性: 类型/频段/武器协同/战略价值    │
│  ├─ 实时态势: 距离/辐射状态                 │
│  └─ 系统参数: 权重/航程/导引头频段          │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  STEP 1: 因子提取与归一化                    │
│  ├─ F1 = lookup(辐射状态)     → [0, 1]     │
│  ├─ F2 = clamp(1-d/120, 0, 1) → [0, 1]     │
│  ├─ F3 = lookup(频段匹配)     → [0, 1]     │
│  ├─ F4 = lookup(武器协同)     → [0, 1]     │
│  └─ F5 = lookup(战略价值)     → [0, 1]     │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  STEP 2: 硬约束检查                          │
│  ├─ F1=0(关机) → 排除, S=0, 跳至STEP 5   │
│  ├─ F3=0(不匹配) → 排除, S=0, 跳至STEP 5  │
│  └─ F2=0(超距) → 排除, S=0, 跳至STEP 5    │
└──────────────┬──────────────────────────────┘
               ▼ (未排除)
┌─────────────────────────────────────────────┐
│  STEP 3: 权重归一化                          │
│  Ŵ_i = W_i / ΣW_j    (确保ΣŴ=1)           │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  STEP 4: 加权求和                            │
│  S = Σ Ŵ_i × F_i                           │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  STEP 5: 强制上浮检查                        │
│  IF F4=1.00 AND S<0.80 → S=max(S, 0.80)   │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  STEP 6: 等级映射                            │
│  Level = map(S_final)  → L1~L5             │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  OUTPUT: (S_final, Level, 各因子明细)         │
└─────────────────────────────────────────────┘
```

### 4.2 全局排序流程

```
┌─────────────────────────────────────────────┐
│  对目标集合 T = {t1, t2, ..., tn} 执行：       │
│                                               │
│  1. 逐个执行单目标计算流程                    │
│  2. 分离为两个子集：                           │
│     T_exclude = {t | Level=1}  (排除集合)     │
│     T_attack   = {t | Level≥2}  (攻击集合)    │
│  3. T_attack 按 S_final 降序排序               │
│     同分差<0.05时按 F2(距离)升序              │
│  4. T_exclude 追加到队列末尾                   │
│  5. 输出完整优先级队列                        │
└─────────────────────────────────────────────┘
```

### 4.3 计算示例

以目标 **T02: MPQ-65 清水 (距离8km, 全功率, 已锁定)** 为例：

```
STEP 1: 因子提取
  F1 = lookup("全功率开机") = 1.00
  F2 = clamp(1 - 8/120, 0, 1) = 0.933
  F3 = lookup("完全匹配", C波段) = 1.00
  F4 = lookup("导弹就绪+已锁定") = 1.00
  F5 = lookup("核心节点", 火控雷达) = 1.00

STEP 2: 硬约束检查
  F1≠0, F3≠0, F2≠0 → 全部通过，未排除

STEP 3: 权重归一化 (使用默认权重，已归一)
  Ŵ = {0.300, 0.250, 0.200, 0.150, 0.100}

STEP 4: 加权求和
  S = 0.300×1.00 + 0.250×0.933 + 0.200×1.00 + 0.150×1.00 + 0.100×1.00
    = 0.300 + 0.233 + 0.200 + 0.150 + 0.100
    = 0.983

STEP 5: 强制上浮
  F4=1.00 但 S=0.983 > 0.80 → 无需上浮

STEP 6: 等级映射
  0.983 ∈ [0.75, 1.00] → LEVEL_5 紧急

OUTPUT: (0.983, LEVEL_5)
```

以目标 **T06: JYL-1 超视距 (距离350km, 全功率, 无武器)** 为例：

```
STEP 1: 因子提取
  F1 = lookup("全功率开机") = 1.00
  F2 = clamp(1 - 350/120, 0, 1) = 0.000  → 触发硬约束
  F3 = lookup("不匹配", HF短波) = 0.00   → (不执行，已被F2排除)
  F4 = lookup("无武器协同") = 0.00
  F5 = lookup("重要节点") = 0.65

STEP 2: 硬约束检查
  F2=0 (350km > 120km) → 排除
  S = 0 (强制归零)

OUTPUT: (0.000, LEVEL_1 排除, 原因: "距离超出航程120km")
```

## 第五章 权重配置策略

### 5.1 默认配置

适用于标准作战场景（单波次攻击，中等电磁环境）。

| 因子 | 权重 | 理由 |
|:----:|:----:|------|
| F1 | **0.30** | 最高：关机=不能打，是其他一切的前提 |
| F2 | **0.25** | 次高：飞不到=白搭，航程是硬约束 |
| F3 | **0.20** | 中等：技术可行性，规划时通常已知且不变 |
| F4 | **0.15** | 较低：风险因素，但有规避手段可化解 |
| F5 | **0.10** | 最低：价值判断是"锦上添花"，不决定能否打 |

### 5.2 场景化调节策略

| 场景 | 调节方向 | 具体调整 | 理由 |
|------|---------|---------|------|
| **深突防** | F1↑ F2↑ | F1=0.35, F2=0.30, F5↓0.05 | 深入敌后方时，只有开机的近距目标才值得攻击，价值权重降低 |
| **饱和攻击** | F5↑ F4↓ | F5=0.20, F4=0.08 | 弹药充足时，优先打高价值节点，不太担心被拦截 |
| **精确打击** | F1↑ F3↑ | F1=0.35, F3=0.25, F2↓0.15 | 弹药有限，确保每发必中，只打确定能截获的目标 |
| **电子压制后** | F1↑↑ F2↑ | F1=0.45, F2=0.30 | 电子战逼出目标后，辐射状态和距离成为最关键因素 |
| **多架接力** | F2↓↓ | F2=0.10 | 接力时距离约束放松，其他因子权重相应提升 |

### 5.3 权重调节操作规程

```
1. 指挥员根据作战意图选择场景预设（或手动调节）
2. 系统自动归一化，无需人工计算
3. 调节任一权重后，所有目标分数实时重算
4. 建议每次只调1-2个权重，避免同时大范围调整导致结果不可解释
5. 调节后观察排序变化，确认符合战术直觉
```

## 第六章 航路规划接口

### 6.1 输入输出接口定义

**输入（从外部系统获取）：**

```
struct TargetInput {
    id:          string    // 目标编号
    name:        string    // 目标名称
    type:        string    // 雷达类型
    freq:        string    // 工作频段
    lng:         float     // 经度
    lat:         float     // 纬度
    dist:        float     // 距离
    state:       enum      // 辐射状态 {FULL,NORMAL,LPI,BLINK,OFF}
    weapon:      enum      // 武器协同 {NONE,NOT_READY,READY_SEARCH,READY_LOCK}
    strategic:   enum      // 战略价值 {CORE,IMPORTANT,NORMAL,MINOR}
}
```

**输出（提供给航路规划器）：**

```
struct ThreatOutput {
    id:              string    // 目标编号
    score:           float     // 威胁分 [0, 1]
    level:           int       // 等级 [1-5]
    isExcluded:      bool      // 是否被排除
    excludeReason:   string    // 排除原因（仅isExcluded=true时）
    overrideFlag:    bool      // 是否触发强制上浮
    overrideReason:  string    // 上浮原因
    factors: {
        F1: {raw, normalized, weight, contribution},
        F2: {raw, normalized, weight, contribution},
        F3: {raw, normalized, weight, contribution},
        F4: {raw, normalized, weight, contribution},
        F5: {raw, normalized, weight, contribution},
    }
    rank:           int       // 全局排名
}
```

### 6.2 航路规划器消费方式

```
航路规划器接收威胁评估输出后：

1. 过滤: 只取 level ≥ 3 的目标纳入航路规划
2. 排序: 按 rank 顺序依次为每个目标规划航路
3. 资源分配:
   - LEVEL_5 目标 → 分配 2-4 架 JWS01（确保摧毁）
   - LEVEL_4 目标 → 分配 1-2 架 JWS01
   - LEVEL_3 目标 → 分配 1 架 JWS01（视弹药余量）
4. 航路约束:
   - LEVEL_5 且 F4≥W3 的目标 → 规划规避航路（绕行拦截区）
   - 其他目标 → 规划直飞航路（最短路径）
5. 航路优化目标:
   - 最小化总飞行距离（节省燃油）
   - 避免不同目标航路之间的时间冲突
   - 满足各目标的时间窗口约束（F1状态可能变化）
```

### 6.3 与仿真推演的联动

```
仿真推演中，目标态势每 2-5 秒更新一次：

1. 目标状态可能变化: 开机→关机, 远→近, 无武器→有武器
2. 每次更新触发重新计算
3. 如果某目标从 LEVEL_3 升至 LEVEL_5:
   → 触发航路重规划告警
   → 指挥员决定是否调整攻击计划
4. 如果某目标从 LEVEL_5 降至 LEVEL_1(关机):
   → 该目标航路标记为"取消"
   → 释放的JWS01资源可重新分配
```

## 附录A：因子独立性验证矩阵

证明五个因子两两之间不存在推导关系：

|  | F1辐射 | F2距离 | F3频段 | F4武器 | F5价值 |
|--|:-----:|:-----:|:-----:|:-----:|:-----:|
| **F1辐射** | — | 独立 | 独立 | 独立 | 独立 |
| **F2距离** | 独立 | — | 独立 | 独立 | 独立 |
| **F3频段** | 独立 | 独立 | — | 独立 | 独立 |
| **F4武器** | 独立 | 独立 | 独立 | — | 独立 |
| **F5价值** | 独立 | 独立 | 独立 | 独立 | — |

验证方法：对每种因子组合检验是否存在"已知其他四个因子值，能否唯一确定第五个"的情况。

- 已知F2=0.5, F3=1.0, F4=0, F5=0.65 → 能否确定F1？不能。F1可能是0(关机)也可能是1(全功率)，距离/频段/武器/价值都不包含辐射状态信息。**通过**
- 已知F1=1, F3=1, F4=1, F5=1 → 能否确定F2？不能。目标可能近(高F2)也可能远(低F2)。**通过**
- 其余组合类似，全部通过。五因子确实正交独立。
