## 1 引言

### 1.1 范围

本需求规格说明书定义了反辐射无人机协同任务规划模块的软件需求，适用于该模块的研制、开发、测试及验收过程。

本文件所描述的功能模块作为防辐射无人机系统的核心组成部分，集成于协同任务规划系统之中，主要实现以下功能：
- 协同任务管理
- 任务场景编辑
- 环境参数设置
- 兵力需求计算
- 作战任务分配
- 巡航路径规划
- 搜索路径规划
- 工作参数规划
- 协同任务规划

### 1.2 定义

| 术语 | 定义 |
|------|------|
| 反辐射无人机 | 利用敌方雷达辐射的电磁信号进行导引、跟踪并摧毁辐射源的无人机 |
| 协同任务规划 | 对多架无人机进行任务分配、路径规划和工作参数设置的系统过程 |
| GIS | 地理信息系统 |
| 导引头 | 反辐射无人机上用于搜索、跟踪目标辐射源的传感器装置 |
| 航点 | 飞行路径中的关键转折点，包含位置坐标、高度、速度等信息 |
| 任务剖面 | 无人机从起飞到完成任务返航的完整飞行过程 |

## 2 总体描述

### 2.1 产品背景

反辐射无人机协同任务规划模块是实现多架反辐射无人机协同作战的关键软件系统。该模块通过集成任务管理、场景编辑、环境设置、兵力计算、任务分配、路径规划及工作参数规划等功能，为指挥员提供完整的作战任务规划能力，提高作战效率和成功率。

### 2.2 用户特征

本模块的主要用户包括：
- 作战指挥员：负责任务规划、审批和下达
- 作战参谋：负责具体任务参数设置和方案制定
- 系统操作员：负责日常操作和维护

### 2.3 约束条件

- 系统应支持实时战场环境数据接入
- 路径规划算法应能在30秒内生成规划方案
- 系统应具备良好的人机交互界面
- 软件应满足军用软件可靠性要求
- 系统应支持7×24小时连续运行

## 3 详细需求说明

### 3.1 协同任务管理功能

#### 3.1.1 功能描述

协同任务管理功能应提供对协同作战任务的创建、编辑、存储、查询和删除等全生命周期管理能力。该功能是整个协同任务规划系统的基础模块，负责管理所有作战任务的基础信息和状态。

#### 3.1.2 任务创建

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| REQ-CTM-001-01 | 系统应提供任务创建入口，在主界面菜单栏提供"新建任务"按钮，支持快捷键Ctrl+N创建 | 必须 |
| REQ-CTM-001-02 | 系统应支持任务名称输入，名称长度范围为1-50个字符，支持中英文、数字及常用符号 | 必须 |
| REQ-CTM-001-03 | 系统应支持任务编号输入，编号格式应为"YYYYMMDD-XXX"，其中YYYYMMDD为日期，XXX为3位序号 | 必须 |
| REQ-CTM-001-04 | 系统应自动检查任务编号唯一性，若编号已存在应提示用户修改 | 必须 |
| REQ-CTM-001-05 | 系统应支持任务描述输入，描述内容长度范围为0-500个字符 | 期望 |
| REQ-CTM-001-06 | 系统应支持创建任务时自动记录创建人、创建时间 | 必须 |

#### 3.1.4 任务目标设置

- 数据加密存储，加密算法应符合国密要求
