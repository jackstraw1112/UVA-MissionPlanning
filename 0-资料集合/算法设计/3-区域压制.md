# 反辐射无人机区域压制搜索路径规划设计方案

## 一、任务场景分类与概述

| 场景类型 | 目标特征 | 搜索复杂度 | 典型代表 |
|---------|---------|-----------|---------|
| **单一固定目标** | 位置已知，不移动 | 低 | 固定雷达站、预警雷达阵地 |
| **单一低速机动目标** | 位置变化慢，移动范围有限 | 中 | 机动雷达、车载雷达 |
| **配有诱饵的单一固定目标** | 主目标固定，诱饵随机发射 | 高 | 带诱饵系统的防空雷达 |
| **小区域多目标** | 密集分布，相互干扰 | 中高 | 防空阵地群、雷达网 |
| **大区域多目标** | 分散分布，需协同搜索 | 高 | 区域防空体系、多部雷达 |

## 二、单一固定目标搜索路径规划

### 2.1 任务特点
- **目标位置**：精确已知（可通过情报或侦察确认）
- **目标行为**：持续辐射或按规律开关机
- **搜索目的**：确认目标状态、获取精确参数、为攻击提供引导

### 2.2 搜索策略

```
┌─────────────────────────────────────────────────────────┐
│                  单一固定目标搜索流程                     │
├─────────────────────────────────────────────────────────┤
│  1. 目标位置已知 → 无需全域搜索                           │
│  2. 采用 8字盘旋或圆形待机搜索                           │
│  3. 逐步缩小半径逼近目标                                  │
│  4. 多角度确认目标特征                                    │
│  5. 转入攻击引导                                          │
└─────────────────────────────────────────────────────────┘
```

### 2.3 路径规划算法

#### 2.3.1 螺旋收缩搜索
- **起始半径**：R_start = 目标最大探测距离 × 1.5
- **半径递减率**：ΔR = 传感器探测宽度 × 0.8
- **搜索高度**：H = 3000m（逐步降低）
- **圈数**：N = R_start / ΔR

#### 2.3.2 多角度8字盘旋
- **盘旋中心**：目标点偏移500-1000m（避开主瓣）
- **盘旋半径**：R = 2-5km（根据目标威胁等级调整）
- **盘旋方向**：交替改变，覆盖不同角度
- **周期**：T = 30-60秒/圈

### 2.4 多机协同方案（可选）

| 无人机数量 | 搜索模式 | 配置说明 |
|-----------|---------|---------|
| 单机 | 螺旋收缩 | 从远到近逐步逼近 |
| 双机 | 交叉8字 | 90度夹角交替盘旋 |
| 三机 | 三角覆盖 | 120度间隔环绕 |

### 2.5 路径示意图

```
        ○ 第3圈
      ○ 第2圈
    ○ 第1圈
    ● 目标
```

## 三、单一低速机动目标搜索路径规划

### 3.1 任务特点
- **目标位置**：大致区域已知，但实时移动
- **移动速度**：通常≤30 km/h（车载雷达）
- **移动范围**：半径5-10km的活动区域
- **搜索目的**：跟踪目标位置，预测移动轨迹

### 3.2 搜索策略

```
┌─────────────────────────────────────────────────────────┐
│                  机动目标搜索流程                         │
├─────────────────────────────────────────────────────────┤
│  1. 根据情报划定可能活动区域                             │
│  2. 采用分区搜索覆盖整个活动区                           │
│  3. 发现目标后转入跟踪模式                               │
│  4. 多机交替跟踪，保持连续覆盖                           │
│  5. 动态更新目标位置概率地图                             │
└─────────────────────────────────────────────────────────┘
```

### 3.3 路径规划算法

#### 3.3.1 概率网格搜索
- **区域栅格化**：将活动区域划分为100×100m网格
- **先验概率**：根据历史移动规律设定初始概率
- **搜索顺序**：按概率从高到低遍历网格
- **贝叶斯更新**：每次搜索后更新概率分布

#### 3.3.2 螺旋扩展搜索（目标丢失后）
- **起始点**：最后发现目标的位置
- **螺旋参数**：半径按R_n = R_0 × √n递增
- **覆盖密度**：每圈保证2-3次扫描重叠

### 3.4 多机协同跟踪

```
                   时间轴 →
UAV-01 ──────┬─────┬─────┬─────┬─────┬─────
UAV-02        └─────┬─────┬─────┬─────┘
UAV-03              └─────┬─────┬─────
```

- **接力跟踪**：当一架无人机续航不足时，另一架接替
- **三角定位**：三机同时接收信号，实时解算位置
- **预测引导**：根据运动趋势提前飞向下一个可能位置

### 3.5 搜索参数配置

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 活动区域半径 | 10 km | 根据目标机动能力 |
| 网格大小 | 100 m | 兼顾精度与效率 |
| 扫描间隔 | 30秒 | 匹配目标移动速度 |
| 接力时间 | 15分钟 | 考虑无人机续航 |

## 四、配有诱饵的单一固定目标搜索路径规划

### 4.1 任务特点
- **主目标**：固定位置，有规律辐射
- **诱饵**：随机发射，信号特征相似
- **干扰类型**：诱饵弹、拖曳式诱饵、转发式诱饵
- **搜索目的**：识别真假目标，确定主目标精确位置

### 4.2 搜索策略

```
┌─────────────────────────────────────────────────────────┐
│                诱饵目标识别搜索流程                       │
├─────────────────────────────────────────────────────────┤
│  1. 多角度同步探测，获取信号到达时间差                    │
│  2. 信号特征分析（频率稳定性、脉冲特性）                  │
│  3. 运动特征分析（固定vs运动）                            │
│  4. 多机交叉定位验证                                      │
│  5. 确认主目标后集中搜索                                  │
└─────────────────────────────────────────────────────────┘
```

### 4.3 路径规划算法

#### 4.3.1 多角度交叉探测
- **无人机部署**：围绕目标区域等角度分布（3-4架）
- **探测半径**：R = 8-12 km（避开诱饵杀伤范围）
- **同步要求**：时间同步精度 < 1ms
- **路径模式**：小范围圆弧运动，保持视线角变化

#### 4.3.2 分层识别搜索
```
第一层：远距离探测（15km） → 识别信号存在
第二层：中距离逼近（10km） → 分析信号特征
第三层：近距离确认（5km）  → 多机交叉定位
第四层：精确锁定（3km）    → 确认主目标
```

### 4.4 多机协同方案

| 角色 | 数量 | 任务 | 路径模式 |
|------|------|------|---------|
| 探测机 | 2 | 远距离探测 | 大范围扫描 |
| 分析机 | 2 | 信号特征分析 | 中距离盘旋 |
| 定位机 | 2 | 精确交叉定位 | 三角布局 |
| 确认机 | 1 | 最终确认 | 逼近俯冲 |

### 4.5 诱饵识别逻辑

```
if 信号来源 == 固定点:
    可能是 主目标 或 固定诱饵
    if 信号特征 == 稳定:
        高度怀疑是主目标
    else:
        可能是诱饵

if 信号来源 == 运动点:
    可能是 机动诱饵
    if 运动轨迹 == 可控:
        判断为拖曳式诱饵
    else:
        可能是弹射诱饵
```

## 五、小区域多目标搜索路径规划

### 5.1 任务特点
- **区域范围**：直径 ≤ 20 km
- **目标数量**：3-10个，密集分布
- **目标类型**：雷达群、防空阵地
- **搜索目的**：全面覆盖，逐一确认，威胁排序

### 5.2 搜索策略

```
┌─────────────────────────────────────────────────────────┐
│                 小区域多目标搜索流程                      │
├─────────────────────────────────────────────────────────┤
│  1. 区域分割：将小区域划分为若干子区域                    │
│  2. 多机并行：每架负责一个子区域                          │
│  3. 协同覆盖：边界重叠10%保证无遗漏                      │
│  4. 发现共享：实时交换目标信息                            │
│  5. 动态重分配：根据发现情况调整搜索密度                  │
└─────────────────────────────────────────────────────────┘
```

### 5.3 路径规划算法

#### 5.3.1 等面积分区并行搜索
- **分区方法**：Voronoi图划分或K-means聚类
- **搜索模式**：各分区内采用平行扫海
- **扫描方向**：相邻分区垂直，减少边界干扰
- **重叠区域**：边界宽度 = 传感器宽度 × 0.5

#### 5.3.2 螺旋遍历（中心聚焦）
- **起始点**：区域中心（目标最密集处）
- **螺旋向外**：逐步扩大半径
- **密度调整**：中心区域扫描间距减半
- **适用场景**：目标概率中心已知

### 5.4 多机协同方案

| 3-5个 | 3架 | 等面积分区 | 15-20 min |
| 5-8个 | 4架 | 中心螺旋+分区 | 20-25 min |
| 8-10个 | 6架 | 网格覆盖 | 20 min |

### 6.1 任务特点
- **区域范围**：直径 50-200 km
- **目标数量**：10-50个，分散分布
- **目标类型**：区域防空网、多部雷达
- **搜索目的**：快速普查，发现目标，威胁排序

```
│  1. 先验情报获取目标大致分布                             │
│  2. 区域分解为多个小区域                                 │
│  3. 多波次无人机接力搜索                                 │
│  4. 重点区域优先（高威胁）                               │

└─────────────────────────────────────────────────────────┘
```

### 6.3 路径规划算法

#### 6.3.1 层次化区域分解

```
第一层：大区域 → 子区域划分（基于威胁等级）
第二层：子区域 → 网格划分（基于面积）
第三层：网格 → 搜索路径生成（基于覆盖算法）
```

#### 6.3.2 概率优先搜索
- **先验概率**：根据情报设定各子区域概率
- **搜索顺序**：按概率从高到低排序
- **动态更新**：发现目标后更新周边区域概率
- **自适应**：高概率区域加密搜索

#### 6.3.3 多波次接力搜索
```
波次1 (0-60min)：覆盖高概率区域（40%面积）
波次2 (60-120min)：覆盖中概率区域（30%面积）
波次3 (120-180min)：覆盖低概率区域（30%面积）
```

#### 6.5.1 航程优化

- **并行度**：同时搜索区域数 = 无人机数

## 二、场景识别模块实现

### 2.1 场景特征提取

```python
class ScenarioAnalyzer:
    def extract_features(self, input_data):
        features = {}
        
        # 目标数量特征
        features['target_count'] = len(input_data.targets)
        
        # 目标机动性特征
        mobility_scores = []
        for target in input_data.targets:
            if target.type == 'fixed':
                mobility_scores.append(0)
            elif target.type == 'mobile':
                mobility_scores.append(target.speed / 30.0)  # 归一化
            else:  # decoy
                mobility_scores.append(0.8)
        features['max_mobility'] = max(mobility_scores)
        features['avg_mobility'] = sum(mobility_scores) / len(mobility_scores)
        
        # 区域特征
        area = calculate_area(input_data.region)
        features['area_size'] = area
        features['target_density'] = features['target_count'] / area
        
        # 诱饵特征
        features['has_decoy'] = any(t.type == 'decoy' for t in input_data.targets)
        features['decoy_count'] = sum(1 for t in input_data.targets if t.type == 'decoy')
        
        return features
```

### 2.2 场景分类决策树

```python
def recognize_scenario(features):
    if features['target_count'] == 1:
        if features['has_decoy']:
            return ScenarioType.DECOY_TARGET
        elif features['max_mobility'] > 0.3:
            return ScenarioType.MOBILE_TARGET
        else:

    else:
        if features['area_size'] > 500:  # km²
```

## 三、单一固定目标搜索实现

### 3.1 螺旋收缩算法

```cpp
class SpiralSearchPlanner : public SearchPlanner {
public:
    Path plan(Scenario scenario, UAVGroup uavs) override {
        Path path;
        Point target = scenario.targets[0].position;
        
        // 参数配置
        double startRadius = scenario.targets[0].maxDetectRange * 1.5;
        double minRadius = 2.0;  // km
        double stepRadius = scenario.uavConfig.sensorWidth * 0.8;
        double height = 3000;  // m
        
        // 生成螺旋路径点
        double r = startRadius;
        int turnCount = 0;
        
        while (r > minRadius) {
            // 每圈点数 = 2πr / stepRadius
            int pointsPerCircle = max(8, (int)(2 * M_PI * r / stepRadius));
            
            for (int i = 0; i < pointsPerCircle; i++) {
                double angle = 2 * M_PI * i / pointsPerCircle;
                double x = target.x + r * cos(angle);
                double y = target.y + r * sin(angle);
                
                path.addWaypoint(Waypoint(x, y, height));
            }
            
            r -= stepRadius;
            turnCount++;
```

```cpp
class ProbabilityGrid {
private:
    struct Cell {
        double probability;  // 目标存在概率
        double lastSearchTime;  // 上次搜索时间
        bool searched;  // 是否已搜索
    };
    
    vector<vector<Cell>> grid;
    double cellSize;  // 网格大小 (km)
    double decayRate;  // 概率衰减率
    
public:
    ProbabilityGrid(Region region, double cellSize = 0.1) {
        this->cellSize = cellSize;
        int rows = region.height / cellSize;
        int cols = region.width / cellSize;
        grid.resize(rows, vector<Cell>(cols));
        
        // 初始化概率
        double totalCells = rows * cols;
        double baseProb = 1.0 / totalCells;
        
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                grid[i][j].probability = baseProb;
                grid[i][j].searched = false;
                grid[i][j].lastSearchTime = 0;
            }
        }
    }
    
    // 根据情报更新先验概率
    void updatePrior(vector<IntelPoint> intels) {
        for (auto& intel : intels) {
            int i = (intel.y - region.minY) / cellSize;
            int j = (intel.x - region.minX) / cellSize;
            
            if (i >= 0 && i < rows && j >= 0 && j < cols) {
                grid[i][j].probability += intel.confidence * 0.1;
                normalize();
            }
        }
    }
    
    // 搜索后贝叶斯更新
    void updateAfterSearch(int i, int j, bool found) {
        if (found) {
            grid[i][j].probability = 0.9;
            // 周边区域概率也增加
            for (int di = -2; di <= 2; di++) {
                for (int dj = -2; dj <= 2; dj++) {
                    int ni = i + di, nj = j + dj;
                    if (ni >= 0 && ni < rows && nj >= 0 && nj < cols) {
                        grid[ni][nj].probability += 0.05;
                    }
                }
            }
        } else {
            grid[i][j].probability *= 0.3;  // 未发现则降低概率
        }
        normalize();
    }
    
    // 获取下一个最优搜索点
    Point getNextBestSearchPoint() {
        double maxProb = 0;
        Point best;
        
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                if (!grid[i][j].searched && grid[i][j].probability > maxProb) {
                    maxProb = grid[i][j].probability;
                    best.x = region.minX + j * cellSize + cellSize/2;
                    best.y = region.minY + i * cellSize + cellSize/2;
                }
            }
        }
        
        return best;
    }
};
```

```cpp
        for (int i = 0; i < centers.size(); i++) {

            sub.polygon = computeVoronoiCell(region, centers, i);
            
            subRegions.push_back(sub);
        }
        
        return subRegions;
    }
    
    vector<SubRegion> decomposeByArea(Region region, int uavCount) {
        vector<SubRegion> subRegions;
        
        double totalArea = region.width * region.height;
        double subArea = totalArea / uavCount;
        
        // 等面积分割（按网格）
        int rows = ceil(sqrt(uavCount));
        int cols = ceil((double)uavCount / rows);
        
        double cellWidth = region.width / cols;
        double cellHeight = region.height / rows;
        
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                int idx = i * cols + j;
                if (idx >= uavCount) continue;
                
                SubRegion sub;
                sub.minX = region.minX + j * cellWidth;
                sub.maxX = region.minX + (j + 1) * cellWidth;
                sub.minY = region.minY + i * cellHeight;
                sub.maxY = region.minY + (i + 1) * cellHeight;
                sub.assignedUAV = idx;
                
                subRegions.push_back(sub);
            }
        }
        
        return subRegions;
    }
```

```cpp
        sort(conflicts.begin(), conflicts.end(), 
             [](auto& a, auto& b) { return a.time < b.time; });
        
        for (auto& conflict : conflicts) {
            // 优先级高的保持，优先级低的调整
            if (paths[conflict.uav1].priority > paths[conflict.uav2].priority) {
                adjustPath(paths[conflict.uav2], conflict);
            } else {
                adjustPath(paths[conflict.uav1], conflict);
            }
        }
    }
    
private:
    void adjustPath(Path& path, Conflict& conflict) {
        // 局部调整：速度调节或绕行
        int idx = path.getIndexAtTime(conflict.time);
        
        // 方法1：降低速度延迟到达
        path.segments[idx].speed *= 0.8;
        
        // 方法2：局部绕行（如果速度调整不够）
        if (distance(path.getPositionAtTime(conflict.time), conflict.position) < MIN_SAFE_DISTANCE) {
            Point detourPoint = findDetourPoint(path, idx);
            path.insertWaypoint(idx, detourPoint);
        }
    }
};
```

### 8.2 负载均衡

```cpp
class LoadBalancer {
public:
    void balanceWorkload(vector<Path>& paths, const vector<UAV>& uavs) {
        // 计算各机任务量
        vector<double> workloads;
        for (auto& path : paths) {
            workloads.push_back(path.totalDistance);
        }
        
        double avgWorkload = accumulate(workloads.begin(), workloads.end(), 0.0) / workloads.size();
        
        // 识别过载和轻载
        vector<int> overloaded, underloaded;
        for (int i = 0; i < workloads.size(); i++) {
            if (workloads[i] > avgWorkload * 1.2) {
                overloaded.push_back(i);
            } else if (workloads[i] < avgWorkload * 0.8) {
                underloaded.push_back(i);
            }
        }
        
        // 重新分配任务
        for (int i = 0; i < min(overloaded.size(), underloaded.size()); i++) {
            transferWorkload(paths[overloaded[i]], paths[underloaded[i]], 
                           (workloads[overloaded[i]] - avgWorkload) / 2);
        }
    }
};
```
